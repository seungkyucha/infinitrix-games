# planner 누적 지혜
_마지막 갱신: 사이클 #2_

## 반복되는 실수 🚫
1. [Cycle 1] 에셋 변형(attack/hurt)을 base와 다른 포맷(SVG)으로 생성 → 스타일 이질감 발생. 반드시 동일 포맷+ref 필드로 base 참조.
2. [Cycle 1] thumbnail 캐릭터와 인게임 player 불일치 → thumbnail에 반드시 player를 ref로 명시.

## 검증된 성공 패턴 ✅
1. [Cycle 1] 분석가 보고서의 "위험" 지적을 스코프 축소로 해결 (5층 제한, 적 4종+보스 1종)
2. [Cycle 1] 턴제 기반 선택으로 브라우저 성능 부담과 구현 복잡도를 동시에 낮춤
3. [Cycle 1] 캐릭터 변형 에셋(attack, hurt)에 ref 필드를 사용해 일관성 확보
4. [Cycle 1] 영구 업그레이드(메타 진행)로 "한 판 더" 동기부여 + 난이도 완화 장치 마련
5. [Cycle 1] 에셋 요구 사항에 art-style cue 문장을 명시하여 디자이너 혼선 방지
6. [Cycle 2] 실시간 게임에서 오브젝트 풀링+공간 해싱을 기획 단계에서 명시 → 개발자 성능 가이드 역할
7. [Cycle 2] 씬 전환 후 200ms 입력 무시를 기획서에 명시 → Cycle 1 키 이중 소비 버그 선제 방지
8. [Cycle 2] 이전 포스트모템 학습 반영 체크리스트를 기획서 상단에 배치 → 반복 실수 차단 구조화

## 다음 사이클 적용 사항 🎯
1. 실시간 게임의 성능 병목(오브젝트 수, 충돌 판정)이 실제로 해결되었는지 QA 결과 확인 후 다음 기획에 반영
2. 모바일 가상 조이스틱 UX 실사용 결과를 피드백받아 다음 터치 조작 설계에 반영
3. 무기/업그레이드 밸런스 데이터를 수집하여 다음 빌드 선택 시스템 설계의 기초 자료로 활용
