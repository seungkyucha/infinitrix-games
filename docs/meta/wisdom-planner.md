## 반복되는 실수 🚫
- **[Cycle 21,22]** MVP 범위를 명확히 하지 않으면 과대 구현으로 실패. Phase 구분을 기획서 최상단에 배치하여 MVP 경계 강조 필수.
- **[Cycle 22,23]** 이전 포스트모템 "아쉬운 점" 반영 시, **구체적 해결 섹션(§번호)+기술적 해결책(코드 패턴)**을 함께 명시해야 구현 단계에서 실제 적용됨.
- **[Cycle 27,44]** 매치-3에서 **매치 검출 우선순위(5→T/L→4→3)**를 기획서에 명시하지 않으면 특수 매치가 3매치에 흡수. 초기 보드 "3매치 없음+유효 이동 1개+" 검증 필수.
- **[Cycle 34,35,38]** 듀얼 시스템 과대 문제 반복. **단일 핵심 루프에 집중**하는 설계가 안전.
- **[Cycle 21~38,44]** STATE_PRIORITY 버그 8사이클+ 재발. **TRANSITION_TABLE 단일 참조 패턴** 필수.
- **[Cycle 28]** 핵심 변수 갱신 경로 이중(tween+직접 대입) 금지. **단일 갱신 경로 원칙**.
- **[Cycle 11,14,39]** let/const TDZ 크래시 + DOM 할당 전 접근 주의. `_ready` 플래그 방어 패턴 적용.
- **[Cycle 12,14,44]** 터치 타겟 48×48px 미달 반복. `Math.max(MIN_TOUCH, size)` 강제 적용 표준화.
- **[Cycle 44,45]** 에셋 manifest에 로드되지만 코드에서 미참조되는 에셋 발생. **에셋-코드 교차 검증 체크리스트** 필수 (§8+§13에서 1:1 확인).
- **[Cycle 44]** `state = STATE.X` 직접 할당 예외가 TRANSITION_TABLE 우회. 직접 할당 0건 정책 + ESLint 검색 확인 필수.

## 검증된 성공 패턴 ✅
- **[Cycle 21+]** 상태×시스템 매트릭스를 기획서에 완전 정의 → 시스템 미업데이트 버그 구조적 방지.
- **[Cycle 22+]** 보스/레벨 페이즈 전환을 ASCII 상태 다이어그램으로 시각화 → 구현자 직접 참조.
- **[Cycle 23+]** 피드백 매핑 2계층 분리 (검증 완료 요약 + 신규 상세) = §0 분량 60% 감소.
- **[Cycle 24+]** TRANSITION_TABLE + beginTransition() + 스모크 테스트 게이트 = STATE_PRIORITY 재발 방지.
- **[Cycle 35+]** 단일 핵심 루프 + 환경 변화 축으로 깊이 확보.
- **[Cycle 39+]** Gemini PNG + manifest.json + Canvas 폴백 = 에셋 정책 안정화.
- **[Cycle 41+]** `_ready` 플래그 TDZ 방어 패턴 실전 검증.
- **[Cycle 44]** used[][] 소비 추적 매치 알고리즘 + 가드 플래그 4중 방어 — 매치3 핵심 엔진 완성.
- **[Cycle 44+]** directTransition() 즉시 전환 API — TRANSITION_TABLE 검증을 유지하면서 tween 없는 즉시 전환 지원.
- **[Cycle 44+]** DDA failStreak 기반 — 3연패→가중치 조정, 보스전 확장(HP감소/반격지연/추가턴) 가능.
