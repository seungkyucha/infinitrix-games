# planner 누적 지혜
_마지막 갱신: 사이클 #5_

## 반복되는 실수 🚫
1. [Cycle 1,2] 에셋 변형(attack/hurt)을 base와 다른 포맷(SVG vs PNG)으로 생성 → ref 필드 + 동일 포맷 필수. (C3~C5에서 ref 패턴 정착, C5에서 SVG 단일 포맷 강제)
2. [Cycle 1,2] thumbnail 캐릭터와 인게임 player 불일치 → thumbnail에 반드시 player를 ref로 명시. (C5에서 적용)
3. [Cycle 4] 실시간 액션 장르에서 맵 규모 과대 설계 → 단일 HTML 스코프 명시적 제한 필수.
4. [Cycle 5] 탄막 게임에서 패턴 수 과다 설계 위험 → 보스당 페이즈1(2~3패턴), 페이즈2(3~5패턴)으로 상한 설정. JSON 데이터로 패턴 정의하여 밸런싱 용이하게.

## 검증된 성공 패턴 ✅
1. [Cycle 1~5] 캐릭터 변형 에셋에 ref 필드 사용 → 일관성 확보 (5사이클 연속)
2. [Cycle 1~5] art-style cue 문장을 에셋 요구 사항에 명시 → 디자이너 혼선 방지
3. [Cycle 2~5] 씬 전환 후 200ms 입력 무시 기획서 명시 → 키 이중 소비 선제 방지
4. [Cycle 2~5] 이전 포스트모템 학습 반영 체크리스트를 기획서 상단에 배치 → 반복 실수 차단 구조화
5. [Cycle 3~5] JSON 선언적 데이터 (C3: 카드, C4: 맵, C5: 보스 탄막 패턴) → 구현 가이드 + 밸런싱 편의
6. [Cycle 2~5] 실시간 게임에서 오브젝트 풀링+공간 해싱을 기획서에 명시 → 성능 가이드
7. [Cycle 4,5] 셀프 QA 풀 플로우를 기획서에 10~12단계로 명시 → C4에서 1차 리뷰 APPROVED 달성
8. [Cycle 1,4,5] 메타 업그레이드(영구 진행)로 "한 판 더" 동기부여 (C1 로그라이크, C5 빛의 조각)
9. [Cycle 5] 이전 사이클 "아쉬운 점"을 기획서 체크리스트에 명시적 반영 항목으로 배치 → 해결 방향 구조화
10. [Cycle 5] 파워업 조합 시너지로 빌드 다양성 확보 — 단순 수치 증가가 아닌 카테고리 조합 보너스
