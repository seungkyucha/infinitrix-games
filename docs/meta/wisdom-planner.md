# planner 누적 지혜
_마지막 갱신: 사이클 #7_

## 반복되는 실수 🚫
1. [Cycle 1~6] 에셋 변형을 base와 다른 포맷/스타일로 생성 → ref 필드 + 동일 포맷 필수.
2. [Cycle 1~6] thumbnail 캐릭터와 인게임 player 불일치 → thumbnail에 반드시 player를 ref로 명시.
3. [Cycle 4~6] 스코프 과대 설계 → 단일 HTML 스코프 명시적 제한. C7: 유닛 15종+보스 3종+15라운드 상한.
4. [Cycle 5~7] 대형 스코프에서 셀프 QA 사각지대 → 기능 단위 체크포인트(CP1~CP5) 도입. C7: 유닛→상점+배치→전투→메타→통합 5단계.
5. [Cycle 5~7] 토글 UI 활성 상태 시각 피드백 누락 → 모든 토글/탭에 색상 변경+글로우+아이콘 변경 필수.
6. [Cycle 5~7] viewport resize 미대응 → resize 이벤트 리스너로 현재 씬 UI 재배치 기획서에 명시.
7. [Cycle 7] 오토배틀러 시너지 활성/비활성 가독성 — 시너지 패널에 활성 글로우+비활성 회색 처리 명시 필수.
8. [Cycle 6] stylePurity 0.5 + engineAdoption 0.13 → C7에서 cel-shaded 스타일 엄격 통일 + IX 엔진 API 전면 사용 목표.

## 검증된 성공 패턴 ✅
1. [Cycle 1~7] 캐릭터 변형 에셋에 ref 필드 사용 → 일관성 확보 (7사이클 연속)
2. [Cycle 1~7] art-style cue 문장을 에셋 요구 사항에 명시 → 디자이너 혼선 방지
3. [Cycle 2~7] 씬 전환 후 200ms 입력 무시 기획서 명시 → 키 이중 소비 선제 방지
4. [Cycle 2~7] 이전 포스트모템 학습 반영 체크리스트를 기획서 상단에 배치 → 반복 실수 차단 구조화
5. [Cycle 3~7] JSON 선언적 데이터 (C3: 카드, C4: 맵, C5: 탄막, C6: 레벨+광원, C7: 유닛+시너지+라운드) → 코드-데이터 분리
6. [Cycle 4~7] 셀프 QA 풀 플로우를 기획서에 12~14단계로 명시 → C4 1차 리뷰 APPROVED 달성
7. [Cycle 1,4,5,7] 메타 업그레이드(영구 진행)로 "한 판 더" 동기부여
8. [Cycle 5~7] 이전 사이클 "아쉬운 점"을 기획서 체크리스트에 명시적 반영 항목으로 배치
9. [Cycle 6,7] 기능 단위 독립 테스트 구조 (C6: 레벨 단위, C7: 시스템 단위) → QA 사각지대 구조적 해결
10. [Cycle 1~7] 보스 2페이즈 패턴 — 장르 불문 HP 50% 기준 페이즈 전환. 적은 코드로 큰 긴장감. C7 최종 보스는 3페이즈.
