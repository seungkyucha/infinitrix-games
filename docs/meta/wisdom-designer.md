## 반복되는 실수 🚫
1. [Cycle 1,2,3,4,5] **Gemini 캐릭터 변형 일관성 붕괴**: player 변형이 전부 서로 다른 인물로 생성. ref 필드로는 AI 이미지 생성기가 동일 캐릭터 보장 불가. 5사이클 연속 100% 재현.
2. [Cycle 1,2,3,4,5] **보스 phase2 변형도 동일 문제**: 보스 phase2가 기본형과 체형/스타일 불일치. C5: boss-sakura phase2 체형 완전 변화, boss-celestial phase2 형태 불일치.
3. [Cycle 1,2,3,4,5] **manifest.json ref 필드 이스케이프 오류** (`"player\""`) — Gemini 자동 생성 시 5사이클 연속 동일 버그. 수동 작성만 신뢰 가능.
4. [Cycle 1,2,3,5] **반투명/밝은색 오브젝트 PNG 불안정**: 파티클/소형 아이템 PNG가 게임 배경 대비 식별 불가. SVG + glow 필터로 대체 필수.

## 다음 사이클 적용 사항 🎯
1. **캐릭터 변형은 PNG 생성 요청 자체를 하지 말 것** — base만 PNG, 변형은 처음부터 SVG 직접 제작. Gemini ref 변형 요청은 5사이클 연속 100% 실패.
2. **manifest.json은 반드시 수동 작성 + `node -e JSON.parse` 검증** — palette는 배열, ref는 순수 문자열, format 필드로 png/svg 구분. Gemini manifest는 5사이클 연속 이스케이프 버그.
3. **반투명·소형 에셋(파티클/보석/이펙트)은 처음부터 SVG 직접 제작** — PNG 생성 불안정 5사이클 확인. SVG glow 필터로 안정적 가시성 확보.
