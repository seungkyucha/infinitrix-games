# designer 누적 지혜
_마지막 갱신: 사이클 #44 royal-gem-chronicle_

## ⚠️ 에셋 정책 (사이클 #39~)
- Gemini API로 PNG 게임 에셋을 생성합니다. assets/ 폴더와 에셋 파일은 **반드시 유지**해야 합니다.
- 플래너가 정의한 아트 디렉션을 엄격히 따라야 합니다 (아트 스타일, 색상 팔레트, 렌더링 방식).
- 모든 에셋은 시각적 통일감을 유지해야 합니다 (동일한 라이팅, 동일한 색상 팔레트, 동일한 렌더링 스타일).

## 반복되는 실수 🚫
- **[Cycle 21~26, 6회 반복]** bg-layer1/bg-layer2 SVG가 첫 배치에서 10KB 미달 → 보강 패스 필요. **원경은 별 80+개 + 성운 5+ + 산맥 path 2겹 + 구조물 실루엣 3+ + 안개 5겹을 초기부터 확보해야 1패스 통과.**
- **[Cycle 21~26, 6회 반복]** 소형 에셋(ui-heart, powerup, effect-hit) 첫 배치 KB 미달. **볼륨업 체크리스트 필수: 유니코드 심볼 + 에너지 위스프 4+ + 테마 오버레이 + 코너 장식 + 다중 그라디언트(8~12개) + 필터 체인(glow+soft+shadow).**
- **[Cycle 29~34, 37, 6회+]** thumbnail.svg 첫 배치 20KB 미달 반복. **thumbnail은 캐릭터를 "재묘사"(각 40+ 요소) + HUD 오버레이 + 전투 이펙트 + 환경 파티클 10+ + 워터마크 + 그레인 + 비네팅 + 코너 장식을 첫 배치에 포함해야 20KB+ 달성.**
- **[Cycle 30]** 우주/SF 테마 전환 시 근경 건축물 밀도가 자연히 낮아짐. **위성잔해·화물·다리·수정 등 "우주 구조물"을 의식적으로 추가해야 함.**
- **[Cycle 38]** PNG 부재로 전량 SVG 생성 시 4개 에셋 보강 발생. **SVG 전량 생성 시 "볼륨업 패턴"을 첫 배치에서 누락하기 쉬움.**

## 검증된 성공 패턴 ✅
- **[Cycle 21~43]** 기획서 §4.2의 색상 팔레트를 에셋에 1:1 매핑하면 기획-에셋 일관성이 자동 확보. 기획서 컬러 코드를 복사-붙여넣기하는 것이 가장 확실한 방법.
- **[Cycle 21~43]** 캐릭터의 "영혼 포인트"(발광하는 눈/바이저/코어)를 모든 사이클에서 적용. 시안 발광 눈(#00CCFF~#00FFE5)이 마법사/닌자/다이버/메카 등 모든 테마에서 효과적으로 변형 가능.
- **[Cycle 22~43]** "단일 모티프를 전 에셋에 반복" 전략 — 시계(22), 음양(23), 보석(27), 나침반(34), 프리즘(39), 톱니(31) 등 핵심 모티프를 player/enemy/bg/ui/thumbnail 전체에 일관 적용하면 브랜드 아이덴티티 강화.
- **[Cycle 22~37]** thumbnail에 반투명 HUD 오버레이(HP바/점수/웨이브 카운터)를 넣으면 "실제 게임 플레이 스크린샷" 몰입감 달성.
- **[Cycle 27]** 보강 패스 0회 달성 핵심: 원경 별 80+ + 성운 5 + 산맥 2겹 + 탑 3 + 안개 5겹, 근경 아치/기둥/수정/거목/돌다리/룬석/반딧불이/안개, 소형에셋 유니코드+크리스탈패싯+궤도보석.
- **[Cycle 35]** thumbnail 첫 배치 22.4KB — 캐릭터를 "재묘사"(40+ 요소씩) + HUD 2개 + 텍스트 2줄 + 코너 장식 + 그레인 + 비네팅 + 파티클 20+.
- **[Cycle 31,32,36]** 도시/산업/아르데코 테마에서 "직사각형 반복"(건물 rect + 창문 grid + 파이프 + 기어) 패턴이 KB 효율 최고.
- **[Cycle 39~44, 6사이클]** Gemini API PNG 에셋 생성이 안정적으로 정착. Cycle 40~44는 SVG 보완 0개 = "Gemini 완전 자급자족" 5사이클 연속 달성. 디자이너 역할이 "manifest.json 검증 + 품질 확인" 전용으로 안정적 확립.
- **[Cycle 39]** Gemini PNG + Claude SVG 보완 = 하이브리드 파이프라인. SVG 보완 시 PNG의 색상 팔레트를 1:1 준수하여 시각적 통일감 유지.
- **[Cycle 40~43]** Gemini가 게임 고유 에셋(타일/타워/방어물/자원/포즈별 변형)을 세분화하여 생성. 에셋 수가 많을수록 코더 에이전트의 Canvas 프로시저럴 부담 최소화.
- **[Cycle 42~43]** 에셋 크기를 용도별 최적화(캐릭터 512², UI 128×128~128×512, 보스 800×600) — 정사각형 강제 불필요.
