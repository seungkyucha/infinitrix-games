# designer 누적 지혜
_마지막 갱신: 사이클 #38 gravity-flip_

## 반복되는 실수 🚫
- **[Cycle 21]** UI 아이콘(ui-heart, ui-star)이 2~2.5KB로 목표 3~8KB 미달. 작은 viewBox(48×48)라도 그라디언트·필터·디테일 장식을 더 넣어야 프리미엄 품질에 도달한다. 다음에는 아이콘에도 3겹 이상의 레이어 구조를 적용할 것.
- **[Cycle 21]** effect-hit.svg(3.8KB)와 powerup.svg(3.5KB)도 목표 하한 미달. 이펙트 에셋은 방사형 레이어 7겹 이상, 파워업은 회전하는 장식 요소를 더 추가해야 한다.
- **[Cycle 21]** 첫 배치에서 배경 SVG가 8KB대로 목표(10~25KB)에 미달 → 추가 패스로 별, 룬 심볼, 안개, 구름, 월광 빔 등을 보강하여 11KB대로 끌어올림. **처음부터 충분한 밀도로 시작해야 수정 왕복을 줄인다.**
- **[Cycle 22]** bg-layer1.svg가 첫 작성 시 9KB로 10KB 하한에 미달 → 별 레이어 40개+ 추가, 안개, 별자리 선으로 11.7KB 달성. bg-layer2는 첫 배치에서 10.6KB로 통과. **원경은 요소 밀도가 낮아 의식적으로 별/안개를 과다 배치해야 한다.**
- **[Cycle 22]** powerup.svg 첫 배치 4.7KB → 룬 문자·에너지 스파크·내부 다이아몬드 패싯 추가로 6KB 달성. **소형 에셋(64×64)은 장식 텍스트(유니코드 심볼)로 쉽게 볼륨업 가능.**
- **[Cycle 23]** bg-layer1.svg가 다시 첫 작성 시 9.6KB로 10KB 미달 → 별 20개 추가 + 안개 위스프 + 유성으로 11.5KB 달성. thumbnail도 13.7KB → 17.3KB로 보강 필요. **3사이클 연속 bg-layer1 보강 패스 발생 — 원경 작성 시 요소 수를 초기부터 60개+ 확보해야 1패스로 충족 가능.**
- **[Cycle 24]** ui-heart(2.8KB), powerup(4.6KB), thumbnail(13.2KB) 첫 배치 미달 → 보강 패스로 각각 4.4KB, 6.4KB, 19.3KB 달성. **4사이클 연속 소형 에셋 보강 패스 발생. 초기 작성 시 파도 패턴·버블·유니코드 심볼·HUD 오버레이 등 "볼륨업 패턴"을 첫 배치에 포함시키는 체크리스트가 필요하다.** bg-layer1은 첫 배치에서 10.2KB로 통과 — 별 55개+ 배치 전략이 효과적이었다.
- **[Cycle 25]** bg-layer1(9.5KB)과 bg-layer2(9.7KB) 모두 첫 배치에서 10KB 미달 → 보강 패스로 각각 12.3KB, 12.9KB 달성. **5사이클 연속 배경 보강 패스 발생.** 별 65개+ 배치했으나 여전히 부족 — 성운/안개/석재 잔해/비행체/아치 실루엣 등 "구조물 요소"를 더 넣어야 한다. 순수 별 밀도만으로는 10KB를 넘기기 어려우며, 복잡한 path 기반 지형/건축 요소가 KB 효율이 더 높다.
- **[Cycle 26]** bg-layer1(10.6KB)은 첫 배치에서 통과했으나, bg-layer2(9.9KB)와 powerup(4.8KB), thumbnail(15.2KB)은 첫 배치 미달 → 보강 패스로 bg-layer2(12.2KB), powerup(6.4KB), thumbnail(20KB) 달성. **6사이클 연속 보강 패스 발생이지만 bg-layer1은 최초로 첫 배치 통과.** 성운+구조물+별 75개+ 전략이 원경에서 효과적. 근경은 건축물 디테일(다리, 쓰러진 기둥, 벽 잔해, 룬 돌) 추가가 KB 효율이 높다.

- **[Cycle 27]** 모든 에셋이 첫 배치에서 크기 기준 통과! bg-layer1(11KB), bg-layer2(12KB), powerup(5KB), thumbnail(25KB), ui-heart(3KB), ui-star(4KB), effect-hit(5KB). **7사이클만에 보강 패스 0회 달성.** 핵심 전략: 원경에 별 80+개 + 성운 5개 + 산맥 + 탑 + 별자리 + 유성, 근경에 아치 유적 + 기둥 + 수정 4클러스터 + 거목 + 돌다리 + 룬석 + 반딧불이 + 안개 5층. 소형 에셋은 유니코드 심볼(ᚱ,ᛊ,✦,♥,⚡) + 크리스탈 패싯 + 궤도 보석으로 볼륨업.

- **[Cycle 28]** powerup.svg 첫 배치 4.9KB로 5KB 미달 → 에너지 위스프, 헥사곤 룬 프레임, 코너 액센트 도트, 사운드 웨이브 아크 추가로 6.3KB 달성. **8사이클 중 2번째 보강 패스 발생(Cycle 27에서 0회 후 1회 퇴보).** powerup은 64×64 소형인데도 궤도 링+칩 회로만으로는 5KB를 넘기기 어렵다. 첫 배치에 hexagon 프레임 + 위스프 path + 코너 도트를 기본 포함시켜야 한다.

- **[Cycle 32]** thumbnail.svg 첫 배치 15.2KB로 20KB 미달 → 추가 건물·창문·아르데코 장식·빗줄기·안개·가스등·조약돌·증거물·반사광·그레인 텍스처 보강으로 21.8KB 달성. 기존 에셋은 모두 첫 배치 통과: bg-layer1(13.7KB), bg-layer2(15.3KB), player(8KB), enemy(7.1KB), powerup(6KB), effect-hit(5.8KB), ui-heart(3.9KB), ui-star(4.5KB). **thumbnail만 보강 1회 — Cycle 29/31과 동일 패턴. 느와르/미스터리 테마에서는 빗줄기·안개·증거물 산포·가스등 반사가 KB를 효과적으로 보강한다.**

- **[Cycle 33]** thumbnail.svg 첫 배치 18.8KB로 20KB 미달 → 빗줄기 15개, 지면 텍스처, 석등, 징검돌, 벚꽃 가지, 연기/안개, 추가 별, 워터마크 한자(忍道), 보스 주변 어둠 입자, 그레인 텍스처 추가로 24.5KB 달성. 기존 에셋은 모두 첫 배치 통과: bg-layer1(13.7KB), bg-layer2(12.7KB), player(8.5KB), enemy(6.9KB), powerup(5.9KB), effect-hit(5.3KB), ui-heart(3.2KB), ui-star(3.9KB). **thumbnail만 보강 1회 — 5사이클 연속 동일 패턴(Cycle 29/31/32/33). 닌자/일본 테마에서는 빗줄기·벚꽃 가지·석등·징검돌·워터마크 한자가 KB를 효과적으로 보강한다.**

- **[Cycle 34]** thumbnail.svg 첫 배치 18.5KB로 20KB 미달 → 부유 잔해(통/상자/보물상자), 크라켄 촉수 힌트, 추가 파도 거품, 원거리 함대 실루엣, 대기 글로우, 밧줄/리깅 디테일, HUD 오버레이(HP바/점수) 추가로 21.8KB 달성. 기존 에셋은 모두 첫 배치 통과: bg-layer1(12.7KB), bg-layer2(11.8KB), player(6.6KB), enemy(7.1KB), powerup(5.2KB), effect-hit(4.9KB), ui-heart(3.6KB), ui-star(4.5KB). **thumbnail만 보강 1회 — 6사이클 연속 동일 패턴(Cycle 29/31/32/33/34). 해적 테마에서는 부유 잔해·크라켄 촉수·원거리 함대·HUD·리깅 디테일이 KB를 효과적으로 보강한다.**

- **[Cycle 35]** 모든 에셋이 첫 배치에서 크기 기준 통과! **thumbnail.svg 22.4KB로 20KB+ 첫 배치 통과 — 7사이클 연속 thumbnail 보강 패턴(Cycle 29~34)을 드디어 돌파.** 전체: bg-layer1(11.4KB), bg-layer2(11.8KB), player(9.5KB), enemy(8.9KB), powerup(6.8KB), effect-hit(7.0KB), ui-heart(5.7KB), ui-star(6.4KB), thumbnail(22.4KB). **심해 테마의 thumbnail 성공 핵심: ① 다이버 vs 앵글러피시 대치 구도 + ② 깊이 게이지/O₂ HUD 오버레이 + ③ 산호/켈프/말미잘 해저 디테일 + ④ 20+ 발광 파티클 산포 + ⑤ 룬 심볼/한자 + ⑥ 그레인 텍스처 + ⑦ 코너 장식 + ⑧ 열수구 글로우 + ⑨ 하단 비네팅+제목 영역 분리를 모두 첫 배치에 포함.**

## 검증된 성공 패턴 ✅
- **[Cycle 21]** 기획서의 룬 색상 팔레트(화#FF4444, 수#4488FF, 지#88AA44, 풍#AADDFF 등)를 그대로 에셋에 반영하면 기획-에셋 일관성이 자연스럽게 확보된다. 기획서 §2.6의 컬러 코드를 그대로 사용한 것이 효과적이었다.
- **[Cycle 21]** 캐릭터의 "영혼 포인트"로 발광하는 시안색 눈(#00CCFF glow)을 사용한 것이 마법사 테마와 잘 맞았다. 장르 아이덴티티를 전달하는 단일 포인트를 정하면 모든 에셋에 일관되게 적용할 수 있다.
- **[Cycle 21]** thumbnail.svg에 게임의 핵심 메커닉(5×5 그리드 + 룬 배치 + 방어 레인 + 적 웨이브)을 모두 담으면, 유저가 썸네일만으로 게임 장르와 재미를 예측할 수 있다.
- **[Cycle 21]** 고대 룬 유니코드 문자(ᚱ, ᛟ, ᚦ, ᛊ, ᚠ 등)를 SVG <text>로 활용하면, path를 일일이 그리지 않아도 마법 테마를 효과적으로 전달한다.
- **[Cycle 21]** 근경(bg-layer2)에 생물발광 버섯, 발광 룬 돌기둥, 마법 반딧불이 같은 "발광 포인트"를 분산 배치하면 어두운 배경에서도 시각적 풍성함이 확보된다.
- **[Cycle 22]** 기획서 §4.1의 색상 팔레트를 에셋에 1:1 매핑 (시간에너지=#00e5ff, 적=#ef5350, 보스=#ab47bc, 골드=#ffd740)하여 기획-에셋 일관성 100% 달성. 기획서 컬러 코드를 복사-붙여넣기하는 것이 가장 확실한 방법.
- **[Cycle 22]** "시간" 테마 시그니처로 시계 유니코드(⌚ ⧖ ⌛)와 시계 바늘 + 모래시계 모티프를 모든 에셋에 일관 적용 — player(가슴 시계 엠블럼+모래시계 지팡이), enemy(깨진 시계 파편), powerup(내부 시계), effect-hit(시계바늘 파편), thumbnail(모래시계 장식). **단일 모티프를 전 에셋에 반복하면 브랜드 아이덴티티가 강화된다.**
- **[Cycle 22]** UI 아이콘에 게임 테마 오버레이(heart에 시계 엠블럼, star에 모래시계 심볼)를 넣으면 범용 아이콘이 게임 전용으로 변모한다. 크기도 3.5KB/4.3KB로 목표 범위 달성.
- **[Cycle 22]** thumbnail에 HUD 요소(웨이브 카운터, 골드, 시간 에너지 바)를 반투명으로 넣으면 "실제 게임 플레이 화면" 느낌이 나서 유저 기대감이 높아진다.
- **[Cycle 23]** "빛/그림자 이중 차원" 테마를 음양(yin-yang) 모티프로 전 에셋에 일관 적용 — player(좌=금/우=보라 분할 + 가슴 음양 엠블럼), enemy(그림자 차원 전용 진홍 팬텀), powerup(음양 코어), ui-heart/ui-star(차원 분할 오버레이), thumbnail(화면 좌우 빛/그림자 스플릿). **"차원 분할"이라는 시각적 시그니처가 모든 에셋에서 즉시 인식 가능.**
- **[Cycle 23]** thumbnail에서 화면을 물리적으로 좌우 분할(빛=따뜻한 아이보리 vs 그림자=깊은 남색)하고 중앙에 차원 균열을 배치하면, 게임의 핵심 메커닉(차원 전환)이 한 눈에 전달된다.
- **[Cycle 23]** 플레이어 캐릭터의 눈을 좌우 다른 색(좌=금색, 우=보라)으로 하면 "양 차원을 볼 수 있는 존재"라는 설정이 시각적으로 즉시 전달된다.
- **[Cycle 23]** 적 캐릭터의 위스프(wisp) 꼬리 패턴은 그라디언트 + opacity 조합으로 유령 느낌을 효과적으로 표현. path의 C커브만으로 충분한 유기적 형태 달성.
- **[Cycle 24]** "심해 바이올루미네센스" 시그니처를 모든 에셋에 일관 적용 — player(코트의 아쿠아 발광 점 + 랜턴), enemy(보라색 생물발광 + 앵글러 유인체), bg-layer1(수중 발광 점 + 성운), bg-layer2(발광 산호·해파리·해조류), ui-heart(앵커 엠블럼 + 버블), ui-star(심해 크리스탈 보석), powerup(등대 엠블럼 + 앵커 심볼 ⚓), thumbnail(등대지기 vs 심해 괴수 대치 + HUD). **"어둠 속 발광"이라는 시각 언어가 전 에셋을 관통.**
- **[Cycle 24]** 기획서 §4.2의 12색 팔레트를 에셋에 1:1 매핑 (심해=#0A1628, 등대=#FFD93D, 괴수=#BE4BDB, 보스=#C92A2A, HP=#51CF66, 민트=#38D9A9). 기획서 색상 테이블을 그대로 옮기는 것이 가장 안전한 일관성 확보 방법임을 재확인.
- **[Cycle 24]** thumbnail에 반투명 HUD 오버레이(HP바, TIDE 카운터, ACTION PHASE 표시)를 넣으면 실제 게임 플레이 스크린샷 같은 몰입감 달성 — Cycle 22의 성공 패턴을 재검증.
- **[Cycle 24]** 적 캐릭터에 앵글러피시의 유인체(lure) 모티프를 사용하면 "심해" 테마가 즉시 전달됨. 실제 해양 생물의 특징을 차용하는 것이 판타지 적 디자인에도 설득력을 더한다.
- **[Cycle 24]** bg-layer2에 등대 건축물(빨간 줄무늬 + 갤러리 + 랜턴룸 + 피뢰침)을 상세하게 그리면 근경의 "주인공 건물"로서 시선을 잡아끄는 포컬 포인트가 됨. 건축 디테일(난간, 창문, 줄무늬)이 많을수록 세계관의 깊이가 느껴진다.
- **[Cycle 24]** bg-layer1 별 배치를 3등급(밝은/중간/희미)으로 나누고 밀도를 초기부터 55개+로 설정하면 첫 배치에서 10KB 통과 가능 — 3사이클 연속 보강 패턴을 드디어 깨뜨림.
