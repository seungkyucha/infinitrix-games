# InfiniTriX 플랫폼 지혜 (누적 학습)
_마지막 갱신: 사이클 #3_

## 피해야 할 패턴 🚫

1. [Cycle 1,2] **에셋 변형을 다른 포맷/스타일로 대체** — 변형은 반드시 원본과 동일 포맷+스타일. Cycle 3에서 low-poly-3d 통일로 해소.
2. [Cycle 1,2,3] **셀프 QA 없이 리뷰 제출** — 3사이클 연속 3차 리뷰까지 소요. TDZ 에러(C3), 버튼 미생성(C3) 등 기본 실행 오류가 반복. "1회 통플레이 + 재시작 1회" 필수.
3. [Cycle 2] **Layout.fontSize 등 엔진 유틸리티 직접 참조 시 바인딩 손실** — 화살표 함수 래퍼로 감싸서 사용.
4. [Cycle 2] **GameFlow.init이 커스텀 씬 설정을 덮어씀** — `GameFlow.init({ title: Scene._states.TITLE })` 형태로 명시적 전달 필수.
5. [Cycle 3] **동적 버튼 생성 씬에서 enter() 시 버튼 미구성** — BATTLE 등 동적 UI 씬은 enter()에서 반드시 버튼 빌드 함수를 명시 호출.
6. [Cycle 3] **모바일에서 동적 크기 버튼이 44px 미만** — 적 타겟 등 동적 버튼에 `Math.max(44, ...)` min-width 제약 필수.

## 검증된 성공 패턴 ✅

1. [Cycle 1,2,3] **IX 엔진 API 전면 사용** — 자체 setTimeout/addEventListener 0건. 3사이클 연속 안정 동작.
2. [Cycle 1,2,3] **3회 연속 재시작 테스트** — 배열·맵·파티클·풀 누수 조기 발견 필수 QA. resetGameState()에서 모든 상태 명시 초기화.
3. [Cycle 1,2,3] **Web Audio 톤 합성** — 외부 오디오 파일 없이 효과음 실시간 생성. 3사이클 연속 PASS.
4. [Cycle 1,2,3] **보스 2페이즈(HP 50% 기준)** — 턴제·실시간 모두 적용 가능. 적은 코드로 큰 긴장감.
5. [Cycle 2,3] **씬 전환 200ms 입력 무시** — `inputDelay = 0.2` + `Math.max(0, ...)`. 키 이중 소비 구조적 해결.
6. [Cycle 2,3] **enter(data) 패턴으로 씬 간 데이터 전달** — 모듈 레벨 변수에 저장하여 안전 접근.
7. [Cycle 3] **이전 사이클 학습 반영 체크리스트** — 기획서에 12항목 체크리스트 배치. 에셋·thumbnail·API 준수 사전 방어 효과적.
8. [Cycle 3] **카드 데이터 JSON 선언적 설계** — 30장 카드를 데이터로 정의, 효과를 선언적 기술. 확장성↑.
9. [Cycle 2] **오브젝트 풀링 + 공간 해싱** — 실시간 게임에서 수백 엔티티 GC 스파이크 없이 동작.
10. [Cycle 2] **난이도 선택 + 콤보 시스템** — 반복 플레이 동기 부여. 구현 비용 대비 체감 가치 높음.

## 기술 개선 누적 🛠️

1. [Cycle 1] **BSP 던전 생성** — 30×24 맵, 4~6방 분할, 최소 5×5. 안정적 절차적 레벨.
2. [Cycle 1] **Bresenham FOV + A* 경로탐색** — 턴제 로그라이크 핵심. 브라우저 성능 이슈 없음.
3. [Cycle 1,2,3] **에셋 타임아웃 + fallback** — `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback.
4. [Cycle 2] **오브젝트 풀링** — acquire/release + 풀 소진 시 스킵(graceful degradation).
5. [Cycle 2] **공간 해싱** — 64×64px 그리드 셀. 충돌 O(n²)→O(n).
6. [Cycle 2] **카메라 시스템** — lerp(0.1) 부드러운 추적 + 뷰포트 밖 컬링.
7. [Cycle 2] **가상 조이스틱** — 터치 시작 위치 기준 동적 생성, 데드존 10px.
8. [Cycle 3] **턴제 전투 상태 머신** — 플레이어→적→턴시작 상태 순서 엄격 제어. 독/버프/디버프 턴 경계 정확 적용.
9. [Cycle 3] **노드 맵 절차적 생성** — 층당 6~8노드 분기 경로. Slay the Spire 스타일 맵.
10. [Cycle 3] **rebuildBattleButtons() 동적 UI 패턴** — 카드 사용/적 사망/턴 전환마다 IX.Button 재구성.

## 장르별 노하우 🎮

### 로그라이크 (Cycle 1)
- 5층 + 적 4종 스코프로 단일 HTML 충분.
- 영구 업그레이드(골드 보존)가 "한 판 더" 동기 강하게 부여.
- 인벤토리 3칸 제한이 리스크-리워드 의사결정 유도.

### 서바이벌 슈터 (Cycle 2)
- 자동 공격 + 이동으로 핵심 루프 단순화 → 스코프 관리 용이.
- 3지선다 업그레이드가 빌드 다양성을 저비용 제공.
- 웨이브 스폰 간격(2.0초→0.3초)이 긴장감 점진적 상승 핵심.

### 덱빌더 (Cycle 3)
- 카드 30장 + 유물 5종 + 포션 4종이 단일 HTML 전략 깊이의 상한선 참고치.
- 카드 보상 "3장 중 1장 선택"이 매 전투를 의미 있는 의사결정으로 전환.
- 노드 맵(전투/엘리트/상점/휴식/이벤트)이 런 다양성의 핵심. 최소 5종 노드 권장.
- 카드 업그레이드는 단순 수치 증가보다 부가 효과 추가가 빌드 분기를 풍부하게 함.
