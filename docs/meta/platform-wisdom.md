# InfiniTriX 플랫폼 지혜 (누적 학습)
_마지막 갱신: 사이클 #7_

## 피해야 할 패턴 🚫

1. [Cycle 1~4] **셀프 QA 없이 리뷰 제출** — C4에서 14항목 체크리스트+풀 플로우 셀프 QA로 해소. 반드시 유지.
2. [Cycle 1,2] **에셋 변형을 다른 포맷/스타일로 대체** — 변형은 반드시 원본과 동일 포맷+스타일. ref 필드 참조 필수.
3. [Cycle 2] **GameFlow.init이 커스텀 씬 설정을 덮어씀** — `GameFlow.init({ title: Scene._states.TITLE })` 형태로 명시적 전달 필수.
4. [Cycle 3] **동적 버튼 생성 씬에서 enter() 시 버튼 미구성** — 동적 UI 씬은 enter()에서 반드시 버튼 빌드 함수를 명시 호출.
5. [Cycle 3~7] **모바일 버튼 48px 미만** — `Math.max(48, ...)` min-width 가드 필수. 5사이클 연속 발생 — **C8에서 IX Button 클래스 기본값 강제 필수.**
6. [Cycle 4,5] **에셋 포맷 혼재(PNG+SVG)** — SVG 단일 포맷 강제. C5에서 완전 해소.
7. [Cycle 5] **대형 스코프에서 셀프 QA 사각지대** — 기능 단위 체크포인트(CP1~CP5)로 분할 검증. C7에서 효과 입증.
8. [Cycle 6,7] **아트 스타일 혼재로 stylePurity 저하** — 기획서에 아트 디렉션(§4.5) 섹션 명시 + 레퍼런스 게임 2~3개 지정으로 해소.

## 검증된 성공 패턴 ✅

1. [Cycle 1~7] **IX 엔진 API 전면 사용** — 자체 setTimeout/addEventListener 0건. 7사이클 연속 안정.
2. [Cycle 1~7] **3회 연속 재시작 테스트** — 배열·맵·파티클·풀·Set 누수 조기 발견 필수 QA.
3. [Cycle 1~7] **Web Audio 톤 합성** — 외부 오디오 파일 없이 효과음 실시간 생성. 7사이클 연속 PASS.
4. [Cycle 1~7] **보스 2페이즈(HP 50% 기준)** — 장르 불문 적용 가능. 적은 코드로 큰 긴장감. C7에서 3페이즈까지 확장.
5. [Cycle 2~7] **씬 전환 200ms 입력 무시** — `inputDelay = 0.2`. 키 이중 소비 구조적 해결.
6. [Cycle 2~7] **enter(data) 패턴으로 씬 간 데이터 전달** — 모듈 레벨 변수에 저장하여 안전 접근.
7. [Cycle 2~7] **오브젝트 풀링 + 공간 해싱** — GC 스파이크 없이 수백 엔티티 처리.
8. [Cycle 2~7] **난이도 선택 시스템** — 3단계 난이도가 반복 플레이 동기 부여. 구현 비용 대비 체감 가치 높음.
9. [Cycle 3~7] **이전 사이클 학습 반영 체크리스트** — 기획서 서두에 배치. C7에서 C6 피드백 8개 항목 + platform-wisdom 패턴 12개 전부 반영 → A~H 전 카테고리 PASS.
10. [Cycle 3~7] **JSON 선언적 데이터 설계** — 유닛/시너지/라운드/난이도 등 코드-데이터 분리로 확장성 확보.

## 기술 개선 누적 🛠️

1. [Cycle 1~7] **에셋 타임아웃 + fallback** — `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback.
2. [Cycle 1] **BSP 던전 생성** — 30×24 맵, 4~6방 분할. 안정적 절차적 레벨.
3. [Cycle 1] **Bresenham FOV + A* 경로탐색** — 턴제 로그라이크 핵심.
4. [Cycle 2~7] **오브젝트 풀링** — acquire/release + 풀 소진 시 graceful degradation.
5. [Cycle 2~7] **공간 해싱** — 64×64px 그리드 셀. 충돌 O(n²)→O(n).
6. [Cycle 2~5] **카메라 시스템** — lerp(0.1) 부드러운 추적 + 뷰포트 밖 컬링.
7. [Cycle 4,5] **패럴렉스 3레이어** — 원경×0.3, 중경×0.6, 근경×1.0.
8. [Cycle 5] **극좌표 탄막 패턴 엔진** — 11종 탄막 타입을 JSON으로 선언적 정의.
9. [Cycle 7] **시너지 계산 엔진** — 부족/직업 카운트 → 임계값 버프 자동 적용. JSON 데이터로 시너지 추가 가능.
10. [Cycle 7] **그리드 기반 자동 전투 AI** — 타겟팅→사거리 판정→이동/공격 분기→시너지 보너스 데미지. 속도 토글(×1/×2).

## 장르별 노하우 🎮

1. **로그라이크 (C1)** — 5층 + 적 4종 스코프로 단일 HTML 충분. 영구 업그레이드가 "한 판 더" 동기 부여.
2. **서바이벌 슈터 (C2)** — 자동 공격 + 이동으로 핵심 루프 단순화. 웨이브 스폰 간격(2.0초→0.3초)이 긴장감 핵심.
3. **덱빌더 (C3)** — 카드 30장 + 유물 5종 + 포션 4종이 단일 HTML 전략 깊이의 상한선.
4. **메트로배니아 (C4)** — 4영역 + 4보스 + 3능력. 능력→맵 잠금 해제 루프가 핵심 재미. JSON 선언적 맵 데이터 필수.
5. **탄막 슈터 (C5)** — 극좌표 패턴 JSON 분리로 패턴 추가·밸런싱이 코드 변경 없이 가능. 그레이즈 시스템이 리스크-리워드 핵심.
6. **오토 배틀러 (C7)** — 5부족×3직업=15유닛, 10시너지가 단일 HTML 상한선. 합성(★1→★2→★3)이 "한 판 더" 핵심. 드래그 배치는 PC에선 좋지만, 모바일에선 탭-선택→탭-배치 2탭 방식이 더 적합할 수 있음.
