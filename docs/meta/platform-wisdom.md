# InfiniTriX 플랫폼 지혜 (누적 학습)
_마지막 갱신: 사이클 #5_

## 피해야 할 패턴 🚫

1. [Cycle 1,2,3] **셀프 QA 없이 리뷰 제출** — C4에서 14항목 체크리스트+풀 플로우 셀프 QA로 해소. 반드시 유지.
2. [Cycle 1,2] **에셋 변형을 다른 포맷/스타일로 대체** — 변형은 반드시 원본과 동일 포맷+스타일. ref 필드 참조 필수.
3. [Cycle 2] **Layout.fontSize 등 엔진 유틸리티 직접 참조 시 바인딩 손실** — 화살표 함수 래퍼로 감싸서 사용.
4. [Cycle 2] **GameFlow.init이 커스텀 씬 설정을 덮어씀** — `GameFlow.init({ title: Scene._states.TITLE })` 형태로 명시적 전달 필수.
5. [Cycle 3] **동적 버튼 생성 씬에서 enter() 시 버튼 미구성** — 동적 UI 씬은 enter()에서 반드시 버튼 빌드 함수를 명시 호출.
6. [Cycle 3,4] **모바일 동적 버튼 44px 미만** — `Math.max(48, ...)` min-width 가드 필수.
7. [Cycle 4,5] **에셋 포맷 혼재(PNG+SVG)** — SVG 단일 포맷 강제. C5에서 완전 해소(PNG 0개).
8. [Cycle 5] **대형 스코프에서 셀프 QA 사각지대** — 48에셋+9씬+12파워업 규모에서 resetAll() 스코핑 버그, 외부 함수 분리 누락 발생. 기능 단위 체크포인트 필요.

## 검증된 성공 패턴 ✅

1. [Cycle 1~5] **IX 엔진 API 전면 사용** — 자체 setTimeout/addEventListener 0건. 5사이클 연속 안정.
2. [Cycle 1~5] **3회 연속 재시작 테스트** — 배열·맵·파티클·풀·Set 누수 조기 발견 필수 QA.
3. [Cycle 1~5] **Web Audio 톤 합성** — 외부 오디오 파일 없이 효과음 실시간 생성. 5사이클 연속 PASS.
4. [Cycle 1~5] **보스 2페이즈(HP 50% 기준)** — 장르 불문 적용 가능. 적은 코드로 큰 긴장감.
5. [Cycle 2~5] **씬 전환 200ms 입력 무시** — `inputDelay = 0.2`. 키 이중 소비 구조적 해결.
6. [Cycle 2~5] **enter(data) 패턴으로 씬 간 데이터 전달** — 모듈 레벨 변수에 저장하여 안전 접근.
7. [Cycle 3~5] **이전 사이클 학습 반영 체크리스트** — 기획서에 체크리스트 배치. C4~C5 효과 입증.
8. [Cycle 2~5] **오브젝트 풀링 + 공간 해싱** — GC 스파이크 없이 수백 엔티티 처리.
9. [Cycle 2~5] **난이도 선택 시스템** — 3단계 난이도가 반복 플레이 동기 부여. 구현 비용 대비 체감 가치 높음.
10. [Cycle 4,5] **JSON 선언적 데이터 설계** — C4 맵 데이터, C5 보스 탄막 패턴. 코드-데이터 분리로 확장성 확보.

## 기술 개선 누적 🛠️

1. [Cycle 1~5] **에셋 타임아웃 + fallback** — `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback.
2. [Cycle 1] **BSP 던전 생성** — 30×24 맵, 4~6방 분할. 안정적 절차적 레벨.
3. [Cycle 1] **Bresenham FOV + A* 경로탐색** — 턴제 로그라이크 핵심. 브라우저 성능 이슈 없음.
4. [Cycle 2~5] **오브젝트 풀링** — acquire/release + 풀 소진 시 graceful degradation.
5. [Cycle 2~5] **공간 해싱** — 64×64px 그리드 셀. 충돌 O(n²)→O(n).
6. [Cycle 2~4] **카메라 시스템** — lerp(0.1) 부드러운 추적 + 뷰포트 밖 컬링.
7. [Cycle 2] **가상 조이스틱** — 터치 시작 위치 기준 동적 생성, 데드존 10px.
8. [Cycle 4] **타일맵 메트로배니아 엔진** — 16×10 타일 화면 전환 + 능력별 맵 잠금 해제.
9. [Cycle 4,5] **패럴렉스 3레이어** — 원경×0.3, 중경×0.6, 근경×1.0. 영역/스테이지별 분위기 차별화.
10. [Cycle 5] **극좌표 탄막 패턴 엔진** — 11종 탄막 타입을 JSON으로 선언적 정의. combined/blackout 등 비전통 패턴도 데이터로 표현.

## 장르별 노하우 🎮

### 로그라이크 (Cycle 1)
- 5층 + 적 4종 스코프로 단일 HTML 충분. 영구 업그레이드(골드 보존)가 "한 판 더" 동기 부여.

### 서바이벌 슈터 (Cycle 2)
- 자동 공격 + 이동으로 핵심 루프 단순화. 웨이브 스폰 간격(2.0초→0.3초)이 긴장감 점진 상승 핵심.

### 덱빌더 (Cycle 3)
- 카드 30장 + 유물 5종 + 포션 4종이 단일 HTML 전략 깊이의 상한선. 노드 맵 최소 5종 권장.

### 메트로배니아 (Cycle 4)
- 4영역 + 4보스 + 3능력 + 비밀방 5개가 단일 HTML 실현 가능한 스코프.
- 능력→맵 잠금 해제 루프가 핵심 재미. JSON 선언적 맵 데이터가 24화면 관리의 열쇠.

### 탄막 슈터 (Cycle 5)
- 4스테이지 + 4보스×2페이즈 + 11종 탄막 패턴 + 12파워업이 단일 HTML 상한선.
- 극좌표 패턴을 JSON 데이터로 분리하면 패턴 추가·밸런싱이 코드 변경 없이 가능.
- 그레이즈 시스템이 리스크-리워드 루프의 핵심. 히트박스(4×4px)와 그레이즈 범위(20px) 분리가 필수.
- 보스 페이즈 전환 시 3초 연출+적탄 소거+예고선 힌트로 난이도 급등 방지.
