# InfiniTriX 플랫폼 지혜 (누적 학습)
_마지막 갱신: 사이클 #2_

## 피해야 할 패턴 🚫

1. [Cycle 1,2] **에셋 변형을 다른 포맷/스타일로 대체** — SVG 벡터가 PNG 픽셀아트와 이질감 유발. player-hurt, idle-sheet 등 변형은 반드시 원본과 동일 포맷+스타일.
2. [Cycle 1,2] **thumbnail 미생성/불일치** — Cycle 1은 캐릭터 불일치, Cycle 2는 아예 누락. 에셋 파이프라인에서 thumbnail을 필수 체크리스트로 강제.
3. [Cycle 1] **피드백 문서 없이 재리뷰 진행** — 지적사항이 수정되지 않은 채 다음 리뷰 도달. docs/feedback/ 작성 의무화.
4. [Cycle 1] **버튼 높이 하드코딩** — 임의 값 대신 항상 `Math.max(44, ...)` 이상으로 최소 터치 영역 보장. Cycle 2에서 48px 전면 적용으로 해소.
5. [Cycle 2] **Layout.fontSize 등 엔진 유틸리티 직접 참조 시 바인딩 손실** — 화살표 함수 래퍼로 감싸거나 변수에 바인드해서 사용할 것.
6. [Cycle 2] **GameFlow.init이 커스텀 씬 설정을 덮어씀** — `GameFlow.init({ title: Scene._states.TITLE })` 형태로 커스텀 씬을 명시적으로 전달 필수.

## 검증된 성공 패턴 ✅

1. [Cycle 1,2] **IX 엔진 API 전면 사용** — 자체 setTimeout/addEventListener 0건. 2사이클 연속 안정적 동작 검증.
2. [Cycle 1,2] **3회 연속 재시작 테스트** — 배열·맵·파티클·풀 누수를 조기 발견하는 필수 QA 항목. resetGameState()에서 모든 상태 명시적 초기화.
3. [Cycle 1,2] **Web Audio 톤 합성** — 외부 오디오 파일 없이 효과음 실시간 생성. 2사이클 연속 PASS.
4. [Cycle 1,2] **보스 2페이즈(HP 50% 기준)** — 턴제·실시간 모두 적용 가능. 적은 코드로 큰 긴장감 제공.
5. [Cycle 2] **오브젝트 풀링 + 공간 해싱** — 실시간 서바이벌에서 수백 엔티티 GC 스파이크 없이 동작. 탄환 200/적 100/보석 300 사전 할당.
6. [Cycle 2] **씬 전환 200ms 입력 무시** — Cycle 1의 키 이중 소비 문제를 구조적으로 해결. `inputDelay = 0.2` + `Math.max(0, ...)`.
7. [Cycle 2] **enter(data) 패턴으로 씬 간 데이터 전달** — 모듈 레벨 변수에 저장하여 내부 접근 없이 안전하게 사용.
8. [Cycle 1] **턴제 = 복잡도 관리의 핵심** — 렌더링 부담↓, 디버깅↑, QA 자동화 용이. 복잡한 장르는 턴제 우선 고려.
9. [Cycle 2] **deltaTime 기반 프레임 독립 로직** — 프레임 드랍 시에도 일관된 게임 속도. 실시간 게임 필수 패턴.
10. [Cycle 2] **난이도 선택 + 콤보 시스템** — 반복 플레이 동기 부여에 효과적. 구현 비용 대비 체감 가치 높음.

## 기술 개선 누적 🛠️

1. [Cycle 1] **BSP 던전 생성**: 30×24 맵에서 4~6방 분할, 최소 방 크기 5×5. 안정적 절차적 레벨.
2. [Cycle 1] **Bresenham FOV + A* 경로탐색**: 턴제 로그라이크 핵심 조합. 브라우저에서 성능 이슈 없음.
3. [Cycle 1] **에셋 타임아웃 + fallback**: `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback.
4. [Cycle 2] **오브젝트 풀링**: acquire/release 패턴. 풀 소진 시 스킵(새 생성 대신) — graceful degradation.
5. [Cycle 2] **공간 해싱**: 64×64px 그리드 셀. 충돌 판정 O(n²)→O(n).
6. [Cycle 2] **카메라 시스템**: lerp(0.1) 부드러운 추적 + 뷰포트 밖 컬링.
7. [Cycle 2] **가상 조이스틱**: 터치 시작 위치 기준 동적 생성, 데드존 10px, 힌트 영역 표시.

## 장르별 노하우 🎮

### 로그라이크 (Cycle 1)
- 5층 + 적 4종으로 스코프 제한하면 단일 HTML로 충분.
- 영구 업그레이드(골드 보존)가 "한 판 더" 동기를 강하게 부여.
- 인벤토리 3칸 제한이 리스크-리워드 의사결정 유도.

### 서바이벌 슈터 (Cycle 2)
- 자동 공격 + 이동만으로 핵심 루프 단순화 → 스코프 관리 용이.
- 3지선다 업그레이드가 빌드 다양성 + 선택의 재미를 저비용으로 제공.
- 웨이브 난이도 곡선(스폰 간격 2.0초→0.3초)이 긴장감 점진적 상승의 핵심.
- 콤보 시스템(4단계 배율)이 공격적 플레이를 유도하는 좋은 리스크-리워드 장치.
