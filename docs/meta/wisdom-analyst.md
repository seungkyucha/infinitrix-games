- **[Cycle 21~52]** CrazyGames, itch.io 등 게임 포털이 WebFetch를 403/404로 차단. WebSearch 5회 병렬로 완전 대체.
- **[Cycle 25~52]** 10개 장르 조합 전체 충족 후에는 "최소 빈도 조합 강화 + 시장 트렌드 정합도 + 프리미엄 적합도" 다기준 평가로 전환.
- **[Cycle 38~52]** 기존 보고서 존재 시 Read 후 Write 필요. Glob 확인 + Read(limit=5) 선행이 필수 패턴.
- **[Cycle 44~52]** 매치3 집중 모드에서는 장르 로테이션 대신 서브장르 품질 향상이 우선. 기존 매치3 게임 대비 차별화 분석이 핵심.
- **[Cycle 45~52]** manifest-code 교차 검증 체크리스트를 분석 보고서에 포함 필수. C44 미참조 에셋 문제의 재발 방지.
- **[Cycle 47~52]** 라운드별 진화 방향 지시문(중기/후기)이 프롬프트에 명시될 경우, 추천 게임이 해당 목표에 정확히 정합해야 함.
- **[Cycle 49~52]** C49 과잉 기획 실패 후, 복잡도 축소가 품질 우선. 이벤트는 2종 이내로 제한 권장.

## 검증된 성공 패턴 ✅
- **[Cycle 21~52]** Grep 병렬 패턴(`"id":` + `"genre":\s*\[` -A 1)이 45개 게임에서도 안정적 동작. 32사이클 연속 검증.
- **[Cycle 21~52]** WebSearch 5회 병렬 실행이 32사이클 연속 안정적 동작. 5축 구성이 표준.
- **[Cycle 33~52]** WebFetch 완전 생략해도 WebSearch 5회만으로 충분한 시장 데이터 확보 가능.
- **[Cycle 44~52]** 매치3 집중 모드에서는 시장 규모 + Top Revenue + 메카닉 혁신 트렌드를 서브장르 특화 검색으로 확보 가능.
- **[Cycle 45~52]** "이전 라운드 문제→현재 라운드 해결" 대조표 방식이 보고서 설득력을 크게 높임. 9라운드 연속 검증.
- **[Cycle 46~52]** 시즌/테마 → 레벨 목표 → 미니게임 → 이벤트/토너먼트 → 메타 시스템 → PvP → 시즌패스+길드로 자연스러운 진화 경로 확인.
- **[Cycle 47~52]** Royal Match 벤치마크 검색이 레벨 디자인/부스터/미니게임/이벤트/메타/PvP/시즌패스의 구체적 참조 데이터 제공에 효과적.
- **[Cycle 50~52]** 복잡도 축소 + 코어 루프 집중 전략이 후기 라운드 목표에 정합. 이벤트 2종 이내 제한이 품질 보장.
