## 반복되는 실수 🚫
- **[Cycle 21]** CrazyGames, itch.io 등 주요 게임 포털이 직접 크롤링(WebFetch)을 403/404로 차단. 향후 분석에서는 직접 방문 대신 검색 엔진 기반 간접 조사 + 특정 태그 페이지(itch.io/games/html5/tag-*) 활용이 더 효과적.
- **[Cycle 21]** game-registry.json이 10,000토큰을 초과하여 전체 읽기 실패. 19개 게임이 각각 8개 언어 i18n을 포함해 파일이 비대. 향후에는 Grep으로 id/genre/title만 추출하는 패턴을 첫 단계로 사용할 것.
- **[Cycle 21-2]** 이전 사이클 보고서가 이미 존재하는 경우, 기존 보고서를 먼저 확인하고 중복 분석을 피해야 한다. cycle-21-report.md가 이미 존재했으나 덮어쓰기로 처리 — 향후에는 기존 보고서 존재 여부를 먼저 확인하고, 업데이트가 필요한 부분만 수정하는 패턴 적용.
- **[Cycle 24]** 게임 포털 직접 WebFetch는 여전히 불안정(403/404 위험). 5회 병렬 WebSearch로 완전 대체하면 충분한 시장 데이터 확보 가능 — WebFetch 시도 자체를 생략해도 분석 품질에 영향 없음 확인.
- **[Cycle 25]** 장르 매트릭스 10개 조합이 모두 1개 이상 충족된 상태에서는 "공백 해소"가 더 이상 1순위 기준이 되지 못함. "최소 빈도 조합 강화" + "시장 트렌드 정합도" + "프리미엄 요구사항 적합도"의 다기준 평가로 전환해야 하며, 이 전환에 약간의 분석 시간이 추가 소요됨.
- **[Cycle 26]** 최소 빈도 조합 5개가 동률(각 1개)일 때, 시장 트렌드 데이터의 "강도" 차이가 결정적 차별화 요인이 됨. 단순히 "트렌드가 있다"가 아니라 "2026년 최고의 해"(GameSpot TD 평가)처럼 정량적/권위적 표현을 근거로 활용해야 선정 논리가 강화됨.
- **[Cycle 27]** 최소 빈도 4개 동률(arcade+casual, action+puzzle, action+casual, puzzle+strategy 각 1개)에서 "검색량 정량 데이터"(Block Blast 823K 월간 검색)가 트렌드 강도의 가장 강력한 증거. 권위적 매체 평가 + 검색량 데이터 조합이 최적의 선정 논리 근거.
