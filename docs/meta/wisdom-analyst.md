## 반복되는 실수 🚫
- **[Cycle 21]** CrazyGames, itch.io 등 주요 게임 포털이 직접 크롤링(WebFetch)을 403/404로 차단. 향후 분석에서는 직접 방문 대신 검색 엔진 기반 간접 조사 + 특정 태그 페이지(itch.io/games/html5/tag-*) 활용이 더 효과적.

- **[Cycle 21-2]** 이전 사이클 보고서가 이미 존재하는 경우, 기존 보고서를 먼저 확인하고 중복 분석을 피해야 한다. cycle-21-report.md가 이미 존재했으나 덮어쓰기로 처리 — 향후에는 기존 보고서 존재 여부를 먼저 확인하고, 업데이트가 필요한 부분만 수정하는 패턴 적용.
- **[Cycle 24]** 게임 포털 직접 WebFetch는 여전히 불안정(403/404 위험). 5회 병렬 WebSearch로 완전 대체하면 충분한 시장 데이터 확보 가능 — WebFetch 시도 자체를 생략해도 분석 품질에 영향 없음 확인.

- **[Cycle 31]** 모든 10개 조합이 최소 2개 달성(8개 동률)되어 "최소 빈도 해소" 프레임워크가 더 이상 차별화 불가. "테마 다양성 + 시장 트렌드 강도 + 프리미엄 적합도" 3축 평가로 전환해야 하며, 최근 6사이클 미사용 장르 필터링이 사실상 2개(puzzle+casual, action+strategy)로 선택지를 좁혀 효율적 결정 가능. WebFetch 1회(Poki Top 10 상세) 시도 — GamerNotify 기사 성공적 접근으로 포털 데이터 보강. 다만 게임 포털 직접 방문은 여전히 사이트별 불안정(교훈 유지).
- **[Cycle 32]** 7개 조합이 2개로 동률, 3개 조합이 3개인 상황에서 "최근 7사이클 미사용 필터"로 3개 후보(puzzle+casual, arcade+action, casual+strategy)까지 축소. 여기서 "Poki Top 5 내 해당 장르 점유율"이 결정적 — puzzle+casual이 2개(#3, #4)로 압도적. 동률 다수일 때 "포털 Top N 장르 점유율"이 최소 빈도보다 강력한 차별화 도구임을 확인.
