- **[Cycle 21~50]** game-registry.json이 비대(10K+ 토큰). Grep으로 id/genre만 추출하는 패턴을 첫 단계로 사용할 것.
- **[Cycle 25~50]** 10개 장르 조합 전체 충족 후에는 "최소 빈도 조합 강화 + 시장 트렌드 정합도 + 프리미엄 적합도" 다기준 평가로 전환.

- **[Cycle 47~50]** 라운드별 진화 방향 지시문(중기/후기)이 프롬프트에 명시될 경우, 추천 게임이 해당 목표에 정확히 정합해야 함.
- **[Cycle 49~50]** C49 과잉 기획(이벤트 4종+미니게임 4종+30레벨) 실패 후, 복잡도 축소가 품질 우선. 이벤트는 2종 이내로 제한 권장.

## 검증된 성공 패턴 ✅
- **[Cycle 21~50]** Grep 병렬 패턴(`"id":` + `"genre":\s*\[` -A 1)이 43개 게임에서도 안정적 동작. 30사이클 연속 검증.
- **[Cycle 21~50]** WebSearch 5회 병렬 실행이 30사이클 연속 안정적 동작. 5축 구성이 표준.
- **[Cycle 33~50]** WebFetch 완전 생략해도 WebSearch 5회만으로 충분한 시장 데이터 확보 가능.
- **[Cycle 44~50]** 매치3 집중 모드에서는 시장 규모 + Top Revenue + 메카닉 혁신 트렌드를 서브장르 특화 검색으로 확보 가능.
- **[Cycle 45~50]** "이전 라운드 문제→현재 라운드 해결" 대조표 방식이 보고서 설득력을 크게 높임. 7라운드 연속 검증.
- **[Cycle 46~50]** 시즌/테마 변형 → 레벨 목표 체계화 → 미니게임 통합 → 이벤트/토너먼트 → 메타 시스템으로 자연스러운 진화 경로 확인.
- **[Cycle 47~50]** Royal Match 벤치마크 검색이 레벨 디자인/부스터/미니게임/이벤트/메타 시스템의 구체적 참조 데이터 제공에 효과적.
- **[Cycle 50]** C49 실패 후 복잡도 축소 + 메타 시스템 집중 전략이 후기 라운드 목표에 정합. 과잉 기획 방지 패턴 확립.
