# 네온 드리프트 레이서 — 포스트모템

## 한 줄 요약
InfiniTriX 플랫폼 최초의 레이싱 장르 — 네온 사이버펑크 탑다운 드리프트 레이서를 2회차 리뷰에서 APPROVED로 배포했다.

## 무엇을 만들었나
네온 불빛이 번지는 미래도시 위에서 드리프트와 부스트를 구사하며 AI 차량 3대와 순위를 경쟁하는 **탑다운 아케이드 레이싱**이다. 고속 주행 중 브레이크+조향으로 드리프트에 진입하면 부스트 게이지가 차오르고, 50% 이상 채워지면 폭발적 가속을 터뜨리는 "스킬 표현 → 보상" 루프가 핵심이다. 1레이스 3랩, 약 2~3분의 짧은 세션으로 리플레이성이 높다.

3개 트랙(시티 서킷/캐니언 런/스타 링)을 순차 언락하며, 트랙마다 고유 장애물(공사 바리케이드, 모래 구간, 왕복 운석)이 있다. 스타 링 1위 달성 시 하드 모드(AI +20%)가 해금되어 마스터리를 확장한다. 3종 AI 개성(aggressive/balanced/defensive)과 러버밴딩이 매 레이스 다른 경쟁 양상을 만들어낸다.

## 잘 된 점 ✅
- **플랫폼 최초 레이싱 장르 성공**: 기획서 §1~§8의 모든 요구사항(3트랙, 드리프트, 부스트, AI 3대, 언락, 하드모드, 점수)이 빠짐없이 구현되었다. CONFIG 객체 수치가 기획서와 7항목 1:1 일치(MAX_SPEED 300, DRIFT_THRESHOLD 0.7, BOOST_SPEED_MULT 1.6 등).
- **시각적 완성도 우수**: CRT 스캔라인 + 글리치 타이틀, 네온 글로우 트랙 경계선, 드리프트 타이어 자국(링버퍼 600), 연기 파티클(ObjectPool 150) 등 사이버펑크 분위기가 100% Canvas API만으로 구현됨.
- **코드 품질 최상급**: 코드 리뷰 10/10 PASS, 브라우저 테스트 11/11 PASS, 모바일 터치 8/8 PASS. 콘솔 에러 0건, 경고 0건. 모든 물리/충돌/AI 함수가 파라미터 기반 순수 함수(§10 준수). `TweenManager.clearImmediate()`, `beginTransition()` 가드, 게임 루프 try-catch 등 11사이클 누적 교훈이 모두 반영되었다.
- **기존 인프라 호환성 재확인**: TweenManager, ObjectPool, TransitionGuard, SoundManager, delta time 루프가 레이싱 장르에서도 수정 없이 동작. 아키텍처 범용성이 12개 장르로 확장됨.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 12사이클 연속 재발**: 코드 내 에셋 참조는 완전히 0건(순수 Canvas 달성)이라는 진전이 있지만, 물리적 파일(manifest.json + SVG 8개)이 여전히 잔존하여 1회차에서 지적받았다. "코드 레벨 클린 ≠ 파일 시스템 레벨 클린"이라는 Cycle 10 교훈이 또다시 확인됐다.
- **일시정지 버튼 WCAG 미달**: 40×40px로 구현되어 1회차에서 44×44px 수정 요청. 터치 타겟 최소 크기는 이제 기획 단계에서 강제해야 할 항목이다.
- **데드 코드 잔존**: 빈 if 블록(L1078-1080)이 1회차에서 발견됨. 이후 벽 충돌 로직(wallHitCount++, shakeMag, sound)으로 대체하여 해결했지만, 코딩 중 임시 코드 정리 습관이 필요하다.
- **2회 리뷰 소요**: 1회차 통과 목표(기획서 §12)를 달성하지 못했다. 다만 3건 모두 경미한 이슈여서 수정 시간은 짧았다.

## 기술 하이라이트 🛠️
- **Catmull-Rom 보간 트랙**: 웨이포인트 배열을 Catmull-Rom 스플라인으로 보간하여 베지어 곡선 수준의 부드러운 도로를 단순한 데이터 구조로 구현. `closestTrackPoint` 함수로 도로 이탈 판정도 깔끔하게 처리했다.
- **드리프트 물리**: 핸드브레이크 시 횡방향 마찰을 감소시키고 `driftAngle`을 누적하여 관성 슬라이드를 구현. 드리프트 각도와 지속 시간에 비례한 부스트 게이지 충전이 스킬 보상 루프의 핵심이다.
- **AI 개성 + 러버밴딩**: 3대 AI에 aggressive/balanced/defensive 성향을 부여하고, 순위 기반 러버밴딩으로 자연스러운 경쟁감을 유지. 하드모드(AI_HARD_MODE_MULT: 1.20)로 마스터리 확장.
- **Web Audio 네이티브 스케줄링**: `o.start(startT)/o.stop(startT+dur)` 파라미터 기반 스케줄링으로 setTimeout 없이 정확한 효과음 타이밍 달성.
- **뷰포트 컬링**: margin 300px 기반 컬링 + 글로우 제한(차량+경계선+UI만)으로 렌더링 부하 최적화.
