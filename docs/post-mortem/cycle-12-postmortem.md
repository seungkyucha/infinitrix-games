# 미니 드리프트 레이서 — 포스트모템

## 한 줄 요약
InfiniTriX 최초의 레이싱 게임 — 탑다운 드리프트 물리, 5개 테마 트랙, 고스트 레코드까지 단일 HTML로 완성했다.

## 무엇을 만들었나
탑다운 시점에서 관성 기반 차량을 조종하며 드리프트 포인트를 쌓고 랩 타임을 단축하는 아케이드 레이싱 게임이다. 핸드브레이크로 후미를 미끄러뜨리고 카운터 스티어링으로 각도를 유지하며 코너를 돌파하는 물리적 쾌감이 핵심이다. 드리프트 포인트가 부스트 게이지를 충전하고, 부스트로 직선을 가속하면 다음 코너 진입 속도가 올라가는 리스크-리워드 순환이 "한 바퀴만 더"를 부르는 중독 루프를 만든다.

초원, 사막, 설산, 도심 야경, 화산까지 5개 트랙이 각각 고유한 배경색, 노면 효과, 목표 시간을 갖고 있으며, 트랙별 별점(1~3성)과 고스트 레코드가 localStorage에 저장되어 자기 자신과의 기록 경쟁을 유도한다. 12사이클 만에 처음으로 도전한 레이싱 장르이지만, 기존 플랫폼 인프라(TweenManager, ObjectPool, SoundManager, TransitionGuard)가 무리 없이 적용되어 아키텍처 범용성을 다시 한번 확인한 사이클이었다.

## 잘 된 점 ✅
- **드리프트 물리의 완성도가 높다.** 핸드브레이크 → 횡방향 마찰 감소 → driftAngle 누적 → 관성 슬라이드의 일련의 흐름이 자연스럽게 구현되었다. 부스트 순환(DP 500 → 게이지 충전 → MAX_SPEED 초과 가속)도 기획 의도대로 동작한다.
- **26개 코드 리뷰 항목 중 24개 PASS, 브라우저 테스트 12/12 전항 PASS.** 콘솔 에러 0건, 7개 상태 머신(TITLE→TRACK_SELECT→COUNTDOWN→RACING→LAP_CLEAR→RACE_RESULT, PAUSED) 전환이 모두 정상이다.
- **assets/ 문제의 실질적 진전.** 12사이클 연속 재발하던 이슈에서, 이번에는 코드 내 에셋 참조가 완전히 0건이다. 파일은 잔존했지만 `new Image()`, `fetch`, SVG 참조가 전혀 없는 순수 Canvas 드로잉을 달성했다.
- **고스트 시스템이 인상적이다.** 6프레임 간격 기록 → localStorage 저장 → 반투명 렌더링으로 서버 없이 비동기 기록 경쟁을 구현했다.
- **메모리 관리가 깔끔하다.** 연기 파티클 80개 ObjectPool, 타이어 자국 200개 제한으로 GC 스파이크 없이 안정적으로 동작한다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **물웅덩이 조향 반전 미구현 (ISSUE-1).** 기획서 §5.3에 명시된 "puddle 진입 시 1초간 조향 반전"이 빠졌다. 속도 감소와 puddleTimer는 구현되어 있어 ~10줄 추가로 해결 가능하지만, 기획 명세 준수라는 관점에서 아쉽다.
- **assets/ 디렉토리가 여전히 물리적으로 존재한다 (ISSUE-2).** 코드에서는 참조하지 않지만 manifest.json + SVG 8개 파일이 배포에 포함된다. 12사이클째 반복되는 이 문제는 CI/pre-commit 훅 없이는 근본 해결이 불가능하다는 결론이 더욱 명확해졌다.
- **트랙 선택 화면에서 매 프레임 localStorage 접근 (ISSUE-3).** 5개 트랙 × 60fps = 초당 300회 읽기. 성능 영향은 미미하지만 불필요한 I/O다. 진입 시 1회 캐싱이면 해결된다.
- **시간이 있었다면:** 리플레이 저장/공유, AI 상대 차량, 트랙 에디터, 날씨 효과(비/눈 시 노면 마찰 변화) 등을 추가했을 것이다.

## 기술 하이라이트 🛠️
- **웨이포인트 기반 트랙 시스템:** 트랙을 웨이포인트 배열로 정의하고, 도로 폭 80px로 경로를 확장한 뒤, `closestTrackPoint` 기반으로 도로 이탈 판정 + 벽 밀어내기를 수행한다. 이 접근은 베지어 곡선 수준의 매끄러운 도로를 단순한 좌표 배열만으로 구현하는 효율적인 방법이다.
- **lerp 카메라 + 전방 룩어헤드 + 셰이크 감쇠:** 고속 레이싱에서 시야 안정성과 역동적 연출을 동시에 달성한다. Cycle 11 플랫포머의 카메라 시스템 노하우가 레이싱에도 유효했다.
- **드리프트 물리 모델:** 핸드브레이크 → 횡방향 마찰 계수 감소 → driftAngle 누적 → 관성 슬라이드의 4단계가 각각 독립된 수치(마찰, 감쇠, 임계값)로 제어되어 밸런스 조정이 용이하다.
- **미니맵 80×80px:** 우상단 고정, 트랙 윤곽 + 플레이어 위치를 실시간 표시. 작은 구현 비용으로 공간 인지에 큰 도움을 주는 UI 요소다.

## 다음 사이클 제안 🚀
1. **CI pre-commit 훅 실제 등록:** assets/ 디렉토리 존재 시 커밋 실패하도록 `.git/hooks/pre-commit`을 실제로 배포한다. 12사이클간의 교훈을 마침내 실행에 옮길 때다.
2. **시뮬레이션/전략 장르 도전:** 레이싱 물리 구현 경험을 살려, 농장 경영이나 도시 건설 같은 시뮬레이션 장르를 시도해볼 만하다. 아이들(Cycle 11) + 경영(Cycle 8) 노하우와 결합하면 깊이 있는 게임이 나올 수 있다.
3. **공용 엔진 모듈 분리:** 12개 게임에서 copy-paste된 TweenManager, ObjectPool, SoundManager, TransitionGuard를 `shared/engine.js`로 추출. 매 게임마다 try-catch 래핑을 기억해야 하는 현재 구조의 한계를 구조적으로 해결한다.
