# Beat Crafter (Beat Crafter) — Post-Mortem

## One-Line Summary
We built a rhythm puzzle game where "solving the puzzle creates music" — falling note blocks placed on a grid that ring out chords.

## What We Built
Beat Crafter is the first music-based game on the InfiniTriX platform. On a 4x8 grid, 5 types of note blocks (Do, Re, Mi, Fa, Sol) fall down, and when 3 of the same note align horizontally or vertically, Web Audio API plays that note's chord and clears them. Combining Tetris's block placement strategy with match-3's alignment satisfaction and procedural music generation, each game offers a different "composition" experience.

The core fun erupts in chain clears. When one placement causes blocks to vanish, and gravity-repositioned blocks match again, chords stack layer upon layer while scores explode. As levels rise, BPM increases from 80 to 145 and drop speed accelerates, naturally creating "let me try to last longer" survival tension. Levels 1-2 start with 3 note types for beginners, all 5 appear from level 5, and dynamic balance correction generates match-friendly blocks during crises so beginners don't give up.

- **100% implementation of all spec §13.1~§13.4 items, with zero ghost code in variable usage verification.** All 10 major variables (score at 3 locations, level at 6 locations, inputMode at 4 locations, etc.) used exactly as intended.

## Pain Points / Room for Improvement ⚠️
- **9 unused SVGs + manifest.json remain in assets/ directory.** No impact on functionality as code references none of them, but there's no mechanism to automatically clean up files copied from generic templates. A "warn on unreferenced assets in code" feature is needed in the build pipeline.
- **startGame()/goToTitle() directly modify state instead of using beginTransition().** Spec §9 mandates "all state transitions must use beginTransition()," but these two functions bypass it. No race condition due to clearImmediate(), but the TITLE→PLAYING fadeOut 300ms transition animation is missing, causing an abrupt cut.
- **LOADING state is dead code.** With synchronous initialization without external assets, LOADING + renderLoading() never execute. Could be viewed as preserving future extensibility, but currently unnecessary code.
- **Limited balance verification for 3-match in 4-column grid.** Headless testing can't assess actual play balance (too easy/hard). Real-play confirmation is needed to verify whether level-based note pool expansion (3→4→5 types) adjusts difficulty as intended.
- **If we had more time:** Multi-block simultaneous drops (Tetris mino shapes), special blocks (wildcard, bomb), custom scales (major/minor toggle) to deepen strategic depth.

## Technical Highlights 🛠️
The technical highlight of this game is the **combination of MatchSystem efficiency and Web Audio chord generation**. Match detection traverses only 4×8 = 32 cells in an O(32) lightweight algorithm, classifying matches into THREE/FOUR/CROSS tiers. The chain loop of clear → gravity → re-match is naturally handled through tween onComplete recursion, with `sfxClearChord()` layering octave-above chords and bass at each chain level for rich sound.

**Offscreen canvas grid cache** executes grid line drawing only once initially, then implements beat-responsive background using only drawImage() and globalAlpha adjustments. The 5 block types' unique shapes (circle/diamond/triangle/square/star) are designed with color-blindness accessibility in mind. DAS (Delayed Auto Shift) was implemented using dt accumulation to maintain the complete setTimeout elimination principle.

## Suggestions for Next Cycle 🚀
1. **Separate shared engine module (`shared/engine.js`)** — TweenManager(clearImmediate), ObjectPool, TransitionGuard(enterState), listen()+destroy(), Web Audio SFX have been stabilized over 5 cycles. Separating into shared module would drastically reduce boilerplate and propagate bug fixes to all games.
2. **Challenge physics-based genre** — After puzzle(C1)→shooting(C2)→strategy(C3)→runner(C4)→rhythm puzzle(C5), an unexplored territory remains. Adding simple 2D physics (gravity, reflection, friction) on top of existing infrastructure with a mini golf or projectile game would open a completely new experience axis.
3. **Prototype asynchronous competition system** — All 5 games are solo play. Adding competition through a localStorage-based "daily seed" challenge without external servers would greatly strengthen replay motivation.
