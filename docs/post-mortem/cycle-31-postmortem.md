# 철갑 선봉대 (Ironclad Vanguard) — 포스트모템

## 한 줄 요약
스팀펑크 도시 코그하임을 탈환하는 **실시간 전술 액션 로그라이트**를 순수 Canvas로 구현, 3차 리뷰에서 P0~P3 전수 수정 후 APPROVED를 획득했다. 플랫폼 최초 스팀펑크 테마 도입.

## 무엇을 만들었나
철갑 선봉대는 부패한 기계왕 페릭스의 군단에 맞서 레지스탕스 수석 엔지니어로서 3종 클럭워크 유닛(스트라이커/거너/엔지니어)을 실시간 지휘하여 5개 구역을 탈환하는 실시간 전술 액션 로그라이트다. 5구역×3단계 = 15 기본 스테이지 + 히든 존 1 = 총 16 스테이지, 구역 보스 5체 + 히든 보스 "클럭워크 아르케" = 6체 보스전이 게임의 스케일을 보여준다.

핵심 재미는 "유닛 배치 전술 × 실시간 액션 × 로그라이트 빌드 성장"의 3축 복합 시스템이다. 거너를 고지대에 배치하고 스트라이커로 전선을 유지할지, 엔지니어로 소모전을 벌일지 — 매 스테이지 다른 전술 판단이 필요하다. 보스 클리어 시 3택 블루프린트(common 6/rare 5/epic 3 = 14종)와 워크샵 영구 업그레이드 3트리(공격/방어/생산)가 리플레이 가치를 보장한다.

3단 난이도(견습생/기술자/마스터) + DDA 3단계 동적 밸런스 + 5구역별 고유 환경 위험(증기/톱니/용광로/시계탑/에테르) + 다국어(한/영) 지원으로 다양한 플레이어 수준을 수용한다. 스팀펑크 비주얼(회전 기어, 도시 실루엣, CRT 비네팅)은 에셋 0건 순수 Canvas 드로잉만으로 달성했다.

## 잘 된 점 ✅
- **P0~P3 전수 수정 완료**: 1~2차 리뷰에서 지적된 TDZ 크래시(P0), assets/ F1 위반(P1), 터치 타겟 부족(P2), ESCAPE_ALLOWED 미적용(P3) 4건을 3차에서 모두 해결. 수정 품질이 높아 회귀 버그 0건, 신규 버그 0건.
- **ESCAPE_ALLOWED 패턴 완전 적용**: 12개 게임 상태 모두에 대한 허용 전환 딕셔너리 + RESTART_ALLOWED를 깔끔하게 구현. beginTransition() 가드 로직도 정확.
- **10 REGION 코드 구조**: CONFIG→ENGINE→ENTITY→DRAW→WEAPON→COMBAT→ROGUE→STATE→SAVE→MAIN 의존 방향 단방향. 순환 참조 0건.
- **스팀펑크 비주얼 완성도**: 회전 기어 애니메이션, 도시 실루엣, CRT 비네팅 효과 등 에셋 없이 Canvas만으로 시그니처 비주얼을 달성. 5구역별 고유 환경 위험의 수치까지 명시(F84 반영).
- **100% Canvas 코드 드로잉**: ASSET_MAP/SPRITES/preloadAssets 전삭제, 외부 CDN 0건, Math.random 0건(SeededRNG 완전 사용), setTimeout 0건.
- **코드 품질 체크리스트 전항 PASS**: 기능 완성도, 메모리 관리(ObjectPool), 모바일 대응(48px+ 터치 타겟), SeededRNG, hitTest 통합, 다국어 등 17개 항목 모두 통과.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **3차 리뷰까지 소요**: P0 TDZ 크래시가 1~2차 리뷰에서 미수정 상태로 남았다는 것은 수정 프로세스의 개선이 필요함을 의미한다. workshopBonuses 빈 객체 초기화라는 단순한 수정이 3라운드까지 지연된 것은 아쉽다.
- **공용 엔진 미분리 31사이클째**: TweenManager, ObjectPool, SoundManager, InputManager 등이 여전히 게임별 copy-paste. 10 REGION 구조의 export 함수 목록을 명시했으나 실제 추출은 미착수.
- **전투 시스템 밸런스 미검증**: 3종 유닛 × 14종 블루프린트 × 3트리 업그레이드 × 5구역 환경 위험의 조합 공간이 넓지만, 코드 리뷰만으로는 지배적 전략이나 불가능 빌드를 검증할 수 없다. DPS/EHP 시뮬레이터 부재가 여전한 과제.
- **소형 디스플레이(≤400px) 심층 검증 미완**: 375px에서 레이아웃 적응과 버튼 표시를 확인했으나, 320px 극소형 디스플레이에서의 UI 간섭 가능성은 완전히 검증되지 않았다.

## 기술 하이라이트 🛠️
- **TDZ 방지 패턴 진화**: `workshopBonuses: {}` 빈 객체 초기화 후 init()/onStateEnter에서 `getWorkshopBonus()` 호출로 채우는 패턴. 함수 호출이 변수 선언보다 앞설 수 있는 상황을 구조적으로 방지.
- **ESCAPE_ALLOWED + RESTART_ALLOWED 이중 딕셔너리**: 12개 상태 × 허용 전환 목록 + 터미널 상태 탈출 화이트리스트를 분리하여 상태 전환 안전성을 극대화. beginTransition() 내 가드 로직과 결합.
- **환경 위험 시스템**: 5개 구역별 고유 환경 위험(증기 분출/톱니 기어/용암 분사/시간 왜곡/에테르 부패)을 수치까지 명시하여 장식이 아닌 게임플레이 변형 요소로 구현. F84 피드백 직접 반영.
- **프로시저럴 오디오**: SoundManager로 다종 SFX + BGM을 Web Audio API 네이티브 스케줄링으로 구현. setTimeout 0건 유지.
- **ObjectPool + Math.hypot 충돌 감지**: 대량 유닛/적/투사체 동시 관리에 풀링 + 거리 기반 충돌 판정 조합으로 성능 확보.

## 다음 사이클 제안 🚀

2. **밸런스 시뮬레이터 프로토타입**: 유닛 × 블루프린트 × 업그레이드 × 환경 조합의 DPS/EHP를 headless 자동 시뮬레이션으로 검증하는 도구 개발. 전략 장르의 조합 공간이 매 사이클 확대되고 있으므로 시급.
