# 미니 플랫포머 — 포스트모템

## 한 줄 요약
Celeste 스타일의 정밀 플랫포머를 단일 HTML 파일에 구현했으나, 11사이클 연속 재발하는 assets/ 디렉토리 문제가 또다시 발목을 잡았다.

## 무엇을 만들었나
벽점프, 대시, 이중점프로 무너져가는 고대 탑을 돌파하는 **정밀 액션 플랫포머**다. 5개 월드(숲→동굴→하늘→용암→별빛탑) × 5스테이지 = 총 25스테이지를 수작업 레벨 데이터로 구성했다. 코요테 타임(6프레임), 점프 버퍼링(6프레임), 코너 보정(4px), 가변 점프 높이 등 Celeste/Super Meat Boy에서 검증된 입력 보정 기법을 충실히 구현하여, "내가 의도한 대로 움직인다"는 조작감을 달성했다.

즉사 후 0.3초 이내 리스폰이라는 핵심 루프가 "한 번만 더" 중독성을 만들고, 스테이지별 보석 3개 + 타임 어택이 리플레이 가치를 제공한다. 월드별로 새 무브셋(W2: 벽점프, W3: 이중점프, W4: 대시)이 순차 해금되어 매 월드마다 새로운 학습 요소가 추가된다. 플랫폼에 없던 플랫포머 장르를 최초로 채운 게임이다.

## 잘 된 점 ✅
- **정밀 플랫포머 메커닉 완성**: 코요테 타임, 점프 버퍼링, 코너 보정, 가변 점프 높이 등 기획서의 핵심 입력 보정 기법이 모두 정확하게 구현됨. 조작감의 완성도가 이전 사이클 대비 확연히 높다.
- **25개 수작업 레벨의 양과 질**: 5월드 × 5스테이지 문자열 레벨 데이터 완비. 가시(4방향), 용암, 화염기둥, 레이저, 바람, 무너지는 발판, 이동 발판, 중력 반전 존까지 다양한 장애물 배치.
- **Canvas fallback 렌더링**: SVG 에셋이 존재하긴 하지만, 모든 렌더링 함수에 Canvas 코드 폴백이 완비되어 SVG 없이도 게임이 완전히 동작한다.
- **브라우저 테스트 12항목 전부 PASS**: 콘솔 에러 0건, 로드→시작→플레이→사망→리스폰 전체 플로우가 안정적으로 동작.
- **코드 체크리스트 높은 통과율**: 메모리 관리(ObjectPool), AABB 충돌, 모바일 터치, 캔버스 리사이즈, 상태 전환 가드, localStorage 등 19개 항목 중 16개 PASS.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 11사이클 연속 재발** 🔴: 기획서 §12.1에서 "절대 금지"로 강조했음에도 10개 SVG 파일이 생성됐다. Canvas fallback이 있어 기능적 문제는 없지만, 프로세스 차원에서 이 패턴이 끊이지 않는 것은 심각하다. CI/빌드 파이프라인에서 `assets/` 존재 시 빌드 실패하는 강제 장치가 유일한 해결책으로 보인다.
- **gameLoop try-catch 미적용**: 기획서 §12.9에 명시된 방어적 코딩 패턴이 누락됨. 런타임 에러 발생 시 게임 루프가 완전히 정지할 위험.
- **일일 챌린지 모드 미구현**: 기획서 §6.5의 Seeded RNG 기반 일일 챌린지가 완전 누락. `saveData.dailyBest` 필드만 정의되고 사용처 없음.
- **미사용 스프라이트 3건**: enemy, uiHeart, powerup — 로드는 되지만 코드에서 참조하지 않는 불필요한 네트워크 요청.

## 기술 하이라이트 🛠️
이번 게임의 가장 인상적인 기술적 성취는 **AABB 기반 타일맵 충돌의 X→Y 분리 판정**이다. 수평·수직 이동을 분리하여 충돌을 판정하고, 코너 보정(4px)까지 적용하여 "벽 모서리에 걸려서 점프가 안 된다"는 고전적 플랫포머 문제를 해결했다. 월드 인덱스 기반 조건부 무브셋 활성화(`createPlayer()`에서 W2+: 벽점프, W3+: 이중점프, W4+: 대시)는 간결한 구현으로 점진적 메커닉 확장을 달성한 좋은 설계다. lerp 카메라 추적 + look-ahead + 화면 흔들림 + 레벨 경계 clamping 조합의 카메라 시스템도 기존 사이클에 없던 새로운 기술적 도전이었다.
