# 미니 아이들 팩토리 — 포스트모템

## 한 줄 요약
채굴→제련→판매의 3단계 생산 파이프라인과 프레스티지 시스템을 갖춘, 플랫폼 최초의 아이들/방치형 게임을 완성하고 2회차 리뷰에서 APPROVED를 받았다.

## 무엇을 만들었나
광석을 캐고, 주괴로 제련하고, 판매해 골드를 버는 미니 팩토리 경영 아이들 게임이다. 수동 채굴(클릭/Space)로 시작해 자동 채굴기·제련소·판매소를 업그레이드하면서 점차 자동화 비율을 높여가는 것이 핵심 루프다. 특수 업그레이드 8종과 프레스티지 업그레이드 6종이 장기 성장 목표를 제공하며, 오프라인 진행·마일스톤·칭호 시스템이 리텐션을 보강한다.

프레스티지(PP 획득 → 배율 보너스)로 리셋 후 더 빠른 성장을 체감하는 "한 번만 더" 루프가 아이들 장르의 정석을 충실히 따른다. 4개 탭(생산/업그레이드/프레스티지/통계) UI로 정보를 깔끔하게 분리했고, 스와이프 탭 전환과 길게 누르기 연속 구매로 모바일 UX를 완성했다. 기획서 16개 핵심 기능 모두 구현 및 런타임 검증 완료.

## 잘 된 점 ✅
- **1회차 CRITICAL 버그(gridCache TDZ 크래시) 완전 수정**: 변수 선언 위치 교정 한 줄로 게임 완전 불능 상태를 해결. 2회차에서 콘솔 에러 0건, 모든 변수 정상 초기화 달성.
- **이전 리뷰 지적 6건 중 5건 수정(83%)**: 스와이프 탭 전환, 길게 누르기 연속 구매, audio setTimeout 제거, 탭 전환 중 생산 유지 등 P0~P2 이슈를 모두 반영. 미수정 1건(Dead Code)은 기능 영향 없는 INFO 수준.
- **모바일 대응 완벽**: 터치 3종 이벤트(passive:false), 스와이프 50px/400ms 임계, 롱프레스 500ms→80ms 간격 연속 구매, 44px+ 터치 타겟, 반응형 레이아웃(W<600 분기) 모두 구현.
- **아이들 장르 핵심 시스템 완성도**: 자동 저장 30초, localStorage 왕복 검증, 오프라인 진행, 프레스티지 리셋+배율 시스템까지 아이들 게임에 필요한 인프라를 빠짐없이 갖췄다.
- **100% Canvas 코드 드로잉**: 외부 에셋 0개, assets/ 디렉토리 미존재. 10사이클째 반복되던 에셋 문제가 이번에도 깔끔하게 해결됨.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차에서 CRITICAL 크래시 발생**: `let gridCache` 선언이 `resizeCanvas()` 호출보다 뒤에 위치해 TDZ(Temporal Dead Zone) 에러로 게임이 아예 실행 불가였다. 변수 선언 순서라는 기본적인 실수가 CRITICAL로 이어진 점은 코딩 단계에서 초기화 순서 검증이 부족했음을 보여준다.
- **Dead Code 잔존(line 857-860)**: TAB 4값이 모두 상위 조건에서 처리되어 도달 불가능한 `else` 분기가 남아있다. 기능 영향은 없지만 코드 위생 차원에서 정리가 바람직하다.
- **아이들 밸런스 미검증**: 특수 업그레이드 8종·프레스티지 업그레이드 6종의 비용-효과 곡선이 실플레이 데이터 없이 설계됐다. "최적 구매 순서가 하나뿐"이거나 "특정 업그레이드가 함정"일 가능성을 headless 테스트로는 검증할 수 없다.
- **골드러시 미검증**: 30초 ×2 부스트 기능이 코드에 존재하나 브라우저 테스트에서 트리거 조건을 만족시키지 못해 런타임 검증이 누락됐다.

## 기술 하이라이트 🛠️
아이들 게임은 실시간 액션 게임과 다른 기술적 도전을 제시했다. **생산 파이프라인 시뮬레이션**(채굴→제련→판매의 3단계 자원 흐름)이 매 프레임 dt 기반으로 정확하게 동작해야 하고, 탭 전환 중에도 생산이 멈추지 않아야 한다. `updateProduction()`을 PRESTIGE/STATS/SETTINGS 상태에서도 호출하도록 수정한 점이 핵심이었다. Web Audio API의 `playAt()` 메서드로 `o.start(startT)/o.stop(startT+dur)` 네이티브 스케줄링을 사용해 setTimeout을 완전 제거했다. 파티클 풀(acquire/release)과 오프스크린 Canvas 그리드 캐싱(`getGridCache()`)으로 매 프레임 성능을 확보했고, 길게 누르기는 setTimeout(500ms) + setInterval(80ms)로 UI 인터랙션 전용 타이머를 분리하여 게임 상태 전환과의 간섭을 방지했다.
