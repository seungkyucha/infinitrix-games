# 미니 플랫포머 — 포스트모템

## 한 줄 요약
Celeste 스타일의 정밀 플랫포머를 단일 HTML에 25스테이지로 구현하고, 3회 리뷰를 거쳐 11사이클 숙원이던 assets/ 문제까지 완전 해결하여 APPROVED를 받았다.

## 무엇을 만들었나
벽점프, 대시, 이중점프로 무너져가는 고대 탑을 돌파하는 **정밀 액션 플랫포머**다. 5개 월드(숲→동굴→하늘→용암→별빛탑) × 5스테이지 = 총 25스테이지를 수작업 레벨 데이터로 구성했다. 코요테 타임(6프레임), 점프 버퍼링(6프레임), 코너 보정(4px), 가변 점프 높이 등 Celeste/Super Meat Boy에서 검증된 입력 보정 기법을 충실히 구현하여, "내가 의도한 대로 움직인다"는 조작감을 달성했다.

즉사 후 0.4초 이내 리스폰이라는 핵심 루프가 "한 번만 더" 중독성을 만들고, 스테이지별 보석 3개 + 스피드런 타이머가 리플레이 가치를 제공한다. 월드별로 새 무브셋(W2: 벽점프, W3: 이중점프, W4: 대시)이 순차 해금되어 자연스러운 학습 곡선을 형성한다. InfiniTriX 플랫폼에서 최초의 순수 플랫포머 장르이자, casual 60% 편중을 탈피하기 위한 전략적 선택이기도 했다.

## 잘 된 점 ✅
- **정밀 플랫포머 메커닉 완성도 우수**: 코요테 타임, 점프 버퍼링, 코너 보정, 가변 점프 높이, 벽 슬라이드, 대시+잔상 — 기획서의 핵심 입력 보정 기법이 모두 정확하게 구현됨. 외부 물리 라이브러리 없이 AABB X→Y 분리 판정으로 정밀 충돌 달성.
- **모바일 터치 전경로 완벽 대응**: 타이틀→월드선택(◀▶+탭)→게임플레이(D패드+A/B/R)→일시정지(⏸)→클리어(탭) 전 경로가 키보드 없이 터치만으로 플레이 가능. 2회차 지적된 T1/T2/T3 모바일 이슈 3건 모두 수정.
- **코드 리뷰 19개 검사 항목 전부 PASS**: 순수 함수 원칙, beginTransition 통일 전환, ObjectPool 파티클 관리, try-catch 게임 루프, TDZ 방지, clearImmediate(), dt 파라미터 전달 등 10사이클 누적 인프라 전량 준수.
- **assets/ 문제 드디어 완전 해결**: 1회차에서 SVG 10개가 재발했으나, 3회차에서 디렉토리 완전 삭제 + ASSET_MAP/SPRITES/preloadAssets 코드 전량 제거. 모든 비주얼이 Canvas API(fillRect, arc, lineTo, fillText) 코드 드로잉으로 전환 확인.
- **브라우저 테스트 12항목 전부 PASS**: 콘솔 에러 0건, 로드→시작→월드선택→플레이→사망→리스폰→클리어 전체 플로우 안정 동작.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **3회 리뷰가 필요했다**: 1회차 assets/ CRITICAL → 2회차 잔여 코드 CRITICAL → 3회차 APPROVED. 처음부터 Canvas 전용으로 작성했다면 1회차에서 끝낼 수 있었다. 기획서 명시만으로는 11사이클 동안 한 번도 1회차 통과를 달성하지 못했다는 사실이 CI 강제 장치의 필요성을 확정적으로 증명한다.
- **월드별 고유 장애물 6종 미구현**: 무너지는 발판, 낙석, 바람 밀림, 사라지는 구름 발판, 화염 기둥, 레이저 등이 기획되었으나 구현되지 않았다. 코어 메커닉에 영향은 없지만 월드 간 차별화가 약해졌다.
- **일일 챌린지(§6.5) 완전 미구현**: Seeded RNG 기반 프로시저럴 레벨 생성 + UI 모두 누락. 코어 메커닉에 집중하느라 부가 모드까지 여력이 닿지 않았다. 별도 이터레이션으로 분리하는 것이 현실적이다.
- **히든 보석/히든 스테이지 미구현**: 수집 콘텐츠 확장이 없어 마스터리 플레이어의 엔드 콘텐츠가 부족하다.

## 기술 하이라이트 🛠️
가장 인상적인 기술적 성취는 **AABB 기반 타일맵 충돌의 X→Y 분리 판정**이다. 수평·수직 이동을 분리 판정하고, 코너 보정(4px)과 함정 마진(3px)까지 적용하여 플랫포머 고전 문제("벽 모서리에 걸려서 점프가 안 된다")를 해결했다. 월드 인덱스 기반 조건부 무브셋 활성화(`CONFIG.ABILITIES[worldIdx]`)는 간결한 구현으로 점진적 메커닉 확장을 달성한 좋은 설계다.

카메라 시스템도 주목할 만하다. lerp 추적 + look-ahead(이동 방향 선행) + 스크린 셰이크 + 월드 경계 clamping의 4중 조합이 대시 중에도 시야 안정성을 보장한다. SoundManager는 Web Audio API로 9종 효과음을 코드에서 직접 합성하여 외부 오디오 파일 0개를 유지했다. ObjectPool(파티클 150개) + 타일 뷰포트 컬링으로 25개 레벨 전체에서 안정적 성능을 달성했다.
