# 보석 왕국 건설기 — 포스트모템

## 한 줄 요약
매치3 퍼즐과 왕국 건설 메타게임을 결합한 Royal Match 스타일의 프리미엄급 HTML5 게임을 3,204줄 단일 파일로 완성했다.

## 무엇을 만들었나
8×8 보석 그리드에서 매치3 퍼즐을 풀고, 획득한 별로 왕국 5구역(성문/정원/시장/도서관/왕좌실)을 재건하는 건설 메타 매치3 게임이다. 6색 보석, 장애물 10종, 스페셜 조합 6종, 부스터 4종이라는 탄탄한 퍼즐 시스템 위에 30레벨 × 5구역의 메타 진행 구조를 얹어 "한 판만 더"의 동기를 극대화했다.

레벨을 클리어하면 별 1~3개를 받고, 건물/장식 3택 1 선택지가 나타난다. 구역을 완성하면 패시브 보너스와 해금 연출이 펼쳐진다. 여기에 SeededRNG 기반 일일 챌린지 8종과 AI 3명과 겨루는 주간 건설 경쟁까지 갖춰 반복 플레이 동기를 다층으로 설계했다. PNG 에셋 67개, 다국어(ko/en), 왕 캐릭터 감정 표현 3종까지 포함한 완성도 높은 결과물이다.

## 잘 된 점 ✅
- **C49 문제 5건(F1~F5) 전수 해결**: 함수 오버라이드 0건 정책으로 호이스팅 무한 재귀를 근절하고, 18상태를 12상태로 축소해 복잡도를 효과적으로 관리했다.
- **에셋 + 폴백 시스템 완성도**: 67개 PNG 에셋 + manifest.json 동적 로드 + drawAssetOrFallback() 전수 적용. idle 스프라이트 애니메이션(6색×4프레임)과 이펙트 시퀀스 6종으로 비주얼 퀄리티가 역대 최고 수준이다.
- **Puppeteer 브라우저 테스트 5종 전항목 PASS**: 로드, 시작, 보드 조작, 일시정지/게임오버/재시작, 터치 동작까지 자동화 검증 완료. 1회차 CRITICAL 3건 수정이 2회차에서도 안정적으로 유지됨을 확인했다.
- **12상태 TRANSITION_TABLE 완전 검증**: 모든 상태 간 전환 경로를 코드 레벨에서 대조 확인. Stuck 상태 불가능 입증(8-1~8-11 전항목 PASS).

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차 CRITICAL 3건 발생**: gBgCache TDZ, input.flush() 타이밍, LEVEL_FAIL 재시작 미초기화 — 모두 기존에 알려진 패턴(TDZ, flush 순서)의 변형이었다. 사전 체크리스트만으로는 새로운 맥락에서의 재발을 막지 못한다. 린터 자동화가 절실하다.
- **3,204줄 단일 파일의 한계**: 50사이클째 공용 엔진(TweenManager, ObjectPool, TransitionGuard 등) 분리를 미루고 있다. 코드 리뷰와 디버깅 비용이 매 사이클 증가 중이다.
- **주간 건설 경쟁의 AI 깊이 부족**: AI 3명의 진행 속도가 고정 공식 기반이라 실제 경쟁감이 제한적이다. 플레이어 행동 패턴 반영형 AI가 있었다면 리플레이 가치가 높아졌을 것이다.
- **페이드 전환 프록시 패턴의 복잡성**: gFadeProxy와 gFadeAlpha 이중 동기화 구조가 직관적이지 않다. tween 시스템 자체에 전역 변수 직접 조작 기능이 있다면 불필요한 계층이 사라진다.

## 기술 하이라이트 🛠️
- **함수 오버라이드 0건 정책**: C49에서 `function` 선언문 호이스팅으로 9개 함수가 무한 재귀에 빠진 교훈을 반영해, 이번 사이클에서는 오버라이드 자체를 사용하지 않았다. 기능 확장은 if/switch 분기와 별도 함수명으로 처리. 결과적으로 declaration 관련 버그 0건 달성.
- **TRANSITION_TABLE 12상태 최적화**: C49의 18상태에서 이벤트 2종(일일 챌린지+주간 경쟁)으로 축소하면서도 왕국 건설 메타 체인(PLAY→LEVEL_CLEAR→BUILD_SELECT→BUILD_ANIM→KINGDOM_MAP)을 안정적으로 구현했다. deferredTransition 큐가 enterState 데드락을 원천 차단.
- **consumed[][] 매치 소비 추적 + 캐스케이드 tween 체인**: 5→T/L→4→3 우선순위 매치 검출에서 이미 소비된 셀을 추적하여 중복 매치를 방지하고, 중력 낙하→재매칭→tween 콜백 체인으로 연쇄 폭발의 시각적 쾌감을 구현했다.
- **SeededRNG 날짜 시드 일일 챌린지**: Math.random() 0건 정책 하에서 날짜 기반 시드로 매일 동일한 챌린지를 생성. 8종 특수 규칙(SPEED_RUN 포함)과 결합하여 재현 가능한 경쟁 환경 제공.

## 다음 사이클 제안 🚀
- **TDZ/선언 순서 정적 검증 자동화**: Engine 생성자 콜백에서 참조하는 변수의 선언 위치를 린터 규칙으로 사전 검증. C50 BUG-1 같은 패턴의 재발을 코드 작성 단계에서 차단.
- **공용 엔진 모듈 분리 (`shared/engine.js`)**: TweenManager, ObjectPool, TransitionGuard, SeededRNG 등 50사이클간 검증된 인프라를 단일 모듈로 추출. 게임 코드 볼륨을 30~40% 절감하고 리뷰 효율을 높인다.
- **플레이어 행동 반영형 AI**: 주간 경쟁 AI의 진행 속도를 플레이어의 실제 클리어 패턴에 맞춰 동적으로 조정. 압도적 승리/패배 없이 항상 접전이 되는 경쟁 경험을 목표로.
