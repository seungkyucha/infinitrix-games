# 룬 서바이버 — 포스트모템

## 한 줄 요약
18사이클 만에 F1(assets/ 금지) 규칙을 정면 돌파한 첫 성공 사례 — Vampire Survivors 스타일 로그라이크 서바이버를 2,757줄 순수 Canvas로 완성하여 즉시 배포 승인을 받았다.

## 무엇을 만들었나
플레이어는 고대 룬 마법사를 조작하여 360° 전방위에서 몰려오는 몬스터 무리 속에서 생존한다. 이동만 직접 조작하고 무기는 자동 발사되는 서바이버라이크 자동 전투 방식을 채택했다. 적을 처치하면 XP 젬이 드롭되고, 레벨업 시 3택 1 로그라이크 업그레이드를 선택하는 구조다.

10웨이브 시스템 안에 슬라임·박쥐·골렘·마법사·해골전사 5종 적과 크림슨 워든·엘더 리치 2종 보스가 등장한다. 룬볼트·화염 오라·얼음 창·번개 체인·보호 실드 5종 무기는 각 5레벨까지 업그레이드 가능하며, 무기×적 타입 상성 테이블(DMG_TABLE)이 빌드 선택에 전략적 깊이를 더한다.

비주얼은 룬 서클 애니메이션, 글리치 타이틀 이펙트, 보스 등장 비네트, 화면 흔들림, 슬로우모션 연출까지 포함하여 18사이클 중 가장 완성도 높은 액션 연출을 달성했다. Web Audio API로 BGM과 SFX를 모두 절차적으로 생성하여 외부 에셋 0개를 유지했다.

## 잘 된 점 ✅
- **F1 규칙 최초 정면 준수**: 17사이클 연속 재발하던 assets/ 디렉토리 문제를 완전히 해결. 100% Canvas 코드 드로잉으로 외부 에셋 의존 완전 제거.
- **MVP 우선 전략(F25) 성공**: Phase 1(3상태+적1종+무기1종) → Phase 6(BGM/SFX/이펙트)까지 점진 확장 전략이 실제로 동작. Cycle 17의 "구현 0%" 참사를 완벽히 극복.
- **역대 최다 피드백 매핑**: F1~F27까지 27개 누적 교훈을 기획서에 선제 매핑. 2회차 리뷰에서 잔여 이슈 0건 달성.
- **코드 리뷰 전항 PASS**: 7개 게임 상태, 6종 오브젝트 풀, circleCollision 거리 제곱 비교, 4중 가드 플래그 체계, try-catch 게임 루프, localStorage 안전 처리 등 모든 검사 항목 통과.
- **모바일 완전 대응**: 가상 조이스틱(DEADZONE 10px), touchSafe() 48px 강제, 3모드 입력 자동 감지, 전경로 터치 지원.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차 NEEDS_MAJOR_FIX**: 최초 제출 시 assets/ 디렉토리와 에셋 로딩 코드가 남아 있어 MAJOR 4건 + MINOR 2건이 발생. "처음부터 assets/ 없이 작성"이라는 원칙을 세웠음에도 관성적으로 에셋 구조를 생성한 점은 아쉽다.
- **밸런스 검증 부재**: 5종 무기 × 5종 적 × 10웨이브의 밸런스를 headless 테스트로 검증할 수 없다. 특정 무기 조합이 압도적이거나, 후반 웨이브가 불가능에 가까울 수 있다.
- **2,757줄 단일 파일**: 공용 엔진 모듈(TweenManager, ObjectPool, SoundManager 등)이 매 게임마다 복사-붙여넣기되는 문제가 여전히 해결되지 않았다.
- **사운드 체감 품질 미검증**: Web Audio BGM/SFX 코드가 존재하는 것은 확인했으나, 실제 청각 피드백의 쾌감·타이밍·볼륨 밸런스는 코드 리뷰만으로 판단할 수 없다.

## 기술 하이라이트 🛠️
- **6종 ObjectPool 시스템**: enemy, proj, particle, gem, popup, bullet 풀로 대량 엔티티 생성/해제 시 GC 부하를 최소화. 서바이버라이크 장르의 동시 엔티티 수백 개 관리에 필수적인 최적화.
- **offscreen canvas 배경 캐싱**: `buildBgCache()` 패턴으로 배경 레이어를 resizeCanvas() 시에만 재빌드. 매 프레임 배경 재드로잉 비용 제거.
- **DMG_TABLE 상성 시스템**: 무기 5종 × 적 6타입의 데미지 배율 테이블로 빌드 다양성 확보. 간단한 2D 배열 참조만으로 복잡한 상성 관계 구현.
- **timeScale 기반 슬로우모션**: 게임 시간만 감속하고 tween/UI는 정상 속도 유지. 보스 등장·사망 연출에서 극적인 효과.
- **STATE_PRIORITY 맵 + 4중 가드 체계**: 7개 상태 간 전환 우선순위(GAMEOVER > BOSS_INTRO > LEVEL_UP > PAUSED > PLAYING)와 waveClearing/isTransitioning/isLevelingUp/isDying 가드 플래그로 경쟁 조건 완전 차단.
