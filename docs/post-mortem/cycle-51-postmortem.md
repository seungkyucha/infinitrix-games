# 보석 대전 (Gem Arena) — 포스트모템

## 한 줄 요약
Match Masters 스타일 1v1 턴 기반 PvP 매치3에 솔로 어드벤처 30레벨, 왕국 건설, 리그 시스템까지 탑재한 역대급 규모의 퍼즐 게임을 완성했다.

## 무엇을 만들었나
보석 대전은 8×8 보드에서 AI와 교대로 보석을 매칭하며 점수를 겨루는 PvP 매치3 퍼즐이다. 핵심 재미는 "내 콤보가 상대의 악몽이 되는" 장애물 전송 메커니즘에 있다. 4매치 이상 스페셜 보석을 만들면 상대 보드에 얼음·돌블록·체인·독이 전송되어 방해하는데, 매칭할 때마다 공격과 방어를 동시에 고민하는 전략적 긴장감이 살아있다.

솔로 모드는 5구역(성문→정원→시장→도서관→왕좌실) × 6레벨, 총 30개 스테이지로 구성된다. 레벨을 클리어하면 별을 모아 왕국 건물을 건설하는 메타 루프가 돌아가고, 트로피를 누적하면 브론즈→다이아몬드 5단계 리그를 승급한다. 부스터 4종(망치/교환/셔플/추가턴), 장애물 6종, 스페셜 보석 4종(라인H/V/폭탄/레인보우), DDA 3단계, 일일 챌린지까지 — 단일 HTML 파일 4,948줄에 이 모든 것이 들어있다.

## 잘 된 점 ✅
- **C50 피드백 7건 전량 반영**: TDZ 방어, input.flush() 타이밍, 초기화 체크리스트, 섹션 인덱스, gFadeProxy 제거, AI 3단계 행동 모델, 오버라이드 0건 정책 — 모두 코드에서 확인됨
- **Puppeteer 전 항목 PASS**: 타이틀→모드선택→왕국→레벨선택→플레이→게임오버→재시작, PvP 대전, 설정 화면 진입/복귀, 터치, 모바일 반응형까지 전부 통과
- **에셋 시스템 안정화**: manifest.json 기반 60개 PNG 동적 로드 + drawAssetOrFallback() 전수 적용으로 에셋 실패에도 100% 플레이 가능
- **12상태 머신 안정성**: TRANSITION_TABLE 화이트리스트 + deferredQueue로 상태 전환 데드락 완전 방지. 왕국 건설 체인처럼 복잡한 흐름에서도 안정

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1차 리뷰에서 CRITICAL 발생**: SETTINGS 상태의 `tween=false` 설정 오류로 ESC→설정 화면이 검은 화면에 고착. 2줄 수정(164줄 tween=true, returnFromSettings() 정리)으로 해결했지만, 상태×시스템 매트릭스를 수기로 관리하는 한 이런 실수는 반복될 수 있다
- **4,948줄 단일 파일**: 섹션 인덱스로 구조를 잡았지만, 여전히 단일 파일 한계가 있다. 공용 엔진 모듈 분리가 절실하다
- **Phase 2 미구현**: 주간 PvP 토너먼트 32강 브래킷, 리그 보상 보석 스킨, 건설 축하 연출 강화는 시간 부족으로 포기. 다국어(ko/en)도 Phase 2에 있었으나 미구현
- **AI 깊이 검증 한계**: 3단계 AI 행동 모델(탐욕/전략/방어)이 코드에 있지만, AI가 실제로 상위 리그에서 충분히 도전적인지 체감 테스트는 부족

## 기술 하이라이트 🛠️
- **consumed[][] 매치 소비 추적**: 5→T/L→4→3 우선순위 매칭에서 이미 매치된 셀의 중복 카운팅을 방지하는 핵심 자료구조. 50사이클 넘게 안정적으로 작동 중
- **장애물 전송 메커니즘**: PvP에서 스페셜 보석 발동 시 상대 보드에 장애물을 일괄 전송하는 시스템. 캐스케이드 완료 후 전송 타이밍을 맞추는 것이 핵심
- **ObjectPool(scorePopPool)**: 점수 팝업 텍스트를 풀링하여 GC 압박 최소화. 매치3의 빈번한 오브젝트 생성/파괴에 효과적
- **SeededRNG 일일 챌린지**: 날짜 기반 시드로 모든 플레이어가 같은 보드를 플레이. Math.random() 0건 정책 유지
- **deferredTransition 큐**: enterState() 내부에서 동기적으로 다른 상태로 전환하려는 시도를 큐에 넣어 다음 프레임에 처리. 12상태 체인에서 데드락 방지
