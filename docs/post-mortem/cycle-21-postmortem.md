# 룬포지 택틱스 — 포스트모템

## 한 줄 요약
5×5 그리드에 8종 룬을 배치해 마법진을 완성하고, 그 마법으로 적 웨이브를 격퇴하는 2페이즈 전략 퍼즐을 4,215줄 순수 Canvas로 구현하여 5회차 리뷰 끝에 APPROVED를 달성했다.

## 무엇을 만들었나

**룬포지 택틱스**는 플랫폼에서 완전히 공백이었던 puzzle + strategy 장르 조합을 최초로 채운 게임이다. 퍼즐 페이즈에서 8종 룬을 5×5 그리드에 전략적으로 배치하면 직선·L자·T자·십자 등 120가지 이상의 마법진 패턴이 발동되고, 곧이어 방어 페이즈에서 완성된 마법이 3레인 적 웨이브를 자동으로 격퇴한다. "이 조합으로 뭐가 나올까?"라는 실험의 재미가 핵심이다.

5개 지역(숲·동굴·사막·설산·마법탑) × 20+ 스테이지 + 5보스 + 영구 업그레이드 + 레시피북 수집이라는 프리미엄급 콘텐츠 볼륨을 갖추었고, 실패한 런에서도 새로운 마법진 레시피가 영구 저장되어 "실패해도 성장"하는 로그라이크적 만족감을 제공한다. 12개 상태 머신, 한/영 이중 언어, 3단계 난이도(견습생/마법사/대마법사)까지 포함한 완성도 높은 게임이다.

## 잘 된 점 ✅

- **F1~F35 역대 최다 35건 피드백 선제 매핑**: 20사이클 누적 교훈을 카테고리별(에셋/상태머신/코드품질/입력/기타)로 체계화하여 기획서에 반영했고, 플래너 6건 + 디자이너 5건 피드백까지 모두 반영 완료. 피드백 매핑 건수가 32건(Cycle 20) → 35건으로 증가하면서 코드 품질이 비례 향상되는 추세가 6사이클 연속 확인됐다.
- **assets/ 디렉토리 미생성 — 21사이클 만의 진정한 정착**: thumbnail.svg만 존재하고, ASSET_MAP/SPRITES/preloadAssets 코드 자체가 없는 완전한 100% Canvas 코드 드로잉을 달성. 8종 룬·8종 적·5종 보스까지 전부 Canvas API로 직접 그렸다. setTimeout 0건, alert/confirm 0건, 외부 CDN 0건.
- **12상태 디스패치 + ESCAPE_ALLOWED 패턴**: TITLE/STAGE_SELECT/PUZZLE/DEFENSE/RESULT/GAMEOVER/PAUSED/UPGRADE/RECIPE_BOOK/BOSS_INTRO/ENDING/TUTORIAL 12개 상태를 STATE_PRIORITY 맵으로 관리하면서, 6개 역방향 전환 경로도 허용하는 유연한 상태 머신 설계.
- **코드 품질**: 순수 함수 패턴(scanMagicCircles, checkCollision, findLine), 단일 갱신 경로(modifyLives, modifyCrystals, addScore), ObjectPool(파티클 200 + 투사체 50), TweenManager clearImmediate(), ScrollManager(momentum + bounce) 등 21사이클간 검증된 인프라가 완벽히 동작.
- **모바일 완전 대응**: touchSafe 48px, 롱프레스 룬 회수(300ms), 드래그 스크롤(업그레이드/레시피 화면), 그리드 직접 탭 + 인벤토리 탭, touch-action:none 이중 방지 등 터치 입력이 빈틈 없이 구현됐다.

## 아쉬웠던 점 / 개선 가능성 ⚠️

- **5회 리뷰 사이클**: 최종 APPROVED까지 5라운드가 소요됐다. 플래너·디자이너 피드백 반영 과정에서 라운드가 추가된 것이지만, 초회 통과를 목표로 했던 것 대비 라운드 수가 많았다.
- **gridCacheCanvas/Ctx 미사용 변수(P3)**: 오프스크린 캐싱을 위해 선언했으나 실제 구현하지 않은 유령 변수가 잔존. 21사이클째 반복되는 유령 변수 문제로, ESLint `no-unused-vars` 도입 없이는 근절이 어렵다.
- **ObjectPool 예외 안전성(P3)**: `updateAll()` 콜백 내부에서 예외 발생 시 아이템이 풀에 복귀하지 못하는 잠재적 이슈. try-catch 래핑으로 간단히 해결 가능하지만 이번 사이클에서는 미적용.
- **업그레이드 아이콘 캐싱(P4)**: `drawUpgradeIcon()`이 매 프레임 새 경로를 생성하는 비효율. 오프스크린 캐싱으로 최적화 여지가 있다.
- **4,215줄 단일 파일**: 플랫폼 역대 최대 규모. 공용 엔진 모듈 분리(shared/engine.js)가 21사이클째 지연 중이며, TweenManager/ObjectPool/SoundManager/touchSafe()의 copy-paste가 21개 게임에 걸쳐 반복되고 있다.

## 기술 하이라이트 🛠️

이 게임의 기술적 백미는 **마법진 패턴 매칭 알고리즘**이다. `scanMagicCircles()` 순수 함수가 5×5 그리드를 스캔하여 직선3·L자·T자·십자·2×2·대각선 등의 패턴을 실시간으로 감지하고, 매칭된 룬의 속성 조합에 따라 120가지 이상의 마법 효과를 발동한다.

**ScrollManager**도 주목할 만하다. 업그레이드 상점과 레시피북 화면에서 momentum 기반 관성 스크롤(MOMENTUM_DECAY=0.92, MAX_MOMENTUM=30)과 bounce 효과(BOUNCE_FACTOR=0.3)를 Canvas 위에 직접 구현했다. Cycle 20에서 "터치 스크롤 미구현"으로 지적받았던 교훈을 완벽히 반영한 결과다.

**SoundManager**는 Web Audio API 절차적 음향 합성으로 외부 오디오 파일 0건을 유지하면서, 룬 배치/마법진 발동/적 처치/보스 등장 등 상황별 효과음을 ctx.currentTime 기반 네이티브 스케줄링으로 처리한다. setTimeout 완전 배제 10사이클 연속 달성.

## 다음 사이클 제안 🚀
