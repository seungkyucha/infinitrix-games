# 젬 오딧세이 (Gem Odyssey) — 포스트모템

## 한 줄 요약
Royal Match급 레벨 기반 매치3 퍼즐을 8×8 그리드, 6종 장애물, 4종 부스터, 20레벨+보상 스테이지 4개 규모로 완성하여 2회차 리뷰에서 APPROVED를 획득했다.

## 무엇을 만들었나
4개 마법 왕국(보석 정원 → 수정 동굴 → 마법 숲 → 별빛 성)을 여행하며 보석을 매칭하는 레벨 클리어형 매치3 퍼즐이다. 단순 점수 달성 외에 "루비 20개 수집", "얼음 전부 파괴", "젤리 타일 제거" 등 5가지 목표 유형이 존재하여, 같은 보드에서도 목표에 따라 완전히 다른 전략이 요구된다. 제한 턴 안에 목표를 달성하면 별 1~3개를 받고, 별을 모아 왕국 랜드마크를 복원하는 메타 진행이 리플레이 동기를 만든다.

얼음(1~3겹), 체인, 나무 상자, 독(매턴 확산), 돌 블록, 커튼까지 6종 장애물이 5레벨 단위로 점진 도입되어 후반부에는 조합이 꽤 복잡해진다. 망치·셔플·추가 턴·색상 폭탄 4종 부스터와 게임오버 직전 "+5턴 사용하기?" 프롬프트가 전략적 선택의 폭을 넓힌다. 노드 기반 레벨 맵에서 잠금/현재/완료/보너스 노드를 구불구불한 경로로 연결한 시각 디자인은 진행감을 직관적으로 전달한다.

## 잘 된 점 ✅
- **5가지 목표 유형 완전 구현**: 기획서의 score/collect/obstacle/jelly/mixed가 20레벨 전수 반영. 레벨마다 목표가 바뀌어 단조로움 없이 20레벨을 소화할 수 있다.
- **C46 gem-kingdom 피드백 5건 전수 해결**: 부스터 엣지 케이스(F1), 맵-보드 입력 충돌(F2), 극단 난이도(F3), 커튼 장애물 추가(F4), 에셋-코드 교차 검증(F5) — 이전 사이클의 아쉬운 점을 기획서 §0.1에 매핑하고 하나도 빠짐없이 반영한 워크플로우가 돋보인다.
- **에셋 58개 + Canvas 폴백 100%**: manifest.json 58키와 실제 PNG 파일이 1:1 대응. 에셋 로드 실패 시에도 `drawGemFallback` 등으로 게임이 100% 동작한다.
- **BUG-1 수정 완벽 — 2회차 에러 0건**: 1회차에서 발견된 gGrid 빈 배열 접근 크래시를 §17, §45 양쪽 + 낙하 애니메이션에도 가드를 추가하여 완전 해결. 브라우저 테스트 전 흐름 에러 0건.
- **TRANSITION_TABLE 12상태 완전 정의 + 9중 가드 플래그**: 상태 전환 화이트리스트와 `isInputBlocked()` 단일 검증으로 Stuck 상태 0건.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차 BUG-1(반짝임 타이머 gGrid 크래시)**: 타이틀 화면에서 gGrid가 빈 배열인 상태에서의 접근을 놓쳤다. engine._update와 gameUpdate() 양쪽에 중복 코드가 존재하는 구조적 문제가 원인. 리뷰 2회차가 필요했다는 것 자체가 초기 방어 코딩의 부족을 보여준다.
- **§17/§45 중복 코드 미정리**: gameUpdate()와 engine._update에 유사한 로직이 병존. 리뷰에서도 비차단 개선 제안으로 지적. 한 쪽을 삭제하거나 공용 함수로 추출해야 한다.
- **input.flush()의 try-catch 외부 배치**: 에러 발생 시 입력 플러시가 스킵될 수 있는 구조. finally 블록으로 이동하면 더 안전하다.
- **밸런스 자동 검증 부재**: 20레벨+6종 장애물 조합의 난이도를 수동으로만 검증. DDA가 있지만 극단 조합(독+커튼+돌 블록 동시 배치 등)의 클리어율을 정량적으로 모른다.
- **색맹 모드 "준비"만**: getGemShape 함수 구조는 존재하지만 실제 토글 UI 미구현. 6색 구분이 핵심인 게임에서 접근성 옵션은 Phase 2에서 반드시 필요.

## 기술 하이라이트 🛠️
- **매치 검출 우선순위 + used[][] 소비 추적**: 5→T/L→4→3 순서로 검출하되, 한 보석이 여러 매치에 중복 카운트되지 않도록 소비 마킹. 십자 형태에서 가로5+세로3이 아닌 가로5만 인식하는 정확한 처리.
- **2단계 DDA(Dynamic Difficulty Adjustment)**: 기본 DDA(연패 시 턴+2, 목표-2)에 Lv.17+ 극단 전용 DDA(장애물 20% 감소, 독 확산 주기 연장)를 오버레이. 레벨별 `ddaOverride` 필드로 세밀한 튜닝이 가능한 구조.
- **에셋 58개 동적 로딩 + 전수 Canvas 폴백**: manifest.json fetch → 개별 에셋 로드 → 실패 시 Canvas 기반 프로시저럴 드로잉. 보석 6색 + 반짝임 4프레임 스프라이트 시트 애니메이션까지 폴백 커버.
- **노드 기반 레벨 맵 + 관성 스크롤**: 구불구불 경로로 연결된 20+4 노드를 터치 드래그 + 관성 스크롤로 탐색. MAP 상태 전용 InputHandler로 보드 입력과 완전 분리.
- **추가 턴 프롬프트 오버레이**: 게임오버 직전 Canvas 기반 모달로 "+5턴 사용하기?" 제안. confirm/alert 없이 순수 Canvas UI로 구현.
