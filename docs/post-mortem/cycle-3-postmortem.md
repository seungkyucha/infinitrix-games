# 미니 타워 디펜스 — 포스트모템

## 한 줄 요약
8×6 그리드 위에 3종 타워를 배치해 20웨이브를 방어하는 클래식 타워 디펜스를 단일 HTML + Canvas + Web Audio로 구현했으나, 웨이브 클리어 반복 호출 버그로 게임 경제가 붕괴되어 재작업이 필요하다.

## 무엇을 만들었나
판타지 세계 배경의 미니 타워 디펜스 게임이다. 아처(빠른 단일)·마법사(범위 감속)·캐논(폭발 광역) 3종 타워를 각각 3단계까지 업그레이드할 수 있고, 고블린·오크·다크나이트·스피드러너 4종 적이 S자 경로를 따라 밀려온다. 제한된 골드로 어디에 무엇을 배치하느냐가 핵심 전략이며, 한 판 5~10분의 세션 길이를 목표로 했다.

Cycle 1(퍼즐)·Cycle 2(슈팅) 이후 처음으로 전략 장르에 도전했다. TweenManager·ObjectPool·상태 머신 등 이전 사이클에서 검증된 인프라를 재사용하면서, Web Audio API로 절차적 효과음 4종을 새로 추가했다. 또한 Cycle 2에서 미해결이던 `destroy()` 패턴과 `easeOutElastic` 이징도 이번에 완전 구현했다.

## 잘 된 점 ✅
- **Cycle 2 교훈 대부분 반영 성공**: setTimeout 완전 금지(0건), 상태×시스템 매트릭스 코드 주석 포함, TweenManager 모든 상태 업데이트, 점수 "판정 먼저 저장 나중에" 순서 정확 적용. 이전 사이클의 CRITICAL 버그 3건(B1 tween 미업데이트, B3 setTimeout, B4 판정 순서)이 모두 재발하지 않았다.
- **아키텍처 완성도 높음**: 7상태 게임 머신, ObjectPool(적 30·투사체 50·파티클 80), listen() 헬퍼 + destroy() 패턴, 이징 5종 완전 구현. 코드 리뷰에서 PASS 항목이 30개 이상으로 역대 최다.
- **게임 콘텐츠 풍성**: 3종 타워 × 3단계 업그레이드, 4종 적(방어력·감속저항 등 특수 능력), 20웨이브 공식 기반 스케일링, 동적 밸런스 보정(위기 보너스), Web Audio 절차적 SFX 4종까지 외부 에셋 0개로 완성.
- **시각적 완성도**: 순수 Canvas 드로잉으로 타워·적·투사체를 개성 있게 표현하고, 오프스크린 배경 캐시로 렌더링 성능을 확보. 골드 펄스·라이프 셰이크 등 HUD 피드백도 충실.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **B1 [CRITICAL] waveComplete() 반복 호출**: 웨이브 클리어 후 상태 전환 tween(1.5초) 동안 매 프레임 보상이 중복 지급되어 웨이브 1만 클리어해도 골드가 2,380G로 폭증했다. 가드 플래그 1줄이면 해결되는 문제인데 놓친 것이 뼈아프다. "tween 기반 지연 전환"은 성공 패턴이지만, 그 사이 원래 조건이 계속 참인지 반드시 가드해야 한다는 새로운 교훈을 얻었다.
- **B2 [CRITICAL] 게임오버 전환 경쟁**: endGame() tween과 waveComplete() tween이 동시에 실행되면 GAMEOVER가 WAVE_PREP에 덮어씌워져 게임을 끝낼 수 없었다. 상태 전환의 우선순위 체계가 부재했다.
- **B3 consecutiveCleanWaves 미작동**: 동적 밸런스 보정(연속 무손실 클리어 시 난이도 상승)이 기획서에는 있지만 코드에서 사실상 작동하지 않는다. `livesAtWaveStart` 변수가 선언만 되고 갱신되지 않음.
- **B4 SVG 필터 재발**: "SVG 필터 완전 미사용" 원칙을 기획서에 명시했으나 에셋 파일에 feGaussianBlur가 다시 포함됨. Cycle 2 B5와 동일한 패턴의 재발로, 에셋 생성 파이프라인에 자동 검증이 필요하다.
- **시간이 있었다면**: 터치 모드 전용 UI 최적화(버튼 1.5배 확대 등 코드는 있으나 실전 테스트 부족), 무한 모드(20웨이브 이후 계속 플레이), 타워 배치 미리보기 드래그.

## 기술 하이라이트 🛠️
- **Web Audio API 절차적 사운드**: 외부 오디오 파일 없이 OscillatorNode + GainNode 조합으로 4종 효과음을 런타임 생성. 아처(고음 삐), 마법사(주파수 스윕), 캐논(저음 쿵), 처치(팝) 등 각 타워 특성에 맞는 사운드를 만들었다. try-catch 래핑으로 AudioContext 미지원 환경에서도 게임플레이에 영향 없음.
- **상태×시스템 매트릭스**: 7상태 × 9시스템 매트릭스를 기획서에 정의하고 코드 주석으로 이중 포함. Cycle 2의 CONFIRM_MODAL tween 버그가 재발하지 않은 직접적 원인이다.
- **S자 경로 + First 타겟팅**: 웨이포인트 인덱스 + 구간 내 진행률을 결합해 "가장 앞선 적"을 정확히 계산. 타워 디펜스의 핵심인 타겟팅 로직을 깔끔하게 구현했다.
- **game.destroy() 패턴 표준화**: registeredListeners 배열로 모든 addEventListener를 추적하고, destroy()에서 cancelAnimationFrame + 리스너 해제 + 풀 정리 + AudioContext 닫기를 일괄 처리. SPA 환경 메모리 누수 방지.
