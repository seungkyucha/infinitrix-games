# 어비스 키퍼 (Abyss Keeper) — 포스트모템

## 한 줄 요약
심해 등대지기가 되어 낚시·채집(캐주얼)과 괴수 격퇴(액션)를 번갈아 수행하는 이중 페이즈 서바이벌을 순수 Canvas로 구현하고, 2차 리뷰 2회차 만에 APPROVED를 달성했다.

## 무엇을 만들었나
어비스 키퍼는 끝없는 심해의 마지막 등대를 지키는 수호자의 이야기다. 낮에는 타이밍 기반 낚시 미니게임과 표류물 채집으로 자원을 모아 등대를 강화하고(캐주얼 페이즈), 밤에는 심연에서 밀려오는 괴수들을 작살·음파포·그물 3종 무기로 격퇴하며 등대를 방어한다(액션 페이즈). 두 페이즈가 조수(Tide) 단위로 자연스럽게 순환하면서, 캐주얼 유저는 낚시와 건설에, 액션 유저는 전투에 몰입할 수 있는 구조다.

15개 조수 스테이지, 3개의 심연 보스, 히든 스테이지 "마리아나 해구", 등대 업그레이드 트리, 무기 제작, 날씨/시간대 변화까지 — 단일 HTML 파일 안에 SeededRNG 프로시저럴 파도 패턴과 동적 난이도 조정 시스템을 갖춘 풍성한 콘텐츠를 담았다. 플랫폼에서 **action + casual = 0개**였던 장르 공백을 이 게임으로 완전히 해소했다.

바이올루미네센스(생물 발광) 파티클, 등대 조명 확산, 3겹 sin 파도 애니메이션 등 심해 분위기를 Canvas API만으로 표현하여, 외부 에셋 없이도 시각적 몰입감을 끌어올렸다.

## 잘 된 점 ✅
- **이중 페이즈 시스템의 완성도**: 기획서에서 의도한 "캐주얼 → 액션 자연스러운 순환"이 16개 게임 상태 + ACTIVE_SYSTEMS 매트릭스(16×12)로 정확히 구현됐다. 조수 전환, 보스 진입, 히든 스테이지 해금까지 모든 상태 전환이 STATE_PRIORITY + RESTART_ALLOWED 패턴으로 경쟁 조건 0건이다.
- **RESTART_ALLOWED 패턴의 기획서 단계 선제 적용(F44)**: Cycle 21~23에서 5라운드 동안 P0으로 남았던 GAMEOVER→TITLE 버그를, 이번에는 기획서 §6.1에 화이트리스트를 명시적으로 선언하여 1차 리뷰에서 바로 해결했다. 기획서 선제 대응의 성공 사례.
- **바이올루미네센스 시각 연출**: 심해 파티클, 등대 조명 확산, 캐주얼→액션 하늘 그라디언트 전환(tween 3초) 등이 Canvas 코드 드로잉만으로 구현되어, assets/ 삭제 후에도 렌더링 품질이 완벽하게 유지됐다.
- **F1/F2 플랫폼 규정 최종 준수**: 외부 에셋 0건, 외부 CDN 0건. assets/ 디렉토리에는 플랫폼 필수 파일(manifest.json + thumbnail.svg)만 존재. ASSET_MAP/SPRITES/preloadAssets 코드 완전 삭제. assets/ 미생성 **8사이클 연속**(Cycle 18~24) 달성.
- **코드 품질**: ObjectPool 패턴, Math.hypot 거리 기반 충돌 감지, rAF + dt 캡(50ms), 프레임당 DOM 접근 0건, F8 패턴(판정 먼저 저장 나중), eval/XSS 0건 등 모든 코드 품질 항목 PASS.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **2차 리뷰 2회차 소요(총 3라운드)**: 1차에서 B1(RESTART_ALLOWED 누락, P0), B2(transAlpha 미연결, P1), B4(터치 48px 미달, P2)가 발견됐고, 2차 1회차에서도 B3(assets/ F1 위반, P1)과 B5(Google Fonts F2 위반, P1)가 잔존했다. 스모크 테스트 게이트 10항목(F47)을 기획서에 명시했으나 초기 구현에서 완전히 관철되지 못했다.
- **Google Fonts 잔존(B5)**: "시스템 폰트만 사용"이 원칙인데 `<link>` 태그 2줄이 남아있었다. 2차 리뷰에서 삭제했으나, 주석 "Google Fonts with fallback"(line 148)이 여전히 잔존한 P3 이슈가 남았다.
- **에셋 코드 잔존(B3)**: ASSET_MAP/SPRITES/preloadAssets가 1차에서 발견됐다. Canvas fallback이 완비되어 삭제만 하면 되지만, "처음부터 생성하지 않는" 원칙이 아직 100% 체화되지 않았음을 보여준다.
- **밸런스 자동 검증 미비(F46)**: 15조수×3난이도×다수 무기 조합의 밸런스를 코드 리뷰만으로 판별할 수 없다. DPS/EHP 산식을 기획서에 명시했으나 자동 시뮬레이터는 아직 부재.
- **단일 파일 규모**: 이중 페이즈 + 보스 + 업그레이드 트리 등 콘텐츠 볼륨이 많아 코드 규모가 대형. 공용 엔진 모듈 분리(shared/engine.js)가 24사이클째 지연 중이다.

## 기술 하이라이트 🛠️
- **16개 게임 상태의 이중 페이즈 시스템**: TITLE/DIFF_SELECT/CASUAL_PHASE/FISHING/UPGRADE/ACTION_PHASE/BOSS_FIGHT/TIDE_RESULT 등 16개 상태를 STATE_PRIORITY + RESTART_ALLOWED + beginTransition으로 무결하게 관리. 캐주얼↔액션 전환이 tween 기반 페이드로 부드럽게 연결된다.
- **SeededRNG 프로시저럴 콘텐츠**: LCG 기반 시드 RNG로 파도 패턴, 어획물, 괴수 스폰이 매 플레이마다 다르면서도 재현 가능. 동적 난이도 조정과 결합되어 쉬움/보통/어려움 3단계를 넘어서는 세밀한 밸런스 곡선을 달성.
- **ACTIVE_SYSTEMS 매트릭스(F7)**: 16개 상태 × 12개 시스템의 완전 정의 매트릭스로, 어떤 상태에서 어떤 시스템이 동작해야 하는지를 기획서와 코드에 이중으로 명시. Cycle 2 B1(CONFIRM_MODAL tween 멈춤) 이래 22사이클간 검증된 패턴의 최대 규모 적용.
- **3종 무기 시스템**: 작살(단일 관통), 음파포(범위 넉백), 그물(감속 설치) — 무기별 차별화된 물리 연산을 Canvas 코드 드로잉과 결합. 1/2/3 키 전환 + 모바일 WPN 터치 버튼(48px+ 보장).
- **바이올루미네센스 파티클 시스템**: sin 곡선 기반 자연스러운 발광 움직임, 등대 조명과 상호작용하는 광원 확산 효과를 Canvas globalAlpha + gradient로 구현.
