# 비트 크래프터 (Beat Crafter) — 포스트모템

## 한 줄 요약
떨어지는 음표 블록을 그리드에 배치하면 화음이 울려 퍼지는, "퍼즐을 풀면 음악이 만들어지는" 리듬 퍼즐 게임을 만들었다.

## 무엇을 만들었나
비트 크래프터는 InfiniTriX 플랫폼 최초의 음악 기반 게임이다. 4×8 그리드 위에 도·레·미·파·솔 5종 음표 블록이 떨어지고, 같은 음 3개를 가로나 세로로 정렬하면 Web Audio API가 해당 음계의 화음을 연주하며 클리어된다. 테트리스의 블록 배치 전략에 매치3의 정렬 쾌감, 그리고 절차적 음악 생성을 결합해 매 게임마다 플레이어가 다른 멜로디를 "작곡"하는 경험을 제공한다.

핵심 재미는 연쇄 클리어(체인)에서 터진다. 한 번의 배치로 블록이 소멸하고, 중력으로 재배치된 블록이 다시 매치되면 화음이 겹겹이 쌓이며 점수가 폭발한다. 레벨이 오르면 BPM이 80에서 145까지 상승하고 낙하 속도가 빨라지면서 "이번엔 더 버텨보자"는 생존 긴장감이 자연스럽게 형성된다. 레벨 1~2에서는 3종 음표로 입문하고, 레벨 5부터 전체 5종이 등장하며, 위기 시 매치 유리 블록을 생성하는 동적 밸런스 보정으로 초보자도 포기하지 않게 설계했다.

## 잘 된 점 ✅
- **4사이클 연속 재발했던 SVG 에셋 문제를 코드 레벨에서 완전히 해결했다.** 기획서 §13.5의 자동 grep 검증 스크립트를 실제 도입하여 금지 패턴 7개 항목 전부 0건 PASS. 코드 내 SVG/이미지 참조가 단 한 건도 없다.
- **Cycle 4 B1 CRITICAL(TweenManager cancelAll+add 경쟁 조건)을 `clearImmediate()` API 분리로 깔끔하게 해결.** resetGame()에서 즉시 정리를 사용하여 경쟁 조건을 원천 차단했고, 리뷰에서 동작 확인 완료.
- **enterState() 패턴이 처음 적용되어 5상태 모두 초기화 로직을 한 곳에서 관리.** Cycle 4의 타이틀 글로우 미복구 같은 문제가 구조적으로 재발 불가능해졌다.
- **기획서 §13.1~§13.4 전 항목 100% 구현, 변수 사용처 전수 검증 유령 코드 0건.** score 3곳, level 6곳, inputMode 4곳 등 10개 주요 변수가 모두 기획 의도대로 정확히 사용되었다.
- **1회차 리뷰 APPROVED, 브라우저 테스트 PASS.** 발견된 이슈 3건 모두 MINOR/COSMETIC으로 배포 차단 사유 없음. 5사이클 중 가장 깔끔한 리뷰 결과.
- **Web Audio 절차적 음악이 핵심 게임플레이로 동작.** 착지 단음 → 클리어 화음 → 체인 옥타브+베이스 → 배경 메트로놈까지 모든 사운드가 코드로 생성되며, 오디오 실패 시에도 시각만으로 정상 동작.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리에 미사용 SVG 9개 + manifest.json이 잔존한다.** 코드에서 일체 참조하지 않으므로 기능 무영향이지만, 범용 템플릿에서 복사되는 이 파일들을 자동으로 정리하는 메커니즘이 없다. 빌드 파이프라인에 "코드 내 미참조 에셋 경고" 기능이 필요하다.
- **startGame()/goToTitle()에서 beginTransition() 대신 state를 직접 변경한다.** 기획서 §9에서 "모든 상태 전환에 beginTransition() 필수"를 명시했으나 두 함수는 이를 우회한다. clearImmediate()로 경쟁 조건은 없지만, TITLE→PLAYING fadeOut 300ms 전환 애니메이션이 누락되어 전환이 뚝 끊긴다.
- **LOADING 상태가 데드 코드다.** 외부 에셋 없이 동기 초기화하므로 LOADING + renderLoading()이 실행될 기회가 없다. 향후 확장성 보존이라고 볼 수도 있지만, 현재로서는 불필요한 코드.
- **4열 그리드에서 3매치 밸런스 검증이 제한적.** headless 테스트에서는 실제 플레이 밸런스(너무 쉬운지/어려운지)를 체감할 수 없다. 레벨별 음표 풀 확장(3→4→5종)이 의도대로 난이도를 조절하는지 실플레이 확인이 필요하다.
- **시간이 있었다면:** 다중 블록 동시 낙하(테트리스 미노 형태), 특수 블록(와일드카드, 폭탄), 커스텀 스케일(메이저/마이너 전환) 같은 확장 요소를 넣어 전략 깊이를 더 높이고 싶었다.

## 기술 하이라이트 🛠️
이번 게임의 기술적 하이라이트는 **MatchSystem의 효율성과 Web Audio 화음 생성의 결합**이다. 매치 판정은 4×8 = 32셀만 순회하는 O(32) 경량 알고리즘으로, 가로/세로 연속 검사 후 THREE/FOUR/CROSS 3단계로 분류한다. 클리어 → 중력 → 재매치의 체인 루프가 tween onComplete 재귀로 자연스럽게 처리되며, 각 체인 단계마다 `sfxClearChord()`가 옥타브 위 화음과 베이스를 겹쳐 풍성한 사운드를 만든다.

**offscreen canvas 격자 캐시**는 매 프레임 반복되는 격자선 드로잉을 최초 1회만 실행하고, 이후에는 drawImage()와 globalAlpha 조절만으로 비트 반응형 배경을 구현한다. 5종 블록의 고유 도형(원/다이아몬드/삼각형/사각형/별)은 색각이상 접근성까지 고려한 설계다. DAS(Delayed Auto Shift)를 dt 누적 방식으로 구현하여 setTimeout 완전 배제 원칙을 유지했다.

## 다음 사이클 제안 🚀
1. **공용 엔진 모듈 분리 (`shared/engine.js`)** — TweenManager(clearImmediate), ObjectPool, TransitionGuard(enterState), listen()+destroy(), Web Audio SFX가 5사이클간 안정화 완료. 공용 모듈로 분리하면 보일러플레이트를 대폭 줄이고 버그 수정이 전 게임에 반영된다.
2. **물리 기반 장르 도전** — 퍼즐(C1)→슈팅(C2)→전략(C3)→러너(C4)→리듬퍼즐(C5) 이후 미개척 영역. 간단한 2D 물리(중력, 반사, 마찰)를 기존 인프라 위에 얹는 미니 골프나 투사체 게임을 시도하면 완전히 새로운 경험축이 열린다.
3. **비동기 경쟁 시스템 프로토타입** — 5게임 모두 솔로 플레이. localStorage 기반 "오늘의 시드" 일일 챌린지로 외부 서버 없이 경쟁감을 추가하면 리플레이 동기가 크게 강화된다.
