# 물들인 하늘 — 포스트모템

## 한 줄 요약
수채화 페인터리 아트 위에 극좌표 탄막 패턴이 펼쳐지는 종스크롤 불릿헬 슈터를 단일 HTML로 구현하고, 6차 리뷰까지 거치며 A~H 전 항목 PASS로 APPROVED를 받았다.

## 무엇을 만들었나
**물들인 하늘**은 먹구름에 잠식된 천상의 채색 정원을 되찾기 위해 빛의 수호자 '아리아'가 빽빽한 탄막 사이를 누비는 종스크롤 탄막 슈터다. 새벽 하늘→석양 들판→별빛 바다→먹구름 성채의 4스테이지를 거치며 각기 다른 극좌표 탄막 패턴(나선형, 방사형, 파동형, 꽃잎형 등 11종)을 구사하는 보스를 격파해야 한다. 4×4px 극소 히트박스로 "불가능해 보이는 탄막도 통과 가능"한 아슬아슬한 회피가 핵심 쾌감이다.

Ori/Gris에서 영감받은 수채화 배경 위에 코랄 핑크·라벤더·아이스블루의 탄막이 쏟아지는 비주얼은 "아름다우면서 긴장감 있는 탄막의 춤"이라는 기획 의도를 충실히 실현했다. 스테이지 사이 3택 파워업(12종)과 카테고리 조합 시너지(4종), 메타 업그레이드(빛의 조각) 시스템이 반복 플레이 동기를 부여하며, 그레이즈 콤보 시스템은 위험에 가까이 갈수록 높은 점수를 주는 리스크-리워드 루프를 만들어낸다.

## 잘 된 점 ✅
- **C4 아쉬운 점 전부 해결**: 에셋 포맷 SVG 통일(PNG 0개), 모바일 가로 모드 유도 오버레이, 보스 페이즈 전환 3초 연출+예고선 힌트+점진적 탄막 밀도 증가(30%씩), 파워업 조합 시너지 4종 — 기획서에서 명시한 C4 피드백 해결 방안을 모두 구현했다.
- **탄막 슈터 장르 완성도**: 11종 탄막 패턴 타입, 4보스×2페이즈, 그레이즈 콤보, 폭탄 시스템, 저속 이동(히트박스 표시)까지 장르 필수 요소를 빠짐없이 담았다. Puppeteer 3회 사이클(수채/먹물/진묵 각 1회) 모두 에러 0건.
- **에셋 57개 SVG 완전 일관성**: painterly-2d 아트 디렉션을 56 manifest + thumbnail까지 일관 유지. 리뷰 G-1, G-2, G-3 전 항목 PASS.
- **17개 버튼 전수 PASS**: 8개 씬에 걸쳐 마우스·터치(48px+)·키보드 3방식 모두 동작 확인.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **6차 리뷰까지 소요**: 초기 리뷰에서 `resetAll()` 스코핑 버그, 에셋 56개 완전 생성, POWERUP/SANCTUARY 외부 함수 분리, SLOW 버튼 기능 미구현 등 HIGH 항목이 발견됐다. 셀프 QA 14항목 프로토콜이 있었음에도 스코프가 커지면서(48에셋, 9씬, 12파워업) 검증 사각지대가 생겼다.
- **viewport resize 미대응**: 브라우저 리사이즈 시 현재 씬 버튼이 재배치되지 않는 문제가 권장 개선으로 남았다. iframe 환경에서는 문제없지만, 독립 실행 시 UX가 깨질 수 있다.
- **SLOW 버튼 시각 피드백 부재**: focusToggle 기능은 동작하나 활성 상태를 시각적으로 표시하지 않아 모바일에서 현재 상태 파악이 어렵다.
- **코드 규모 관리**: 1000~1300줄 예상 대비 실제 규모가 커졌을 가능성. 4스테이지×보스 패턴+12파워업+시너지+메타 업그레이드를 단일 파일에 담는 것은 한계에 근접한다.

## 기술 하이라이트 🛠️
**극좌표 탄막 패턴 엔진**: 11종 탄막 타입(radial, spiral, aimed, fan, wave, rain, burst, combined, gravity, blackout, pattern)을 JSON 선언적 데이터로 정의하고, 런타임에 극좌표→직교좌표 변환으로 렌더링한다. 보스 패턴을 코드가 아닌 데이터로 분리함으로써 패턴 추가·수정이 용이해졌다. `combined` 타입은 여러 패턴을 동시에 조합하고, `blackout`은 시야 차단이라는 비전통적 탄막 기법도 데이터로 표현 가능하다.

**오브젝트 풀 기반 대량 탄막 관리**: enemyBullets(200~400), playerBullets(30~60), particles(100~200) 등 5종 풀로 적탄 300발+파티클 100개 동시 처리를 GC 스파이크 없이 60fps로 유지한다. 공간 해싱(64×64 그리드)으로 충돌 판정을 O(n)으로 최적화하고, 그레이즈 판정(20px)과 히트박스 판정(4×4px)을 동일 해싱 구조에서 처리한다.

**파워업 시너지 시스템**: 12종 파워업을 공격/방어/특수 3카테고리로 분류하고, 동일 카테고리 3개 이상 수집 시 시너지 효과를 발동한다. `checkSynergies()` 함수가 카테고리 카운트를 실시간 집계하여 "빛의 분노"(공격×3)~"조화의 빛"(각 1+) 4종 시너지를 판정한다.

## 다음 사이클 제안 🚀
1. **퍼즐/매치3 장르 도전**: 로그라이크(C1)→슈터(C2)→덱빌더(C3)→메트로배니아(C4)→불릿헬(C5)로 액션·전략 장르를 5사이클간 집중 커버했다. 이제 완전히 다른 인터랙션 축인 퍼즐·리듬 계열을 탐험할 때다.
2. **대형 스코프 관리 전략**: C5에서 48에셋+9씬+12파워업+메타 업그레이드의 스코프가 셀프 QA 사각지대를 만들었다. 다음엔 기능 단위 체크포인트(에셋 → 코어 루프 → 서브시스템 → 통합)를 도입해보자.
3. **런타임 상태 시각화 디버그 도구**: SLOW 버튼 피드백 부재처럼 "상태가 보이지 않는" 문제가 반복된다. 개발 중 오브젝트 풀 사용량, 현재 시너지 상태, 히트박스 등을 실시간 표시하는 디버그 오버레이를 시도해보자.
