# 미니 로그 다이스 — 포스트모템

## 한 줄 요약
이번 사이클에서 첫 시도(프루츠 머지)가 초기화 버그로 좌절한 뒤 재기획하여, 12개 상태 머신 기반 턴제 로그라이트 주사위 게임을 구현하고 1회차 리뷰에서 APPROVED를 달성했다.

## 무엇을 만들었나
미니 로그 다이스는 3층 던전을 탐험하며 주사위를 굴려 전투하는 턴제 로그라이트 게임이다. 3×3 던전 맵에서 방을 선택해 탐험하고, 전투에서는 주사위를 굴린 뒤 장비 슬롯에 배치하여 공격·방어·회복을 결정한다. 12종 장비, 10종 적, 3보스, 6종 이벤트, 6종 메타 업그레이드가 매 런마다 다른 전략을 요구하며, 사망 시 골드 50%를 유지하는 메타 성장 루프가 "한 판만 더"의 중독성을 만든다.

이 게임의 핵심 재미는 주사위 결과를 어디에 배치할지 고민하는 **의사결정의 깊이**에 있다. 높은 눈금을 공격에 쓸지 방어에 쓸지, 장비 특수 효과를 어떻게 조합할지 매 턴 고민이 발생한다. 적의 다음 행동 예고("다음: 공격 2")로 정보에 기반한 전략적 판단이 가능하다. 8개 SVG 에셋(플레이어, 보스, 배경 2레이어, UI 아이콘 등) + 프로시저럴 적 드로잉(10종 도형)으로 시각적 완성도도 갖췄다.

첫 시도였던 프루츠 머지가 이벤트 리스너 초기화 순서 버그로 NEEDS_MAJOR_FIX 판정을 받은 뒤, 재기획을 거쳐 완전히 새로운 게임으로 구현했다는 점에서 이번 사이클은 "실패에서 배워 성공한" 전형적인 복구 사례다.

## 잘 된 점 ✅
- **코드 리뷰 14항목 + 브라우저 테스트 12항목 전원 PASS**: 1회차 리뷰에서 APPROVED를 달성한 것은 28건 피드백 매핑의 효과가 여실히 드러난 결과다. setTimeout 0건, eval/alert 0건, 콘솔 에러 0건.
- **12개 상태 머신의 안정적 구현**: TITLE/META_SHOP/DUNGEON_MAP/BATTLE/REWARD/EVENT/SHOP/REST/GAMEOVER/PAUSE 등 12개 상태가 STATE_MATRIX(12×5)와 STATE_PRIORITY 맵으로 정밀하게 제어된다. beginTransition() 이중 호출 방지까지 완비.
- **에셋 통합 우수**: 8개 SVG 에셋이 manifest.json과 1:1 대응하며 프리로드 완료. 보스/플레이어에 에셋, 일반 적에 프로시저럴 드로잉이라는 하이브리드 접근이 시각적 위계를 자연스럽게 형성.
- **접근성 3중 표현**: 주사위를 숫자 + 도트 + 색상으로 3중 구분하여 색각이상 접근성 확보. 장비 타입별 색상 코딩(ATK 빨강, DEF 파랑, HEAL 초록, SPECIAL 보라)도 직관적.
- **모바일 터치 완전 대응**: touchSafe() 유틸로 모든 버튼 최소 48px 보장, passive:false + touch-action:none, 모바일 Audio Context resume 처리까지 완비.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **첫 시도(fruits-merge) 실패로 사이클 자원 소모**: 프루츠 머지의 초기화 순서 버그는 "이벤트 리스너를 init() 내부로 이동"하는 1건 수정으로 해결 가능했다. 스모크 테스트 게이트가 있었다면 재기획 없이 수정 후 통과할 수 있었을 것이다.
- **사운드 시스템 미확인**: 리뷰 보고서에서 Web Audio 관련 세부 테스트(음악, 효과음 다양성)가 명시적으로 확인되지 않았다. 턴제 로그라이트에서 전투 타격음, 주사위 굴림 소리, 보스 BGM 등이 체감에 큰 영향을 미치므로 사운드 체감 테스트를 별도 항목으로 추가하면 좋겠다.
- **밸런스 실플레이 검증 부재**: 12종 장비 × 10종 적 × 3보스의 밸런스가 headless 테스트로는 검증 불가하다. 자동 시뮬레이션(N회 런 평균 클리어율)이나 분석 리포트 연동이 필요하다.
- **시간이 있었다면**: 장비 조합 시너지(검+방패 세트 효과), 엘리트 적 변이, 잠금 해제 실적 시스템, 일일 챌린지 모드를 추가했을 것이다.

## 기술 하이라이트 🛠️
이번 게임의 가장 흥미로운 기술적 성취는 **12개 상태 × 5개 시스템 매트릭스의 안정적 구현**이다. STATE_MATRIX(L123-136)가 각 상태에서 업데이트할 시스템을 명확히 정의하고, STATE_PRIORITY 맵이 경쟁 조건을 방지한다. TransitionGuard의 `_transitioning` 플래그가 이중 전환을 원천 차단하는 패턴은 Cycle 4에서 시작된 이래 가장 복잡한 상태 머신(12개)에서도 안정적으로 동작함을 증명했다.

ObjectPool(파티클 50 + 팝업 15)과 TweenManager.clearImmediate()의 조합, 순수 함수 패턴(canPlaceDice, calcEquipEffect, getEnemyAction, rollDice 등 전역 의존 없음), Web Audio 네이티브 스케줄링(`ctx.currentTime + offset`), offscreen Canvas 배경 캐시, DPR 기반 고해상도 처리 등 14사이클간 누적된 인프라가 턴제 로그라이트라는 새 장르에서도 수정 없이 동작했다.
