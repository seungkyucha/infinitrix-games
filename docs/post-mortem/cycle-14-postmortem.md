# 미니 던전 다이스 — 포스트모템

## 한 줄 요약
첫 시도(프루츠 머지)가 초기화 버그로 좌절한 뒤 재기획하여, 주사위 4종 × 5층 던전 17전투 × 보스 3페이즈의 턴제 로그라이트를 완성하고 2차 리뷰에서 APPROVED를 달성했다.

## 무엇을 만들었나
미니 던전 다이스는 주사위를 굴려 던전을 탐험하는 턴 기반 로그라이트다. 매 전투에서 3~6개의 주사위를 굴린 뒤 공격/방어/회복 슬롯에 배치하여 전투를 해결하고, 5층 보스를 처치하면 승리한다. 공격(빨강)·방어(청록)·회복(파랑)·만능(노랑) 4종 주사위를 전략적으로 운용하는 것이 핵심이다.

"이번 턴, 높은 눈금을 공격에 몰빵할까 방어를 두텁게 할까?" — 매 턴 이 한 가지 의사결정에 모든 전략이 집중된다. 층 클리어 보상으로 주사위 면 업그레이드와 새 주사위 획득이 가능해 "내 주사위가 강해진다"는 성장 쾌감이 있고, 재굴림(턴당 1회, 층당 2회) 시스템이 운과 전략 사이의 긴장감을 만든다. 한판 3~5분이라 "한판만 더" 중독성이 높다.

이번 사이클은 프루츠 머지(수박게임 클론)가 이벤트 리스너 초기화 순서 버그로 NEEDS_MAJOR_FIX를 받은 뒤, 완전히 새로운 게임으로 재기획한 복구 사례다. 첫 시도의 실패 원인("init() 외부 이벤트 리스너 등록")을 분석하고 28건 피드백을 재매핑하여 2차 리뷰에서 APPROVED를 달성했다.

## 잘 된 점 ✅
- **2차 리뷰 APPROVED — 코드 리뷰 9항목 + 브라우저 테스트 8항목 + 모바일 8항목 전원 PASS**: 기획서 §0의 17건 피드백 매핑(F1~F17)이 효과적으로 작동했다. setTimeout 0건, eval/alert 0건, 콘솔 에러 0건 달성.
- **9개 상태 머신 안정 동작**: TITLE→DUNGEON_MAP→DICE_ROLL→DICE_PLACE→BATTLE_RESOLVE→REWARD→GAME_OVER/VICTORY/PAUSED 전환이 TransitionGuard 패턴으로 무결하게 관리된다. beginTransition() 이중 호출 방지도 완비.
- **에셋 하이브리드 전략 성공**: SVG 8개(플레이어·보스·배경·UI 아이콘)를 프리로드하되, 모든 사용처에 `if (SPRITES.xxx)` 가드 + Canvas 폴백을 완비하여 에셋 없이도 100% 동작. 시각적 품질과 견고함을 동시에 달성.
- **ObjectPool + offscreen Canvas 캐시**: 파티클 50개·팝업 20개 사전 할당으로 GC 부담 최소화. 배경 bgCache로 매 프레임 재생성 방지.
- **모바일 완벽 대응**: touchstart/touchmove/touchend 3종 + passive:false + touch-action:none. 모든 버튼 48×48px 이상 강제. 턴제라 가상 조이스틱 불필요 — 탭만으로 전체 플로우 완결.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **첫 시도(fruits-merge) 실패로 사이클 자원 소모**: 프루츠 머지의 초기화 순서 버그는 "이벤트 리스너를 init() 내부로 이동"하는 1건 수정으로 해결 가능했다. 스모크 테스트 3단계 게이트가 있었다면 재기획 없이 수정만으로 통과 가능했을 것이다.
- **assets/ 디렉토리 잔존**: F6("assets/ 생성 금지")를 기획서에 명시했으나 에셋 디렉토리가 여전히 물리적으로 존재한다. 코드에서 폴백이 완비되어 기능 이슈는 없지만, 14사이클 연속 반복되는 이 패턴은 CI 파이프라인 강제 없이는 해결 불가능함이 다시 확인되었다.
- **밸런스 검증 부재**: 주사위 4종 × 적 편성 × 보상 조합의 밸런스가 headless 테스트로는 검증 불가하다. 자동 시뮬레이션(N회 AI 랜덤 런 → 클리어율/층별 사망률 통계)이 턴제 게임 밸런스 개선의 다음 단계.
- **사운드 체감 미검증**: Web Audio 코드 존재와 모바일 resume는 확인했으나, 전투 타격음·주사위 굴림·보스 BGM 등 실제 청각 피드백의 품질은 리뷰 항목에 포함되지 않았다.

## 기술 하이라이트 🛠️
가장 흥미로운 기술적 성취는 **14사이클 누적 인프라의 장르 범용성 재증명**이다. TweenManager, ObjectPool, TransitionGuard, SoundManager(Web Audio 네이티브 스케줄링), delta time 게임 루프(33ms 상한), DPR 고해상도 처리 — 이 모든 것이 액션/아케이드에서 시작해 턴제/아이들/레이싱/플랫포머를 거쳐, 이번 턴제 로그라이트에서도 수정 없이 동작했다.

기획서 §0의 피드백 매핑 17건(F1~F17)이 체계적으로 이전 사이클 교훈을 반영한 점도 주목할 만하다. CONFIG.MIN_TOUCH_TARGET 직접 참조(F1), setTimeout 완전 배제(F2), init() 내부 이벤트 등록 강제(F3), 버튼 너비·높이 독립 48px 보장(F4) 등 반복 지적 사항을 기획 단계에서 선제 차단한 것이 2차 리뷰 APPROVED의 핵심 요인이다. 12개 순수 함수(F12)와 게임 루프 try-catch(F13)도 안정성에 크게 기여했다.
