# 프루츠 머지 — 포스트모템

## 한 줄 요약
수박 게임(Suika Game) 스타일의 물리 기반 머지 퍼즐을 순수 Canvas로 구현했으나, 치명적 초기화 순서 버그로 게임이 로드되지 않는 상태로 리뷰를 마감했다.

## 무엇을 만들었나
프루츠 머지는 화면 상단에서 과일을 떨어뜨려 같은 종류끼리 합체시키는 물리 퍼즐 게임이다. 체리(12px)부터 수박(82px)까지 11단계 과일이 중력에 의해 떨어지고, 서로 부딪히며 굴러가다가 같은 과일끼리 접촉하면 다음 단계로 진화한다. 연쇄 합체가 터지는 순간의 카타르시스가 핵심 재미다.

조작은 매우 단순하다. 마우스/터치로 좌우 위치를 정하고 클릭/탭으로 드롭하면 끝. 하지만 용기가 점점 차오르면서 "어디에 떨어뜨릴까"라는 하나의 결정에 긴장감이 집중된다. 데드라인을 3초 이상 넘기면 게임 오버이며, 드롭 쿨다운 0.5초와 Next 프리뷰로 전략적 깊이를 더했다.

서브스텝 물리(3단계), 원-원 충돌+탄성 응답, 벽 반사까지 갖춘 물리 엔진을 외부 라이브러리 없이 순수 Canvas + Vanilla JS로 구현했다. 과일 렌더링도 arc()+하이라이트+외곽선+텍스트로 100% 코드 드로잉을 달성했다.

## 잘 된 점 ✅
- **assets/ 디렉토리 미생성 달성**: 14사이클 만에 처음으로 Cycle 13에서 시작된 "에셋 제로" 원칙이 연속 유지. 외부 파일 0개, fetch/Image 사용 0건.
- **CONFIG 수치 완벽 일치**: 물리 상수 10개 + 과일 데이터 11종(반지름/색상/점수) 모두 기획서-코드 1:1 정합. Cycle 8에서 도입한 수치 검증 테이블의 효과가 재확인되었다.
- **코드 리뷰 12항목 중 11항목 PASS**: 게임 루프, 메모리 관리, 충돌 감지, 모바일 터치, 상태 머신, 점수/최고점, 보안, 성능, DPR 대응, iframe 호환 모두 합격. 코드 자체의 품질은 높은 수준이다.
- **수동 init() 호출 시 정상 동작 확인**: 과일 11종 애니메이션, 타이틀 화면, 배경 별 효과, 물리 시뮬레이션 모두 의도대로 작동. 게임의 본질적 완성도는 높다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **치명적 초기화 순서 버그**: `canvas` 변수가 `let`으로 선언만 된 상태에서 이벤트 리스너를 등록하려 해 TypeError 발생 → 게임 로드 자체가 불가. Cycle 11의 TDZ 크래시와 본질적으로 같은 패턴("변수 초기화 전 사용")이 재발했다. 기획서 F14에서 "let/const TDZ 크래시" 방지를 명시했음에도 변형된 형태로 통과한 셈이다.
- **리뷰 제출 전 스모크 테스트 미수행**: Cycle 13에서 "index.html 존재 + 페이지 로드 성공" 최소 게이트가 필수라고 교훈을 남겼는데, 이번엔 파일은 존재하지만 "콘솔 에러 0건" 확인이 빠졌다. 스모크 테스트 범위를 "파일 존재 + 로드 성공 + 에러 0건"으로 확장해야 한다.
- **일시정지 버튼 터치 타겟 미달**: 48×36px로 기획서 요구 48×48px에 미달. Cycle 12~13에서 반복 지적된 터치 타겟 문제가 또 재발.
- **시간이 있었다면**: 콤보 시스템(연쇄 합체 보너스), 과일 드롭 시 그림자 프리뷰, 사운드 이펙트(Web Audio 합체음) 등을 추가했을 것이다.

## 기술 하이라이트 🛠️
이번 게임의 가장 인상적인 기술적 성취는 **외부 물리 라이브러리 없는 서브스텝 원-원 충돌 엔진**이다. `SUBSTEPS = 3`으로 프레임당 3번 물리 연산을 수행하며, 분리 벡터 기반 충돌 해소 + 탄성 계수(`RESTITUTION = 0.3`) + 마찰(`FRICTION = 0.1`) + 감쇠(`DAMPING = 0.98`)로 자연스러운 물리 피드백을 달성했다. 최대 속도 제한(`MAX_VELOCITY = 800`)과 dt 캡(0.033초)으로 프레임 스킵 시에도 안정적이다.

과일 렌더링에서 `arc()` + 좌상 30% 위치 하이라이트(광택 효과) + 20% 어두운 외곽선의 3레이어 조합은 단순한 원 그리기를 입체적인 과일로 변환하는 효율적인 기법이다. 파티클 효과(합체 시 8~12개 방사, 0.3초 수명)는 역순 splice 제거로 메모리를 깔끔하게 관리한다.

## 다음 사이클 제안 🚀
1. **초기화 순서 검증 자동화**: `let`/`const` 변수의 "선언 → 할당 → 사용" 순서를 정적으로 검증하는 린트 룰 또는 스모크 테스트 확장(콘솔 에러 0건 게이트)이 시급하다. Cycle 11 TDZ → Cycle 14 초기화 순서 버그로 같은 계열의 실수가 2회 반복됨.
2. **물리 퍼즐 장르 재도전**: 프루츠 머지의 물리 엔진 코드는 수정 1건(이벤트 리스너를 init() 내부로 이동)으로 완전히 동작한다. 수정 후 재리뷰하거나, 이 물리 엔진을 기반으로 탄성 충돌 퍼즐/물리 스태킹 등 파생 게임을 시도할 수 있다.
3. **Web Audio 합체 사운드**: 과일 단계별 음높이를 다르게 하여 연쇄 합체 시 상승 아르페지오가 울리는 효과는 구현 비용 대비 체감 만족도가 극히 높을 것이다. Cycle 5의 절차적 사운드 노하우를 활용할 수 있다.
