# 미니 커피숍 타이쿤 — 포스트모템

## 한 줄 요약
작은 커피 노점에서 프랜차이즈 3호점까지 성장시키는 하이브리드 캐주얼 타이쿤을 만들었다 — InfiniTriX 최초의 경영/아이들 장르.

## 무엇을 만들었나
플레이어가 직접 재료를 클릭해 음료를 제조하는 **수동 바리스타 미니게임**과, 바리스타를 고용하고 매장을 확장하는 **아이들 경영**이 결합된 하이브리드 타이쿤이다. 하루 30초 실시간으로 30일을 플레이하며(총 15~20분), 에스프레소 2종에서 시작해 스페셜 블렌드 6종까지 메뉴를 확장하고, 좌석·머신·인테리어를 업그레이드하며, 2호점·3호점을 오픈해 아이들 수익을 쌓아간다.

핵심 재미는 세 가지다. 첫째, 주문 버블에 표시된 재료를 정확한 순서로 클릭하는 **제조의 쾌감**—완벽 서빙 시 50% 팁 보너스와 ★PERFECT 이펙트가 즉각 보상한다. 둘째, 초라한 노점(의자 2개)에서 샹들리에 달린 프리미엄 카페(의자 8개)로 변하는 **시각적 성장**. 셋째, 제한된 골드로 "메뉴 확장 vs 바리스타 고용 vs 인테리어"를 고르는 **경영 의사결정**이다. 고객 4타입(일반/급한/VIP/단골)의 인내심과 팁 배율 차이가 우선순위 판단에 전략적 깊이를 더한다.

## 잘 된 점 ✅
- **플랫폼 장르 공백 완전 해소**: 퍼즐→슈팅→전략→러너→리듬→물리→액션에 이어 경영/아이들까지 8장르를 커버. Cycle 7 포스트모템에서 제안한 "마지막 공백"을 정확히 채웠다.
- **CONFIG 수치 정합성 22/22 (100%)**: 기획서의 모든 밸런스 수치가 코드 상수와 1:1 일치. Cycle 7에서 지적된 수치 불일치(크리티컬 30%→25%) 문제가 이번엔 완전히 해결됐다. 기획서 §13.5 수치 정합성 검증 테이블이 효과적이었다.
- **순수 함수 18개 전수 검증 PASS**: 전역 변수 직접 참조 0건(MENUS 불변 상수 참조만 허용). Cycle 7에서 확립한 "기획서에 함수 시그니처 사전 정의 + 리뷰 전수 검증" 2단계 접근이 2사이클 연속 성공.
- **금지 패턴 8/8 PASS + 필수 패턴 7/7 PASS**: setTimeout, confirm/alert, 외부 에셋 참조 모두 0건. 100% Canvas 코드 드로잉 원칙 완벽 준수(코드 레벨).
- **브라우저 테스트 11/11 PASS**: 콘솔 에러 0건, 모든 화면 정상 렌더링, 터치 이벤트 완비, localStorage 정상 동작.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 8사이클 연속 재발**: 코드에서 참조하지 않지만 9개 SVG + manifest.json이 또 잔존. Cycle 7에서 "pre-commit 훅 강제가 유일한 해결책"이라고 선언했고, 이번 기획서 §13.1에 훅 스크립트까지 명세했는데도 실제 파이프라인에 적용되지 않아 재발했다. **기획서에 쓰는 것과 실제로 동작시키는 것은 다르다**는 교훈이 8번째 확인됐다.
- **PAUSED 상태 전환 일관성 미흡**: `beginTransition()` 경유 원칙을 세웠지만 PAUSED 진입은 `enterState()` 직접 호출, 복귀는 `enterState()` + `state =` 이중 할당. 기능적 버그는 아니지만, 다른 상태에서도 일시정지가 가능해지면 문제가 될 수 있는 설계 부채다.
- **drawTitle dt 하드코딩**: `drawTitle(ctx, 0.016)` — 60fps 전제 하드코딩. 고/저 주사율 디스플레이에서 타이틀 애니메이션 속도가 달라진다. 작은 이슈지만 프레임 독립적 로직이라는 원칙에 어긋난다.
- **시간이 있었다면**: 고객별 단골 메모리(자주 주는 메뉴 기억), 날씨 시스템(비 오는 날 고객 감소), 경쟁 카페 이벤트 등 경영 깊이를 더할 수 있었을 것이다.

## 기술 하이라이트 🛠️
이번 게임의 기술적 하이라이트는 **다중 시스템 동시 운용 아키텍처**다. CustomerManager(고객 스폰/이동/대기/이탈), BrewingSystem(수동 제조 판정), BaristaAI(자동 제조), DayTimer, ParticleSystem, AudioManager가 상태×시스템 매트릭스에 따라 정확히 on/off된다. 특히 바리스타 AI는 "가장 오래 기다린 고객 우선 + 레벨별 제조 속도/메뉴 범위 차등"으로 플레이어 수동 제조와 자연스럽게 공존한다.

매장 확장 시스템도 흥미롭다. 2호점/3호점은 완전 아이들 수익(`idleIncome = baseRate × (1 + interiorLv × 0.3)`)으로, PLAYING 상태의 1호점 수동 플레이와 분리되면서도 Day 종료 정산에 합산된다. Web Audio BGM은 Cmaj7→Fmaj7→G7→Cmaj7 재즈 코드 진행을 sine+triangle으로 절차 생성하여, 커피숍 분위기를 외부 에셋 없이 연출했다.
