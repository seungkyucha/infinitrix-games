# 네온 헥스 드롭 — 포스트모템

## 한 줄 요약
Hextris에서 영감을 받은 6방향 낙하 블록 퍼즐을, 1454줄 단일 HTML로 완전 구현하고 2차 리뷰에서 APPROVED를 획득했다.

## 무엇을 만들었나
화면 중앙의 정육각형(코어)을 좌우로 회전시켜 6방향에서 동시에 떨어지는 색상 블록을 정렬하는 낙하 블록 퍼즐이다. 같은 색 블록 3개 이상이 한 변에 연속 인접하면 소멸되고, 소멸 후 위 블록이 내려오면서 연쇄(Cascade)가 발생한다. "어느 방향을 먼저 받을까"라는 순간 판단과 의도치 않은 연쇄의 럭키 모먼트가 핵심 재미다.

레벨이 오를수록 낙하 속도가 빨라지고(2초/블록 → 0.4초/블록), 활성 색상이 3색에서 6색까지 증가하여 자연스러운 난이도 곡선을 만든다. 한 판 1~3분의 짧은 세션, 좌우 탭만으로 조작하는 직관성 덕분에 모바일에서도 쾌적하게 플레이할 수 있다. InfiniTriX 플랫폼의 "낙하 블록" 서브장르 공백을 해소하는 첫 게임이다.

## 잘 된 점 ✅
- **2차 리뷰 APPROVED 달성**: Cycle 15의 5회 리뷰에서 대폭 개선. 기획서 §0에서 F1~F23 피드백 23건을 선제 매핑한 워크플로우가 효과를 발휘했다.
- **assets/ 디렉토리 미생성**: 16사이클 만에 가장 깨끗한 파일 구조 달성. index.html 단일 파일, 외부 의존성 0개, 100% Canvas 코드 드로잉. 1차 리뷰에서 지적된 assets/ 문제도 2차에서 완전 해결.
- **순수 함수 패턴 완전 적용**: `getFallSpeed()`, `addScorePure()`, `findMatches()`, `applyGravityPure()`, `rotateStacksPure()` 등 모든 게임 로직 함수가 파라미터 기반, 부작용 없음.
- **setTimeout 0건**: Web Audio `ctx.currentTime + offset` 네이티브 스케줄링만 사용. 연쇄 처리도 TweenManager `onComplete` 콜백으로 처리하여 완벽한 setTimeout 배제 달성.
- **4개 상태 화면 + 콘솔 에러 0건**: TITLE/PLAYING/PAUSED/GAMEOVER 모두 Puppeteer 테스트에서 정상 렌더링 확인.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **레벨별 활성 색상 수 미세 불일치**: 기획서 §2.2는 레벨 1~3에서 3색(R, B, G)을 명시했으나, 코드 `getActiveColors()`는 4색(R, O, G, B)을 반환. 배포 차단 수준은 아니지만, 초반 난이도가 의도보다 약간 높다. 기획-코드 수치 정합성 검증이 색상 배열까지 내려가야 함을 재확인.
- **PAUSED → GAMEOVER 전환 경로 잔존**: `allowedTransitions`에 이론적으로 불필요한 전환 경로가 포함. 실제 트리거는 불가능하지만, 상태 전환 화이트리스트의 "최소 필요 경로만 허용" 원칙이 더 엄격해질 필요가 있다.
- **1차 리뷰에서 CRITICAL 2건 발생**: index.html 미존재 + assets/ 디렉토리라는 기존 반복 패턴이 또다시 1차에서 나타났다. 스모크 테스트 게이트가 자동화되어 있었다면 1차에서 바로 잡을 수 있었다.
- **밸런스 실측 검증 한계**: 레벨별 낙하 속도, 색상 수 변화에 따른 체감 난이도 곡선을 headless 테스트로는 검증할 수 없다. 자동 시뮬레이션 도구가 없는 한 밸런스는 주관적 판단에 의존한다.

## 기술 하이라이트 🛠️
- **Pointy-top 헥사곤 좌표계**: 6변 각각에 최대 4층 블록 스택을 관리하는 순환 배열 구조. `findCircularRuns()`로 순환 배열 3+ 연속 매칭을 감지하고, `applyGravityPure()`로 소멸 후 중력 낙하를 재귀적으로 처리하는 패턴이 헥사곤 퍼즐의 핵심 알고리즘이다.
- **ObjectPool 이중 운용**: 파티클 100개 + 팝업 20개를 풀링하여 GC 스파이크 없이 연쇄 소멸 시각 효과를 처리. `releaseAll()`로 상태 전환 시 한꺼번에 정리.
- **Offscreen Canvas 배경 캐싱**: `cacheBg()`로 그라디언트 + 그리드를 오프스크린에 미리 그리고, `resizeCanvas()` 시에만 재빌드. 매 프레임 배경 재생성 비용을 완전 제거.
- **TransitionGuard + allowedTransitions**: `STATE_PRIORITY` 맵 + 화이트리스트 + `isTransitioning` 가드의 3중 방어로, `isClearing`, `isRotating` 가드 플래그와 결합하여 블록 소멸/회전/상태 전환 간 경쟁 조건을 원천 차단.
- **DPR 대응 + 모바일 터치 완전 지원**: `canvas.width = W * dpr` + `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)` 조합으로 고해상도 디스플레이 대응. 좌/우 절반 탭 = 회전, 하단 1/4 스와이프 = 하드드롭, 상단 ⏸ 버튼의 직관적 터치 레이아웃.
