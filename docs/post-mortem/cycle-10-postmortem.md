# 미니 카드 배틀러 — 포스트모템

## 한 줄 요약
Slay the Spire의 핵심 루프를 단일 HTML 파일 1905줄에 응축한, 플랫폼 최초의 턴제 덱빌딩 로그라이크를 완성했다.

## 무엇을 만들었나
어둠의 탑 3층(숲→동굴→탑)을 카드 전투로 돌파하는 로그라이크 덱빌딩 게임이다. 매 턴 손패 5장 중 마나 한도 내에서 공격·방어·스킬·파워 카드를 조합하고, 전투 승리 시 3장 중 1장을 골라 덱을 강화해 나간다. "이번 턴 올인 공격? 아니면 방어 쌓기?"라는 의사결정이 매 턴 발생하고, 덱이 점점 강해지면서 시너지가 터지는 순간의 쾌감이 핵심 재미다.

카드 30종(공격10/방어8/스킬8/파워4), 적 10종(일반5+엘리트2+보스3), 14개 게임 상태, 상점·이벤트·휴식·보상 시스템, 카드 업그레이드, 디버프 4종(취약/약화/출혈/가시), 보스 페이즈 전환, 언락 시스템 6개, Seeded RNG까지 — 1런 10~15분에 상당한 깊이를 담아냈다. 9사이클간 실시간 게임만 만들었던 플랫폼에서 턴제 + 카드/덱빌딩이라는 완전히 새로운 축을 개척했다는 점에서 의미가 크다.

## 잘 된 점 ✅
- **코드 리뷰 15/15 전항 PASS + 브라우저 테스트 12/12 전항 PASS**: 기획서 핵심 요구사항 13/14 충족. 모바일 터치, DPR 대응, localStorage, Web Audio, 보안 체크까지 빈틈없이 통과했다.
- **100% Canvas 코드 드로잉**: 적 10종을 slime·goblin·skeleton 등 Canvas shape로 직접 렌더링. 8사이클 연속 재발했던 에셋 로딩 문제가 코드 레벨에서 완벽히 해소됐다.
- **이전 피드백 18건 완전 반영**: 기획서 §0에서 platform-wisdom 피드백 18건을 명시적으로 매핑하고, `beginTransition()` 경유 원칙, `isTransitioning` 가드, `clearImmediate()`, 금지 패턴 0건 등 모두 충실히 구현됐다.
- **턴제 아키텍처 호환성 검증**: 9사이클간 실시간 게임으로 검증된 인프라(TweenManager, ObjectPool, TransitionGuard)가 턴제로 장르가 전환되어도 수정 없이 동작함을 확인했다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 잔존 (MINOR-01)**: 코드에서 에셋을 전혀 참조하지 않지만, 9개 SVG + manifest.json이 물리적으로 존재. 기획서 §0 #5에서 "생성 자체를 금지"라고 명시했음에도 디렉토리가 생성됐다. 코드 레벨은 클린하나 파일 시스템 레벨에서 기획 위반이 남아, 최종 판정이 APPROVED가 아닌 NEEDS_MINOR_FIX로 떨어졌다.
- **덱빌딩 밸런스 미검증**: 30종 카드의 시너지 밸런스, 3층 난이도 곡선, 보스 페이즈 전환 타이밍 등은 headless 테스트로 검증할 수 없는 영역이다. 자동 시뮬레이션(N회 자동 플레이 → 승률/카드 선택률 통계)이 없으면 "특정 빌드가 압도적" 같은 밸런스 이슈를 잡기 어렵다.
- **공용 엔진 모듈 10사이클째 미분리**: TweenManager, ObjectPool, TransitionGuard 등이 10게임에서 copy-paste 상태. 보일러플레이트 누적이 한계에 달했다.

## 기술 하이라이트 🛠️
턴제 게임이라는 특성이 기술적으로 흥미로운 차이를 만들었다. 실시간 게임의 프레임 단위 충돌 판정·물리 시뮬레이션 대신, **14개 상태의 복잡한 전환 그래프**(TITLE→MAP→PRE_BATTLE→PLAYER_TURN→ENEMY_TURN→REWARD→SHOP→REST→EVENT→VICTORY→DEFEAT 등)를 안정적으로 관리하는 것이 핵심 과제였다. 모든 전환이 `beginTransition()` 경유 + `isTransitioning` 가드로 보호되고, 즉시 전환도 `beginTransition(s, {immediate:true})`로 통일했다. `render(ctx, dt, timestamp)` 시그니처 일원화, LCG 기반 Seeded RNG, 보스 페이즈 전환(phase2hp/phase3hp 기반 패턴 교체) 등 턴제 특화 설계가 돋보인다. 무엇보다 이전 사이클에서 검증된 인프라가 장르가 완전히 달라져도 그대로 작동했다는 점이 플랫폼 아키텍처의 성숙도를 증명한다.

## 다음 사이클 제안 🚀
1. **공용 엔진 모듈 분리**: 10게임에 걸쳐 검증된 TweenManager, ObjectPool, TransitionGuard를 `shared/engine.js`로 추출. 보일러플레이트 500줄+ 감소와 버그 수정 일괄 반영이 가능해진다.
2. **수정 회귀 방지 자동화**: 코드 수정 후 전체 플로우 회귀 테스트(상태 전환 전경로 순회)를 Puppeteer 스크립트로 자동화. 개별 수정 검증만으로는 시그니처 전파 누락 같은 이슈를 잡을 수 없다.
3. **미개척 장르 도전**: 10장르 커버 완료 후, 내러티브 기반 어드벤처나 생태계 시뮬레이션 같은 색다른 방향을 시도해볼 만하다.
