# 젬 시즈 (Gem Siege) — 포스트모템

## 한 줄 요약
C44 매치3 엔진을 기반으로 보스 배틀 5종, 신규 장애물 3종, 보스 도감 메타 레이어를 추가하여 20레벨 규모의 전략적 매치3 퍼즐을 완성했다.

## 무엇을 만들었나
젬 시즈는 어둠의 군세가 보석 왕국을 침략하는 설정 위에 세워진 **보스 배틀 매치3 퍼즐**이다. 플레이어는 8×8 그리드에서 6색 보석을 매칭하여 보스의 HP를 깎되, 보스가 매 턴 반격(독 타일 생성, 얼음 2겹 강화, 체인 잠금, 색 셔플 등)하므로 단순 매칭이 아닌 전략적 판단이 필요하다. 독 슬라임 왕부터 카오스 드래곤까지 5종 보스가 각자 고유한 반격 패턴과 약점을 가지고 있어, 보스별로 다른 공략법을 찾는 재미가 핵심이다.

기존 C44의 3종 장애물(얼음/나무상자/체인)에 독 타일(매 턴 확산), 텔레포터(위치 교환), 잠금석(특정 색만 해제) 3종을 추가해 총 6종 장애물이 레벨 디자인의 폭을 넓혔다. 보스 격파 시 도감에 등록되는 메타 레이어는 "다음 보스는 누구인가"라는 수집 동기를 부여하며, 전체 격파 시 '왕국 수호자' 칭호가 해금된다. 스페셜 보석 조합 6종(무지개+무지개=보드 전체 클리어 등)이 보스전에서 결정적 한 방의 쾌감을 제공한다.

## 잘 된 점 ✅
- **C44 엔진 재활용의 위력**: 매치 검출(5→T/L→4→3), 캐스케이드 tween 체인, 데드록 셔플, 가드 플래그 등 핵심 인프라를 그대로 이어받아 보스 배틀이라는 상위 시스템에 집중할 수 있었다. 엔진 안정성 덕에 2차 리뷰에서 바로 APPROVED.
- **보스 컷신 연출 품질**: 보스 인트로(실루엣 줌인→이름 슬라이드→HP바→번개 파티클)와 격파(깜빡임→백색 플래시→VICTORY 바운스→보석 비→도감 카드) 연출이 기획대로 구현됨. C44의 F3(캐슬 복원 연출 미흡) 피드백을 확실히 해소.
- **C44 피드백 4건 전수 반영**: 미참조 에셋(F1), state= 직접 할당(F2), 컷신 연출(F3), 밸런스 미검증(F4) 모두 해결. 피드백 매핑 워크플로우가 44사이클째 안정적으로 동작.
- **6중 가드 방어**: 기존 4중(swapLocked/cascadeInProgress/resolving/goalChecking)에 poisonSpreading, bossCountering을 추가해 보스 반격과 독 확산 중 입력 차단을 완벽히 처리.
- **코드 품질**: Math.random() 0건(SeededRNG), setTimeout 0건(tween onComplete 100%), alert/confirm/prompt 0건, 에셋 50개 전수 로드+Canvas 폴백 100%.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **밸런스 자동 검증 부재**: 20레벨×DDA 조합의 난이도 곡선을 수동으로만 확인했다. headless 시뮬레이터로 N회 자동 플레이하여 클리어율을 정량 측정하는 시스템이 여전히 없다. 45사이클째 미착수.
- **공용 엔진 모듈 미분리**: TweenManager, ObjectPool, TransitionGuard 등 45개 게임에서 복사-붙여넣기된 코드가 여전히 단일 HTML 안에 있다. 기술 부채가 누적 중이다.
- **보스 하드모드 미구현**: Phase 2에 계획된 2회차 하드모드, 히든 스테이지, 영웅 스킬 시스템은 시간 제약으로 미구현. 보스 도감 보상(스킬 언락)까지 이어졌으면 리플레이 가치가 더 높았을 것.
- **독 타일 밸런스 체감**: 독 타일 확산이 후반 레벨에서 상당히 압박적인데, 시뮬레이터 없이는 "적절한 긴장감"과 "좌절"의 경계를 정밀하게 잡기 어려웠다.

## 기술 하이라이트 🛠️
- **TRANSITION_TABLE 화이트리스트**: 8개 상태 간 허용 전환만 명시적으로 등록. `beginTransition()`과 `directTransition()` 두 경로 모두 테이블을 참조하므로, 잘못된 상태 전환이 구조적으로 불가능하다. C21부터 반복된 STATE_PRIORITY 버그를 C44에서 근절한 뒤 C45에서 보스 상태(BOSS_INTRO, BOSS_DEFEAT) 확장에도 안정적으로 동작.
- **보스 반격 시스템 아키텍처**: 각 보스의 `counterPattern` 함수가 보드 조작(독 추가, 얼음 강화, 체인 배치, 색 셔플)을 수행하며, `gBossCountering` 가드 플래그로 반격 애니메이션 중 입력을 차단. 약점 시스템(`weaknessMultiplier`)은 특정 매치 조건에 데미지 배율을 적용하여 보스별 차별화된 공략법을 제공.
- **에셋 50개 + 스프라이트 시트 애니메이션**: 6색 보석×6프레임 반짝임 시트, 이펙트 시트 3종을 Gemini PNG로 생성. manifest.json 동적 로드 + 프로그레스 바 연동 로딩 화면. onerror=resolve 패턴으로 에셋 실패 시에도 Canvas 폴백으로 100% 동작.
- **DDA 확장**: 기존 failStreak 기반에 보스전 전용 파라미터(bossHpReduction -5~20%, bossCounterDelay 1턴, extraTurns +2)를 추가하여 보스전 좌절 방지.
