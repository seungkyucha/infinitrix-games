# 스타 가디언 — 포스트모템

## 한 줄 요약
Cycle 1의 퍼즐에서 완전히 방향을 틀어, 네온 우주 배경의 종스크롤 슈팅 게임을 만들고 TweenManager·ObjectPool 등 핵심 시스템을 검증했다.

## 무엇을 만들었나
**스타 가디언**은 우주를 배경으로 플레이어 전투기를 조종해 웨이브별로 밀려오는 적 함대를 격파하는 종스크롤 슈팅 게임이다. 스카우트·파이터·탱크·다트 4종의 적과 3페이즈 보스전, 무기 강화(4단계)·HP 회복·쉴드 3종 파워업, 콤보 시스템과 점수 이정표까지 아케이드 슈팅의 핵심 재미 요소를 단일 HTML 파일(1,751줄)에 담았다.

3레이어 패럴랙스 별 배경 위에 네온 글로우 이펙트, 스크린 셰이크, 폭발 파티클이 어우러져 시각적 완성도가 높다. 키보드·마우스·터치(가상 조이스틱+오토파이어) 3종 입력을 모두 지원해 데스크톱과 모바일에서 즉시 플레이 가능하다. 한 판 3~7분의 짧고 강렬한 세션으로 "한 판 더"의 중독성을 노렸다.

## 잘 된 점 ✅
- **Cycle 1의 모든 문제(M1~M4)를 완전 해결했다.** Google Fonts 제거(M1), 미사용 에셋 정리(M2), Canvas 확인 모달로 confirm() 대체(M3), TweenManager로 애니메이션 시스템 구축(M4). 이전 사이클의 교훈이 기획 단계부터 체크리스트로 반영된 좋은 사례다.
- **TweenManager + ObjectPool 아키텍처가 실전에서 검증되었다.** Tween은 적 등장/폭발/파워업/웨이브 텍스트 등 10곳 이상에서 활용되었고, ObjectPool은 총알·적·파티클 등 6종 엔티티의 GC 압박을 최소화했다. 이 두 시스템은 앞으로 모든 게임의 표준 인프라가 될 수 있다.
- **20개 테스트 항목 전체 PASS, 콘솔 에러 0건.** 에셋 8/8 로드 성공, 게임 시작→플레이→게임오버→재시작 전체 플로우 정상 동작. MINOR 이슈 3건도 실질적 플레이 영향 없음. Cycle 1 대비 코드 품질이 한 단계 올라갔다.
- **기획서 충실도가 매우 높다.** 적 4종+보스의 이동/공격 패턴, 파워업 드롭 확률, 웨이브 스케일링 공식까지 기획서에 HEX·수식을 명시한 전략이 다시 한번 효과를 증명했다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **자동 발사 분기 미구현(M1).** 기획서는 "터치 환경 오토파이어 ON, 키보드는 수동"이었으나 코드에서 `shouldFire` 변수를 조건 분기에 사용하지 않아 키보드에서도 항상 자동 발사된다. UX 측면에서는 오히려 편리하지만, 기획 의도와의 불일치는 앞으로 "입력 모드 자동 감지" 로직으로 해결할 수 있다.
- **TweenManager.cancelAll()의 참조 안전성(M2).** 현재 동시 호출 경로가 없어 버그는 아니지만, update 도중 cancelAll 호출 시 참조 불일치 가능성이 있다. 복잡한 게임에서는 deferred cancel 패턴이 필요할 수 있다.
- **이벤트 리스너 cleanup 미구현.** 단일 페이지 게임이라 실질적 문제는 없으나, 플랫폼이 SPA 구조로 진화하면 메모리 누수 원인이 될 수 있다. 게임 종료 시 리스너 해제 패턴을 표준화할 필요가 있다.
- **이징 함수가 4종으로 기획서(5종) 대비 1개 부족.** easeOutElastic이 누락된 것으로 보인다. 점수 이정표 텍스트 등에서 활용 예정이었던 함수인데, 다음 사이클에서 보완하면 좋겠다.

## 기술 하이라이트 🛠️
이번 사이클의 가장 큰 기술적 성과는 **재사용 가능한 게임 인프라 2종의 실전 검증**이다.

**TweenManager**는 `add(target, props, duration, easing, onComplete)` API로 어떤 객체든 속성 보간 애니메이션을 걸 수 있다. 적 등장, 폭발, 파워업 획득, 웨이브 텍스트, HP바 감소, 피격 플래시, 배경 색상 전환, 모달 등장 등 10곳 이상에서 활용되며 Cycle 1의 최대 아쉬움이었던 "움직임 없는 게임"을 근본적으로 해결했다.

**ObjectPool**은 acquire/release 패턴으로 총알·적·파티클 등 6종 엔티티를 풀링한다. 매 프레임 수십 개의 객체가 생성/소멸되는 슈팅 게임에서 GC 스파이크 없이 60fps를 안정적으로 유지할 수 있었다. 역순 순회 기반의 안전한 release 패턴도 잘 동작했다.

**보스 3페이즈 AI**도 주목할 만하다. 직선 연사→원형 확산→추적탄을 4초 간격으로 순환하며, HP 50% 이하에서 속도가 가속되는 2단계 난이도를 구현했다. 상태 머신 위에 상태 머신을 얹는 패턴(게임 상태 6종 + 보스 페이즈 3종)이 깔끔하게 동작했다.

## 다음 사이클 제안 🚀
1. **전략/시뮬레이션 장르 도전** — 퍼즐(Cycle 1)과 액션(Cycle 2)을 거쳤으니, 타워 디펜스나 자원 관리 같은 전략 장르로 TweenManager·ObjectPool의 범용성을 검증해보자.
2. **입력 모드 자동 감지 시스템** — 키보드/마우스/터치 입력을 자동 감지해서 UI와 동작을 자동 전환하는 범용 InputManager를 설계하면, 매 게임마다 입력 코드를 반복 작성하지 않아도 된다.
3. **사운드 시스템 도입 검토** — 2개 사이클 동안 비주얼은 성숙했지만 사운드는 전무하다. Web Audio API 기반의 절차적 사운드(procedural SFX)로 외부 에셋 없이 효과음을 생성하는 실험을 해볼 때다.
