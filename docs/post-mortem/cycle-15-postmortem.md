# 보석 매치 블리츠 — 포스트모템

## 한 줄 요약
InfiniTriX 플랫폼의 최대 서브장르 공백이었던 Match-3 퍼즐을 8×8 보석 그리드 + 30스테이지 + 특수 보석 3종으로 구현하고, 3회차 리뷰에서 APPROVED를 달성했다.

## 무엇을 만들었나
**보석 매치 블리츠**는 8×8 보석 그리드에서 같은 색 3개 이상을 연결하여 폭발시키는 클래식 Match-3 퍼즐 게임이다. 보석을 스와이프로 교환하면 매칭이 성립하고, 빈 칸에 위 보석이 낙하하며 연쇄 폭발(Cascade)이 터진다. 의도하지 않은 연쇄가 화면 가득 폭발로 이어지는 "럭키 모먼트"가 핵심 재미다.

4개 일렬 매치로 라인 폭탄, L/T형 매치로 범위 폭탄, 5개 일렬로 색상 폭탄이 생성되며, 특수 보석끼리의 조합 6종(색상+색상 = 그리드 전체 폭발 등)이 전략적 깊이를 더한다. 30스테이지를 점수 달성(1~10), 특정 색 N개 제거(11~20), 특수 보석 N개 생성(21~30)의 3가지 목표 유형으로 돌파하며, 제한된 이동 수 안에서 매 수가 중요한 긴장감을 유지한다.

네온 다크 테마의 화려한 시각 연출, 크리스탈 파티클 배경, Web Audio API 효과음 8종이 한 판 1~2분의 "한판만 더" 중독 루프를 완성한다. 1,449줄 단일 HTML에 100% Canvas 코드 드로잉으로 외부 에셋 0개를 달성했다.

## 잘 된 점 ✅
- **플랫폼 최대 공백 정확히 충족**: Match-3은 HTML5 게임 시장 부동의 1위 장르였고, InfiniTriX에서 유일하게 빠져 있던 서브장르. puzzle 5→6으로 장르 다양성이 향상됐다.
- **기획서 피드백 매핑 19건(F1~F19) 역대 최고 효과율 74%**: 선제 대응한 19건 중 14건이 초회부터 완벽 준수. assets/ 금지(F3), cellSize 48px(F6), init() 내부 이벤트(F7), setTimeout 금지(F11) 등 누적 교훈이 실질적으로 작동했다.
- **코드 품질 — 15사이클 인프라 총동원**: ObjectPool(파티클 40/팝업 20), TweenManager, Fisher-Yates 셔플 + 데드록 자동 셔플, `beginTransition()` 우선순위 가드, 순수 함수 패턴, Web Audio 네이티브 스케줄링(`ctx.currentTime + offset`), 게임 루프 try-catch 래핑 등이 안정적으로 동작.
- **기획서 F1~F22 전체 준수**: 보안(eval 0건, alert 0건), 성능(localStorage 매 프레임 접근 0건, bgCache 캐싱), 메모리 관리(ObjectPool, `_toAdd` 큐) 모두 리뷰 PASS.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 15사이클 연속 재발**: 기획서 F3 "절대 금지" 명시에도 1회차에서 SVG 8개 + manifest.json 발견. 2회차에서 삭제, 3회차에서 `thumbnail.svg` 위치까지 정리하여 최종 APPROVED. 15사이클 데이터가 "CI/pre-commit 훅 실등록 없이는 해결 불가"를 통계적으로 확정했다.
- **cellSize 48px 미달 재발**: 400px 뷰포트에서 45px로 산출. `Math.max(48, ...)` 1줄 수정으로 해결됐지만, 기획서 명시 + 코드 구현의 괴리가 터치 타겟에서 반복되는 패턴이다.
- **스테이지 11~30 목표 유형 1회차 미구현**: 기획서 §2.4에 명세된 3유형 목표 중 score만 구현되고 color/special이 누락. Cycle 12 "절반 구현" 패턴의 재발. 기능별 세부 체크리스트(A: ✅, B: ☐, C: ☐) 단위의 구현 확인이 필요하다.
- **밸런스/사운드 실측 검증 불가**: 30스테이지 난이도 곡선과 효과음 8종의 체감 품질이 headless 테스트로는 검증 불가능. 자동 시뮬레이터 + 사운드 체감 테스트 리뷰 항목 신설이 시급하다.
- **3회 리뷰 사이클 소요**: 1회차(6건 지적) → 2회차(assets/ 2파일 잔존) → 3회차(완전 해결). 초회 APPROVED까지의 효율성을 높이려면 리뷰 제출 전 스모크 테스트 게이트가 필수.

## 기술 하이라이트 🛠️
**연쇄 폭발(Cascade) 엔진**: 가로·세로 스캔으로 3+매치 감지 → L/T형·4연속·5연속 특수 보석 판정 → 폭발 애니메이션 → 중력 낙하 → 재매칭의 파이프라인이 TweenManager와 결합되어 부드러운 시각 피드백을 제공한다. `isProcessing` + `_transitioning` 이중 가드로 연쇄 중 콜백 충돌 0건을 달성했다.

**3유형 목표 시스템**: `LEVEL_OBJECTIVES[30]` 배열 + `colorRemoved[6]` 추적 + `specialsCreated` 카운터 + `isObjectiveComplete()` 판정 함수로 score/color/special 3유형 목표를 깔끔하게 분리. 새 목표 유형 확장이 배열 항목 추가만으로 가능한 구조다.

**offscreen 배경 캐시(`bgCache`) + touchSafe() 전면 적용**: 매 프레임 배경 재생성 비용을 절감하면서, `Math.max(CONFIG.MIN_TOUCH_TARGET, size)` 패턴이 모든 셀과 버튼에 적용되어 400px 뷰포트에서도 48px 터치 타겟 보장. delta time 상한(`dt = Math.min(0.033)`)으로 프레임 스킵 안정성도 확보했다.
