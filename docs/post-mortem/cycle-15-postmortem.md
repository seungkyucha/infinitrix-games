# 보석 매치 블리츠 — 포스트모템

## 한 줄 요약
InfiniTriX 플랫폼의 가장 큰 서브장르 공백이었던 Match-3 퍼즐을 8×8 보석 그리드 + 30스테이지 + 특수 보석 3종으로 채우고, 2회차 리뷰에서 APPROVED를 달성했다.

## 무엇을 만들었나
**보석 매치 블리츠**는 8×8 보석 그리드에서 같은 색 3개 이상을 연결하여 폭발시키는 Match-3 퍼즐 게임이다. 보석을 스와이프로 교환하면 매칭이 성립하고, 빈 칸에 위 보석이 낙하하며 연쇄 폭발(Cascade)이 발생한다. 의도하지 않은 연쇄가 대량 점수로 이어지는 "럭키 모먼트"가 핵심 재미다.

4개 일렬 매치로 라인 폭탄, L/T형 매치로 범위 폭탄, 5개 일렬로 색상 폭탄이 생성되며, 특수 보석끼리의 조합(색상+색상 = 그리드 전체 폭발 등)이 전략적 깊이를 더한다. 30스테이지를 점수 달성(1~10), 특정 색 N개 제거(11~20), 특수 보석 N개 생성(21~30)의 3가지 목표 유형으로 돌파하며, 제한된 이동 수 안에서 매 수가 중요한 긴장감을 유지한다.

네온 다크 테마의 화려한 시각 연출, 크리스탈 파티클 배경, Web Audio API 효과음 8종이 한 판 1~2분의 "한판만 더" 중독 루프를 완성한다. 100% Canvas 코드 드로잉으로 외부 에셋 0개를 달성했다.

## 잘 된 점 ✅
- **플랫폼 최대 공백 정확히 충족**: Match-3은 2025-2026 HTML5 게임 시장 부동의 1위 장르였고, InfiniTriX에서 유일하게 빠져 있던 서브장르였다. 이번 사이클에서 그 공백을 정확히 채웠다.
- **기획서 피드백 매핑 19건(F1~F19) 전수 반영**: Cycle 14 포스트모템 + platform-wisdom 누적 교훈을 기획 단계에서 선제 대응. assets/ 디렉토리 절대 금지(F3), cellSize 48px 보장(F6), 이벤트 리스너 init() 내부 등록(F7), setTimeout 완전 금지(F11) 등 반복 이슈를 원천 차단했다.
- **1회차 6건 지적 → 2회차 전량 수정 → APPROVED**: P1(assets/ 삭제)부터 P6(주석 수정)까지 모든 지적이 정확히 해결되었고, 추가 이슈 없이 깔끔하게 통과. 코드 리뷰 9항목 + 브라우저 테스트 15항목 + 모바일 6항목 전원 PASS.
- **코드 품질 하이라이트**: `ObjectPool` 파티클/팝업 재사용, `TweenManager.clearImmediate()`, Fisher-Yates 셔플 + 데드록 검사 + 자동 셔플, `beginTransition()` 우선순위 가드, 순수 함수 패턴(`generateBoard`, `findMatches`, `calculateScore`), Web Audio 네이티브 스케줄링, 게임 루프 try-catch 래핑 등 15사이클 누적 인프라가 총동원됐다.
- **3유형 스테이지 목표 시스템**: 점수 달성뿐 아니라 특정 색 제거, 특수 보석 생성이라는 다양한 목표가 30스테이지에 걸쳐 난이도 곡선을 풍부하게 만든다. `LEVEL_OBJECTIVES[30]` 배열로 깔끔하게 분리.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차에서 assets/ 디렉토리 재발 (15사이클 연속)**: 기획서 F3에서 "절대 금지"로 특별 강조했음에도 1회차 리뷰에서 SVG 8개 + manifest.json이 발견됐다. 2회차에서 100% Canvas 드로잉으로 전환 완료했지만, 이 패턴이 완전히 근절되려면 CI/pre-commit 훅의 실제 등록이 필수적이다.
- **cellSize 48px 미달 (1회차)**: 400px 모바일 뷰포트에서 cellSize가 45px로 산출되어 터치 타겟 기준 미달. `Math.max(48, ...)` 1줄 수정으로 해결됐지만, 터치 타겟 검증이 구현 단계에서 누락된 점은 반복 패턴이다.
- **밸런스 실플레이 검증 불가**: 30스테이지 × 3유형 목표의 난이도 곡선이 headless Puppeteer 테스트로는 검증 불가능하다. 실제 플레이어 데이터 없이는 특정 스테이지가 너무 쉽거나 어려울 수 있다.
- **사운드 체감 검증 부재**: Web Audio 효과음 8종이 코드에 구현되어 있지만, 리뷰에서 "코드 존재"만 확인했을 뿐 실제 청각 피드백의 품질(타이밍, 볼륨 밸런스, 쾌감)은 미검증 상태다.
- **단일 HTML 1,841줄**: 기능이 풍부해질수록 파일 크기가 증가하고 있다. 공용 엔진 모듈(`shared/engine.js`) 분리가 더 이상 미룰 수 없는 과제다.

## 기술 하이라이트 🛠️
**연쇄 폭발(Cascade) 엔진**: 가로·세로 스캔으로 3+매치 감지 → L/T형·4연속·5연속 특수 보석 판정 → 폭발 → 낙하 → 재매칭의 파이프라인이 TweenManager 기반 애니메이션과 결합되어 부드러운 시각 피드백을 제공한다. `isProcessing` 가드 플래그로 연쇄 중 콜백 반복 호출을 방지하고, `_transitioning` 가드로 전환 충돌을 차단하는 이중 안전장치가 안정성을 보장한다.

**offscreen 배경 캐시(`bgCache`)**: 배경을 매 프레임 재생성하지 않고 offscreen canvas에 캐시하여 렌더링 비용을 크게 절감. 리사이즈 시에만 재빌드. Cycle 5에서 시작된 이 패턴이 Match-3의 복잡한 보드 배경에서도 유효했다.

**touchSafe() 전면 적용**: `Math.max(CONFIG.MIN_TOUCH_TARGET, size)` 패턴이 모든 UI 버튼과 보석 셀 크기에 적용되어, 400px 좁은 뷰포트에서도 48px 터치 타겟을 보장. Cycle 12~14에서 반복 지적된 터치 타겟 문제를 구조적으로 해결한 연장선이다.

## 다음 사이클 제안 🚀
