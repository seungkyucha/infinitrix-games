# 그래비티 플립 (Gravity Flip) — 포스트모템

## 한 줄 요약
중력 반전 한 번으로 연구소를 탈출하는 레이지 플랫포머 — Level Devil 트렌드에 정면 대응하면서 arcade+casual 장르 최장 미사용 공백을 해소한 사이클.

## 무엇을 만들었나

중력 실험 사고로 붕괴하는 연구소에서 탈출하는 **원터치 중력 반전 레이지 플랫포머**다. 탭/클릭 한 번으로 중력 방향을 180° 뒤집어 장애물을 회피하고, 적대적으로 변하는 환경(바닥 소멸, 천장 낙하, 스파이크 돌출)을 돌파해 출구 포탈에 도달한다. Poki 2026년 3월 1위인 Level Devil의 "안전해 보이는 곳이 함정"이라는 철학을 직접 차용하면서, 중력 반전이라는 물리 기반 메카닉으로 차별화했다.

5개 구역(실험동→복도→반응로→중력실→핵심부) × 3 스테이지 = 15 메인 + 보스 5종 + 히든 제로그래비티 스테이지 3개 = **총 23 스테이지** 구성이다. 업그레이드 트리는 중력 제어/대시/타임 슬로우 3갈래 × 5단계로 빌드 다양성을 확보했고, 즉사+즉부활(0.3초) 루프로 "분노→재도전→성취감" 사이클을 극대화하는 설계다. SeededRNG 기반 절차적 장애물 패턴 조합으로 매 플레이마다 미세한 변형이 발생해 패턴 암기만으로는 클리어할 수 없다.

이전 사이클(#37 골드러시 택틱스)의 듀얼 시스템(퍼즐+건설) 과부하 문제를 교훈 삼아, **단일 핵심 루프(뒤집기→돌파→탈출)**에 집중했다. 상태 머신도 기존 5상태에서 **4상태(TITLE/MAP/PLAY/BOSS)**로 더 단순화하여 코드 복잡도를 낮추는 방향을 잡았다.

## 잘 된 점 ✅

- **이전 사이클 교훈의 체계적 반영**: F1~F28까지 28개 피드백 매핑을 기획서에 명시하고, Cycle 37의 "아쉬웠던 점" 4건을 모두 해결 섹션에 연결했다. TRANSITION_TABLE 자동 검증, assets/ CI 강제 차단, 밸런스 검증 수학 공식 — 3가지 제안이 모두 §6.1, §14.3, §8에 반영되었다.
- **단일 핵심 루프 집중**: Cycle 35 planner 교훈대로 서브시스템을 업그레이드 트리 하나로 줄이고, 핵심 메카닉 함수를 5개 이내로 제한하는 설계가 코드 복잡도를 효과적으로 통제한다.
- **4상태 TRANSITION_TABLE 단일 정의**: STATE_PRIORITY 7번째 재발(Cycle 36)을 교훈 삼아, 플랫포머 특성에 맞게 MAP/SHOP 통합 4상태로 최소화하고 `Object.keys(TRANSITION_TABLE).length === 4` 검증 기준을 기획서에 명시했다.
- **수학적 난이도 곡선 공식화**: 장애물 밀도·속도·안전 구간·코인 수·기준 시간을 스테이지 번호 n의 수학 공식으로 정의하여 밸런스 사전 검증이 가능하도록 설계. Cycle 37의 밸런스 시뮬레이터 부재 문제를 구조적으로 개선했다.
- **BFS 도달 가능성 검증 + 프로시저럴 생성**: 스테이지 생성 시 BFS로 출구 도달 가능성을 자동 검증하고, 실패 시 seed+1로 최대 10회 재생성하는 안전장치가 "불가능한 맵" 문제를 원천 차단한다.
- **보스 약점 타이밍 윈도우 ASCII 시각화**: Cycle 37 planner 제안을 반영하여 보스별 약점 노출 타이밍을 기획서에 ASCII 다이어그램으로 명시. 완성 가능성을 사전 검증했다.

## 아쉬웠던 점 / 개선 가능성 ⚠️

- **리뷰 미완료 (PENDING)**: Cycle 37에 이어 2사이클 연속 전체 코드 리뷰가 완료되지 않은 상태다. 게임 코드가 아직 생성되지 않았거나 구현이 진행 중인 것으로 보인다. 기획서의 완성도에 비해 실제 구현 상태를 확인할 수 없는 점이 가장 큰 아쉬움이다.
- **assets/ 38사이클 연속 재발 우려**: 기획서에 F1/F24로 명시하고 스모크 테스트 FAIL 게이트까지 설계했지만, 37사이클간 데이터가 "기획서 명시만으로는 1차 리뷰에서 반드시 재발"함을 증명한다. CI 강제 차단이 실제로 동작하는지가 관건이다.
- **밸런스 자동 시뮬레이터 부재 지속**: 수학 공식 기반 밸런스 설계는 진전이지만, 실제 "극단 업그레이드 빌드(대시 올인 + 슬로우 올인)" 조합에서의 클리어 가능성은 자동 시뮬레이터 없이 검증 불가. 38사이클째 미해결.
- **공용 엔진 미분리 38사이클째**: TweenManager, ObjectPool, SoundManager, hitTest(), touchSafe() 등이 여전히 게임별 복사-붙여넣기. 기획서 §5에 명시된 INIT_EMPTY, TW, SND 패턴이 이전 37개 게임과 동일한 코드의 반복이다.
- **10종 장애물 + 5보스의 콘텐츠 볼륨**: 단일 핵심 루프에 집중한다고 했으나, 장애물 10종(T1~T10) + 보스 5종 + 히든 3개의 콘텐츠 양은 여전히 상당하다. Phase 1 MVP 경계를 명확히 잡은 점은 좋지만, 실제 구현 시 Phase 2 범위가 과대해질 수 있다.

## 기술 하이라이트 🛠️

- **원터치 중력 반전 물리 시스템**: 단일 입력(탭)으로 중력 방향을 180° 전환하는 핵심 메카닉. 캐릭터가 자동 우측 이동(120px/s)하는 오토러너 구조에 중력 반전을 결합하여, 조작은 극도로 심플하면서도 타이밍 판단의 깊이를 확보했다.
- **SeededRNG 세그먼트 기반 스테이지 생성**: 스테이지를 8개 세그먼트(300px) 조합으로 구성하고, 사전 정의 패턴 20종에 ±2타일/30% 변형을 적용. 난이도 배분(`[E, E, M, M, M, H, H, E]`)으로 "마지막은 출구 전 안전"이라는 레이지 게임의 감정 곡선을 구조화했다.
- **카메라 시스템 4모드**: 플레이어 중심 수평 스크롤(좌측 1/3 고정) + 보스전 줌아웃(1.0→0.7) + 사망 시 화면 흔들림+슬로모션 + 클리어 줌인(1.0→1.3) — 4가지 카메라 모드가 게임 이벤트와 동기화되어 연출 수준을 높인다.
- **DDA 4단계 "정보 추가 제공" 방식**: 연속 사망 5/10/20회에 따라 장애물 속도 감소, 유령 경로 힌트, 위험 글로우 강화를 단계적으로 제공. 함정 비활성화가 아닌 "더 많은 정보"로 도움을 주는 방식이라 레이지 게임의 도전감을 훼손하지 않는다.
- **TRANSITION_TABLE 정적 검증**: 스모크 테스트에서 모든 전환 쌍(from→to)을 호출 검증하는 방식으로 STATE_PRIORITY 7번째 재발 문제의 자동 감지를 시도. `Object.keys(TRANSITION_TABLE).length === 4` 정적 검증 기준도 포함.
