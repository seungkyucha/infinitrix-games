# 크로노 시즈 — 포스트모템

## 한 줄 요약
시간 마법(감속/가속/역전)을 실시간으로 시전하며 타워를 전략적으로 배치하는 하이브리드 타워 디펜스를 구현하여, 플랫폼의 arcade + strategy 장르 공백을 해소했다.

## 무엇을 만들었나
크로노 시즈는 시간의 균열에서 쏟아지는 크로노비스트를 막는 크로노맨서의 이야기를 다룬 타워 디펜스 게임이다. 플레이어는 7종 타워를 그리드에 배치하는 전략(strategy) 메카닉과, Q/W/E로 감속·가속·역전 3종 시간 마법을 실시간 시전하는 아케이드(arcade) 메카닉을 동시에 운용한다. 고대→중세→산업혁명→미래→시간의끝, 5시대를 넘나들며 각 시대의 고유한 적과 보스를 상대한다.

핵심 긴장감은 시간 에너지 관리에서 비롯된다. 마법은 강력하지만 에너지가 한정되어, "지금 감속을 쓸까, 보스전까지 아낄까?"라는 순간 판단이 매 웨이브의 승패를 좌우한다. 5시대 × 4스테이지 + 5보스 + 히든 스테이지의 풍부한 콘텐츠 위에, 영구 업그레이드 트리와 난이도 3종(쉬움/보통/어려움)이 리플레이 가치를 더한다.

## 잘 된 점 ✅
- **arcade + strategy 장르 공백 최초 해소**: 플랫폼에 0개였던 이 조합을 성공적으로 채워, 기존 mini-tower-defense(순수 전략)와 명확히 차별화했다.
- **14상태 머신 — 플랫폼 역대 최다**: TITLE부터 TUTORIAL까지 14개 상태가 STATE_PRIORITY + ESCAPE_ALLOWED + beginTransition 체계로 경쟁 조건 0건 달성. 가드 플래그 패턴이 15사이클 연속(Cycle 4~22) 안정 동작을 확인했다.
- **F1~F38 역대 최다 38건 피드백 매핑**: 22사이클간 축적된 모든 교훈을 기획서 §0에 매핑하여, 핵심 게임플레이 7항목 전원 PASS를 달성했다.
- **이슈 제로 최종 배포**: P1·P2·P3 이슈 0건으로 즉시 배포 가능 상태. 1차 리뷰에서 P1 2건이 있었으나, 4라운드에 걸쳐 모든 이슈를 해결했다.
- **assets/ 미생성 5사이클 연속**: Cycle 18부터 시작된 에셋 제로 원칙이 완전히 정착되었다. thumbnail.svg만 존재하며, 외부 CDN 0건, setTimeout 0건(11사이클 연속) 유지.
- **한/영 이중 언어 지원**: TEXT.ko + TEXT.en 객체 기반 완전 로컬라이제이션이 Cycle 21에 이어 2사이클 연속 적용.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **4라운드 리뷰 소요**: 1차에서 P1 2건(CRITICAL급) + P2 2건이 발생했고, 2차~3차에서 P3 어드바이저리가 잔존하여 총 4라운드(2차 리뷰 3회차)까지 필요했다. 초회 APPROVED를 달성하지 못한 점은 아쉽다.
- **menuY 레이아웃 이슈가 3회차까지 존재**: 480px 모바일 뷰포트에서 타이틀 메뉴 버튼이 잘리는 문제가 P3-ADV로 분류되어 3회차에서야 완전 수정. 다양한 뷰포트 크기의 UI 레이아웃 테스트를 기획 단계에서 더 체계적으로 포함해야 한다.
- **밸런스 검증 미비**: 5시대 × 4스테이지 × 난이도 3종 × 타워 7종 × 마법 3종의 방대한 조합 공간에 대한 자동 밸런스 검증 수단이 없다. 코드 리뷰만으로는 "특정 시간 마법 조합이 압도적"이거나 "후반 시대가 불가능"한지 판별할 수 없다.
- **단일 파일 규모 지속 증가**: 정확한 줄수는 미확인이나, 14상태 + 5시대 + 7타워 + 보스 체계의 복잡도를 감안하면 Cycle 21(4,215줄)을 넘었을 가능성이 높다. 공용 엔진 모듈 분리가 22사이클째 지연 중이다.
- **다중 이해관계자 피드백 프로세스**: 플래너·디자이너 피드백이 리뷰 라운드를 연장시키는 패턴이 Cycle 21에 이어 재발. 피드백을 리뷰 전에 통합하는 프로세스가 아직 확립되지 않았다.

## 기술 하이라이트 🛠️
- **시간 마법 시스템**: 감속/가속/역전 3축 마법을 timeScale 분리로 구현. 게임 시간만 변조하고 UI·사운드는 실시간 유지하는 구조가 서바이버라이크(Cycle 18) 슬로우모션 패턴을 타워 디펜스에 성공적으로 확장했다.
- **14상태 머신 안정 동작**: STATE_PRIORITY + ESCAPE_ALLOWED(역방향 전환 경로) + beginTransition + enterState(PAUSED 예외) 조합이 역대 최다 상태에서도 무결하게 동작. 특히 PAUSED→GAMEPLAY 즉시 전환과 GAMEOVER 우선순위 처리가 자연스럽다.
- **오프스크린 캐싱 4중 구조**: buildBgCache(시대별 배경) + buildGridCache(그리드 라인) + buildIconCache(타워/마법 아이콘) + buildUICache(HUD 프레임)로 매 프레임 재드로잉 비용을 극적으로 절감. resize/스테이지 전환 시에만 재빌드.
- **menuY 하한 바운드**: `Math.min(H * 0.72, H - (3 * (btnH + 12) + 40))`로 소형 뷰포트에서도 UI 요소가 화면 내에 수용되도록 보장하는 반응형 레이아웃 패턴.
- **ScrollManager 재활용**: Cycle 21에서 도입한 모멘텀+바운스 물리 스크롤이 업그레이드·도감 화면에서 네이티브 앱 수준의 터치 UX를 제공.
