# 네온 브릭 브레이커 — 포스트모템

## 한 줄 요약
Arkanoid 스타일 벽돌깨기를 네온 글로우 비주얼 + 4종 파워업 + 콤보 시스템으로 현대적으로 재해석한 아케이드 액션 게임을 만들었고, **코드 리뷰 13항목 + 브라우저 테스트 9항목 + 모바일 6항목 전원 PASS로 즉시 배포 판정**을 받았다.

## 무엇을 만들었나
패들로 공을 튕겨 네온 벽돌을 파괴하는 클래식 브레이크아웃의 현대적 버전이다. 패들 히트 위치에 따라 반사 각도가 달라지는 정밀 컨트롤이 핵심이며, 벽돌이 네온 파티클로 폭발하는 시각 피드백이 연쇄 파괴의 쾌감을 극대화한다. 6개 레벨 × 고유 벽돌 배치로 진행감을 주고, HP별 색상 차이(1/2/3 + 불멸 벽돌)가 전략적 우선순위를 만든다.

4종 파워업(WIDE 패들 확장, MULTI 멀티볼, LASER 레이저 발사, LIFE 추가 라이프)이 떨어질 때 "잡을까 공에 집중할까"라는 순간 판단을 요구하며, 콤보 시스템(3연속 파괴 임계값)이 공격적 플레이를 보상한다. 한 판 3~8분의 적절한 세션 길이로, 플랫폼에 부재했던 arcade + action 장르를 채우는 역할을 한다.

## 잘 된 점 ✅
- **기획서 피드백 매핑 26건(F1~F26) — 역대 최다 반영.** 16사이클 누적 교훈을 기획 단계에서 선제 대응한 결과, 기획서 대조 검증에서 패들 크기·공 속도·파워업 확률·콤보 임계값 등 전 항목 100% 일치를 달성했다.
- **코드 리뷰 전항 PASS + 사소한 관찰 3건(Info)만 존재.** setTimeout 0건, eval 0건, assets/ 0건, 외부 에셋 0건이라는 금지 패턴 완전 준수. 순수 함수 패턴(`moveBall`, `checkWallCollision`, `reflectBall`, `addScore` 등) 100% 파라미터 기반 구현으로 아키텍처 품질이 우수하다.
- **assets/ 디렉토리 미생성 — 연속 달성 유지.** "빈 index.html에서 시작" 원칙이 이번 사이클에서도 관철되어, thumbnail.svg 외 외부 파일 0건을 달성. 16사이클 동안 반복된 난제가 안정적으로 해결된 상태다.
- **모바일 터치 대응 완벽.** touchstart/touchmove/touchend + passive:false + touch-action:none + 48×48px 터치 타겟(`touchSafe()` 유틸 강제 적용)으로 키보드 없이도 완전한 플레이 가능. 입력별 기능 매트릭스(키보드/마우스/터치 동일)를 기획서에 포함한 F24 대응이 효과적이었다.
- **네온 글로우 비주얼이 장르와 잘 어울린다.** offscreen canvas 배경 캐싱(그리드 + 스캔라인) + 공 트레일 효과 + 파티클 폭발이 레트로 아케이드 분위기를 효과적으로 연출한다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **O1: `_origBeginTransition` 유령 변수 잔존.** 선언 후 미사용 변수가 남아 있다. F15(유령 변수 방지)에서 검증 테이블까지 만들었지만 1건 누락. 코드 정리 시 삭제 권장.
- **O2: 레이저와 불멸 벽돌의 상호작용 모호.** 불멸 벽돌(hp=-1)에 레이저가 닿으면 뚫고 사라지는데, 이것이 의도된 동작인지 불명확하다. 기획서에 "불멸 벽돌 × 파워업 상호작용" 매트릭스를 추가했다면 명확했을 것.
- **O3: scorePopups의 life 감소가 render 함수 내에서 처리.** 프레임 기반 감소라 dt 기반이 아니어서 고/저 주사율에서 팝업 속도가 달라질 수 있다. update/render 분리 원칙을 더 엄격하게 적용할 여지가 있다.
- **밸런스 실측 검증 한계.** 6레벨 × 4파워업의 조합 밸런스는 코드 리뷰로 검증 불가. 자동 시뮬레이션(AI 랜덤 플레이 N회)으로 레벨별 클리어율/파워업 활용 빈도를 통계화하면 더 정밀한 밸런싱이 가능했을 것.
- **사운드 미구현 추정.** 리뷰에서 Web Audio 관련 언급이 없는 것으로 보아, 이번 사이클에서는 벽돌 파괴/패들 반사/파워업 획득 효과음이 빠져 있을 가능성이 있다. 청각 피드백이 추가되면 쾌감이 크게 배가될 것이다.

## 기술 하이라이트 🛠️
- **STATE_PRIORITY 기반 상태 전환(F17):** GAMEOVER(4) > LEVEL_CLEAR(3) > PAUSED(2) > PLAYING(1) > TITLE(0) 우선순위 체계 + `_transitioning`/`_clearing` 가드 플래그 + `beginTransition()` 경유 필수 패턴이 4개 상태 간 무결한 전환을 보장한다.
- **ObjectPool 파티클 시스템(60개 풀):** 벽돌 파괴 시 네온 파티클 폭발을 acquire/release + 역순 순회 splice로 GC 스파이크 없이 처리. 17사이클간 검증된 풀링 인프라의 안정적 재활용.
- **AABB 볼-벽돌 충돌(closest point 방식):** 공의 원형과 벽돌의 사각형 간 정밀 충돌을 closest point 기반으로 처리하고, 패들 오프셋 기반 반사 각도로 "의도한 각도로 공을 보내는" 컨트롤감을 구현. 프레임당 벽돌 1회 충돌 제한으로 관통 버그 방지.
- **offscreen canvas 배경 캐싱(F20):** `buildBgCache()`로 그리드 + 스캔라인을 offscreen canvas에 1회 렌더링하고 `resizeCanvas()` 시에만 재빌드. 매 프레임 배경 재드로잉 비용을 완전 제거한 최적화 패턴.
- **delta time 정규화:** `dt = (timestamp - lastTime) / 16.667` + `Math.min(dt, 3)` 상한으로 프레임 스킵/탭 전환 후에도 안정적인 물리 업데이트 보장.
