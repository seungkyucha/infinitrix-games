# 미니 골프 어드벤처 — 포스트모템

## 한 줄 요약
외부 물리 라이브러리 없이 순수 2D 벡터 연산만으로 벽 반사·마찰·포탈을 구현한 10레벨 탑다운 미니 골프 게임을 만들었다.

## 무엇을 만들었나
마우스/터치 드래그로 공의 방향과 힘을 조절해 최소 타수로 홀에 넣는 탑다운 2D 미니 골프 게임이다. 10개 레벨을 순서대로 진행하며, 레벨마다 벽 반사, 모래 구간(마찰 증가), 물 해저드(페널티), 움직이는 벽, 포탈 같은 새로운 요소가 하나씩 등장한다. 궤적 프리뷰로 1차 반사까지 미리 볼 수 있어 "이 각도면 벽에 튕겨서 홀에 들어간다"는 계획적 쾌감이 핵심이다.

InfiniTriX 플랫폼 최초의 물리 기반 게임이라는 점이 가장 큰 의미다. 기존 5개 게임(퍼즐/슈팅/전략/러너/리듬퍼즐)에 없던 물리 메커닉을 추가하면서, 난이도를 easy로 설정해 진입 장벽을 낮췄다. 별 3개 시스템과 홀인원 보너스(+500점)가 리플레이 동기를 부여한다.

## 잘 된 점 ✅
- **기획서 충실도 100% 달성**: 22개 체크항목 전체 PASS. 2회차 재리뷰에서 APPROVED를 받았으며, 1회차 지적사항(assets/ 잔존, 퍼펙트 반사 보너스 미구현) 2건 모두 깔끔하게 수정했다.
- **금지 패턴 완전 제거**: 4사이클 연속 재발했던 SVG 에셋 문제가 드디어 0건으로 종결되었다. assets/ 디렉토리 자체를 생성하지 않는 원칙이 효과적이었고, 3단계 자동 검증(코딩 전/중/후)도 성과를 냈다.
- **물리 연산의 정확성**: 법선 벡터 기반 벽 반사, 점-선분 거리 충돌 판정, 마찰 감속 모델이 수학적으로 정확하게 구현되었다. 관대한 홀 판정(시각 16px, 판정 20px)은 "아슬아슬하게 들어간" 쾌감을 잘 살렸다.
- **Cycle 5 교훈의 실질적 반영**: 단일 갱신 경로 원칙(ball.x/y는 물리 루프에서만), setTimeout 완전 제거, enterState() 패턴, clearImmediate() API 모두 안정적으로 계승되었다.
- **터치 대응 완벽**: 드래그 에이밍이라는 조작 특성상 터치와 마우스가 자연스럽게 통합되었고, 키보드 없이도 모든 핵심 플레이가 가능하다.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1회차에서 assets/ 잔존 재발**: 자동 검증 3단계를 기획했음에도 1회차 리뷰에서 assets/ 디렉토리가 발견되었다. 검증 스크립트의 실행 타이밍 강제화가 여전히 과제다.
- **speed() 함수의 결합도**: 전역 `ball` 객체에 직접 의존하는 `speed()` 함수가 파라미터화되지 않아 테스트 용이성이 떨어진다. 순수 함수 패턴으로 리팩터링하면 단위 테스트가 가능해진다.
- **Space 가속 시 물리 스킵 가능성**: 4배속 가속 중 벽 충돌이 프레임 스킵될 수 있는 구조다. 현재 3회 반복 루프로 대응하고 있으나, 서브스텝 물리가 더 견고한 해법이다.
- **레벨 에디터 부재**: 10개 레벨이 JS 배열 리터럴로 하드코딩되어 있다. 시간이 있었다면 비주얼 레벨 에디터나 JSON 기반 레벨 로더를 만들어 확장성을 높였을 것이다.

## 기술 하이라이트 🛠️
이번 게임의 핵심 기술은 **외부 라이브러리 없는 2D 물리 시뮬레이션**이다. Vector2 클래스를 직접 구현하여 법선 벡터 반사(`V' = V - 2·dot(V,N)·N`)와 에너지 손실(×0.85)을 처리했다. 점-선분 거리 함수(`pointToSegmentDist`)로 공-벽 충돌을 감지하고, 마찰 계수(0.985/0.95)로 감속 모델을 구현했다.

궤적 프리뷰 시스템도 흥미로운데, 실제 물리를 20스텝 시뮬레이션하면서 1차 반사까지 점선으로 표시한다. 이는 플레이어에게 "예측 가능한 물리"라는 신뢰감을 주면서도, 2차 반사 이후는 숨겨 도전 요소를 유지하는 균형 잡힌 설계다.

포탈 시스템은 쿨다운 30프레임으로 무한 루프를 방지하고, 움직이는 벽은 sin 함수 기반 왕복으로 예측 가능한 타이밍 퍼즐을 만들었다. offscreen canvas로 잔디 텍스처를 캐시하여 매 프레임 배경 재드로잉을 피한 것도 실용적인 최적화였다.
