# 젬 나이트메어 — 포스트모템

## 한 줄 요약
C47 매치3 엔진 위에 Royal Match식 미니게임 4종과 왕 캐릭터 내러티브를 통합하여, 단순 퍼즐을 넘어선 **내러티브 미니게임 통합 매치3**를 완성했다.

## 무엇을 만들었나

보석 왕국 4개 지역을 여행하며 20레벨을 클리어하는 매치3 퍼즐이다. 핵심 차별점은 5레벨마다 자동 진입하는 "왕의 위기" 미니게임 4종(물 상승 탈출, 불길 회피, 함정 미로, 드래곤 보스)으로, 반복 피로를 해소하면서 캐릭터에 감정적 몰입을 만든다. 왕이 공포에 떠는 표정에서 구출 후 환한 미소로 바뀌는 연출이 "내가 왕을 구했다"는 보상감을 준다.

기술적으로는 C47 젬 오딧세이의 검증된 엔진(8×8 6색, 스페셜 조합 6종, 장애물 7종, 부스터 4종)을 계승하면서, `safeGridAccess()` 래퍼, `coreUpdate()` 단일화, `finally { input.flush() }` 등 C47에서 아쉬웠던 코드 구조 문제 3건을 모두 해결했다. 80개 PNG 에셋, 왕 4포즈 스프라이트, 보석 애니메이션, 지역별 BGM 스케일까지 갖춘 프로덕션급 완성도다.

## 잘 된 점 ✅
- **미니게임 4종 완전 통합**: 기획서 §0의 핵심 신규 기능이 STATE.MINI_INTRO/MINI_GAME/MINI_RESULT 상태 머신으로 깔끔하게 구현됨. DDA도 미니게임까지 확장(추가 시간+난이도 감소)
- **1차 리뷰 버그 3건 완벽 수정**: GOAL_CHECK 데드락, RESULT→LEVEL_INTRO 전환 누락, 렌더 루프 정지 — 2차 Puppeteer 테스트에서 회귀 0건
- **코드 품질 대폭 개선**: TRANSITION_TABLE 16상태 화이트리스트, 10중 가드 플래그 `isInputBlocked()` 단일 함수, SeededRNG(Math.random() 0건). 리뷰어가 코드 구조/품질 9/10, 기획 적합성 10/10 부여
- **색맹 모드 실제 구현**: C47에서 "준비만"이었던 것을 F5 토글 + GEM_SHAPES 마커 6종 + 하이콘트라스트 배경으로 완성. 6색 매치3에서 접근성 필수 기능
- **80개 에셋 + 전수 Canvas 폴백**: `drawAssetOrFallback()` 패턴으로 에셋 로드 실패에도 게임 100% 동작 보장

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **1차 리뷰에서 치명적 버그 3건 발생**: GOAL_CHECK 동기 호출 데드락은 enterState()에서 상태 전환 중 또 다른 전환을 트리거하는 고질적 패턴. 48사이클째에도 여전히 발생한다는 것은 설계 단계에서 "enterState 안에서 절대 전환하지 않는다"는 규칙을 기계적으로 강제할 장치가 필요하다는 뜻
- **밸런스 자동 시뮬레이터 미착수**: 기획서 F4에 DEV_AUTO_PLAY 간이 시뮬레이터를 명세했지만 실제 구현 여부 미확인. 20레벨×7종 장애물 조합의 난이도 밸런스는 여전히 수동 검증에 의존
- **공용 엔진 모듈 분리 48사이클째 지연**: TweenManager, ObjectPool, TransitionGuard 등 48개 게임에서 복사-붙여넣기. 매 사이클 같은 버그를 반복 수정하는 비용이 누적되고 있음
- **모바일 대응 9/10**: 터치 시스템은 완비되었으나, 미니게임 4종의 모바일 UX(특히 미로 스와이프, 드래곤 보스 보석 매칭+회피 동시 조작)에 대한 실기기 테스트가 부족

## 기술 하이라이트 🛠️

**TRANSITION_TABLE 16상태 화이트리스트**가 이번 사이클의 핵심 안전장치다. TITLE→MAP→LEVEL_INTRO→PLAY→GOAL_CHECK→RESULT에 MINI_INTRO→MINI_GAME→MINI_RESULT 3상태가 추가되어 총 16상태 전환 경로를 명시적으로 관리한다. `directTransition()`에서 허용 전환만 실행하고, BUG-1 수정 시 `GUARDS.transitioning` 가드를 오히려 제거하여 중첩 호출을 안전하게 허용하는 역발상이 인상적이었다.

`safeGridAccess(r, c)` 래퍼는 C47의 gGrid 빈 배열 크래시(BUG-1)를 근본적으로 해결한다. 모든 그리드 접근이 이 래퍼를 경유하므로, 보드 미생성 상태에서의 접근이 구조적으로 불가능하다. `coreUpdate()` 단일화와 `finally { input.flush() }`와 함께 "방어적 프로그래밍 3종 세트"로 자리잡았다.

미니게임 DDA(`getMiniGameAdjustment()`)도 눈에 띈다. 메인 매치3의 failStreak 기반 DDA와 별개로, 미니게임별 추가 시간/난이도 감소를 독립 적용하여 두 시스템이 서로 간섭하지 않는 설계다.
