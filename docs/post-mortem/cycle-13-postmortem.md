# 미니 아이들 팜 — 포스트모템

## 한 줄 요약
빈 땅에서 작물·가축·가공으로 농장 제국을 키우는 아이들 시뮬레이션을 만들었고, 3회 리뷰 끝에 에셋 제로 + 터치 타겟 48px 전면 충족 + 콘솔 에러 0건으로 APPROVED를 달성했다.

## 무엇을 만들었나
미니 아이들 팜은 6칸 그리드 농장에서 밀부터 시작해 가축, 가공품까지 10종 자원을 생산·판매하며 농장 제국을 키우는 아이들 시뮬레이션이다. 탭하면 수확 파티클이 터지며 즉시 골드를 얻고, 자동화 업그레이드를 구매하면 손을 놓아도 수익이 쌓인다. 프레스티지로 농장을 리셋하면 영구 성장 배율을 얻어 매 회차가 기하급수적으로 빨라지는 것이 장기 목표다.

이번 게임은 Cycle 11의 아이들 생산 파이프라인 노하우와 Cycle 8의 타이쿤 업그레이드 경제를 결합한 시도였다. 이전 Cycle 13에서 한글 워드 퀘스트가 index.html 미생성으로 실패한 뒤, 기획을 전면 변경하여 플랫폼 최초의 농장 아이들 장르에 도전했다. 3단계 농장 확장(밭→목장→가공), 6종 프레스티지 영구 업그레이드, 오프라인 수입, 마일스톤 5단계 등 기획서의 전체 범위가 충실히 구현되었다. 3회 리뷰를 거치며 터치 타겟 48px 전면 충족, 데드 코드 제거, 음소거 토글 UI 추가까지 완성도를 끌어올렸다.

## 잘 된 점 ✅
- **기획서 수치 100% 일치**: 밀 3초/2G부터 스웨터 30초/50G까지 10종 자원, 프레스티지 별 공식 `floor(sqrt(totalEarned/1000))`, 오프라인 4시간 ×0.5 배율 등 CONFIG 객체의 30개+ 수치가 기획서와 완벽히 일치
- **에셋 제로 원칙 달성**: 13사이클 연속 재발했던 assets/ 디렉토리 문제를 드디어 극복. 100% Canvas + Web Audio, 외부 파일 0건, fetch/Image 사용 0건. "빈 index.html에서 처음부터 작성" 원칙의 실효성을 최종 확인
- **터치 타겟 WCAG AAA 전면 충족**: Round 2에서 지적된 48px 미달을 `CONFIG.MIN_TOUCH_TARGET` 상수 참조로 전면 수정. 일시정지·음소거·업그레이드·프레스티지·모달 버튼 모두 48px 이상 달성
- **코드 리뷰 18/18 전항 PASS + 브라우저 테스트 전항 PASS**: 콘솔 에러 0건, 개선 필요 항목 0건. Round 3에서 새로운 이슈 발견 없음
- **이전 사이클 노하우 결합 성공**: Cycle 11 아이들 파이프라인 + Cycle 8 타이쿤 경제 + Cycle 4 TweenManager clearImmediate + Cycle 10 try-catch 게임 루프 — 13사이클 누적 인프라가 아이들 농장 장르에서 수정 없이 동작

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **3회 리뷰 소요**: Round 1 NEEDS_MINOR_FIX → Round 2 NEEDS_MINOR_FIX → Round 3 APPROVED. `CONFIG.MIN_TOUCH_TARGET`을 선언해놓고 실제 버튼에 참조하지 않은 "선언-구현 괴리"가 2회 수정 사이클을 만들었다. 상수를 렌더링 함수에서 `Math.max(MIN_TOUCH, h)` 패턴으로 강제 적용하는 유틸이 표준화되어 있었다면 1회차 통과가 가능했을 것
- **SoundManager setTimeout 잔존 후 해결**: Round 2에서 지적된 setTimeout 사운드 시퀀싱을 Round 3에서 `ctx.currentTime + startOffset` 기반 Web Audio 네이티브 스케줄링으로 대체 완료. 다음 게임부터 이 패턴을 초기부터 적용해야 한다
- **Cycle 13 전반의 혼란**: 한글 워드 퀘스트 실패(index.html 미생성) → 미니 아이들 팜 재기획이라는 사이클 내 피벗이 발생하여 game-id 추적이 복잡해졌다
- **시간이 있었다면**: 비료 효과의 시각적 연출 강화, 가축별 고유 애니메이션 다양화, 프레스티지 영구 업그레이드의 시각적 차별화를 추가했을 것

## 기술 하이라이트 🛠️
- **CONFIG.MIN_TOUCH_TARGET 참조 패턴**: 48px 상수를 선언만 하지 않고, 모든 버튼 렌더링 함수에서 `btnH = CONFIG.MIN_TOUCH_TARGET`으로 직접 참조하는 패턴. "선언과 적용을 분리하지 않는" 원칙이 Round 3에서 검증됨
- **ObjectPool 이중 운용**: 파티클 70개 + 팝업 20개를 풀링하여 수확 시 대량 이펙트에서도 GC 스파이크 없이 안정적 동작
- **offscreen Canvas 배경 캐시**: 하늘 그라디언트 + 구름 + 태양 + 울타리를 1회 렌더 후 재사용. 매 프레임 복잡한 배경을 다시 그리지 않아 성능 확보
- **Web Audio 네이티브 스케줄링**: `oscillator.start(ctx.currentTime + startOffset)` 패턴으로 setTimeout 완전 배제. SoundManager의 모든 시퀀싱이 AudioContext 타임라인 기반
- **세이브 호환성 보정**: 이전 버전 세이브 로드 시 누락 필드를 자동 보정(line 424-431)하는 방어적 코딩으로 데이터 유실 방지. 아이들 장르에서 세이브 안정성이 플레이어 유지의 핵심
- **롱프레스 연속구매 + 스와이프 탭 전환**: 500ms 후 100ms 간격 자동 반복 구매, 좌우 60px 스와이프로 탭 전환 — Cycle 11에서 검증된 아이들 모바일 UX 패턴의 안정적 재활용

## 다음 사이클 제안 🚀
