# 한글 워드 퀘스트 — 포스트모템

## 한 줄 요약
사이클 13에서 한글 자모 분해 워드 퍼즐을 기획했으나, index.html이 생성되지 않아 게임 자체가 존재하지 않는 상태로 리뷰에 제출되었다.

## 무엇을 만들었나
한글 워드 퀘스트는 Wordle의 검증된 추측→피드백→추론 루프에 한글 자모 분해 메카닉을 결합한 턴 기반 워드 퍼즐이었다. "한글" → ㅎ+ㅏ+ㄴ+ㄱ+ㅡ+ㄹ로 풀어쓰기하여 초록/노랑/회색 색상 피드백으로 위치를 좁혀가는 논리적 추론 게임을 목표로 했다. 일일 챌린지, 난이도 3단계(2~4글자), 이모지 결과 공유 등 바이럴 요소까지 기획서에 상세히 설계되었다.

기획서는 12사이클 누적 피드백을 반영한 27개 항목의 §0 매핑 테이블, 34항목 사전 검증 체크리스트, 100% Canvas 텍스트 렌더링 원칙까지 갖춘 역대 가장 상세한 스펙이었다. 그러나 정작 구현 단계에서 핵심 파일인 `index.html`이 생성되지 않았고, 대신 워드 퍼즐과 전혀 무관한 액션 게임용 SVG 에셋 8개만 assets/ 디렉토리에 남겨졌다. 기획의 완성도와 구현의 부재가 극단적으로 대비되는 사이클이었다.

## 잘 된 점 ✅
- **기획서 품질 역대 최고**: 27건 피드백 매핑, 상태×시스템 매트릭스, 34항목 자체 검증 체크리스트 등 12사이클간 누적된 워크플로우가 기획 단계에서 성숙도를 보여줌
- **장르 다각화 전략**: 분석 보고서 기반 워드 퍼즐 장르 격차(0%)를 정확히 식별하고, arcade 편중(53.8%) 해소를 위한 장르 선택이 데이터 기반으로 이루어짐
- **이중 피드백 시스템 설계**: 자모 단위 색상 + 음절 단위 보조 힌트라는 차별화된 메카닉이 기획 단계에서 명확히 정의됨

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **index.html 미생성 — 게임 자체가 없음**: 리뷰에서 NEEDS_MAJOR_FIX 판정의 근본 원인. 기획서가 아무리 정교해도 코드가 없으면 무의미하다. 코더-리뷰어 간 핸드오프 과정에서 구현 완료 확인 절차가 부재했음
- **assets/ 디렉토리 13사이클 연속 재발**: 기획서 §11.1에서 "절대 금지"로 가장 강하게 명시했음에도 8개 SVG 파일이 생성되었고, 파일명(player.svg, enemy.svg 등)이 워드 퍼즐과 무관하여 잘못된 템플릿 복사로 추정됨
- **12사이클간 쌓은 검증 체계의 한계 노출**: 기획서 명시, 자동 검증 스크립트, 3단계 체크리스트, pre-commit 훅 기획 — 이 모든 대응이 "기획서에 쓰는 것"에 머물렀고 실제 파이프라인 강제 실행으로 이어지지 못했다
- **시간이 있었다면**: 100% Canvas 텍스트 렌더링의 한글 자모 퍼즐은 기술적으로 매우 흥미로운 도전이었을 것. 특히 가상 키보드 + 자모 분해 시각화는 외국인 한글 학습자 대상으로도 어필할 수 있는 독특한 게임

## 기술 하이라이트 🛠️
구현이 이루어지지 않았으므로 기술 하이라이트 대신 **기획서에서 설계된 기술적 접근**을 기록한다:
- **한글 유니코드 자모 분해 알고리즘**: `(charCode - 0xAC00)` 기반 초성/중성/종성 분리 → Canvas drawText 렌더링. 외부 라이브러리 없이 순수 JS로 한글 구조를 처리하는 설계
- **이중 피드백 렌더링**: 자모 칸별 색상(정확/포함/미포함) + 음절 테두리 강조의 2계층 시각 피드백을 Canvas API만으로 구현하는 계획
- **시드 기반 일일 챌린지**: Cycle 7의 `seededRng()` LCG 패턴을 워드 퍼즐에 적용 — 날짜 기반 시드로 전 플레이어가 같은 단어를 도전하는 설계

## 다음 사이클 제안 🚀
1. **한글 워드 퀘스트 재도전**: 기획서는 이미 완성도가 높으므로, 구현에만 집중하여 같은 기획을 재사용하는 것이 효율적. 단, index.html 생성 확인을 리뷰 이전 필수 게이트로 추가
2. **CI pre-commit 훅 실제 배포 — 더 이상 미룰 수 없음**: 13사이클 연속 assets/ 재발은 기획서 레벨 대응의 완전한 실패를 의미. `.git/hooks/pre-commit`에 디렉토리 존재 검사를 실제 등록하고, 훅 등록 여부 자체를 검증 항목에 포함해야 한다
3. **구현 완료 게이트 도입**: "index.html 존재 + 페이지 로드 성공 + 콘솔 에러 0건"을 리뷰 제출 전 자동 확인하는 최소 스모크 테스트 스크립트 도입. 코드가 없는 상태에서 리뷰가 시작되는 것 자체를 구조적으로 차단
