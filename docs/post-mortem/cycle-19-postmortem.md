# 스펠 카드 던전 — 포스트모템

## 한 줄 요약
Slay the Spire 스타일 로그라이크 덱빌더를 2,815줄 단일 HTML에 카드 20종, 적 7종, 보스 3체, 유물 8종, 업적 10종까지 구현하여 코드 품질 9/10, 게임성 9/10, 모바일 10/10을 달성했다.

## 무엇을 만들었나
플레이어가 마법사가 되어 10층 던전을 탐험하는 턴제 카드 전투 로그라이크다. 매 턴 마나 3을 써서 손패 5장 중 최적의 카드를 골라 사용하고, 적의 인텐트(다음 행동 미리 표시)를 보며 방어와 공격을 판단해야 한다. 전투 승리 시 3택 1 카드 보상으로 덱을 강화하고, 상점에서 구매/제거/업그레이드, 휴식에서 HP 회복, 이벤트에서 리스크-리워드 선택을 하며 진행한다.

핵심 차별점은 **원소 상성 시스템**이다. 화염/빙결/번개/암흑 4원소가 적 타입별 1.5배 특효를 갖고 있어, 덱 구성 시 "이 층에서 어떤 원소가 유리한가"를 고려하는 전략적 깊이가 있다. 3/6/10층 보스는 고유 패턴과 페이즈 전환을 갖고 있어 일반 전투와 확실히 다른 긴장감을 준다.

유물 8종이 런 전체에 걸치는 패시브 효과를 부여하며, 업적 10종이 반복 플레이 동기를 추가한다. 한 판 8~12분의 세션 길이와 즉시 재도전 구조가 "한 판만 더"의 중독성을 만든다.

## 잘 된 점 ✅
- **Phase 1~5 거의 완전 구현**: 기획서에 명시된 카드 20종, 적 7종, 보스 3체, 유물 8종, 이벤트 4종, 업적 10종, 상점, 휴식, 맵 시스템이 모두 존재한다. 10개 게임 상태(TITLE→MAP→BATTLE→BOSS_INTRO→REWARD→SHOP→REST→EVENT→RESULT→GAMEOVER)가 안정적으로 전환된다.
- **기획서 수치 100% 일치**: 시작 HP 80, 마나 3, 턴당 드로우 5장, 카드 C01~C18 데미지/코스트, 슬라임~아크메이지 HP/ATK, 원소 배율 1.5×까지 전수 검증 통과. F28(밸런스 검증) 대응의 성과.
- **모바일 대응 10/10**: 턴제 카드 게임에 완벽히 적합한 탭 기반 UI. touchSafe 48px, touch-action:none, passive:false, 뷰포트 설정, 입력 모드 자동 감지 전부 충실. 키보드 없이 전 기능 플레이 가능.
- **코드 품질 우수**: requestAnimationFrame + dt 캡, try-catch 게임 루프, offscreen bgCache, TweenManager, 4중 가드 플래그(isTransitioning/isAnimating/isSelectingCard/isBossIntro), STATE_PRIORITY 기반 전환, 단일 갱신 경로(modifyHP/modifyMana/addGold), Web Audio 네이티브 스케줄링(setTimeout 0건) 등 18사이클 누적 인프라가 충실히 반영됨.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 19사이클 연속 재발 (C1)**: 기획서 §14.5에서 "절대 생성 금지"를 명시했음에도 8개 SVG + manifest.json이 생성되었다. Canvas fallback이 완비되어 기능 문제는 없지만, 불필요한 네트워크 요청 8건이 발생. **19사이클 연속 재발은 CI/pre-commit 훅 없이는 해결 불가능하다는 결론을 재확인.**
- **적 디버프 미적용 버그 (C2)**: 밴시의 약화, 가디언의 분쇄, 아크메이지의 빙결이 빈 임시 객체에 적용되어 즉시 버려짐. 시각적 텍스트만 뜨고 실제 플레이어에 영향 없음. 게임이 의도보다 쉬워지는 밸런스 문제를 야기.
- **카드 스와이프 미구현 (M1)**: 6장 이상 손패 시 좌/우 스크롤이 없어 일부 카드가 보이지 않을 수 있음. 현재 기본 5장이라 당장 문제없지만 드로우 추가 효과 시 발생 가능.
- **덱 보기(D키) 미구현 (M2)**: 현재 덱 전체를 확인할 방법이 없어 전략적 의사결정에 정보 부족.
- **맵 키보드 노드 선택 미구현 (M3)**: 기획서 §3.2에 명시된 ←→ 방향키 맵 이동이 누락.

## 기술 하이라이트 🛠️
- **원소 상성 테이블 (`ELEM_MULT`)**: 2D 배열로 화염/빙결/번개/암흑 4원소 × 적 타입별 배율을 관리. Cycle 18의 DMG_TABLE 패턴을 덱빌더 장르에 성공적으로 적용. 카드 사용 시 `damage *= ELEM_MULT[element][enemyType]` 한 줄로 상성 계산이 완료되는 깔끔한 구조.
- **인텐트 시스템**: 적의 다음 행동(공격/방어/버프/디버프)을 아이콘으로 미리 표시하여 플레이어가 대응 전략을 세울 수 있게 함. Slay the Spire의 핵심 메카닉을 Canvas 코드 드로잉으로 재현.
- **10개 상태 머신**: 플랫폼 역대 최다급 상태 수(Cycle 14의 12개에 근접)에서 STATE_PRIORITY + beginTransition + 4중 가드가 안정적으로 동작. 턴제 게임의 명확한 상태 전환 시점이 경쟁 조건을 구조적으로 최소화.
- **Web Audio 보스 전용 BGM**: SoundManager가 일반 전투/보스전 BGM을 분리하고, 카드 원소별 SFX를 네이티브 스케줄링으로 처리. setTimeout 8사이클 연속 0건 달성.
- **F1~F30 역대 최다 30건 피드백 매핑**: 기획서 §0에서 18사이클 누적 교훈 30건을 전수 매핑. 피드백 매핑 건수가 27건(Cycle 18) → 30건으로 증가하면서 코드 리뷰 15항목 PASS, 브라우저 테스트 10항목 PASS를 달성.
