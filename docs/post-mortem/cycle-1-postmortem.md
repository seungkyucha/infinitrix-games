# 픽셀 심연 — 포스트모템

## 한 줄 요약
InfiniTriX 플랫폼의 첫 번째 게임으로, BSP 던전 생성과 A* 적 AI를 갖춘 턴제 로그라이크 던전 크롤러를 단일 HTML 파일로 완성했다.

## 무엇을 만들었나

**픽셀 심연**은 16-bit 픽셀아트 스타일의 턴제 로그라이크입니다. 플레이어는 매 런마다 BSP 알고리즘으로 랜덤 생성되는 5층 던전을 탐험하며, 슬라임·박쥐·해골 전사·마법사 4종의 적과 전투하고, 최종 보스 '다크 나이트'를 처치하는 것이 목표입니다.

핵심은 **영구 사망 + 메타 성장** 루프입니다. 죽으면 처음부터지만, 골드는 영구 보존되어 로비 상점에서 HP·공격력·시야·행운 업그레이드를 구매할 수 있습니다. "한 판 더" 중독성을 의도했고, 리뷰어가 "30초 내 재미 전달"을 PASS 판정한 것으로 보아 핵심 루프는 성공적이었습니다.

인벤토리(3칸), 함정 타일, 원거리 적(마법사 LoS 판정), 보스 2페이즈 시스템까지 기획 대부분이 구현되었습니다. 첫 사이클치고 스코프가 컸지만 턴제라는 선택 덕분에 복잡도를 관리할 수 있었습니다.

## 잘 된 점 ✅

- **IX 엔진 완벽 준수**: GameFlow/Scene/Button/Input 모든 엔진 API를 올바르게 사용했고, 자체 setTimeout/addEventListener 호출 0건. 첫 게임에서 엔진 표준을 확립한 좋은 선례.
- **재시작 안정성**: 3회 연속 재시작 테스트에서 HP/층/점수/골드/인벤토리 모두 완벽 초기화. 배열·맵·트윈·파티클 누수 없음.
- **게임플레이 완성도**: BSP 던전 생성, A* 경로탐색 AI, Bresenham FOV, 보스 2페이즈, 영구 업그레이드 — 로그라이크의 핵심 시스템이 모두 동작. QA에서 D 섹션 전 항목 PASS.
- **에셋 품질**: player.png, boss, 배경(bg-lobby 등) 대부분이 16-bit 픽셀아트 스타일을 정확히 구현. 특히 bg-lobby.png은 "매우 높은 품질" 평가.
- **JS 에러 0건**: 전체 QA 과정에서 콘솔 에러가 단 한 건도 발생하지 않았음.

## 아쉬웠던 점 / 개선 가능성 ⚠️

- **버튼 최소 크기 미준수**: "뒤로"(36px), "타이틀"(40px) 2개 버튼이 44px 미달. 1차 리뷰에서 지적됐지만 2차까지 미수정. 보조 버튼이라 키보드로 대체 가능하나, 모바일 터치 UX에서 문제.
- **에셋 일관성 깨짐**: thumbnail.png의 캐릭터(갈색머리 소년)가 실제 player.png(헬멧 기사)과 불일치. enemy-slime이 SVG 벡터로 픽셀아트 스타일과 이질감. player-attack/hurt, boss-attack도 SVG 대체 상태.
- **피드백 루프 부재**: 1차→2차 리뷰 사이에 피드백 문서가 없어 수정이 이루어지지 않음. 에이전트 간 커뮤니케이션 프로세스 개선 필요.
- **턴 키 이중 소비**: LOBBY→PLAY 전환 시 Space가 D-pad ⏸에도 소비되어 turnCount 1에서 시작. 작은 버그지만 첫 인상에 영향.

## 기술 하이라이트 🛠️

- **BSP 던전 생성**: 이진 공간 분할로 4~6개 방을 만들고 복도로 연결하는 절차적 생성. 매 런마다 다른 맵 구조를 보장하면서도 플레이 가능한 레이아웃을 안정적으로 생성.
- **A* 경로탐색 + FOV 연동**: 적 AI가 시야 내일 때만 A*로 추적하고, 시야 밖에서는 랜덤 배회. Bresenham's line 기반 Raycasting으로 벽 뒤 적이 보이지 않는 긴장감 구현.
- **턴제 렌더링 최적화**: requestAnimationFrame 루프를 유지하되 턴 변화 시에만 실제 재렌더링. 불필요한 프레임 연산을 줄여 브라우저 부담 최소화.
- **Web Audio 톤 합성**: 별도 오디오 파일 없이 Web Audio API로 hit/score/gameover 등 효과음을 실시간 생성. 단일 HTML 파일 제약 내에서의 영리한 접근.
