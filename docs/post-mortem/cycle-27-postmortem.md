# 엘리멘탈 캐스케이드 (Elemental Cascade) — 포스트모템

## 한 줄 요약
6원소 매치-3 보석 퍼즐과 RPG 전투를 결합한 로그라이크, 단일 HTML 4,238줄에 5지역·7보스·13유물·18스펠을 담아냈다.

## 무엇을 만들었나
엘리멘탈 캐스케이드는 8×8 보석 그리드에서 6원소(화염/물결/대지/바람/빛/암흑)를 매칭해 마나를 생성하고, 축적된 마나로 18종 스펠을 시전하여 봉인된 던전의 몬스터와 보스를 처치하는 **원소 매치-3 로그라이크 RPG**다. 3매치→4매치(십자 폭발)→5매치(원소 폭발)→L/T매치(특수 효과)로 이어지는 캐스케이드 콤보의 시각적 쾌감이 핵심이다.

5개 지역(화산/심해/숲/하늘/심연)을 순차 공략하며, 지역마다 고유 환경 효과(화염 추가 데미지, 물결 보석 자동 생성, 번개 고정 데미지 등)가 전략을 변화시킨다. 전투 사이 3택 유물 선택(common 6/rare 4/epic 3)과 영구 업그레이드 3트리(공격/방어/유틸)가 로그라이크 빌드 다양성을 제공하고, 히든 보스 "원소의 근원"까지 도달하면 진엔딩을 볼 수 있다. 3단 난이도(수습/마법사/대마법사), DDA 동적 밸런스, 한/영 다국어까지 지원한다.

## 잘 된 점 ✅
- **기획서 100% 구현 달성**: 8×8 그리드, 6원소 상성, 매치 5종(3/4/5/L/T), 마나 생성량, 5지역·7보스·13유물·업그레이드 3트리, 3난이도, DDA, SeededRNG, 다국어 — 기획서 CONFIG 수치가 코드와 1:1로 정합.
- **보스 이중 지급 버그 근본 해결**: `bossRewardGiven` 플래그 + `checkBattleEnd()→checkEnemiesDefeated()` 위임 구조로 보상 단일 경로 확보. 코드 중복도 해소되어 유지보수성이 크게 향상.
- **비주얼 품질**: 적 유형별 고유 비주얼 5종, 보스 6원소 장식, 지역별 환경 이펙트(용암·기포·낙엽·번개·왜곡), 패럴랙스 별 배경, 보스 줌인 연출, 콤보 골드 글로우, 카메라 쉐이크 — 에셋 0건 100% Canvas 프로시저럴 드로잉으로 달성.
- **코드 품질 우수**: 스모크 테스트 12/12 PASS, 브라우저(puppeteer) 테스트 콘솔 에러 0건, Math.random 0건(SeededRNG 완전), setTimeout 0건(16사이클 연속), hitTest() 단일 함수로 모든 터치/클릭 판정 통합.
- **4회차 리뷰에서 APPROVED**: 1~3회차 지적 P1~P5 전부 수정 유지 + P2 보스 이중 지급 완전 해결. 회귀 버그 0건.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **4회 리뷰 사이클 소요**: 1차에서 assets/ 불법 SVG 8건(P0)과 RESTART_ALLOWED 데드코드, 스와이프 결함, 터치 버튼 높이 미달이 동시 발견. 자동 스모크 테스트 게이트가 있었다면 2회 이내로 단축 가능했을 것.
- **공용 엔진 분리 27사이클째 미착수**: TweenManager/ObjectPool/SoundManager/InputManager가 여전히 단일 파일 내 copy-paste. 4,238줄(역대 최대급) 단일 HTML은 유지보수 한계에 근접.
- **밸런스 자동 검증 부재**: 6원소×4매치타입×13유물×5적종×7보스 조합 공간이 방대하지만, DPS/EHP 캡(200%/150%)과 DDA 폴백에 의존할 뿐 실제 자동 시뮬레이션은 미구현.
- **assets/ 1차 재발**: Cycle 18~26까지 10사이클 연속 정리 기록이 위기에 처했다가 2차에서 복구. CI 강제 차단 없이는 이 문제가 27사이클째에도 1차에서 재발한다는 사실을 재확인.
- **히트 영역·드로잉 좌표 분리 문제**: Cycle 26에서 지적된 drawTitle() 언어 토글 12px 오프셋 패턴이 이번에도 hitTest() 단일 함수로 개선되었으나, 기존 레거시 코드의 좌표 분리 패턴은 완전히 청산되지 않았다.

## 기술 하이라이트 🛠️
- **매치 검출 알고리즘**: 5매치→T→L→4→3 우선순위 스캔 + 재귀적 캐스케이드 처리. `isProcessing` 가드 + `_transitioning` 이중 가드로 콜백 충돌 0건.
- **SeededRNG 완전 사용**: `G.rng.next()` 전용, Math.random 문자열 0건. 보석 배치, 적 스폰, 유물 선택지 모두 시드 기반으로 리플레이 재현 가능.
- **bossRewardGiven 플래그 패턴**: 보상 지급 경로를 `checkEnemiesDefeated()` 단일 함수로 통합하고, 플래그 가드로 어떤 호출 경로에서든 최초 1회만 지급을 보장하는 구조. 상태 전환과 보상 로직의 분리가 깔끔.
- **hitTest() 단일 함수**: 기존 터치/클릭 판정이 함수마다 분산되던 패턴(F60)을 `hitTest(x, y, rect)` 단일 함수로 완전 통합. 히트 영역 동기화 문제를 구조적으로 예방.
- **18상태 수준의 게임 깊이**: 타이틀/난이도/전투/보스인트로/보스전/유물선택/상점/일시정지/게임오버/승리 등 다수 상태를 STATE_PRIORITY + beginTransition + RESTART_ALLOWED 조합으로 무결 관리. 가드 플래그 패턴 20사이클 연속 안정.
- **devicePixelRatio + Math.min(rawDt, 33.33)**: 고해상도 디스플레이 대응 + 2프레임 스킵 캡으로 다양한 환경에서 안정적 렌더링.
