# 네온 대시 러너 — 포스트모템

## 한 줄 요약
네온 사이버펑크 도시를 질주하는 무한 러너를 만들었으나, TweenManager 경쟁 조건으로 게임 시작 자체가 불가능한 치명적 버그가 발견되어 재작업이 필요하다.

## 무엇을 만들었나
3개 레인을 전환하고 점프로 장애물을 넘으며 코인을 모으는 무한 러너 게임이다. 절차적 청크 생성으로 매 게임마다 다른 장애물 배치가 나오고, 거리에 비례해 속도가 올라가면서 "한 판만 더"의 중독성을 노렸다. 배리어·스파이크·레이저·드론 4종 장애물과 자석·쉴드·x2 3종 파워업으로 단순한 조작 속에 전략적 깊이를 더했다.

시각적으로는 네온 사이버펑크 테마를 순수 Canvas 드로잉으로 구현했다. 3레이어 패럴랙스 배경(별 하늘·건물 실루엣·3레인 도로)이 속도감을 강화하고, 코인 획득·충돌·Near Miss 등 다양한 이펙트가 플레이의 쾌감을 증폭시킨다. 플랫폼 최초의 casual 장르 진입이기도 하다.

Cycle 3에서 지적된 tween 상태 전환 가드 패턴을 `beginTransition()` + `STATE_PRIORITY` 맵으로 표준화하고, 유령 변수 방지 체크리스트를 도입한 점은 의미 있는 아키텍처 진화였다.

## 잘 된 점 ✅
- **TransitionGuard 패턴 완성**: `STATE_PRIORITY` 맵 + `beginTransition()` 헬퍼로 상태 전환 경쟁 조건의 근본 해결책을 설계하고 구현 완료. GAMEOVER(99)가 모든 전환을 우선하는 구조가 잘 동작한다.
- **유령 변수 근절**: `inputMode` 4개 사용처, `nearMissCount`, `timeSinceLastPowerup`, `consecutiveSafeDist` 등 기획서에 명시된 모든 변수가 실제로 갱신·사용됨을 리뷰에서 확인. Cycle 2~3의 반복 이슈를 드디어 해결.
- **절차적 레벨 생성 성공**: 6종 패턴 + 안전 레인 보장 규칙 + 동적 밸런스 보정이 기획 의도대로 동작. 매 플레이가 다른 경험을 제공하면서도 "반드시 피할 수 있는" 공정성을 유지한다.
- **기획서 체크리스트 14/16 PASS**: 금지 패턴 grep 검사, 매트릭스 준수, 이징 5종 완전 구현, destroy() 패턴, Canvas 모달 등 대부분의 품질 기준을 통과.
- **강제 진입 시 게임플레이 완성도 높음**: B1만 우회하면 장애물·코인·충돌·HUD·Near Miss·파워업·게임오버 연출 등 모든 게임 시스템이 정상 동작.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **[B1] 게임 시작 불가 — CRITICAL**: `resetGame()`이 `tw.cancelAll()`을 호출한 직후 새 tween을 추가하면, deferred `_pendingCancel` 플래그가 신규 tween까지 삭제해버린다. `transitioning`이 영구 true로 고정되어 유저가 게임을 시작할 수 없다. TweenManager의 deferred 패턴을 3사이클째 사용하면서도 "cancelAll 직후 add" 시나리오를 테스트하지 않은 것이 근본 원인이다.
- **[B2] SVG 에셋 재발 — Cycle 2→3→4 세 번째**: 기획서에 "SVG 완전 미사용"을 명시했음에도 에셋 9개가 생성되었다. Canvas 폴백 코드가 있어 기능적 문제는 없지만, `feGaussianBlur` 포함은 3사이클 연속 재발로 자동 검증의 부재가 명확한 원인이다.
- **코인 콤보 보너스 미구현**: 기획서 §7.1의 "연속 코인 5개 → +20점" 메커니즘이 빠졌다. 총 누적만 세고 연속 판정 로직이 없다.
- **타이틀 글로우 tween 비복구**: GAMEOVER→TITLE 복귀 시 `pulseTitle()`이 재호출되지 않아 글로우가 멈춘다. 작은 문제지만 시각적 완성도를 떨어뜨린다.

## 기술 하이라이트 🛠️
**절차적 청크 생성 시스템**이 이번 게임의 기술적 핵심이다. 400px 단위 청크를 거리 기반 난이도에 따라 6종 패턴(SINGLE_BARRIER, SPIKE_ROW, DOUBLE_BARRIER, LASER_GATE, DRONE_PATROL, GAUNTLET)에서 가중 랜덤으로 선택한다. "안전 레인 1개 이상 보장" 규칙과 "연속 레이저 금지" 같은 안전장치가 절차적 생성의 가장 큰 위험인 "불가능한 배치"를 방지한다.

**패럴랙스 3레이어 배경**은 offscreen canvas 캐시를 활용해 매 프레임 건물 실루엣을 다시 그리지 않고 drawImage 오프셋만 변경한다. 네온 창문(3×3px 랜덤 색상 사각형)과 도로 양옆 글로우 스트립이 적은 드로잉 비용으로 사이버펑크 분위기를 만들어낸다.

**Near Miss 시스템**은 장애물 통과 시 y 거리 < 40px 조건으로 "아슬아슬 회피"를 판정하고, 슬로모 효과(gameSpeed × 0.8, 200ms tween)와 +30점 팝업으로 즉각 보상한다. 단순한 판정이지만 긴장감과 보상감을 동시에 높이는 효과적인 메커니즘이다.

## 다음 사이클 제안 🚀
1. **TweenManager `clearImmediate()` 메서드 추가**: deferred cancel과 즉시 clear를 분리해야 한다. `cancelAll()` 직후 `add()`가 안전하게 동작하도록 API를 개선하고, 이를 공용 유틸로 확정하자.
2. **에셋 자동 검증 스크립트 실제 도입**: 3사이클 연속 SVG 재발은 "기획서 명시"만으로는 해결 불가임을 증명했다. 빌드/리뷰 단계에서 `grep -rn "feGaussianBlur\|\.svg\|setTimeout\|confirm(" games/` 자동 실행이 필요하다.
3. **리듬/음악 장르 도전**: Web Audio API 절차적 사운드가 2사이클 연속 안정적으로 동작했으므로, 음악 동기화(비트 매칭) 기반 게임을 시도해볼 시점이다. casual 장르의 확장이기도 하다.
