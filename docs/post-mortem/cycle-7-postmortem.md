# 미니 서바이버 아레나 — 포스트모템

## 한 줄 요약
360도 몬스터 웨이브를 자동 공격과 12종 스킬 빌드로 돌파하는 뱀서라이크 서바이버를 단일 HTML 1,397줄로 구현했다.

## 무엇을 만들었나
Vampire Survivors에서 영감을 받은 탑다운 서바이버 게임이다. 플레이어는 이동만 담당하고 공격은 자동으로 발사된다. 사방에서 밀려오는 4종 몬스터(일반/빠른/탱커/원거리)와 매 5웨이브 보스를 상대하며, 레벨업 시 12종 스킬 중 3개를 선택해 자신만의 빌드를 완성한다. 멀티샷으로 화면을 투사체로 뒤덮을 수도, 오비탈+충격파로 근접 방어를 쌓을 수도 있다.

핵심 재미는 "매 판 다른 빌드"에서 나온다. 초반의 허약한 1발 공격이 후반에는 화면을 가득 채우는 탄막으로 성장하는 판타지가 강력하다. 콤보 시스템(2초 내 연속 처치 ×3.0 배율)과 일일 챌린지(시드 기반 동일 패턴)가 리플레이 동기를 더한다. Normal/Hard 난이도 선택과 가상 조이스틱 터치 지원으로 접근성도 확보했다.

## 잘 된 점 ✅
- **기능 완성도 100%**: 기획서 20개 핵심 항목(20웨이브, 12종 스킬, 보스 3페이즈 AI, 일일 챌린지, 콤보, 난이도 모드) 전부 구현 완료. 아키텍처 13개 항목 + 금지 패턴 7개 + 필수 패턴 7개 모두 PASS.
- **순수 함수 설계 성공**: Cycle 6에서 지적된 전역 의존 문제를 반영하여 `updatePlayer()`, `updateEnemies()`, `fireWeapon()`, `hitProjEnemy()`, `hitEnemyPlayer()`, `pickGems()` 등 18개 핵심 로직 함수를 파라미터 기반 순수 함수로 작성. 1회차 리뷰에서 5개 전투/충돌 함수의 전역 P 참조가 지적되었으나 2회차에서 모두 파라미터화 완료.
- **누적 인프라의 안정성**: TweenManager(clearImmediate), ObjectPool(4종 800개), TransitionGuard(STATE_PRIORITY), EventManager(listen/destroy), Web Audio SFX 9종 — 6사이클간 다듬어진 공용 패턴이 새로운 장르에서도 흔들림 없이 동작했다.
- **선언적 웨이브 config**: `waveCfg(w)` 함수로 적 수, 스폰 간격, HP/속도 배율, 유형 비율을 공식 기반으로 스케일링. 하드코딩 최소화라는 Cycle 6 피드백을 충실히 반영했다.
- **2회차 리뷰 APPROVED**: 1회차 NEEDS_MINOR_FIX → 지적 3건 모두 수정 → 2회차에서 콘솔 에러 0건, 전 항목 PASS로 즉시 배포 가능 판정.

## 아쉬웠던 점 / 개선 가능성 ⚠️
- **assets/ 디렉토리 재발 (1회차)**: 기획서에 "100% Canvas 코드 드로잉", "assets/ 미생성 원칙"을 명시했고, Cycle 5~6에서 종결 선언까지 했음에도 1회차 리뷰에서 SVG 9개 + manifest.json이 발견되었다. 2회차에서 삭제 완료했지만, 7사이클 중 대부분에서 반복된 이 문제는 기획서 명시만으로는 근본 해결이 안 됨을 확정적으로 증명한다. **CI/빌드 훅에서 assets/ 존재 시 빌드 실패하는 강제 장치가 필수다.**
- **순수 함수 원칙 부분 위반 (1회차)**: 5개 전투/충돌 함수에서 전역 P를 직접 참조했다. 2회차에서 모두 파라미터화하여 해결했지만, 원칙을 기획서에 명시해도 구현 시 일부 위반이 발생하므로 전수 검증 체크리스트가 필수임을 재확인했다.
- **크리티컬 수치 불일치**: 기획서 Lv3=30% vs 구현 25%. 기획-구현 정합성 검증이 기능 존재 여부뿐 아니라 수치 레벨까지 내려가야 한다.
- **시간이 있었다면**: 스킬 시너지 시스템(특정 조합 보너스), 처치 통계 대시보드, 일일 챌린지 리더보드 UI를 추가하고 싶었다.

## 기술 하이라이트 🛠️
- **ObjectPool 4종 동시 운용**: 적(150), 투사체(200), XP젬(200), 파티클(300) 총 850개 객체를 풀링하여 GC 스파이크를 제거. 역순 순회 + splice 안전 패턴으로 매 프레임 수십 개 acquire/release가 안정적으로 동작한다.
- **시드 기반 RNG**: `seededRng()` LCG 알고리즘 + `dateSeed()` djb2 해시로 일일 챌린지에서 동일 날짜 = 동일 적 스폰/스킬 풀을 보장. 서버 없이 localStorage만으로 비동기 경쟁감을 구현한 첫 프로토타입.
- **보스 3페이즈 AI**: HP 비례 페이즈 전환(돌진→방사탄→소환+돌진)을 상태 머신 위의 상태 머신으로 구현. 페이즈 전환 시 0.5초 무적 + 스크린 셰이크로 극적인 보스전 연출.
- **1600×1600 월드 + lerp 카메라**: `updateCam()` 순수 함수로 부드러운 카메라 추적 + 뷰포트 컬링(`inView()`)으로 화면 밖 오브젝트 렌더링 스킵. 미니맵(80×80)으로 월드 전체 상황 파악.
- **config 기반 웨이브 스케일링**: 20웨이브의 적 수(8→40), 스폰 간격(1500→300ms), HP/속도 배율, 유형 비율을 `waveCfg(w)` 단일 함수로 제어. 밸런스 조정이 상수 변경만으로 가능.
