# 프리즘 브레이크 (Prism Break) — 사이클 #39 게임 기획서

> **1페이지 요약**: 어둠에 잠식된 프리즘 왕국을 되찾기 위해, 크리스탈을 배치·회전하여 빛 경로를 만들고 그림자 적을 소멸시키는 **액션 퍼즐**. 빛 굴절 퍼즐 + 실시간 그림자 적 전투의 하이브리드로, Poki 2026 3월 #1 Level Devil의 action+puzzle 트렌드와 CrazyGames Top 20의 퍼즐-플랫포머 4작 인기에 직접 대응한다. 5구역 × 3스테이지 = 15 메인 + 보스 3종(MVP) + 히든 2개 = **총 20스테이지**. 업그레이드 트리(빛 강도/굴절 범위/크리스탈 종류 3갈래×5단계) + SeededRNG 크리스탈 배치 + BFS 경로 검증. **action+puzzle 11사이클 최장 미사용 해소 + 기존 다크 테마 2작(phantom-shift, shadow-rift)과 정반대 밝은 비주얼로 극적 차별화.**

> **MVP 경계**: **Phase 1**(핵심 루프: 크리스탈 배치→빛 굴절→그림자 소멸→구역 정화, 구역 1~3 + 보스 3체 + 업그레이드 Lv1~3 + DDA 4단계 + 코인 시스템 + 기본 내러티브) → **Phase 2**(구역 4~5 + 히든 2스테이지 + 업그레이드 Lv4~5 + 날씨/시간대 연출 + 전체 내러티브 + 다국어 완성). **Phase 1만으로도 완전한 빛 굴절 액션 퍼즐 경험 제공 필수.**

## §1. 게임 개요 및 핵심 재미 요소

### 핵심 컨셉

**"빛을 조종하라, 그림자를 소멸시켜라"** — 크리스탈을 배치하고 회전하여 빛 경로를 설계하는 퍼즐 + 그림자 적이 실시간으로 접근하는 액션의 하이브리드.

### 스토리

프리즘 왕국은 빛으로 에너지를 얻는 문명이었으나, '그림자 침식(Shadow Corruption)'이 빛의 원천인 핵심 프리즘을 잠식했다. 왕국은 어둠에 빠졌고, 주민들은 그림자 괴물로 변해버렸다. 마지막 빛의 수호자(Lux)는 남은 크리스탈 파편을 모아 빛을 복원하고, 5개 구역을 정화하여 핵심 프리즘에 도달해야 한다.

### 핵심 재미 요소

1. **공간 퍼즐**: 크리스탈을 그리드에 배치하고 각도를 조절하여 광원에서 목표(그림자 코어)까지 빛 경로를 연결
2. **실시간 긴장감**: 그림자 적이 지속적으로 광원을 향해 이동 — 빛에 닿으면 소멸하지만, 광원에 도달하면 에너지 감소
3. **유레카 모먼트**: 복잡한 반사 경로가 한 번에 연결되어 다수의 적을 동시 소멸시키는 쾌감
4. **빌드 다양성**: 3갈래 업그레이드 트리로 "강한 단일 빔" vs "약한 다중 빔" vs "특수 색상 빔" 전략 분기
5. **프로시저럴 리플레이**: SeededRNG 기반 크리스탈 배치 + 적 스폰 패턴 변형

### 단일 핵심 루프 (Cycle 35 planner 교훈 적용)

```
크리스탈 배치(퍼즐) → 빛 경로 활성화 → 그림자 적 소멸(액션) → 구역 정화 → 업그레이드
```

서브시스템은 **업그레이드 트리 하나**로 제한. 핵심 메카닉 함수 5개 이내:
1. `placeCrystal(grid, pos, type)` — 크리스탈 배치
2. `rotateCrystal(grid, pos, angle)` — 크리스탈 회전
3. `traceLight(grid, source, crystals)` — 빛 경로 계산 (레이캐스팅)
4. `checkShadowHit(lightPath, enemies)` — 그림자 적 충돌 판정
5. `updateEnemies(enemies, dt, target)` — 적 이동/AI 업데이트

## §2. 게임 규칙 및 목표

### 주요 목표
- 5개 구역(수정 동굴→프리즘 궁전→무지개 다리→그림자 요새→빛의 핵심부) 각 3스테이지를 클리어하여 프리즘 왕국을 정화
- 각 스테이지에서 광원의 빛을 크리스탈로 굴절시켜 **그림자 코어**(스테이지 정화 목표)에 도달시키면 클리어

### 기본 규칙
1. **그리드 시스템**: 각 스테이지는 12×8 그리드. 일부 셀은 벽/장애물로 차단
2. **광원**: 스테이지 좌측에 고정된 빛 발사 장치. 항상 우측으로 빛 발사
3. **크리스탈**: 빈 셀에 배치 가능. 빛이 닿으면 설정 각도로 반사/굴절
4. **그림자 코어**: 스테이지 목표 지점. 빛을 연결하면 정화
5. **그림자 적**: 맵 가장자리에서 스폰하여 광원을 향해 이동. 빛 경로에 닿으면 소멸. 광원에 도달하면 에너지 -1
6. **에너지**: 스테이지당 초기 에너지 5. 0이 되면 스테이지 실패. 실패해도 즉시 재도전 가능 (코인 유지)
7. **코인**: 적 소멸 시 드롭. 업그레이드에 사용

### 크리스탈 종류 (7종, MVP Phase 1 기준)

| ID | 이름 | 효과 | 해금 구역 |
|----|------|------|----------|
| C1 | 반사 크리스탈 | 빛을 90° 반사 (4방향 설정) | 구역 1 시작 |
| C2 | 분산 프리즘 | 빛을 2갈래로 분리 (45°/315°) | 구역 1-2 |
| C3 | 집속 렌즈 | 2개 빛을 1개 강화 빔으로 합침 | 구역 2-1 |
| C4 | 색상 필터 (R) | 빛을 빨간색으로 변환 (빨간 코어 정화용) | 구역 2-3 |
| C5 | 색상 필터 (B) | 빛을 파란색으로 변환 (파란 코어 정화용) | 구역 3-1 |
| C6 | 시간 크리스탈 | 빛 통과 시 주변 2칸 적 슬로우 (50%) | 구역 3-2 |
| C7 | 폭발 크리스탈 | 빛 통과 시 주변 3×3 범위 적 즉사 (쿨다운 5초) | 구역 3-3 |

### 승리/패배 조건
- **스테이지 클리어**: 그림자 코어에 올바른 색상의 빛 연결 + 해당 웨이브의 적 전멸 (또는 생존 시간 경과)
- **스테이지 실패**: 에너지 0
- **게임 클리어**: 5번째 구역 보스 격파 (Phase 2) / 3번째 구역 보스 격파 (Phase 1 MVP)

## §4. 시각적 스타일 가이드

### §4.1 코드/에셋 원칙

- **assets/ 디렉토리 절대 생성 금지** (F1, F24): 모든 그래픽은 inline SVG 리터럴 또는 Canvas 프로시저럴 렌더링
- **외부 CDN/폰트 0건** (F2): 시스템 폰트만 사용 (`'Segoe UI', system-ui, sans-serif`)
- **SVG 필터 제한 사용** (F3~7 사이클 재발 교훈): `feGaussianBlur` 최소화, blur는 Canvas `ctx.filter` 대체. SVG는 도형+그라디언트만 사용
- 스모크 테스트: `grep -r "assets/" index.html` 결과 0건 → FAIL 게이트

### §4.2 색상 팔레트

### §5.2 메인 루프 (60fps 기준)

```
매 프레임 (requestAnimationFrame):
  1. dt 계산 (deltaTime, cap 33ms)
  2. TW.update(dt)                    ← TweenManager 글로벌 업데이트
  3. POOL.update(dt)                  ← ObjectPool 파티클 업데이트
  4. switch(G.state):
     TITLE:  drawTitle(ctx)
     MAP:    drawMap(ctx, G.zone, G.stage, G.upgrades)
     PLAY:   updatePlay(dt) → drawPlay(ctx)
     BOSS:   updateBoss(dt) → drawBoss(ctx)
  5. drawUI(ctx, G)                   ← 공통 UI 오버레이
  6. SND.update()                     ← 사운드 큐 처리
```

**updatePlay(dt) 상세:**
```
1. 웨이브 타이머 갱신
2. 적 스폰 (스폰 간격에 따라)
3. updateEnemies(enemies, dt, lightSource) — 이동 + AI
4. traceLight(grid, source, crystals) → G.lightPaths 갱신
5. checkShadowHit(lightPaths, enemies) — 적 피격 판정 (F8: 판정 먼저)
6. 에너지 감소 체크 (적이 광원 도달)
7. 코어 정화 체크 (빛이 코어 도달 + 적 전멸)
8. 코인/점수 갱신 (F8: 저장 나중에)
9. DDA 판정 (실패 횟수 기반)
```

**tween 규칙 (F4, F5, F13, F14):**
- `setTimeout` 사용 0건 → 모든 지연은 `TW.add()` + `onComplete`
- 모든 tween 콜백에 가드 플래그 (`waveClearing = true`) 설정
- `TW.clearImmediate()` API로 즉시 정리 (deferred 경쟁 조건 방지)
- 각 값의 갱신 경로는 1개만 존재 (tween OR 직접 대입, 절대 중복 금지)

## §6. 상태 머신

### §6.1 TRANSITION_TABLE 단일 정의 (F6, F21, F25)

```javascript
const TRANSITION_TABLE = {
  TITLE: ['MAP'],                    // 타이틀 → 맵
  MAP:   ['PLAY', 'TITLE'],          // 맵 → 플레이, 맵 → 타이틀
  PLAY:  ['MAP', 'BOSS', 'PLAY'],    // 플레이 → 맵(일시정지), 보스, 재시작
  BOSS:  ['MAP'],                    // 보스 → 맵(클리어)
};

// 정적 검증: 정확히 4상태
console.assert(Object.keys(TRANSITION_TABLE).length === 4, '상태 수 불일치');

function beginTransition(from, to, onComplete) {
  if (!TRANSITION_TABLE[from]?.includes(to)) {
    console.warn(`Invalid transition: ${from} → ${to}`);
    return false;
  }
  // 페이드 아웃 → 상태 변경 → 페이드 인
  TW.add({ target: G, prop: 'fadeAlpha', from: 1, to: 0, dur: 300,
    onComplete: () => {
      G.state = to;
      if (onComplete) onComplete();
      TW.add({ target: G, prop: 'fadeAlpha', from: 0, to: 1, dur: 300 });
    }
  });
  return true;
}
```

## §8. 난이도 시스템

### 수학적 난이도 곡선 (F26)

스테이지 번호 `n` (1~15, 보스 제외):

| 파라미터 | 공식 | n=1 | n=8 | n=15 |
|---------|------|-----|-----|------|
| 적 수 | `5 + floor(n × 1.5)` | 6 | 17 | 27 |
| 적 이동속도 (기본) | `30 + n × 3` px/s | 33 | 54 | 75 |
| 빈 셀 수 (크리스탈 배치 가능) | `max(6, 20 - n)` | 19 | 12 | 6 |
| 웨이브 수 | `1 + floor(n / 4)` | 1 | 3 | 4 |
| 코인 드롭량 | `floor(n × 0.5) + 1` | 1 | 5 | 8 |
| 기준 클리어 시간 | `30 + n × 6` 초 | 36 | 78 | 120 |

### DDA (Dynamic Difficulty Adjustment) — 4단계

| 단계 | 조건 (같은 스테이지 실패 횟수) | 보정 |
|------|---------------------------|------|
| 0 (기본) | 실패 0~1회 | 없음 |
| 1 (약간 쉽게) | 실패 2~3회 | 적 이동속도 -15%, 에너지 +1 |
| 2 (쉽게) | 실패 4~5회 | 적 이동속도 -25%, 에너지 +2, 빛 경로 힌트 (1개 크리스탈 위치 표시) |
| 3 (매우 쉽게) | 실패 6회+ | 적 이동속도 -40%, 에너지 +3, 빛 경로 힌트 (2개) |

> ⚠️ Cycle 35 planner 교훈: "난이도 자체는 유지하되 정보를 더 제공하는" DDA. 적을 줄이는 대신 힌트를 제공하여 퍼즐 본연의 재미 보존.

## §10. 프로시저럴 생성 시스템

### §10.1 스테이지 생성 규칙

1. 12×8 그리드 기본 템플릿 로드 (구역별 3종 사전 정의)
2. SeededRNG로 벽 위치 ±2셀 변형
3. 적 스폰 포인트 랜덤 배치 (가장자리 4면)
4. 코어 위치 랜덤 (우측 4열 중 택 1)
5. BFS 도달 가능성 검증 (§10.2)
6. 실패 시 seed+1로 재생성 (최대 10회)

### §10.2 BFS 빛 경로 도달 가능성 검증 (F25, F27)

```
입력: grid[12][8], source(0, y), cores[(cx, cy)]
출력: boolean (모든 코어에 빛 도달 가능 여부)

알고리즘:
1. 광원에서 BFS 시작 — 빈 셀을 통해 4방향 탐색
2. 각 빈 셀에 가상 크리스탈(최적 각도) 배치 가정
3. 빛 경로가 최소 1개 코어에 도달 가능한 경로 존재 확인
4. 필요한 크리스탈 수 ≤ 배치 가능 인벤토리 수 확인
5. 모든 조건 충족 시 true, 아니면 seed+1 재생성
```

> ⚠️ Cycle 23 planner: "도달 가능성 검증" 단계 필수 포함.
> ⚠️ Cycle 26 planner: "약점 위치는 BFS 경로에서 최소 1셀 이격" — 보스전에서 보스 약점 위치가 빛 경로를 물리적으로 차단하지 않도록 검증.

## §11. 점수 시스템

### §11.1 점수 계산 (F8: 판정 먼저, 저장 나중에)

```javascript
// 1단계: 판정
const stageScore = baseScore + timeBonus + comboBonus + noDamageBonus;
```

| 항목 | 계산 | 비고 |
|------|------|------|
| 기본 점수 | `적_소멸수 × 100` | — |
| 시간 보너스 | `max(0, (기준시간 - 클리어시간) × 10)` | 빠를수록 높음 |
| 콤보 보너스 | `연속_소멸 × 50 × 콤보_카운트` | 빛 1경로로 다수 적 소멸 |
| 무피격 보너스 | `에너지 === 5 ? 500 : 0` | 퍼펙트 보너스 |
| 보스 격파 보너스 | `보스HP × 200` | 보스전 한정 |

### §11.2 랭킹 (localStorage)

- 스테이지별 최고 점수 + 클리어 시간 저장
- 전체 스코어 = 모든 스테이지 최고 점수 합산
- 별 등급: ★(클리어), ★★(시간 보너스 달성), ★★★(무피격)

## 부록 A: 극단 빌드 밸런스 검증

### 빌드 1: 파워 올인 (빛 Lv5 + 굴절 Lv0 + 크리스탈 Lv0)

```
DPS = 기본데미지(2) × 관통(×2적) + 범위(×0.5 인접) + 태양폭발(3데미지 전체)
= 스테이지 n=15 기준: 적 27체, HP 평균 2.5
태양폭발 1회 = 27 × 3 = 81 데미지 (대부분 즉사)
나머지: 일반 빔으로 처리 (크리스탈 3개, 배치 유연성 낮음)
예상 클리어 시간: ~70초 (태양폭발 쿨다운 대기 포함)
```

### 빌드 2: 컨트롤 올인 (빛 Lv0 + 굴절 Lv5 + 크리스탈 Lv0)

```
배치 유연성 극대화: 45° 회전 + 인벤토리 4개 + 웨이브 중 재배치 + 무한 반사
데미지: 기본 1 (낮음), 경로 수 많음 (8방향 반사로 복잡 경로)
예상 클리어 시간: ~100초 (데미지 부족, 다경로로 보완)
```

### 빌드 3: 크리스탈 마스터 (빛 Lv0 + 굴절 Lv0 + 크리스탈 Lv5)

```
특수 크리스탈 강화: 폭발 쿨 3초 + 시간 범위 3칸 + 레인보우 캐논
데미지: 폭발(3×3 즉사 매 3초) + 레인보우(전색 정화)
```

## 부록 B: SeededRNG 구현 참고

```javascript
class SeededRNG {
  constructor(seed) { this.s = seed; }
  next() {
    this.s = (this.s * 1664525 + 1013904223) & 0xFFFFFFFF;
    return (this.s >>> 0) / 0xFFFFFFFF;
  }
  int(min, max) { return min + Math.floor(this.next() * (max - min + 1)); }
  pick(arr) { return arr[this.int(0, arr.length - 1)]; }
  shuffle(arr) {
    const a = [...arr];
    for (let i = a.length - 1; i > 0; i--) {
      const j = this.int(0, i);
      [a[i], a[j]] = [a[j], a[i]];
    }
    return a;
  }
}
```
