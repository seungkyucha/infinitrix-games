## 1. 게임 개요 및 핵심 재미 요소

**폴리 스파이어**는 Slay the Spire에서 영감받은 싱글 플레이어 로그라이크 덱빌더입니다. 플레이어는 기본 덱 10장으로 시작하여, 3개 층(Floor)을 거치며 전투·이벤트·상점·휴식 노드를 선택하고, 카드를 획득·제거하며 자신만의 덱을 구축합니다.

### 핵심 재미 요소
1. **덱 빌딩의 쾌감**: 매 전투 후 3장 중 1장을 선택하는 보상 시스템. "이 카드를 넣으면 내 덱이 어떻게 변할까?" 매 선택이 전략적 의사결정.
2. **카드 콤보 발견**: 방어 카드로 블록을 쌓고, 스킬 카드로 드로우를 늘리고, 공격 카드로 폭발적 딜을 넣는 시너지 발견의 재미.
3. **리스크-리워드 노드 선택**: 엘리트(강적+보상↑) vs 상점(골드 소모) vs 휴식(HP 회복) 중 어디로 갈지 매 갈림길이 도박.
4. **보스 2페이즈 긴장감**: 각 층 보스가 HP 50% 이하에서 패턴이 바뀌며, "거의 잡았다"에서 역전당하는 극적 전개.
5. **런마다 다른 경험**: 카드 보상 풀이 랜덤이므로 매 런마다 다른 빌드를 시도. 한 판 더 하고 싶은 동기.

## 2. 게임 규칙 및 목표

### 2-1. 게임 목표
- 3개 층을 돌파하고 최종 보스 **"폴리곤 코어"**를 쓰러뜨리면 승리.
- 각 층은 6~8개 노드로 구성된 맵이며, 마지막 노드는 항상 보스.

### 2-2. 전투 규칙

**턴 구조:**
1. **플레이어 턴 시작**: 에너지 3 충전, 덱에서 5장 드로우
2. **카드 사용**: 에너지를 소모하여 손패 카드를 사용 (공격/방어/스킬)
3. **턴 종료**: "턴 종료" 버튼 클릭 → 미사용 손패를 버리기 더미로 이동
4. **적 턴**: 적이 예고된 행동(공격/방어/버프)을 실행
5. **턴 반복**: 한쪽 HP가 0이 될 때까지 반복

**핵심 수치:**
- 플레이어 최대 HP: 70 (쉬움 80, 어려움 60)
- 에너지: 매 턴 3 (일부 카드/유물로 증가 가능)
- 손패 제한: 10장
- 블록(방어): 턴 시작 시 0으로 초기화 (데미지는 블록 → HP 순으로 적용)

### 2-3. 덱 관리
- **시작 덱**: 타격(Attack) 5장 + 수비(Defend) 4장 + 강타(Heavy Strike) 1장 = 10장
- **뽑기 더미(Draw Pile)**: 셔플된 덱. 소진 시 버리기 더미를 셔플하여 보충.
- **버리기 더미(Discard Pile)**: 사용한 카드 + 턴 종료 시 남은 손패.
- **소멸(Exhaust)**: 일부 강력한 카드는 사용 후 소멸 (버리기 더미에 안 감).

### 2-4. 노드 유형
| 노드 | 아이콘 | 설명 |
|------|--------|------|
| ⚔️ 일반 전투 | 검 | 일반 적 1~2체 전투. 승리 시 카드 보상 3장 중 1장 선택 + 골드 |
| 💀 엘리트 전투 | 해골 | 강화 적 전투. 보상 강화 (희귀 카드 확률↑ + 유물 드랍) |
| 🏪 상점 | 가방 | 카드 구매(50~150G), 카드 제거(75G), 포션 구매(50G) |
| 🔥 휴식 | 모닥불 | HP 25% 회복 또는 카드 1장 업그레이드(강화) 중 택 1 |
| ❓ 이벤트 | 물음표 | 랜덤 이벤트 (보상/위험 택 1~3) |
| 👑 보스 | 왕관 | 층 보스 전투. 승리 시 다음 층 진입 |

## 4.5. 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**low-poly-3d** — Low-poly 3D render style: flat-shaded polygons, geometric forms, PS1/N64 era look, visible triangulation.

### 스타일 세부 정의
- 모든 캐릭터와 오브젝트는 **낮은 폴리곤 수의 3D 모델을 렌더링한 듯한** 외형
- **평면 셰이딩(flat shading)**: 부드러운 그라데이션 대신 면마다 단일 색상
- **가시적 삼각형 분할**: 폴리곤 경계가 살짝 보이는 느낌
- **PS1/N64 시대 향수**: 약간의 텍스처 어긋남, 정점 스냅 느낌 (과하지 않게)
- **밝고 포화된 색감**: 로우폴리 특유의 장난감 같은 친근함

### 레퍼런스 게임
1. **Slay the Spire** — 카드 UI 레이아웃, 전투 화면 구성, 노드 맵 구조 참고
2. **Superflight / Astroneer** — 로우폴리 3D 아트 스타일, 평면 셰이딩, 기하학적 자연 환경 참고

> ⚠️ 모든 에셋은 이 통일된 low-poly-3d 아트 디렉션을 엄격히 따라야 함. 스타일 혼재 절대 금지.

## 5. 핵심 게임 루프 (프레임 기준 로직 흐름)

### 씬 구조
```
TITLE → DIFF_SELECT → MAP → BATTLE → REWARD → MAP → ... → BOSS → VICTORY/GAMEOVER
                              ↕ EVENT
                              ↕ SHOP
                              ↕ REST
```

### 턴제 전투 루프 (BATTLE 씬)
```
┌─ 턴 시작 ─────────────────────────────────────────────┐
│  1. 블록 → 0 초기화                                     │
│  2. 에너지 → 3 충전                                      │
│  3. 유물 "턴 시작" 효과 발동                              │
│  4. 뽑기 더미에서 5장 드로우 (애니메이션 0.3초)            │
│  5. 적 의도 표시 (아이콘: 검/방패/해골)                    │
├─ 플레이어 입력 대기 ──────────────────────────────────────┤
│  A. 카드 선택 → 대상 선택 → 카드 효과 적용 (애니 0.2초)    │
│     - 에너지 부족 시: 카드 흔들림 + 경고음                  │
│     - 적 사망 시: 사망 이펙트 (0.3초)                      │
│  B. 포션 사용 (턴 소모 안 함)                              │
│  C. "턴 종료" 클릭/키                                     │
├─ 적 턴 ──────────────────────────────────────────────────┤
│  6. 손패 → 버리기 더미 이동 (애니 0.2초)                   │
│  7. 각 적 행동 실행 (순서대로, 각 0.4초)                   │
│     - 공격: 데미지 숫자 팝업 + 화면 흔들림                  │
│     - 방어: 블록 수치 표시                                  │
│     - 버프/디버프: 아이콘 표시                              │
│  8. 독/지속피해 적용                                       │
│  9. 전투 종료 체크 (적 전멸 → REWARD / 플레이어 사망 → GAMEOVER) │
└─ 다음 턴 시작 ────────────────────────────────────────────┘
```

### 맵 씬 (MAP)
- 노드를 선택하면 해당 씬으로 전환 (200ms 입력 무시 적용)
- 현재 위치에서 연결된 노드만 선택 가능 (라인으로 연결 표시)

### 프레임 렌더링 (60fps 기준)
- 턴제이므로 물리 시뮬레이션 없음
- 애니메이션은 트윈 기반: 카드 이동, 데미지 팝업, 셰이크, 페이드
- 파티클은 기하학적 조각 (삼각형/다이아몬드) 최대 20개 동시

## 6. 난이도 시스템

### 3단계 난이도 선택 (게임 시작 시)

| 항목 | 쉬움 | 보통 | 어려움 |
|------|------|------|--------|
| 플레이어 HP | 80 | 70 | 60 |
| 시작 골드 | 120 | 99 | 80 |
| 적 HP 배율 | ×0.85 | ×1.0 | ×1.2 |
| 적 공격력 배율 | ×0.85 | ×1.0 | ×1.15 |
| 보상 카드 선택지 | 4장 | 3장 | 3장 |
| 상점 가격 | ×0.8 | ×1.0 | ×1.2 |
| 포션 칸 | 3 | 2 | 2 |
| 엘리트 수 (층당) | 1 | 1~2 | 2 |

### 층별 난이도 곡선
- **1층**: 적 1체 위주, HP/공격 낮음. 덱 빌딩 학습 구간.
- **2층**: 적 1~2체, 디버프/힐 패턴 등장. 덱 효율이 중요해짐.
- **3층**: 적 2체, 고HP/고공격. 완성된 덱의 시너지가 승리를 결정.

## 10. 기술 구현 참고 사항

### IX 엔진 API 준수 사항 (Cycle 1,2 학습)
- `GameFlow.init({ title: Scene._states.TITLE })` — 커스텀 씬 명시 전달
- Layout.fontSize 등 유틸리티는 화살표 함수 래퍼로 감싸서 사용
- 씬 전환 시 `inputDelay = 0.2` + `Math.max(0, ...)` 적용
- 모든 UI 버튼: `Math.max(48, ...)` 최소 터치 영역
- 자체 setTimeout/addEventListener 사용 금지 — IX API로 대체
- `enter(data)` 패턴으로 씬 간 데이터 전달

### 재시작 안정성 (3회 연속 테스트 필수)
`resetGameState()`에서 초기화할 항목:
- 덱 (시작 덱으로 복원)
- HP, 블록, 에너지
- 골드, 점수
- 유물, 포션
- 현재 층, 맵 데이터
- 전투 상태 (적 목록, 턴 수, 독/버프 등)
- 뽑기 더미, 버리기 더미, 소멸 더미

### 카드 데이터 구조 (JSON 기반)
```javascript
const CARDS = {
  strike: { name: '타격', cost: 1, type: 'attack', rarity: 'common',
            effect: { damage: 6 }, desc: '피해 6' },
  defend: { name: '수비', cost: 1, type: 'defense', rarity: 'common',
            effect: { block: 5 }, desc: '블록 5' },
  // ... 30장 정의
};
```

### 에셋 로딩
- `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback (Cycle 1 검증 패턴)
- 카드 아트가 로딩 실패 시 카드 프레임 + 텍스트만으로 플레이 가능하도록 설계

### Web Audio 효과음 (톤 합성)
- 카드 사용: 짧은 시안 톤 (주파수 상승)
- 피격: 낮은 임팩트 사운드
- 블록: 메탈릭 클랭 사운드
- 레벨업/보상: 상승 아르페지오
- 보스 등장: 낮은 드론 + 임팩트
