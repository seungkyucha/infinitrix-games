## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
플레이어는 마법사가 되어 10층짜리 마법 던전을 탐험한다. 매 층에서 **전투/이벤트/상점/휴식** 중 경로를 선택하고, 턴제 카드 전투로 몬스터를 처치한다. 전투 승리 시 새 카드를 획득하고, 상점에서 카드를 구매/제거/업그레이드하며 덱을 강화한다. **유물**은 런 전체에 걸치는 패시브 효과를 부여한다.

**Slay the Spire 스타일**의 로그라이크 덱빌더 — 2025~2026 최고 인기 전략 장르.

### 1.2 핵심 재미 요소
1. **덱빌딩 전략**: 매 전투 후 3택 1 카드 보상. 덱을 어떻게 구성하느냐가 승패를 결정
2. **턴제 전투 긴장감**: 제한된 마나로 어떤 카드를 사용할지 매 턴 고민. 적의 다음 행동이 미리 표시(인텐트 시스템)되어 대응 전략 수립
3. **유물 시너지**: 최대 3개 유물이 덱과 시너지를 이루며 빌드의 방향성을 결정
4. **보스전 스펙터클**: 3층·6층·10층에 고유 패턴 보스 등장. 전용 연출 + 특수 보상
5. **로그라이크 리플레이**: 매 런마다 다른 카드·유물·적 조합. 무한 리플레이 가치
6. **한 판 8~12분**: 10층 클리어 또는 체력 0 시 종료. 즉시 재도전 가능

### 1.3 범위 관리 전략 (F25 대응)
> ⚠️ Cycle 17에서 과대 범위로 구현 0%, Cycle 18에서 MVP 우선으로 성공. 이번에도 MVP 우선을 엄수한다.

**MVP 구현 순서** (반드시 이 순서로 구현):
1. **Phase 1 (핵심 — 이것만으로 실행 가능)**: TITLE → BATTLE → GAMEOVER 3상태 + 기본 카드 5종(공격3/방어2) + 슬라임 1종 + 턴 루프(드로우→사용→적턴→결과)
2. **Phase 2**: MAP 상태 + 10층 맵 생성 + 경로 선택 UI + 전투 보상(3택 1 카드)
3. **Phase 3**: 적 7종 + 카드 20종 + 상점/휴식/이벤트 노드
4. **Phase 4**: 보스 3체(3F/6F/10F) + BOSS_INTRO 상태 + 유물 시스템(8종)
5. **Phase 5**: 업적 시스템(10종) + RESULT 상태 + 파티클/이펙트
6. **Phase 6**: BGM + 효과음(8종) + 화면 흔들림/카드 애니메이션

Phase 1~2가 완성되면 **실행 가능한 게임**이다. Phase 3 이후는 점진 확장.

### 1.4 장르 균형 기여
- **strategy 장르 직접 보강** (현재 순수 strategy 단독 1개 → 2개)
- 최근 3사이클 액션 연속(arcane-bastion, rune-survivor) → 전략 중심으로 방향 전환
- mini-card-battler(단순 카드 전투)와 완전 차별화: 덱빌딩 + 로그라이크 진행 + 유물 시스템

## §2. 게임 규칙 및 목표

### 2.1 승리 조건
- 10층 던전의 최종 보스(아크메이지)를 처치하면 **클리어**

### 2.2 패배 조건
- 플레이어 HP가 0 이하가 되면 **게임 오버**

### 2.3 턴제 전투 규칙

#### 플레이어 턴
1. 턴 시작 시 **마나 3** 회복 (최대 마나까지)
2. 드로우 파일에서 **5장** 드로우
3. 마나를 소비하여 **카드 사용** (순서 자유, 여러 장 사용 가능)
4. "턴 종료" 버튼 클릭 → 미사용 카드는 버린 카드 더미로 이동

#### 적 턴
1. 인텐트(다음 행동 미리 표시)에 따라 행동 실행
2. 공격/버프/디버프 중 하나를 수행
3. 적 턴 종료 → 다음 플레이어 턴

#### 드로우 파일 소진
- 드로우 파일이 비면, 버린 카드 더미를 **셔플**하여 새 드로우 파일로

### 2.4 카드 시스템

#### 카드 속성
| 속성 | 설명 |
|------|------|
| name | 카드 이름 |
| type | attack / skill / power |
| cost | 마나 비용 (0~3) |
| rarity | common / uncommon / rare |
| element | fire / ice / lightning / arcane / neutral |
| effect | 데미지, 방어, 버프, 디버프 등 |
| description | 효과 설명 텍스트 |
| upgraded | 업그레이드 여부 (false/true) |

#### 카드 타입
- **Attack (공격)**: 적에게 데미지를 입힘
- **Skill (스킬)**: 방어도 획득, 드로우 추가, 디버프 부여 등
- **Power (파워)**: 사용 시 전투 내 영구 효과 부여 (턴마다 자동 적용)

### 4.2 배경 구성 (offscreen canvas 캐싱 — F10 대응)
- **Layer 0 (far)**: 깊은 던전 벽면 — 격자 패턴 + 어두운 그라데이션
- **Layer 1 (mid)**: 횃불 빛 + 벽 균열 디테일
- **Layer 2 (near)**: 전투 영역 바닥 — 석재 타일 패턴
- **Layer 3 (foreground)**: 파티클 이펙트 레이어 (매 프레임 렌더)

캐싱 전략: Layer 0~2는 `buildBgCache()`에서 offscreen canvas에 한 번 그린 후 `resizeCanvas()` 호출 시에만 재빌드. Layer 3만 매 프레임 렌더링.

## §5. 핵심 게임 루프

### 5.1 메인 루프 (F12 대응)
```javascript
function gameLoop(timestamp) {
  try {
    const dt = Math.min((timestamp - lastTime) / 1000, 0.1); // 100ms 상한
    lastTime = timestamp;

    update(dt);   // 상태 변경은 여기서만 (F26)
    render();     // 순수 출력 함수 (F26)
  } catch(e) {
    console.error('GameLoop error:', e);
  }
  requestAnimationFrame(gameLoop);
}
```

### 5.2 update(dt) 흐름
```
update(dt):
  tweenManager.update(dt)
  particleManager.update(dt)

  switch(state):
    TITLE:      updateTitle(dt)
    MAP:        updateMap(dt)
    BATTLE:     updateBattle(dt)
    BOSS_INTRO: updateBossIntro(dt)
    REWARD:     updateReward(dt)
    SHOP:       updateShop(dt)
    REST:       updateRest(dt)
    EVENT:      updateEvent(dt)
    RESULT:     updateResult(dt)
    GAMEOVER:   updateGameover(dt)
    PAUSED:     // 아무것도 하지 않음 (tween은 멈춤)
```

### 5.3 render() 흐름
```
render():
  drawBackground(state)     // offscreen 캐시 사용

  switch(state):
    TITLE:      renderTitle()
    MAP:        renderMap()
    BATTLE:     renderBattle()
    BOSS_INTRO: renderBossIntro()
    REWARD:     renderReward()
    SHOP:       renderShop()
    REST:       renderRest()
    EVENT:      renderEvent()
    RESULT:     renderResult()
    GAMEOVER:   renderGameover()
    PAUSED:     renderPaused()  // 위에 오버레이

  particleManager.render(ctx)
  renderTransition()          // 전환 오버레이
```

### 5.4 가드 플래그 체계 (F5 대응)
```javascript
const GUARDS = {
  isTransitioning: false,   // 화면 전환 중
  isAnimating: false,       // 카드 사용/적 행동 애니메이션 중
  isSelectingCard: false,   // 카드 선택 UI 활성화 중
  isBossIntro: false,       // 보스 등장 연출 중
};
```
모든 상태 전환 함수 진입부에 가드 체크:
```javascript
function changeState(newState) {
  if (GUARDS.isTransitioning) return;
  if (STATE_PRIORITY[newState] < STATE_PRIORITY[currentState]) return;
  GUARDS.isTransitioning = true;
  beginTransition(newState); // F23
}
```

### 7.2 데미지 공식 (F28 대응)
```
최종 데미지 = (기본 데미지 + 강화 스택) × 원소 배율 × (적이 취약 ? 1.5 : 1.0)
방어 적용 = max(0, 최종 데미지 - 적 방어도)
적 공격력 = 기본 공격력 × (둔화 중 ? 0.75 : 1.0) × (약화 중 ? 0.75 : 1.0)
```

### 7.3 적 데이터 테이블

## §13. 사운드 시스템 (Web Audio API)

### 13.1 BGM (절차적 생성)
- **던전 BGM**: 낮은 드론 + 미스터리한 아르페지오. C-minor 기반, BPM 70
- 보스전: BPM 110으로 가속 + 베이스 드럼 추가
- 볼륨: 0.15 (기본), 보스전 0.20

### 13.2 효과음 (8종 — F29 대응)
| SFX | 트리거 | 생성 방법 | 볼륨 |
|-----|--------|-----------|------|
| 카드 드로우 | 카드 드로우 시 | 짧은 "스윽" (고주파 화이트 노이즈 0.1초) | 0.2 |
| 카드 사용 | 카드 플레이 시 | 원소별 다른 음색 (fire=따뜻한 사인파, ice=차가운 톱니파) | 0.25 |
| 적 피격 | 데미지 적중 | "퍽" (노이즈 + 저주파 0.15초) | 0.3 |
| 적 사망 | 적 HP 0 | 하강 글리산도 (0.3초) | 0.3 |
| 플레이어 피격 | 플레이어 피격 | "쿵" (저주파 사인파 0.2초) | 0.35 |
| 보스 등장 | 보스전 시작 | 극적인 상승 코드 (0.8초) | 0.4 |
| 골드 획득 | 골드 획득 | "차링" (높은 사인파 연속 3음 0.2초) | 0.2 |
| 레벨업/보상 | 카드 보상 선택 | 상승 아르페지오 (0.4초) | 0.25 |

### 13.3 SFX 타이밍 매핑 (F29 대응)
```
카드 사용: 카드 애니메이션 시작 프레임에 즉시 재생
적 피격: 데미지 계산 완료 + 히트 이펙트 시작 시점에 재생 (카드 사용 후 200ms)
적 사망: 사망 파티클 시작 시점에 재생
보스 등장: BOSS_INTRO 상태 진입 즉시 재생
```

### 15.2 순수 함수 원칙 (F3 대응)

## §17. 맵 생성 알고리즘

### 17.1 절차적 맵 생성
```
1층: 전투 노드 2~3개 (첫 전투는 항상 슬라임)
2층: 전투 2개 + 이벤트 1개
3층: 보스 1개 (고정)
4층: 전투 2개 + 상점 1개
5층: 전투 2개 + 휴식 1개
6층: 보스 1개 (고정)
7층: 전투 2개 + 이벤트 1개
8층: 전투 2개 + 상점 1개
9층: 전투 3개 (엘리트 높은 확률)
10층: 보스 1개 (고정)
```

### 17.2 경로 연결 규칙
- 각 층에 2~3개 노드
- 노드 간 연결: 같은 층 내 노드에서 다음 층의 인접 노드로 1~2개 연결
- 보스 층(3/6/10)은 단일 노드, 모든 이전 노드가 연결
- 플레이어는 연결된 경로만 선택 가능
