# 섀도우 리프트 (Shadow Rift) — 사이클 #29 게임 기획서

> **1페이지 요약**: 차원 균열로 연결된 상호연결 맵을 탐험하며 능력을 언락하고 보스를 처치하는 **메트로이드바니아 로그라이트**. 5개 존(폐허/수정/심연/용암/공허) × 3방 = 15 기본 방 + 히든 존 2방 = **총 17 스테이지**. 존 보스 5체 + 히든 보스 "Void Weaver" = **6체 보스**. 5단계 능력 게이팅(대시→벽점프→차원전환→시간정지→중력반전)으로 비선형 탐험 경로 제공. 방 클리어 시 3택 아티팩트(유물) 선택(common 6/rare 4/epic 3 = 13종), 영구 업그레이드 3트리(Shadow/Rift/Echo), SeededRNG 절차적 방 변형, 3단 난이도(탐험가/전사/전설), DDA 동적 밸런스, 다국어(한/영). **action+puzzle 조합을 1→2개로 강화**하며, Poki 2026.3 #1 Level Devil(action+puzzle)의 트렌드를 계승한다.

> **MVP 경계**: Phase 1(핵심 루프: 탐험→전투→능력 해금→퍼즐→보스, 존 1~2 + 보스 2체 + 능력 3종(대시/벽점프/차원전환) + 아티팩트 6종 + 업그레이드 기본 트리) → Phase 2(존 3~5 + 보스 3체 + 히든 보스 + 능력 2종(시간정지/중력반전) + 전체 내러티브 + 아티팩트 7종). Phase 1만으로도 완전한 게임 경험 제공 필수.

## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
섀도우 리프트는 차원 균열로 연결된 상호연결 맵을 탐험하며 능력을 해금하고 보스를 처치하는 **메트로이드바니아 로그라이트**다. 5개 존의 인터커넥티드 방을 비선형으로 탐험하며, 새로운 능력을 얻을 때마다 이전에 접근할 수 없었던 경로가 열린다. **탐험의 발견 재미**, **액션 전투의 긴장감**, **퍼즐의 사고 쾌감**이 삼중 축을 이룬다.

### 1.2 핵심 재미 3축
1. **능력 게이팅 탐험 (Explore & Unlock)**: 대시→벽점프→차원전환→시간정지→중력반전 5단계 능력을 해금하며 맵이 점진적으로 열린다. "아, 여기는 벽점프가 있어야 가는 곳이구나!" → 능력 획득 후 되돌아가는 메트로이드바니아의 핵심 쾌감.
2. **액션 전투 + 환경 퍼즐**: 근접/원거리 공격 + 능력 콤보로 적을 처치. 방마다 배치된 환경 퍼즐(스위치, 이동 플랫폼, 차원 게이트)을 능력으로 해결하여 진행.
3. **로그라이트 빌드 다양성**: 방 클리어 시 3택 아티팩트 선택으로 매 런마다 다른 빌드. "대시 특화", "시간정지 특화", "중력 조작 특화" 등 다중 전략 경로.

### 1.3 스토리/내러티브
- **배경**: 차원 경계를 수호하는 "리프트 워커" 에코가 갑작스러운 차원 균열("섀도우 리프트")의 확산을 감지한다. 균열에서 쏟아지는 그림자 존재들이 5개 차원을 침식하고 있으며, 균열의 중심에는 고대 직물자 "보이드 위버"가 깨어나고 있다.
- **존별 이야기**: 각 존에는 이전 리프트 워커의 기록이 남아있다.
  - **폐허 존(Ruin)**: 첫 번째 워커 "아쉬"의 기록 — 균열의 시작
  - **수정 존(Crystal)**: 두 번째 워커 "루미나"의 기록 — 차원 반사 발견
  - **심연 존(Abyss)**: 세 번째 워커 "프리즘"의 기록 — 시간의 균열
  - **용암 존(Magma)**: 네 번째 워커 "코어"의 기록 — 중력 이상 현상
  - **공허 존(Void)**: 다섯 번째 워커 "실루엣"의 기록 — 보이드 위버와의 조우
- **스테이지 간 텍스트 대화**: Canvas 연출 3줄 + 포트레이트 (총 15회 + 히든 2회)
- **엔딩 분기**:
  - 5존 정화 → **"균열 봉인"** 일반 엔딩: 섀도우 리프트를 봉인하고 차원의 평화를 되찾는다.
  - 히든 보스 "Void Weaver" 격파 → **"차원 통합"** 진엔딩: 보이드 위버는 사실 차원들을 하나로 합치려는 최초의 리프트 워커였고, 에코는 대화로 새로운 균형을 창조한다.

## §2. 게임 규칙 및 목표

### 2.1 승리 조건
- **일반 클리어**: 5개 존의 보스를 모두 처치하고 중앙 포탈에서 균열 봉인
- **진 클리어**: 5보스 처치 후 해금되는 히든 존에서 Void Weaver 격파

### 2.2 패배 조건
- HP가 0이 되면 현재 런 종료 → 영구 업그레이드 유지, 아티팩트는 리셋
- 런 시작 시 HP 100 + 업그레이드 보너스

### 2.3 핵심 규칙
1. **방(Room) 기반 진행**: 맵은 17개 방(15 기본 + 2 히든)으로 구성. 방마다 적/퍼즐/보스 존재
2. **능력 게이팅**: 특정 방 진입에 특정 능력 필요 (§10.2 경로 맵 참조)
3. **아티팩트 선택**: 방 클리어 시 3택 중 1개 선택 (common/rare/epic 가중치)
4. **영구 업그레이드**: 런 종료 시 획득 "에코 포인트"로 3트리 업그레이드
5. **세이브**: 존 보스 처치 시 자동 세이브. 방 단위 체크포인트 없음 (로그라이트 긴장감)

- **에코 (플레이어)**: 8방향 포즈 (정면/후면/좌/우 × 정지/이동)

  - 클로크: bezierCurveTo로 바람에 날리는 효과 (이동 시 반대 방향으로 흔들림)

  - 대시 잔상: 0.3초간 반투명 잔상 3개 (alpha: 0.6→0.3→0.1)
  - 벽점프: 벽면에서 스파크 파티클 5개
  - 차원전환: 전신 실루엣이 보라↔시안 교차 글로우
- **적 디자인 (존별 차별화)**:
  - 폐허: 균열에서 나온 돌 골렘 (사각형 기반, 관절 분리 애니)
  - 수정: 부유하는 수정 요정 (삼각형 결정체 회전)
  - 심연: 촉수 그림자 (bezierCurveTo 웨이브 애니메이션)
  - 용암: 마그마 슬라임 (원형 + 끓는 표면 패턴)
  - 공허: 글리치 팬텀 (노이즈 효과 + 깜빡임)
- **보스 디자인** (600×400px급 대형):
  - 폐허 보스 "석화의 왕": 거대 골렘, 코어 부위 3개 (빨강→노랑→파랑)
  - 수정 보스 "프리즘 마녀": 수정 날개 4쌍, 반사 배리어
  - 심연 보스 "심해 군주": 거대 촉수 8개 + 중앙 안구
  - 용암 보스 "용암 타이탄": 용암 갑옷 + 바닥 분출
  - 공허 보스 "차원 파쇄자": 공간 왜곡 에어리어 + 분신
  - 히든 보스 "Void Weaver": 4페이즈 변형 (각 존 테마 흡수)

### 4.5 카메라 시스템
- **기본**: 플레이어 중심 추적 (smooth lerp, factor=0.08)
- **보스전**: 보스 등장 시 줌아웃(1.0→0.75, 2초 tween) + 보스 방향 팬
- **능력 획득**: 줌인(1.0→1.3) + 슬로모션(0.3초) + 줌아웃 복원
- **방 전환**: 방향별 슬라이드 전환 (0.5초 tween)
- 구현: `cam = {x, y, zoom, shakeX, shakeY}` — 모든 드로잉에 `cam` 오프셋 적용

### 4.6 배경 인터랙티브 요소
- **파괴 가능 오브젝트**: 나무 상자(HP 2), 수정 덩어리(HP 4), 용암 바위(HP 6)
  - 파괴 시 파티클 분산 + 랜덤 드롭(체력/에코포인트)
- **이동 플랫폼**: 수평/수직 왕복 (tween 기반, 1~3초 주기)
- **스위치**: 밟으면 활성화 → 연결된 문/플랫폼 동작
- **차원 게이트**: 차원전환 능력으로 활성화 → 숨겨진 경로 개방

### 4.7 날씨/시간대 효과
- **폐허**: 모래 폭풍 파티클 (수평 이동, 20개)
- **수정**: 반짝이는 결정 파티클 (랜덤 위치, 15개)
- **심연**: 부유하는 거품 파티클 (위로 올라감, 12개)
- **용암**: 불꽃 파티클 + 하단 용암 파도 (sin wave)
- **공허**: 글리치 스캔라인 + 노이즈 스트립

### 5.2 메인 루프 (60fps)
```
function gameLoop(timestamp) {
  const dt = Math.min((timestamp - lastTime) / 1000, 0.05); // 프레임 캡
  lastTime = timestamp;

  // 1. 입력 처리 (InputManager.poll)
  // 2. 시스템 업데이트 (ACTIVE_SYS[state]에 따라)
  //    - TWEEN: tw.update(dt)
  //    - PHYSICS: updatePhysics(dt, player, room, cam)
  //    - ENEMY: updateEnemies(dt, enemies, player, room)
  //    - ABILITY: updateAbilities(dt, player, abilityState)
  //    - PUZZLE: updatePuzzles(dt, room, switches, platforms)
  //    - COMBAT: updateCombat(dt, player, enemies, projectiles)
  //    - CAMERA: updateCamera(dt, cam, player, room)
  //    - PARTICLE: pool.update(dt)
  //    - DDA: updateDDA(stats)
  //    - SAVE: autoSave 체크
  // 3. 충돌 검출
  // 4. 상태 전환 체크
  // 5. 렌더링
  //    - 배경 (존별 + 날씨 효과)
  //    - 방 타일/플랫폼/스위치
  //    - 적/보스
  //    - 플레이어
  //    - 파티클
  //    - UI 오버레이
  // 6. requestAnimationFrame(gameLoop)
}
```

**핵심 원칙**:
- `setTimeout` 0건 (F4, 18사이클 연속). 모든 지연은 `tw.add()` + `onComplete`
- `Math.random` 0건 (F18). 모든 랜덤은 `SeededRNG.next()`
- 가드 플래그로 tween 콜백 1회 실행 보장 (F5): `if (guard.transitioning) return;`
- 단일 값 갱신 경로 (F14): HP, 점수 등 핵심 변수는 setter 함수 경유

## §7. 전투 시스템

### 7.1 기본 전투
- **근접 공격**: 전방 부채꼴(60°) 범위, 대미지 = baseDMG × abilityMul × artifactMul
  - 공격 모션: 0.25초, 3프레임 애니메이션
  - 쿨다운: 0.4초
- **원거리 공격**: 직선 투사체, 사거리 300px, 대미지 = baseDMG × 0.6
  - 투사체 속도: 400px/s
  - 쿨다운: 1.5초
- **능력 콤보**:
  - 대시 + 근접 = "대시 슬래시" (1.5× 대미지 + 넉백)
  - 시간정지 + 원거리 = "시간의 화살" (관통 + 2× 대미지)
  - 중력반전 + 근접 = "그래비티 해머" (범위 대미지 + 스턴 1초)

### 7.2 적 AI 패턴
| 적 타입 | HP | 대미지 | 이동 패턴 | 공격 패턴 |
|---------|-----|--------|----------|----------|
| 순찰자 | 30 | 10 | 좌우 순찰 | 접촉 대미지 |
| 추적자 | 50 | 15 | 플레이어 추적 | 돌진 (3초 쿨) |
| 사격수 | 40 | 12 | 고정 위치 | 투사체 발사 (2초 쿨) |
| 방어자 | 80 | 8 | 느린 접근 | 정면 실드 (공격 반사) |
| 소환자 | 60 | 5 | 도주 | 소환 (10초 쿨, 순찰자 2체) |

### 7.3 대미지 계산 공식
```
finalDMG = baseDMG × abilityMultiplier × artifactMultiplier × comboDMG × ddaFactor
방어 = max(0, finalDMG - enemyDEF × 0.3)
크리티컬 = baseCritRate(10%) + artifactBonus, 크리 배율 1.5×
```

### 7.4 충돌 검출
- AABB(Axis-Aligned Bounding Box) 기반
- `hitTest(ax, ay, aw, ah, bx, by, bw, bh)` 단일 함수 (F16)
- 근접 공격: 부채꼴 판정 → 각도 체크 + 거리 체크
- 투사체: AABB 매 프레임 갱신

### 8.2 DPS/EHP 밸런스 산식 (F74)

```
플레이어 DPS = baseDMG × atkSpeed × hitRate × comboMul × artifactMul
  - hitRate 가정: 탐험가 80%, 전사 65%, 전설 50%
  - comboMul 가정: 평균 1.2 (능력 콤보 20% 적중)
  - artifactMul 캡: 최대 2.0× (DPS 200% 상한 — F74)

적 EHP = HP × (1 + DEF×0.003)
  - 방 클리어 목표 시간: 20~40초 (초반), 30~60초 (후반)

보스 EHP = HP × (1 + DEF×0.005) × phaseCount
  - 보스 클리어 목표 시간: 60~120초 (존 보스), 180~300초 (히든 보스)

DDA 폴백 (3단계):
  1. 3번 연속 사망 → 적 HP/DMG 10% 감소
  2. 5번 연속 사망 → 적 HP/DMG 20% 감소 + 체력 물약 드롭률 2×
  3. 7번 연속 사망 → 적 HP/DMG 30% 감소 + "그림자 보호" 부여 (1히트 무적)
  클리어 시 DDA 리셋
```

> **가정 명시**: hitRate/comboMul은 난이도별 차등. 가정이 틀릴 경우 DDA 3단계가 자동 보정 (F24 Cycle 24 교훈).

### 8.3 아티팩트 캡 (F74)
- **DPS 캡**: 최대 200% 증가 (baseDMG의 3배 초과 불가)
- **시너지 캡**: 최대 150% 증가 (동일 타입 아티팩트 3개 이상 시 효과 체감)
- **캡 초과 아티팩트 제외**: `applyArtifact()`에서 캡 검증 후 초과 시 선택지에서 제외 (F27 Cycle 27)

## §15. 사운드 시스템

### 15.1 Web Audio API 프로시저럴 사운드

| # | 사운드 | 타입 | 생성 방법 |
|---|--------|------|----------|
| 1 | 존별 BGM (5종) | 배경음악 | OscillatorNode(사인/사각/톱니) + GainNode 루프 |
| 2 | 근접 공격 | SFX | 고주파 버스트(800Hz→200Hz, 0.1초) |
| 3 | 원거리 공격 | SFX | 중주파 파형(400Hz, 0.15초) + 에코 |
| 4 | 대시 | SFX | 바람 소리(노이즈 + 밴드패스 필터, 0.2초) |
| 5 | 벽점프 | SFX | 짧은 탭(1200Hz, 0.05초) |
| 6 | 차원전환 | SFX | 역방향 리버브(저→고, 0.4초) |
| 7 | 시간정지 | SFX | 딥 드론(80Hz, 0.5초) + 페이드 |
| 8 | 중력반전 | SFX | 상승/하강 글리산도(200→800Hz→200Hz, 0.3초) |
| 9 | 적 피격 | SFX | 짧은 노이즈 버스트(0.08초) |
| 10 | 보스 등장 | SFX | 저주파 드럼 롤(50Hz, 1.5초 크레센도) |
| 11 | 보스 격파 | SFX | 코드 분산(C-E-G, 0.8초) + 리버브 |
| 12 | 아티팩트 획득 | SFX | 반짝임 아르페지오(C5→E5→G5→C6, 0.6초) |
| 13 | 능력 해금 | SFX | 팡파르(3음 상승, 1.0초) |

> **SFX**: `SoundManager.playSFX(type)` — 모든 스케줄링 Web Audio 네이티브.
