# 그래비티 플립 (Gravity Flip) — 사이클 #38 게임 기획서

> **1페이지 요약**: 중력 실험 사고로 붕괴하는 연구소에서 탈출하는 **원터치 중력 반전 레이지 플랫포머**. 탭/클릭 한 번으로 중력 방향을 뒤집어 장애물을 회피하고, 적대적으로 변하는 환경(바닥 소멸, 천장 낙하, 스파이크 돌출)을 돌파하여 출구에 도달한다. 5개 구역(실험동→복도→반응로→중력실→핵심부) × 3 스테이지 = 15 메인 + 중력 보스 5종 + 히든 제로그래비티 스테이지 3개 = **총 23 스테이지**. 업그레이드 트리(중력 제어/대시/타임 슬로우 3갈래×5단계) + 절차적 장애물 패턴 조합 + 즉사+즉부활 루프. **arcade+casual 10게임 최장 미사용 해소 + Level Devil(Poki 2026 3월 #1) 레이지 플랫포머 트렌드 직접 대응.**

> **MVP 경계**: **Phase 1**(핵심 루프: 원터치 중력 반전 → 장애물 돌파 → 출구 도달, 구역 1~3 + 보스 3체 + 업그레이드 Lv1~3 + DDA 3단계 + 코인 시스템 + 기본 내러티브) → **Phase 2**(구역 4~5 + 보스 2체 + 히든 3스테이지 + 업그레이드 Lv4~5 + 날씨/시간대 연출 + 전체 내러티브 + 다국어 완성). **Phase 1만으로도 완전한 레이지 플랫포머 경험 제공 필수.**

## §1. 게임 개요 및 핵심 재미 요소

### 핵심 컨셉

**"탭 한 번의 긴장감, 즉사 한 번의 중독"** — Level Devil의 "적대적 환경" 철학 + 중력 반전이라는 물리 기반 핵심 메카닉.

### 핵심 재미 요소 (5가지)

1. **원터치 중력 반전**: 탭/클릭 한 번으로 중력이 180° 뒤집힘. 캐릭터가 천장과 바닥을 오가며 장애물을 회피. 심플한 조작 = 직관적 이해 + 깊은 마스터리
2. **적대적 환경 (Hostile Environment)**: Level Devil 스타일 — 바닥이 사라지고, 천장이 내려앉고, 벽에서 스파이크가 돌출. "안전해 보이는 곳이 함정"이라는 기대 전복
3. **즉사+즉부활 루프**: 죽으면 0.3초 내 스테이지 처음부터 재시작. 로딩 없는 즉시 재시도 = 분노→재도전→성취감 사이클
4. **절차적 장애물 패턴**: SeededRNG 기반으로 각 플레이마다 장애물 타이밍이 미세하게 변화. "외웠다"고 안심하면 변형 패턴에 당함
5. **연구소 탈출 내러티브**: 구역마다 연구소 붕괴 진행 → 환경이 점점 더 적대적으로 변화. 시각적 연출이 난이도 상승과 동기화

### 장르 분석

| 항목 | 값 |
|------|-----|
| 주 장르 | arcade (빠른 반응, 점수 추구, 패턴 돌파) |
| 부 장르 | casual (원터치 조작, 짧은 세션, 접근성 높음) |
| 서브장르 | 레이지 플랫포머 (Level Devil식 적대적 환경) |
| 플레이 세션 | 1스테이지 15~90초 (즉사 포함) |
| 타겟 유저 | 캐주얼 브라우저 게이머 + 레이지 게임 애호가 |

## §2. 게임 규칙 및 목표

### 기본 규칙

1. **캐릭터는 자동으로 우측 이동**한다 (수평 속도 120px/s 기본)
2. **탭/클릭으로 중력 반전**: 중력이 아래→위 또는 위→아래로 전환
3. **장애물에 닿으면 즉사**: 스파이크, 낙하, 크러셔, 레이저 등
4. **출구(포탈)에 도달하면 스테이지 클리어**
5. **코인 수집**: 스테이지 내 코인을 모아 업그레이드 구매
6. **별 등급**: 클리어 시간 + 사망 횟수 + 코인 수집률로 ★1~3 부여

### 스테이지 목표

- **메인 목표**: 출구 포탈에 도달하여 스테이지 클리어
- **서브 목표**: 코인 전량 수집, 시간 내 클리어(★3), 무사망 클리어
- **히든 목표**: 특정 스테이지에서 숨겨진 경로 발견 → 히든 스테이지 해금

### 보스전 규칙

- 보스는 구역의 3번째 스테이지 클리어 후 도전
- 보스가 특정 패턴으로 환경을 변형 → 플레이어는 중력 반전 타이밍으로 회피
- 보스 약점 노출 타이밍에 특정 위치 도달 = 1히트
- 3히트로 보스 격파

```
// ... 이하 모든 변수 선언 완료 후 init() 호출
```

### §5.2 프레임 루프 (gameLoop)

```
function gameLoop(timestamp) {
  try {
    const dt = Math.min((timestamp - lastTime) / 1000, 0.05); // 50ms 캡
    lastTime = timestamp;

    // 1. 입력 처리 (Input)
    processInput(G.state, INPUT);

    // 2. Tween 갱신 (모든 상태에서 실행) [F7]
    TW.update(dt);

    // 3. 상태별 업데이트
    switch (G.state) {
      case 'TITLE': updateTitle(dt); break;
      case 'MAP':   updateMap(dt); break;
      case 'PLAY':  updatePlay(dt); break;
      case 'BOSS':  updateBoss(dt); break;
    }

    // 4. 파티클 갱신 (모든 상태에서 실행)
    updateParticles(dt);

    // 5. 카메라 갱신
    CAM.update(dt);

    // 6. 렌더링
    render(ctx, G, CAM);

    // 7. 사운드 갱신
    SND.update(dt);

    requestAnimationFrame(gameLoop);
  } catch(e) {
    console.error('GameLoop Error:', e);
    requestAnimationFrame(gameLoop); // 크래시 복구
  }
}
```

**핵심 원칙**:
- setTimeout 0건 [F4] — 모든 지연은 `TW.add()` onComplete로 처리
- 가드 플래그 [F5] — `G.transitioning = true` 설정 후 1회만 실행
- clearImmediate() [F13] — `TW.clearImmediate()` 후 `TW.add()` 즉시 안전
- 단일 갱신 경로 [F14] — 예: `G.speed`는 tween으로만 갱신, 직접 대입 0건
- SeededRNG [F18] — `Math.random()` 0건, 모든 랜덤은 `RNG.next()`

## §8. 난이도 시스템

### §8.1 수학적 난이도 곡선 [F26]

```
스테이지 번호 n (1~15):
  장애물 밀도  = 0.3 + 0.05 × n              (0.35 ~ 1.05)
  장애물 속도  = 80 + 8 × n                   (88 ~ 200 px/s)
  안전 구간   = max(1.0, 3.0 - 0.13 × n)     (2.87 ~ 1.05 초)
  코인 수     = 5 + floor(n × 0.8)            (5 ~ 17)
  기준 시간   = 30 + n × 4                     (34 ~ 90 초)
```

### §8.2 DDA (Dynamic Difficulty Adjustment) — 4단계

| 등급 | 조건 (연속 사망) | 효과 | 시각적 표현 |
|------|----------------|------|-----------|
| 0 (기본) | — | 기본 난이도 | — |
| 1 (쉬움) | 5회 연속 사망 | 장애물 속도 -15%, 안전 구간 +20% | — (표시 없음) |
| 2 (매우 쉬움) | 10회 연속 사망 | 추가: 유령 경로 힌트 표시 [Cycle 35 planner] | 반투명 점선 경로 |
| 3 (가이드) | 20회 연속 사망 | 추가: 위험 장애물 붉은 글로우 강화 | 붉은 발광 효과 |

> DDA는 "함정 비활성화"가 아닌 **"정보 추가 제공"** 방식 — 레이지 게임의 도전감 보존 [Cycle 35 planner 교훈]

### §8.3 업그레이드 효과와 난이도 보정

업그레이드로 인한 난이도 하락을 보정하기 위해:
```
보정 난이도 = 기본 난이도 × (1 + 0.02 × 총 업그레이드 레벨)
```
업그레이드를 많이 할수록 후반 스테이지가 약간 더 어려워져 도전감 유지.

## §10. 프로시저럴 생성 시스템

### §10.1 SeededRNG [F18]

```javascript
class SeededRNG {
  constructor(seed) { this.s = seed; }
  next() { this.s = (this.s * 1664525 + 1013904223) & 0xFFFFFFFF; return (this.s >>> 0) / 0xFFFFFFFF; }
  range(min, max) { return min + this.next() * (max - min); }
  int(min, max) { return Math.floor(this.range(min, max + 1)); }
}
```

### §10.2 스테이지 생성 + BFS 검증 [F25, F27]

```
generateStage(zoneId, stageId, seed):
  1. RNG 초기화: new SeededRNG(seed)
  2. 세그먼트 배열 생성: 난이도 공식에 따라 E/M/H/X 배분
  3. 각 세그먼트에 사전 정의 패턴(20종) 중 선택
  4. 패턴 내 변형 요소 랜덤화 (±2타일, 30% 변형)
  5. 코인 배치 (난이도 공식에 따른 수량)
  6. ★★★ BFS 도달 가능성 검증 ★★★
     - 시작점 → 출구까지 BFS (중력 반전 가능 이동 시뮬레이션)
     - 검증 실패 시 seed + 1로 재생성 (최대 10회)
```

## §11. 점수 시스템

### §11.1 점수 계산 [F8]

```
스테이지 점수 = 기본점수 + 코인보너스 + 시간보너스 + 무사망보너스

기본점수 = 100 × 구역번호
코인보너스 = 수집코인 × 10
시간보너스 = max(0, (기준시간 - 클리어시간)) × 5
무사망보너스 = (사망 == 0) ? 500 : 0
```

**판정 순서** [F8]:
1. 점수 계산 (현재 런 기준)
2. 최고 기록 비교 → 갱신 여부 판정
3. localStorage 저장 (판정 완료 후)

### §11.2 총합 점수

- 전체 점수 = 모든 클리어 스테이지 점수 합계
- 별 총 합계 = 진행도 표시 (최대 69개: 23 스테이지 × ★3)

### §12.1 Web Audio API 프로시저럴 생성

모든 사운드는 `AudioContext` + `OscillatorNode` + `GainNode`로 프로시저럴 생성. `new Audio()`, `<audio>` 태그, 외부 파일 0건.

### §12.2 BGM (4종 루프)

| BGM | 상태 | BPM | 특성 |
|-----|------|-----|------|
| title_bgm | TITLE | 90 | 부드러운 신스 패드, 미스터리한 분위기 |
| map_bgm | MAP | 100 | 밝은 아르페지오, 탐험 느낌 |
| play_bgm | PLAY | 130 | 긴장감 있는 베이스라인 + 신스 리드 |
| boss_bgm | BOSS | 150 | 강렬한 드럼 + 디스토션 베이스 |

### §12.3 SFX (10종)

| SFX | 트리거 | 파형 |
|-----|--------|------|
| flip | 중력 반전 | 짧은 상승/하강 스위프 (sawtooth, 0.1s) |
| land | 착지 | 저음 펄스 (square, 0.05s) |
| die | 사망 | 노이즈 버스트 + 하강 (0.3s) |
| coin | 코인 수집 | 밝은 삼각파 2음 (triangle, 0.15s) |
| clear | 스테이지 클리어 | 상승 아르페지오 (sine, 0.5s) |
| boss_hit | 보스 타격 | 깊은 임팩트 (square + 노이즈, 0.2s) |
| boss_die | 보스 격파 | 폭발 노이즈 + 승리 팡파레 (0.8s) |
| ui_select | UI 선택 | 짧은 클릭 (sine, 0.05s) |
| dash | 대시 사용 | 바람 스위프 (sawtooth, 0.2s) |
| warning | 위험 경고 | 경보음 2음 반복 (square, 0.4s) |

### §12.4 사운드 스케줄링 [Cycle 28 planner]

모든 사운드 타이밍은 `audioCtx.currentTime` 기반. `setTimeout` 사용 0건:
```javascript
SND.play = function(name) {
  const osc = audioCtx.createOscillator();
  osc.start(audioCtx.currentTime);
  osc.stop(audioCtx.currentTime + duration);
};
```

## 부록 A: 극단 빌드 검증

### 빌드 1: 중력 특화 (Gravity Max)
- 중력 제어 Lv5 + 나머지 Lv1
- 전환 속도 200%, 360° 미세조정, 전환 시 무적 0.2초
- **예상 효과**: 장애물 회피 극대화, 스테이지 클리어 시간 최소화
- **검증**: 스테이지 15 기준 — 기본 클리어 90초 → 중력 특화 시 ~45초 (50% 단축)
- **클리어 시간 하한 10초**: 장애물 배치 밀도와 스테이지 길이로 물리적 하한 보장

### 빌드 2: 대시 특화 (Dash Max)
- 대시 파워 Lv5 + 나머지 Lv1
- 연속 대시 3회, 대시 중 무적, 잔상 폭발
- **예상 효과**: 장애물 무시하고 돌파, 대신 코인 수집률 하락
- **검증**: 스테이지 15 기준 — 대시 쿨다운(3초)으로 연속 무적 시간 제한, ~50초 클리어

### 빌드 3: 슬로우 특화 (Slow Max)
- 타임 슬로우 Lv5 + 나머지 Lv1
- 3초 슬로우, 빠른 충전, 코인 2배, 패턴 프리뷰
- **예상 효과**: 정보 극대화로 안전 플레이, 코인 수집 최적
- **검증**: 스테이지 15 기준 — 슬로우 사용 시간 포함 ~70초 (시간 소요 증가)

**결론**: 세 빌드 모두 클리어 시간 15~90초 범위 내. 극단 빌드도 밸런스 파괴 없음.
