## §0. MVP 범위 정의

### Phase 1 — 이번 사이클 구현 범위 (MVP)

- **★신규: 왕 캐릭터** — 레벨 맵에서 진행 마커, 미니게임 주인공, 스토리 내러티브

- 20레벨 + 보상 4개 + 미니게임 4개 = 28 스테이지
- **★신규: 코드 구조 개선** — `coreUpdate()` 단일화, `safeGridAccess()` 래퍼, `finally { input.flush(); }`
- Web Audio BGM + 효과음 16종+
- Canvas 풀스크린 + devicePixelRatio + 동적 리사이즈
- 다국어 ko/en

## §1. 게임 개요 및 핵심 재미 요소

### 컨셉
마법 왕국 4개 지역을 여행하며 보석을 매칭하는 레벨 클리어형 매치3 퍼즐에, Royal Match식 "King's Nightmare" 미니게임을 통합한 **내러티브 미니게임 통합 매치3**. 5레벨마다 왕이 위기 상황(물, 불, 함정, 드래곤)에 빠지며, 플레이어의 보석 매칭 실력이 왕의 운명을 결정한다. 매치3의 전략적 깊이 + 미니게임의 세션 다양성 + 캐릭터 내러티브의 감정적 몰입이 삼위일체로 작동한다.

### 핵심 재미 요소
1. **왕을 구출하는 긴장감**: 5레벨마다 등장하는 "왕의 위기" 미니게임이 반복 피로를 해소하고, 매치3 실력이 왕의 생사를 좌우하는 감정적 보상을 제공
2. **다양한 레벨 목표의 전략적 선택**: "루비 20개 수집" vs "얼음 전부 파괴" vs "점수 달성" — 같은 보드에서도 목표에 따라 완전히 다른 전략
3. **연쇄 폭발의 쾌감**: 캐스케이드 → 콤보 → 스페셜 조합의 도미노 폭발. 스페셜 조합 6종의 화려한 연출
4. **장애물 6종의 퍼즐적 깊이**: 얼음 겹겹이, 독 퍼짐, 커튼 뒤 미지의 보석, 돌 블록 우회 — 조합이 만드는 복잡성
5. **부스터로 역전**: 전략적 부스터 사용이 곧 실력. 미니게임 클리어 시 보너스 부스터 획득으로 선순환
6. **왕국 여행 + 왕 캐릭터 내러티브**: 보석 정원 → 수정 동굴 → 마법 숲 → 별빛 성, 왕이 지역별 위기를 극복하며 왕관을 되찾는 스토리

## §2. 게임 규칙 및 목표

### §2.1 보석 그리드 시스템

- **그리드 크기**: 8×8 (64칸)
- **보석 종류**: 6색 기본 보석 + 3종 스페셜 보석
- **6색 보석**: 루비(빨강), 사파이어(파랑), 에메랄드(초록), 토파즈(노랑), 아메시스트(보라), 다이아몬드(하늘)
- **초기 보드 생성 규칙**:
  1. SeededRNG로 보석 배치
  2. 3매치 이상 존재 시 해당 칸 재생성 (while 루프, 최대 1000회)
  3. BFS로 유효 이동 1개 이상 존재 확인. 없으면 전체 재생성
  4. 장애물 칸은 레벨 데이터에서 미리 정의된 위치에 배치
  5. 커튼 칸은 보석은 존재하나 시각적으로 가려진 상태로 시작

### §2.2 매칭 규칙

**매치 검출 우선순위** (⚠️ 반드시 이 순서로 — [C27,44] 반복 실수 방지):
1. **5자 매치** → 무지개 보석 생성
2. **T자/L자 매치** → 폭탄 보석 생성
3. **4자 매치** → 줄 파괴 보석 생성 (스왑 방향에 따라 가로/세로)
4. **3자 매치** → 단순 제거

**used[][] 소비 추적**: 한 보석이 여러 매치에 중복 카운트되지 않도록 소비 마킹. 예: 십자 형태에서 가로5+세로3이 아닌, 가로5 우선 인식.

**캐스케이드 루프**:
```
매칭 검출 → 제거 애니메이션(200ms) → 장애물 영향 처리 → 스페셜 생성 → 낙하(300ms) → 리필(200ms) → 재검출
```
- 캐스케이드 횟수 무제한
- 각 캐스케이드에서 콤보 카운터 증가 (+1)
- 캐스케이드 중 입력 차단 (`cascadeInProgress` 가드)

### §2.3 레벨 목표 시스템

5가지 목표 유형 (C47 계승):

| 유형 | 설명 | 아이콘 | 예시 |
|------|------|--------|------|
| `score` | 점수 N점 이상 달성 | 별 아이콘 | "15,000점 달성" |
| `collect` | 특정 색 보석 N개 매칭 | 보석 아이콘+숫자 | "루비 25개 수집" |
| `obstacle` | 특정 장애물 N개 파괴 | 장애물 아이콘+숫자 | "얼음 15개 파괴" |
| `jelly` | 젤리 타일 전부 제거 | 젤리 아이콘 | "젤리 12칸 모두 제거" |
| `mixed` | 위 유형 2~3개 조합 | 다중 아이콘 | "루비 10개 + 얼음 8개" |

**젤리 타일**: 보드 특정 칸 아래에 깔린 젤리. 해당 칸에서 매칭 발생 시 1겹 제거. 1~2겹 존재.

**레벨 목표 UI**: 상단 좌측에 현재 레벨 목표 아이콘 + 남은 수량 실시간 표시. 목표 달성 시 체크마크 + 초록색 전환 + 달성 SFX.

**레벨 클리어 조건**: 모든 목표 달성 AND 남은 턴 > 0
**레벨 실패 조건**: 턴 소진 AND 목표 미달성

| 커튼 | 커튼 제거+아래 보석 공개 | 인접 매칭 효과만 | 줄 위 커튼 제거 | 범위 내 커튼 제거 | 간접 영향 없음 |
| 독 | 독 제거 | 독 칸은 보석 없어 직접 타겟 불가 | 줄 위 독 제거 | 범위 내 독 제거 | 독 칸은 보석 없어 직접 타겟 불가 |
| 돌 블록 | 효과 없음 (파괴 불가) | 영향 없음 | 돌에서 멈춤 (관통 불가) | 돌 블록은 파괴 불가 | 영향 없음 |

### §2.5 부스터 시스템

| 부스터 | 효과 | 무료 지급 | 사용 조건 |
|--------|------|-----------|-----------|
| 🔨 망치 | 선택한 타일 1개 즉시 파괴 (장애물 포함, 돌 블록 제외) | Lv.5,10,15 클리어 시 + 미니게임 클리어 보너스 | 턴 소모 없음 |
| 🔄 셔플 | 보드의 모든 보석 무작위 재배치 (장애물은 유지) | Lv.3,8,13 클리어 시 | 턴 소모 없음, 즉시 발동 |
| ⏳ 추가 턴 | 남은 턴 +5 | Lv.7,12,17 클리어 시 | 아무 때나 사용 가능, 즉시 발동 |
| 🌈 색상 폭탄 | 보드에서 색 하나 탭 → 해당 색 전체 제거 | Lv.10,20 클리어 시 + 미니게임 최고 등급 보너스 | 턴 소모 없음 |

**부스터 사용 흐름**:
1. 하단 부스터 바에서 아이콘 탭 → `boosterActive = true` 가드 활성화 → 상태 `BOOSTING` 진입
2. 보드에서 대상 타일 탭 (망치/색상폭탄) 또는 즉시 발동 (셔플/추가턴)
3. 연출 재생 (200~500ms) → 상태 `ANIMATING`
4. 캐스케이드 필요 시 `CASCADING` → 완료 후 `GOAL_CHECK`
5. `boosterActive = false` → 일반 입력 복귀

**부스터 엣지 케이스 (C47 F1 계승 + 강화)**:
- 색상 폭탄 발동 중 캐스케이드 발생 시: `colorBombResolving` 가드로 추가 캐스케이드 지연
- 망치 + 다겹 장애물: 최상위 1겹만 파괴 (얼음 3겹 → 2겹, 상자 2겹 → 1겹)
- 망치 + 돌 블록: 효과 없음 + "파괴 불가" 텍스트 팝업 + 부스터 소모 안 함
- 셔플 + 장애물: 장애물 위치는 유지, 장애물 위가 아닌 보석만 셔플
- 셔플 + 커튼: 커튼 아래 보석도 셔플 대상 (공개 안 됨)
- 추가 턴 + 턴 0: 즉시 +5 적용 → PLAY 상태 복귀 (FAIL_ANIM 진입 방지)
- 색상 폭탄 + 스페셜 보석: 해당 색 중 스페셜이면 스페셜 효과 발동 후 제거
- **★ 미니게임 상태에서 부스터 바 비활성**: MINI_GAME 상태 진입 시 `boosterBarEnabled = false` 가드. 부스터 UI 완전 숨김

**부스터 저장**: localStorage에 부스터 보유 수량 저장. try-catch 래핑. 게임 시작 시 로드.

### §2.8 미니게임 시스템 (★ C48 핵심 신규)

5레벨 클리어마다 "왕의 위기" 미니게임이 자동으로 등장한다. 미니게임은 매치3 보드 위에 오버레이 형태로 진행되며, 보석 매칭 결과가 미니게임 액션에 직접 연동된다.

**미니게임 공통 규칙**:
- 제한 시간: 30초 (화면 상단 타이머 바)
- 미니게임 중 부스터 사용 불가 (`boosterBarEnabled = false`)
- 성공 등급: ⭐(30초 내 완료) / ⭐⭐(20초 내) / ⭐⭐⭐(15초 내)
- 실패 시: 부스터 보상 없이 맵 복귀 (레벨 자체는 클리어 유지). 재도전 가능
- 보상: ⭐→망치 1개, ⭐⭐→셔플+추가턴 1개씩, ⭐⭐⭐→색상 폭탄 1개 추가

#### 미니게임 1: 물 상승 탈출 (Lv.5 후)
- **상황**: 왕이 수몰되는 방에 갇힘. 물이 아래서 올라오고 있다
- **메카닉**: 화면 하단에 4×4 간이 보드. 매칭할 때마다 플랫폼 블록 1개 생성 → 왕이 위로 올라감
- **물 상승 속도**: 1초당 1칸씩 상승 (30초에 30칸)
- **승리 조건**: 왕이 상단 탈출구 도달 (20칸 이상 높이)
- **시각**: 보드 좌측에 왕+물+플랫폼 수직 타워. 물이 점점 차오르는 연출
- **입력**: 4×4 보드에서 일반 매치3 스왑 (3매치만 — 스페셜 없음)

#### 미니게임 2: 불길 회피 (Lv.10 후)
- **상황**: 동굴에서 용암이 흘러나와 왕이 탈출해야 함
- **메카닉**: 3레인 러너 형태. 왕이 상단으로 달리고, 좌/우 스와이프로 레인 변경. 불길이 무작위 레인에 등장. 화면 하단 4×4 보드에서 매칭하면 물 폭탄 발사 → 불길 1개 제거
- **승리 조건**: 제한 시간 내 왕 HP 1 이상 유지 (HP 3, 불길 피격 시 -1)
- **시각**: 상단에 러닝 왕+불길 레인, 하단에 4×4 보드
- **입력**: 왕 이동 = 좌/우 화살표 or 스와이프 + 보드 매칭 = 탭/클릭 스왑

#### 미니게임 3: 함정 미로 (Lv.15 후)
- **상황**: 마법 숲의 함정 미로에 갇힌 왕. 경로가 막혀 있다
- **메카닉**: 5×5 미로 그리드. 벽으로 막힌 경로를 매칭으로 개방. 화면 하단 4×4 보드에서 3매치 → 벽 1개 파괴. 4매치 → 벽 2개 파괴
- **승리 조건**: 왕이 입구(좌상)에서 출구(우하)까지 경로 도달
- **시각**: 상단에 미로+왕 아이콘, 하단에 4×4 보드
- **입력**: 매칭으로 벽 파괴 후, 왕은 자동으로 BFS 최단 경로 이동 (플레이어는 어떤 벽을 파괴할지 선택하지 않음 — 매칭 색상에 따라 벽 파괴 위치 결정: 루비=좌, 사파이어=우, 에메랄드=상, 토파즈=하, 아메시스트/다이아몬드=가장 가까운 벽)

#### 미니게임 4: 드래곤 보스 (Lv.20 후)
- **상황**: 별빛 성의 드래곤이 왕관을 지키고 있다
- **메카닉**: 화면 상단에 드래곤 (HP 20). 화면 하단 4×4 보드에서 매칭 → 데미지. 3매치=1dmg, 4매치=3dmg, 5매치=5dmg. 드래곤은 3초마다 반격: 보드에 얼음 장애물 1~2개 생성
- **승리 조건**: 드래곤 HP를 0으로
- **드래곤 반격 패턴**:
  - HP 20~11: 3초마다 얼음 1개 생성
  - HP 10~1: 2초마다 얼음 1~2개 생성 + 보드 1줄 셔플
- **시각**: 상단에 드래곤 + HP 바, 하단에 4×4 보드. 공격 시 보석이 드래곤에게 날아가는 tween 연출
- **입력**: 4×4 보드 일반 매치3 스왑

**미니게임 4×4 보드 규칙**:
- 보석 4색 (6색 중 루비, 사파이어, 에메랄드, 토파즈만 사용 — 4×4에서 6색은 데드락 빈도 과다)
- 스페셜 보석 없음 (4매치 → 단순 추가 보상, 스페셜 생성 X)
- 캐스케이드 존재 (메인 보드와 동일 로직, 다만 스페셜 미생성)
- 데드락 시 자동 셔플

## §3. 조작 방법

### §3.1 마우스
- **보석 스왑**: 보석 클릭 → 인접 보석 클릭 (또는 드래그)
- **부스터 사용**: 하단 부스터 아이콘 클릭 → 보드에서 대상 클릭
- **레벨 맵**: 노드 클릭으로 레벨 선택, 드래그로 맵 스크롤
- **미니게임 왕 이동**: 좌/우 화살표 키 또는 왕 양옆 클릭

### §3.2 터치
- **보석 스왑**: 탭 선택 → 스와이프 방향으로 스왑
- **부스터 사용**: 부스터 탭 → 대상 타일 탭
- **레벨 맵**: 노드 탭, 스와이프로 맵 스크롤
- **미니게임 왕 이동**: 좌/우 스와이프 (미니게임 영역)

### §3.3 키보드
- **방향키**: 커서 이동 (보드) / 왕 이동 (미니게임)
- **Space/Enter**: 보석 선택/스왑 확정
- **1~4 키**: 부스터 1~4번 활성화
- **ESC**: 부스터 취소 / 일시정지 메뉴
- **A 키**: 색맹 모드 토글

### §3.4 터치 안전 가이드
- 모든 인터랙티브 요소: `Math.max(CONFIG.MIN_TOUCH_48, size)` 강제 적용
- 메인 보석 셀 크기: 최소 48×48 CSS px
- 미니게임 4×4 보석 셀 크기: 최소 52×52 CSS px (터치 정확도 확보)
- 부스터 버튼: 최소 56×56 CSS px (하단 UI 영역 확보)
- 레벨 맵 노드: 최소 48×48 CSS px + 노드 간 최소 24px 간격

### §3.5 색맹 모드 (★ C48 신규 — F5 해결)
- **활성화**: 설정 메뉴 토글 또는 A키 단축키
- **보석 형태 마커**: 각 보석 중앙에 고유 형태 오버레이
  - 루비 → ◆ (다이아몬드)
  - 사파이어 → ● (원)
  - 에메랄드 → ⬡ (육각)
  - 토파즈 → ♥ (하트)
  - 아메시스트 → ▲ (삼각)
  - 다이아몬드 → ★ (별)
- **배경**: 보드 그리드 셀 교대로 명도 차이 강화 (체스보드 패턴)
- **구현**: `renderGem()` 함수에서 `CONFIG.colorBlindMode` 플래그 확인 → 흰색 아웃라인 형태 마커 추가 드로잉
- **저장**: localStorage에 색맹 모드 on/off 저장

- **레벨 맵 배경**: 판타지 왕국 조감도. 푸른 초원+구불구불한 강+원경 성. 파스텔 톤 수채화 느낌
- **게임 보드 배경**: 지역 테마별 변화 (정원/동굴/숲/성). 보드 외곽은 장식 프레임(골드+보석 장식)
- **미니게임 배경**: 지역 테마 변형 — 물 상승(정원 지하실), 불길(동굴 용암), 함정(숲 미로), 드래곤(성 탑)
- **UI 배경**: 반투명 다크 오버레이 + 골드 테두리 패널

### 오브젝트 형태
- **보석**: 둥근 다이아몬드/원형 기반. 내부에 빛 굴절 + 가장자리 하이라이트
- **장애물**: 각각 고유 형태 (얼음=투명 사각, 체인=금속 사슬, 상자=나무 질감, 커튼=반투명 천, 독=보라 연기, 돌=거친 회색)
- **부스터 아이콘**: 골드 원형 뱃지 안에 아이콘 (망치/화살표/모래시계/무지개)
- **왕 캐릭터**: 작은 왕관 쓴 둥근 캐릭터. 큰 눈, 친근한 표정. 미니게임에서 다양한 감정 표현

## §4.5. 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**"glossy casual fantasy"** — 로얄 매치/캔디 크러시 급 밝고 화려한 글로시 캐주얼 판타지

### 상세 정의
- **보석**: 3D 느낌의 2D 렌더링. 부드러운 그라디언트 + 광택 반사(specular highlight) + 내부 빛 산란(subsurface scattering 느낌) + 가장자리 림라이트. 각 보석마다 고유한 커팅 형태 (루비=둥근 다이아몬드, 사파이어=타원, 에메랄드=육각, 토파즈=하트, 아메시스트=물방울, 다이아몬드=라운드 브릴리언트)
- **배경**: 따뜻한 동화풍 판타지. 파스텔+수채화 질감. 구름, 나무, 성이 부드러운 곡선으로 표현. 지역마다 고유 분위기 (정원=밝고 따뜻, 동굴=신비로운 보라, 숲=깊고 초록, 성=장엄한 금+네이비)
- **왕 캐릭터**: 디즈니/로얄 매치식 둥근 캐릭터 디자인. 큰 눈+작은 코+밝은 미소. 왕관+망토+보석 장식. 미니게임마다 표정이 변함 (당황, 공포, 결의, 환희)
- **미니게임 환경**: 각 지역 테마의 위험한 변형. 정원→침수된 지하실(푸른 물+이끼), 동굴→용암 분출(붉은 조명), 숲→덩굴 함정(어두운 초록), 성→드래곤 둥지(보라+불꽃)
- **UI**: 골드 프레임 + 보석 장식. 버튼은 둥글고 입체적(버튼 위 하이라이트 + 아래 그림자). 미니게임 타이머 바는 빨강→초록 그라디언트
- **이펙트**: 밝고 생동감 있는 파티클. 보석 색상과 매칭되는 파티클 색상. 글로우+스파클 조합

### §4.6 색맹 모드 아트 가이드
- 형태 마커는 흰색 아웃라인 (두께 2px), 보석 중앙에 배치
- 활성화 시 보드 배경을 #1a1a2e 어두운 톤으로 전환하여 보석 대비 극대화
- 장애물 아이콘에도 보조 텍스트 마커 추가 (얼음=❄, 체인=🔗, 독=☠ 등 — Canvas drawText)

2. **Candy Crush Saga** (King) — 보석 반짝임, 스페셜 조합 연출, UI 레이아웃 참고

### §5.1 메인 루프 — coreUpdate() 단일화 (F1, F2 해결)

```javascript
// ★ C48 핵심 개선: 단일 업데이트 경로
function gameLoop(timestamp) {
  const dt = Math.min(timestamp - lastTime, 16);
  lastTime = timestamp;

  try {
    coreUpdate(dt);  // ★ 단일 진입점 — 중복 코드 0건
    coreRender();
  } catch (e) {
    console.error('[GameLoop Error]', e);
  } finally {
    input.flush();  // ★ F3 해결: finally 블록에서 입력 플러시 보장
  }

  requestAnimationFrame(gameLoop);
}

function coreUpdate(dt) {
  if (!_ready) return;  // TDZ 방어

  tweenManager.update(dt);
  particleSystem.update(dt);

  switch (currentState) {
    case STATE.LOADING:     updateLoading(dt); break;
    case STATE.TITLE:       updateTitle(dt); break;
    case STATE.MAP:         updateMap(dt); break;
    case STATE.LEVEL_INTRO: updateLevelIntro(dt); break;
    case STATE.PLAY:        updatePlay(dt); break;
    case STATE.ANIMATING:   updateAnimating(dt); break;
    case STATE.CASCADING:   updateCascading(dt); break;
    case STATE.BOOSTING:    updateBoosting(dt); break;
    case STATE.GOAL_CHECK:  updateGoalCheck(dt); break;
    case STATE.CLEAR_ANIM:  updateClearAnim(dt); break;
    case STATE.FAIL_ANIM:   updateFailAnim(dt); break;
    case STATE.RESULT:      updateResult(dt); break;
    case STATE.PAUSED:      break;  // Paused: tween/particle 업데이트 스킵
    case STATE.MINI_INTRO:  updateMiniIntro(dt); break;
    case STATE.MINI_GAME:   updateMiniGame(dt); break;
    case STATE.MINI_RESULT: updateMiniResult(dt); break;
  }
}
```

### §5.2 PLAY 상태 상세 흐름
```
1. 유저 스왑 입력 → isInputBlocked() 체크
2. 유효 스왑인지 검증 (인접 칸 + 매칭 가능 여부 + 체인/얼음 이동 제한 체크)
3. 스왑 tween 실행 (200ms) → ANIMATING 상태
4. 매치 검출 (우선순위: 5→T/L→4→3)
5. 매칭 없으면 스왑 복귀 tween → PLAY 복귀
6. 매칭 있으면:
   a. 제거 애니메이션 (200ms) + 점수 가산 + 장애물 영향 처리 + 커튼 제거
   b. 스페셜 보석 생성 (해당 시) + 생성 연출 (100ms)
   c. 낙하 tween (300ms)
   d. 리필 tween (200ms)
   e. 재매칭 검출 → 있으면 6a로 (캐스케이드)
   f. 캐스케이드 종료 → 독 확산 처리 (update에서만!) → 턴 수 -1
7. GOAL_CHECK 상태 진입
   a. 모든 목표 달성 → CLEAR_ANIM
   b. 턴 소진 + 목표 미달 → FAIL_ANIM (단, 추가 턴 부스터 보유 시 "+5턴 사용하기?" 프롬프트 먼저 표시)
   c. 그 외 → PLAY 복귀
```

3. [600~1000ms] 실패 SFX + 페이드 → RESULT 상태 전환
   - RESULT에서 "재시도" + "맵으로" 버튼

### §6.3 DDA (Dynamic Difficulty Adjustment) — 2단계 시스템 (C47 계승)

```javascript
const DDA = {
  failStreak: 0,
  maxAdjust: 3,

  onLevelFail() {
    this.failStreak = Math.min(this.failStreak + 1, this.maxAdjust);
  },
  onLevelClear() {
    this.failStreak = Math.max(this.failStreak - 1, 0);
  },

  // === 기본 DDA (전 레벨) ===
  getAdjustedTurns(baseTurns) {
    return baseTurns + this.failStreak * 2;  // 1연패:+2, 2연패:+4, 3연패:+6
  },
  getAdjustedGoal(baseGoal) {
    return Math.max(1, baseGoal - this.failStreak * 2);
  },
  shouldOfferFreeShuffle() {
    return this.failStreak >= 2;
  },
  shouldOfferFreeBooster() {
    return this.failStreak >= 3;
  },

  // === 극단 레벨 전용 DDA (Lv.17+ 전용) ===
  getHardLevelAdjustment(levelId) {
    if (levelId < 17 || this.failStreak < 2) return null;
    return {
      obstacleReduction: Math.floor(this.failStreak * 0.2 * 100) / 100,
      poisonSpreadInterval: this.failStreak >= 3 ? 2 : 1,
      extraBooster: this.failStreak >= 3 ? 'hammer' : null
    };
  },

  // ★ 미니게임 DDA (신규)
  getMiniGameAdjustment(miniId) {
    if (this.failStreak < 1) return null;
    return {
      extraTime: this.failStreak * 5,  // 연패당 +5초
      reducedDifficulty: this.failStreak >= 2  // 2연패: 물 상승 속도 감소 / 드래곤 HP 감소 등
    };
  }
};
```

### §6.4 간이 밸런스 시뮬레이터 (F4 해결 — 개발 전용)
```javascript
// CONFIG.DEV_AUTO_PLAY = true 시 활성화 (프로덕션에서는 false)
function autoPlaySimulation(levelId, iterations = 100) {
  const results = { clears: 0, totalTurns: 0, minTurns: Infinity, maxTurns: 0 };
  for (let i = 0; i < iterations; i++) {
    const sim = createHeadlessLevel(levelId);
    while (!sim.isComplete()) {
      const bestMove = sim.findBestMove(); // BFS 기반 최적 이동
      sim.executeMove(bestMove);
    }
    if (sim.isCleared()) {
      results.clears++;
      results.totalTurns += sim.turnsUsed;
      results.minTurns = Math.min(results.minTurns, sim.turnsUsed);
      results.maxTurns = Math.max(results.maxTurns, sim.turnsUsed);
    }
  }
  console.table({
    levelId,
    clearRate: (results.clears / iterations * 100).toFixed(1) + '%',
    avgTurns: (results.totalTurns / results.clears).toFixed(1),
    range: `${results.minTurns}~${results.maxTurns}`
  });
}
```

function safeSave(key, value) {
  try { localStorage.setItem(key, JSON.stringify(value)); }

## §12. 성능 최적화

- **offscreen canvas 배경 캐싱**: 배경 + 보드 프레임을 별도 캔버스에 캐싱. resize 시만 재빌드
- **ObjectPool**: 파티클, tween 객체 풀링. `pool.acquire()` / `pool.release()` 패턴
- **devicePixelRatio 대응**: `canvas.width = canvas.clientWidth * dpr`, 모든 드로잉에 dpr 스케일 적용
- **동적 리사이즈**: `window.onresize` 디바운스 200ms + 캐시 무효화 + 보드 좌표 재계산
- **미니게임 전용 렌더러**: MINI_GAME 상태에서는 메인 보드 렌더링 비활성. 미니게임 4×4 보드 + 환경만 렌더링 → GPU 부하 감소
- **에셋 지연 로드**: 미니게임 에셋(bg-mini-*, king-*, mini-*)은 미니게임 진입 직전에 로드 시작 (LEVEL_INTRO에서 preload). LOADING 상태에서는 기본 에셋만
