## 1. 게임 개요 및 핵심 재미 요소

### 컨셉
같은 색상의 블록을 합쳐 **무지개 색상 단계를 올라가는** 퍼즐 게임입니다. 2048의 슬라이드 머지 메카닉에 **색상 진화 시스템**과 **콤보 체인 보너스**를 더해 시각적 쾌감과 전략적 깊이를 동시에 제공합니다.

### 핵심 재미 요소
1. **색상 진화의 만족감** — 빨강에서 시작해 보라까지, 무지개를 완성해 나가는 시각적 성취감
2. **콤보 체인** — 한 번의 스와이프로 연쇄 머지가 발생하면 콤보 배수가 올라가며 화면 이펙트 폭발
3. **"한 판만 더" 중독성** — 게임 오버 후 최고 점수 갱신 욕구로 즉시 재시작
4. **직관적 조작** — 스와이프 한 방향이면 모든 블록이 움직이므로 진입 장벽 제로

## 2. 게임 규칙 및 목표

### 2.1 기본 규칙
- **5×5 그리드** 위에서 진행
- 매 턴마다 플레이어가 **상/하/좌/우 중 한 방향**으로 스와이프
- 모든 블록이 해당 방향으로 끝까지 밀림
- **같은 색상 블록**이 밀리는 방향으로 충돌하면 **하나로 합쳐지며 다음 단계 색상으로 진화**
- 한 번의 스와이프에서 같은 행/열의 블록은 **한 번만** 머지 (2048과 동일)
- 머지 후 **랜덤 빈 칸**에 새 블록 1개 생성 (1단계 빨강 90%, 2단계 주황 10%)

### 2.2 색상 단계 (7단계 무지개)

| 단계 | 색상 | HEX 코드 | 머지 점수 |
|------|------|----------|----------|
| 1 | 빨강 (Red) | `#FF6B6B` | 2 |
| 2 | 주황 (Orange) | `#FFA06B` | 6 |
| 3 | 노랑 (Yellow) | `#FFD93D` | 18 |
| 4 | 초록 (Green) | `#6BCB77` | 54 |
| 5 | 파랑 (Blue) | `#4D96FF` | 162 |
| 6 | 남색 (Indigo) | `#6B5BFF` | 486 |
| 7 | 보라 (Violet) | `#C56BFF` | 1458 |

> **보라(7단계) + 보라 = 무지개 블록(특수)** → 2916점 + 보드 위의 가장 낮은 단계 블록 3개 제거 (보너스 이벤트)

### 2.3 콤보 시스템
- 한 번의 스와이프에서 머지가 **2회 이상** 발생하면 콤보 발동
- 콤보 배수: `머지 횟수 × 1.5` (소수점 버림)
- 예: 3회 머지 → 4배 보너스, 5회 머지 → 7배 보너스
- 콤보 발생 시 화면에 "COMBO ×N!" 텍스트 이펙트

### 2.4 게임 목표
- **최고 점수 달성** (무한 플레이, 명확한 엔딩 없음)
- **보라 블록 생성** (또는 무지개 블록 달성)이 사실상의 목표

### 2.5 게임 오버 조건
- 25칸(5×5) 모두 채워져 있고
- 상하좌우 어느 방향으로도 **머지 가능한 인접 블록 쌍이 없을 때**
- 게임 오버 시: 최종 점수 표시, 최고 점수 갱신 여부 확인, 재시작 버튼

## 4. 시각적 스타일 가이드

### 4.1 전체 분위기
- **미니멀 + 모던** — 깔끔한 라운드 사각형, 부드러운 그림자
- **다크 배경 + 비비드 블록** — 블록 색상이 돋보이도록

### 4.2 색상 팔레트

| 요소 | HEX | 용도 |
|------|-----|------|
| 배경 | `#1A1A2E` | 메인 배경 (진한 남색) |
| 그리드 배경 | `#16213E` | 그리드 영역 배경 |
| 빈 칸 | `#0F3460` | 빈 셀 표시 |
| 텍스트 (주) | `#FFFFFF` | 점수, 제목 |
| 텍스트 (부) | `#A0A0B0` | 설명, 보조 텍스트 |
| 콤보 텍스트 | `#FFD93D` | 콤보 이펙트 (노란색) |
| 블록 색상 | 2.2 표 참조 | 7단계 무지개 |

### 4.3 오브젝트 형태
- **블록:** 라운드 사각형 (border-radius: 12px), 내부에 색상 단계 숫자 또는 아이콘 없이 **색상만으로 구분**
- **블록 크기:** 그리드 셀의 90% 크기, 셀 사이 간격 8px
- **그리드:** 라운드 사각형 컨테이너 (border-radius: 16px), 약간의 내부 패딩
- **점수판:** 그리드 상단, 현재 점수 + 최고 점수 나란히 배치
- **무지개 블록(특수):** 7색 그라데이션 테두리 + 펄스 애니메이션

### 4.4 애니메이션
| 애니메이션 | 지속 시간 | 설명 |
|-----------|----------|------|
| 블록 이동 | 150ms | ease-out 슬라이드 |
| 머지 | 200ms | 스케일 1.0 → 1.2 → 1.0 (탄성 바운스) |
| 새 블록 등장 | 200ms | 스케일 0 → 1.0 (팝업) |
| 콤보 텍스트 | 800ms | 위로 떠오르며 페이드아웃 |
| 게임 오버 오버레이 | 300ms | 페이드인 |

## 5. 핵심 게임 루프 (프레임 기준 로직 흐름)

```
[대기 상태]
    │
    ▼
[입력 감지] ← 키보드/터치/마우스 이벤트
    │
    ▼
[입력 유효성 검증]
    │  - 애니메이션 진행 중이면 무시 (입력 잠금)
    │  - 해당 방향으로 이동 가능한 블록이 있는지 확인
    │  - 이동 불가능하면 대기 상태로 복귀
    │
    ▼
[블록 이동 계산] (순수 로직, 프레임 무관)
    │  1. 이동 방향에 따라 행/열 순회 순서 결정
    │  2. 각 블록을 해당 방향 끝까지 이동
    │  3. 인접 동일 색상 블록 머지 처리
    │  4. 머지 횟수 카운트 (콤보 계산용)
    │  5. 새 그리드 상태 확정
    │
    ▼
[애니메이션 재생] (~150ms, requestAnimationFrame)
    │  - 블록 이동 트윈 애니메이션
    │  - 머지 바운스 애니메이션
    │  - 콤보 텍스트 표시 (해당 시)
    │
    ▼
[새 블록 생성]
    │  - 빈 칸 목록에서 랜덤 선택
    │  - 1단계(90%) 또는 2단계(10%) 블록 배치
    │  - 등장 애니메이션 재생 (~200ms)
    │
    ▼
[게임 상태 확인]
    │  - 빈 칸 있으면 → [대기 상태]로 복귀
    │  - 빈 칸 없으면 → 인접 머지 가능 여부 확인
    │     - 가능하면 → [대기 상태]로 복귀
    │     - 불가능하면 → [게임 오버]
    │
    ▼
[게임 오버]
    │  - 최고 점수 비교 및 localStorage 저장
    │  - 오버레이 표시 (최종 점수, 최고 점수, 재시작 버튼)
    │
    ▼
[재시작] → 그리드 초기화 → 초기 블록 2개 배치 → [대기 상태]
```

### 렌더링 루프 (60fps)
```javascript
function gameLoop(timestamp) {
    // 1. 애니메이션 업데이트 (이동, 머지, 이펙트)
    updateAnimations(timestamp);

    // 2. 캔버스 클리어
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // 3. 그리드 배경 그리기
    drawGridBackground();

    // 4. 블록 그리기 (현재 위치 + 애니메이션 오프셋)
    drawBlocks();

    // 5. 이펙트 그리기 (콤보 텍스트, 파티클 등)
    drawEffects();

    // 6. UI 그리기 (점수, 최고 점수)
    drawUI();

    requestAnimationFrame(gameLoop);
}
```

## 6. 난이도 시스템

### 6.1 동적 난이도 곡선

이 게임은 **명시적 난이도 선택 없이** 게임 진행에 따라 자연스럽게 난이도가 올라갑니다.

| 조건 | 변화 | 효과 |
|------|------|------|
| 기본 상태 | 새 블록 90% 1단계, 10% 2단계 | 쉬운 시작 |
| 점수 500+ | 새 블록 80% 1단계, 15% 2단계, 5% 3단계 | 약간의 복잡성 추가 |
| 점수 2000+ | 새 블록 70% 1단계, 20% 2단계, 10% 3단계 | 중급 난이도 |
| 점수 5000+ | 매 턴 새 블록 1개 → 가끔 2개 (20% 확률) | 공간 압박 증가 |
| 점수 10000+ | 2개 블록 생성 확률 35% | 고급 난이도 |

### 6.2 난이도 밸런스 원칙
- **처음 2분:** 여유롭게 플레이하며 규칙 학습
- **3~5분:** 공간 관리 전략이 필요해짐
- **5분 이후:** 한 수 한 수 신중하게, 긴장감 유지
- 무지개 블록(7단계 두 개 머지) 달성은 **숙련 플레이어 기준 10~15분**

## 7. 점수 시스템

### 7.1 점수 계산

```
턴 점수 = Σ(머지된 블록의 머지 점수) × 콤보 배수
```

| 항목 | 계산 |
|------|------|
| 기본 머지 점수 | 단계별 고정 (2.2 표 참조: 2, 6, 18, 54, ...) |
| 콤보 배수 | 1회 머지 = ×1, 2회 = ×3, 3회 = ×4, 4회 = ×6, 5회+ = ×7 |
| 무지개 보너스 | 2916점 + 낮은 단계 블록 3개 제거 |

### 7.2 점수 이정표 (선택적 알림)

| 점수 | 이정표 |
|------|--------|
| 100 | "좋은 시작!" |
| 500 | "머지 마스터!" |
| 2000 | "색상 전문가!" |
| 5000 | "무지개 사냥꾼!" |
| 10000 | "전설의 퍼즐러!" |

### 7.3 최고 점수 저장
- `localStorage.setItem('colorMergePuzzle_bestScore', score)` 사용
- 게임 시작 시 로드, 게임 오버 시 갱신
- 상단 UI에 항상 표시

## 8. 기술 구현 가이드

### 8.1 파일 구조
```
/games/color-merge-puzzle/index.html   ← 단일 파일, 모든 것 포함
```

### 8.2 기술 스택
- **HTML5 Canvas** — 게임 렌더링
- **Vanilla JavaScript** — 게임 로직 (프레임워크 없음)
- **CSS** — 최소한의 레이아웃 (canvas 감싸는 컨테이너)
- **localStorage** — 최고 점수 저장

### 8.3 반응형 대응
- Canvas 크기: `min(화면 너비 - 40px, 500px)` 정사각형
- 모바일: 화면 전체 너비 활용
- 데스크톱: 최대 500px로 제한 (중앙 정렬)
- `window.resize` 이벤트에서 캔버스 재조정

### 8.4 성능 목표
- 60fps 유지 (requestAnimationFrame)
- 초기 로딩: 0.5초 이내
- 파일 크기: 30KB 이하 (gzip 전)

## 10. 향후 확장 가능성 (Cycle 2+)

> 아래는 Cycle 1 MVP에 포함하지 않으며, 포스트모템 결과에 따라 결정합니다.

1. **타임 어택 모드** — 3분 제한 시간 내 최고 점수 도전
2. **일일 챌린지** — 매일 동일 시드로 동일 초기 배치, 전 유저 동일 조건
3. **테마 스킨** — 파스텔, 네온, 모노크롬 등 색상 테마 변경
4. **언두 기능** — 마지막 1수 되돌리기 (광고 시청 보상으로)
5. **글로벌 리더보드** — Supabase 연동 순위표
