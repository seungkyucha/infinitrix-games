## 1. 게임 개요 및 핵심 재미 요소

**픽셀 심연**은 16-bit 픽셀아트 스타일의 턴제 로그라이크 던전 크롤러입니다.

플레이어는 고대 던전의 심연을 탐험하며, 매 층마다 랜덤 생성되는 던전에서 몬스터를 처치하고 아이템을 수집합니다. 죽으면 처음부터 다시 시작하지만, 획득한 골드로 영구 업그레이드를 구매하여 점점 더 깊은 층에 도달할 수 있습니다.

### 핵심 재미 요소
- **"한 판 더" 중독성**: 매 런마다 다른 던전 구조 + 아이템 조합으로 새로운 경험
- **리스크-리워드 의사결정**: "이 방에 들어갈까? 포션을 지금 쓸까 아껴둘까?"
- **메타 성장**: 런마다 쌓이는 골드로 영구 업그레이드 → 점진적 진행감
- **전략적 턴제 전투**: 적의 패턴을 읽고, 지형을 활용한 포지셔닝

### 이전 사이클 학습 반영
> 첫 사이클이므로 이전 포스트모템은 없습니다. 분석가 보고서의 권고를 반영합니다:
> - **분석가 권고**: 턴제 기반으로 복잡도를 관리 → ✅ 실시간이 아닌 턴제 채택
> - **분석가 권고**: HTML5 브라우저 성능 부담 최소화 → ✅ 턴제라 렌더링 부담 적음
> - **분석가 위험 지적**: 첫 게임으로 복잡도 높음 → ✅ 5층 제한, 적 4종류로 스코프 축소

## 2. 게임 규칙 및 목표

### 최종 목표
- **5층 던전의 최하층 보스를 처치**하면 승리 (엔딩 화면)
- 도중 HP가 0이 되면 **영구 사망** (Permadeath) → 로비로 복귀

### 핵심 규칙
1. **턴제 이동**: 플레이어가 한 칸 이동/공격하면, 모든 적도 한 턴 행동
2. **시야 시스템**: 플레이어 주변 5칸 반경만 밝게 표시 (안개 속 탐험)
3. **층 구조**: 각 층에 계단을 찾아 내려가면 다음 층 진입
4. **아이템**: 바닥의 아이템 위를 밟으면 자동 획득
5. **골드 영구 보존**: 런 중 획득한 골드는 사망해도 유지 → 로비에서 영구 업그레이드 구매

### 게임 흐름
```
[로비] → 던전 입장 → [1층] → 계단 → [2층] → ... → [5층 보스] → 승리
                          ↓ (사망)
                       [게임오버] → 골드 정산 → [로비] (영구 업그레이드)
```

- UI는 중세 판타지풍 프레임 (석재/나무 테두리)

## 4.5. 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**`pixel-art-16bit`** — Crisp 16-bit pixel art, limited palette, dithering, retro SNES/Genesis feel. Sharp pixels, no anti-aliasing.

### 세부 방향
- **해상도**: 게임 내 논리 해상도 480×320 → Canvas에서 2x 업스케일 (960×640)
- **팔레트 제한**: 전체 게임에서 24색 이내 사용 (DawnBringer 32 팔레트 기반)
- **디더링**: 그림자, 조명 그라데이션에 체커보드 디더링 사용
- **안티앨리어싱 금지**: 모든 에셋은 날카로운 픽셀 경계 유지
- **애니메이션**: 2~4프레임의 간결한 스프라이트 애니메이션
- **폰트**: 픽셀 비트맵 폰트 (한글은 도트체 스타일)

### 레퍼런스 게임
1. **Shattered Pixel Dungeon** — 던전 크롤러 UI/UX, 타일맵 구조, 아이템 아이콘 스타일
2. **Nuclear Throne** — 2등신 캐릭터 비율, 적 디자인의 실루엣 차별화

## 5. 핵심 게임 루프 (턴 기준 로직 흐름)

로그라이크는 실시간이 아닌 **턴제**이므로, 프레임 기반이 아닌 **턴 기준** 로직입니다.

### 한 턴의 처리 순서
```
1. [입력 대기] 플레이어 입력 수신 (이동/공격/아이템/대기)
2. [플레이어 행동]
   - 이동: 목표 타일이 빈 공간이면 이동, 벽이면 무시
   - 공격: 목표 타일에 적이 있으면 데미지 계산
   - 아이템: 인벤토리에서 소비 아이템 사용
   - 대기: 턴만 넘김
3. [아이템 획득] 플레이어 위치에 아이템이 있으면 자동 획득
4. [적 AI 행동] (모든 적 순차 처리)
   a. 시야 내 적: 플레이어를 향해 A* 경로탐색으로 이동/공격
   b. 시야 밖 적: 랜덤 배회 또는 정지
5. [상태 업데이트]
   - HP 체크 (플레이어 사망 → 게임오버)
   - 적 HP 체크 (사망 → 골드/아이템 드롭)
   - 계단 도달 체크 → 다음 층 생성
6. [렌더링]
   - 시야 계산 (Raycasting FOV)
   - 타일맵 렌더링 (바닥/벽/장식)
   - 엔티티 렌더링 (플레이어/적/아이템)
   - UI 업데이트 (HP바, 인벤토리, 미니맵, 층 정보)
   - 데미지 숫자 팝업 애니메이션
7. [1번으로 돌아감]
```

### 던전 생성 알고리즘 (BSP)
```
1. 전체 맵 영역을 BSP(이진 공간 분할)로 4~6개 방으로 분할
2. 각 방의 크기를 랜덤 조정 (최소 5x5, 최대 10x10)
3. 인접 방을 복도로 연결
4. 적 배치: 방마다 0~3마리 (층 깊이에 비례)
5. 아이템 배치: 방마다 0~1개 (확률 기반)
6. 계단 배치: 가장 먼 방에 하향 계단
7. 보스층(5층): 큰 단일 방 + 보스 1마리
```

## 6. 난이도 시스템

### 층별 난이도 변화 (3단계 이상)

| 층 | 난이도 | 적 종류 | 적 HP | 적 공격력 | 아이템 빈도 | 특수 기믹 |
|----|--------|---------|-------|-----------|-------------|-----------|
| 1층 | 쉬움 | 슬라임만 | 8 | 2 | 높음 (50%) | 없음 — 튜토리얼 안내 |
| 2층 | 보통 | 슬라임 + 박쥐 | 12~15 | 3~4 | 보통 (35%) | 함정 타일 등장 |
| 3층 | 어려움 | 박쥐 + 해골 전사 | 18~22 | 5~6 | 보통 (30%) | 어둠 구간 (시야 축소) |
| 4층 | 매우 어려움 | 해골 전사 + 마법사 | 25~30 | 7~9 | 낮음 (20%) | 원거리 공격 적 등장 |
| 5층 | 보스 | **다크 나이트** 보스 | 80 | 12 | 보스 드랍만 | 보스 패턴 전투 |

### 보스 "다크 나이트" 패턴
- **페이즈 1** (HP 100~50%): 플레이어를 추적하며 근접 공격 (2턴에 1회)
- **페이즈 2** (HP 50% 이하): 이동 속도 증가 (매 턴 공격), 돌진 공격 추가 (직선 3칸)

## 7. 점수 시스템

### 런 점수 계산
```
런 점수 = (처치한 적 × 10) + (도달 층 × 100) + (획득 골드 × 1) + (보스 처치 보너스 500)
```

### 골드 시스템
- 적 처치 시 1~5 골드 드랍
- 던전 내 보물상자에서 10~20 골드
- **골드는 사망해도 영구 보존** → 로비에서 영구 업그레이드 구매

### 영구 업그레이드 (로비 상점)
| 업그레이드 | 비용 | 효과 | 최대 레벨 |
|-----------|------|------|----------|
| 튼튼한 갑옷 | 30/60/100 | 시작 HP +5/+10/+20 | 3 |
| 날카로운 검 | 40/80/130 | 시작 공격력 +2/+4/+7 | 3 |
| 탐험가의 눈 | 50/100 | 시야 범위 +1/+2 | 2 |
| 행운의 부적 | 60/120 | 아이템 드랍률 +10%/+20% | 2 |

### 게임오버 화면 표시
- 최종 점수
- 도달 층
- 처치한 적 수
- 획득 골드 (영구 보존됨)
- 최고 점수 (로컬 스토리지 저장)

## 10. 기술 구현 참고사항

### 단일 index.html 구현 가이드
- **Canvas 크기**: 960×640 (논리 480×320의 2x 스케일)
- **타일 크기**: 32×32px (논리), 렌더링 시 2x 스케일
- **맵 크기**: 각 층 30×24 타일 (화면에 15×10 타일 표시, 카메라 따라감)
- **렌더링**: HTML5 Canvas 2D API (requestAnimationFrame으로 루프, 턴 변화 시만 재렌더)
- **시야**: 간단한 Raycasting (Bresenham's line) — 플레이어에서 각 타일까지 직선에 벽이 있으면 차단
- **경로탐색**: A* 알고리즘 (적 AI용)
- **저장**: localStorage에 영구 업그레이드 + 최고 점수 저장
- **에셋 로딩**: 모든 이미지를 base64 인라인 또는 외부 PNG 파일로 로드

### 상태 머신
```
LOBBY → PLAYING → GAME_OVER → LOBBY
                → VICTORY → LOBBY
```
