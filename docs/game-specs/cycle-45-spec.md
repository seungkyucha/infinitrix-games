## §1. 게임 개요 및 핵심 재미 요소

### 컨셉
어둠의 군세가 보석 왕국을 침략한다. 플레이어는 보석의 힘을 매칭하여 적 보스를 격파하고 왕국을 수호하는 **보스 배틀 매치3 퍼즐**. Royal Match의 King's Nightmare 모드에서 영감을 받되, 5종 고유 보스의 반격 메카닉과 성장형 장애물(독 타일)이 차별화된 전략적 긴장감을 제공한다.

### 핵심 재미 요소
1. **보스와의 전략적 대결**: 매칭으로 보스 HP를 깎되, 보스가 매 턴 반격(장애물 생성, 보석 잠금, 보드 방해)하므로 단순 매칭이 아닌 전략적 판단 필요
2. **연쇄 폭발의 쾌감**: 캐스케이드 → 콤보 → 스페셜 조합 연쇄. 보스전에서 대형 콤보가 터지면 보스에게 치명적 데미지 + 화면 가득 파티클
3. **성장형 장애물의 긴장감**: 독 타일이 매 턴 1칸씩 확장 — 방치하면 보드를 잠식. 즉각적 제거 vs 보스 데미지 딜레마
4. **보스 도감 수집**: 5종 보스 격파 → 도감 등록 + 왕국 방어 스토리 해금. "다음 보스는 어떤 녀석일까?" 궁금증
5. **스페셜 보석 조합의 깊이**: 무지개+무지개=보드 전체 클리어, 보스전에서 결정적 한 방

## §2. 게임 규칙 및 목표

### §2.1 보석 그리드 시스템

- **그리드 크기**: 8×8 (64칸)
- **보석 종류**: 6색 기본 보석 + 3종 스페셜 보석
- **6색 보석**: 루비(빨강), 사파이어(파랑), 에메랄드(초록), 토파즈(노랑), 아메시스트(보라), 다이아몬드(흰색)
- **초기 보드 생성 규칙**:
  1. 랜덤으로 6색 보석 배치 (균등 가중치 각 ~16.7%, DDA 보정 시 목표색 +3%)
  2. **3매치 없음 검증**: 가로/세로 연속 3개 동색 검출 시 해당 보석 재배치
  3. **유효 이동 검증**: 최소 1개 이상의 유효 스왑이 존재하는지 BFS 탐색
  4. 두 조건 모두 통과할 때까지 반복 (최대 100회, 초과 시 안전 패턴 폴백: 격자형 색 배치)
  5. **보스 레벨**: 보스 반격으로 배치될 장애물 영역을 미리 확보 (가장자리 2칸은 초기 장애물 없음)

### §2.2 매칭 규칙 (검출 우선순위 — V1 유지)

매치 검출은 반드시 아래 우선순위를 따른다. 같은 보석이 여러 패턴에 포함될 경우 **상위 우선순위 패턴이 소비**한다:

| 우선순위 | 패턴 | 생성되는 스페셜 보석 | 점수 배율 | 보스 데미지 배율 |
|----------|-------|---------------------|-----------|-----------------|
| 1 | **5매치** (가로 또는 세로 5연속) | 무지개 보석 (Rainbow) | ×5 | ×3.0 |
| 2 | **T자 매치** (가로3+세로3 교차, 5개) | 폭탄 보석 (Bomb) | ×4 | ×2.5 |
| 3 | **L자 매치** (가로3+세로3 꺾임, 5개) | 폭탄 보석 (Bomb) | ×4 | ×2.5 |
| 4 | **4매치** (가로 또는 세로 4연속) | 줄 파괴 보석 (Line, 방향=매치 직교) | ×3 | ×2.0 |
| 5 | **3매치** (가로 또는 세로 3연속) | 없음 | ×1 | ×1.0 |

**검출 알고리즘** (C44 검증 완료, 그대로 유지):
1. 전체 그리드를 스캔하여 모든 가능한 매치를 리스트에 수집
2. `used[row][col]` 2D boolean 배열 초기화
3. 우선순위 높은 것부터 처리하며, `used[][]`가 true인 보석은 하위 매치에서 제외
4. 스페셜 보석은 스왑된 보석 위치에 생성 (캐스케이드 매치 시 매치 중앙에 생성)

### §2.3 스페셜 보석 효과

| 보석 | 활성화 | 효과 |
|------|--------|------|
| **줄 파괴 (Line)** | 매칭 또는 스왑 참여 | 가로 또는 세로 한 줄 전체 제거 (화살표 방향) |
| **폭탄 (Bomb)** | 매칭 또는 스왑 참여 | 3×3 범위 폭발 제거 |
| **무지개 (Rainbow)** | 아무 보석과 스왑 | 스왑 대상과 같은 색의 보석을 보드 전체에서 모두 제거 |

|------|------|-------------------|

| 무지개 + 폭탄 | 랜덤 1색 전체를 폭탄으로 변환 후 전부 발동 | ×4.0 |
| 무지개 + 무지개 | 보드 전체 제거 (Ultimate Clear) | ×5.0 |

**독 타일 상세 규칙**:
- 매 턴 종료 시(CHECK_GOAL 직전) 독 확장 단계 실행
- 각 독 타일이 인접 4방향 중 랜덤 1방향으로 확장 시도
- 확장 대상 칸에 보석이 있으면 보석 파괴 + 독 타일로 변환
- 확장 대상 칸에 다른 장애물이 있으면 확장 실패 (다음 턴 재시도)
- 독 타일 위에서는 보석 스왑 불가, 새 보석 낙하도 차단
- **보스전**: 보스가 반격으로 독 타일을 직접 배치

**텔레포터 상세 규칙**:
- 항상 쌍(A↔B)으로 배치, 보라색 포탈 이펙트
- 위의 보석이 매칭으로 제거되면, 새 보석은 쌍의 반대편 텔레포터 위에서 낙하
- 일반 낙하와 텔레포터 낙하가 동시 발생 시, 일반 낙하 우선 처리 후 텔레포터 처리
- 레벨 디자인용 고정 오브젝트 (파괴 불가)

### §2.6 보스 시스템 (신규)

#### 보스 공통 규칙
- 보스 레벨에서는 화면 상단에 보스 초상화 + HP바 표시
- 플레이어의 매칭이 보스에게 데미지: `baseDmg × matchMultiplier × comboBonus`
  - `baseDmg = 매칭된 보석 수 × 10`
  - `matchMultiplier` = §2.2 보스 데미지 배율
  - `comboBonus` = 1.0 + (comboCount - 1) × 0.2 (최대 3.0)
- 보스는 **플레이어 턴 종료 후** 반격 실행 (장애물 생성, 보석 변환 등)
- 보스 HP가 0이 되면 격파 → 컷신 재생 → 도감 등록

#### 5종 보스 상세

| # | 보스명 | HP | 등장 레벨 | 반격 패턴 | 약점 |
|---|--------|-----|-----------|----------|------|
| 1 | **독 슬라임 왕** | 800 | Lv 4 | 매 턴 독 타일 1~2개 배치 | 3연속 매치로 독 제거 시 ×2 데미지 |
| 2 | **얼음 마녀** | 1200 | Lv 8 | 매 턴 랜덤 보석 2개를 얼음 2겹으로 동결 | 4매치 이상으로 ×2 데미지 |
| 3 | **체인 골렘** | 1600 | Lv 12 | 매 2턴 랜덤 보석 3개에 체인 부착 | 스페셜 보석 활성화 시 ×2 데미지 |

**보스 등장 컷신** (§5.5 상세):
1. 화면 어두워짐 (alpha 0→0.7, 500ms)
2. 보스 실루엣 중앙 등장 (scale 0→1.2→1.0 바운스 tween, 800ms)
3. 화면 쉐이크 (amplitude 8px, 300ms)
4. 보스명 + HP바 슬라이드인 (왼쪽에서, 400ms)
5. 배경 보라색 번개 파티클 (20개, 1000ms)

**보스 격파 컷신**:
1. 보스 HP 0 → 보스 깜빡임 (200ms × 5회)
2. 대형 폭발 파티클 (50개, 1200ms) + 화면 백색 플래시
3. "VICTORY!" 텍스트 팝업 (scale 0→1.5→1.0, 골드색)
4. 보석 비 내리기 (30개 보석 상단에서 낙하, 2000ms)

## §3. 조작 방법

### §3.1 마우스 조작
- **클릭 스왑**: 보석 A 클릭 → 인접 보석 B 클릭 → 스왑 시도
- **드래그 스왑**: 보석 위에서 마우스 다운 → 인접 방향으로 드래그 → 마우스 업 시 스왑
- **부스터 사용**: 하단 부스터 버튼 클릭 → 대상 보석/영역 클릭
- **실드 사용**: 실드 버튼 클릭 → 즉시 적용 (보스전 한정)

### §3.2 키보드 조작
- **방향키 (↑↓←→)**: 그리드 내 커서 이동
- **Space**: 보석 선택/선택 해제, 두 번째 Space로 스왑 실행
- **Enter**: 부스터 메뉴 토글
- **Escape**: 일시정지 메뉴
- **1/2/3/4**: 부스터 1/2/3/4번 직접 활성화

### §3.3 터치 조작
- **탭 스왑**: 보석 탭 → 인접 보석 탭
- **스와이프 스왑**: 보석 위에서 터치 → 인접 방향으로 스와이프
- **부스터**: 부스터 버튼 탭 → 대상 탭
- **터치 타겟**: 모든 버튼/인터랙티브 요소 최소 48×48px (V8 유지)
  - 렌더링 함수에서 `Math.max(MIN_TOUCH, actualSize)` 강제 적용

## §4.5 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**"Dark Fantasy Glossy Gems"** — C44의 밝은 동화풍에서 **어두운 판타지 배틀**로 전환. 보석은 여전히 글로시하고 화려하지만, 배경은 어둡고 긴장감 있는 분위기. 보석의 빛이 어둠 속에서 더욱 돋보이는 대비 효과.

### 레퍼런스 게임
1. **Royal Match — King's Nightmare** (Dream Games) — 어두운 배경 + 밝은 보석 대비, 긴장감 있는 보스전
2. **Puzzle Quest 3** (505 Games) — 다크 판타지 + 매치3 결합, 보스 초상화와 HP바 UI

### 통일 원칙
- 모든 에셋은 동일한 45도 상단-좌측 광원을 따름
- 보석의 하이라이트(좌상단)+그림자(우하단) 방향 통일
- 배경은 어두운 보라/남색 계열 — 보석과 이펙트가 빛나는 대비
- 보스 에셋은 보라+빨강 톤으로 위협적 분위기
- 이펙트는 밝고 채도 높은 색상 (파티클, 빛줄기, 번개)

### §5.1.1 PLAY 서브 상태 (보스전 확장)

```
IDLE → SWAP → MATCH_CHECK → MATCH_RESOLVE → CASCADE → SETTLE
  ↑                                                      ↓
  ← CHECK_GOAL ← BOSS_COUNTER ← POISON_SPREAD ←────────┘
  ↓

```

| 서브 상태 | 설명 | 가드 플래그 |
|-----------|------|-------------|
| **IDLE** | 플레이어 입력 대기 | — |
| **SWAP** | 보석 스왑 tween (300ms) | `swapLocked = true` |
| **MATCH_CHECK** | 매치 검출 (우선순위 순, used[][] 추적) | — |
| **MATCH_RESOLVE** | 매칭 보석 제거 tween + 점수/데미지 팝업 (400ms) | `resolving = true` |
| **CASCADE** | 낙하 + 리필 tween (350ms/칸) | `cascadeInProgress = true` |
| **SETTLE** | 재매칭 검사 → 매치 있으면 MATCH_CHECK로 | — |
| **POISON_SPREAD** | 독 타일 확장 처리 (보스전/독 레벨) | `poisonSpreading = true` |
| **BOSS_COUNTER** | 보스 반격 애니메이션 (보스전 한정, 800ms) | `bossCountering = true` |
| **CHECK_GOAL** | 목표 달성 확인 / 턴 소진 확인 / 보스 HP 확인 | `goalChecking = true` |

**유효하지 않은 스왑 처리**: SWAP 후 MATCH_CHECK에서 매치 없음 → 보석 원위치 tween (300ms) → IDLE (턴 소모 안 함)

**데드록 처리**: SETTLE에서 매치 없음 + 유효 이동 없음 → 자동 셔플 (Fisher-Yates) + "No moves! Shuffling..." 팝업

### §5.5 보스 컷신 연출 상세 (F3 해결)

```
t=0ms:    배경 어두워짐 (alpha tween 0→0.7, 500ms, easeInQuad)
t=300ms:  보스 실루엣 중앙 등장 (scale tween 0→1.2→1.0, 800ms, easeOutBack)
t=800ms:  화면 쉐이크 (amplitude 8px, duration 300ms, decay linear)
t=1000ms: 보스명 텍스트 "독 슬라임 왕" (왼쪽→중앙, 400ms, easeOutCubic)
t=1400ms: HP바 슬라이드인 (위→아래, 300ms, easeOutCubic)
t=1700ms: 번개 파티클 20개 방출 (800ms)
t=2500ms: directTransition(STATE.PLAY)
```

**보스 격파 (BOSS_DEFEAT 상태, 3초)**:
```
t=0ms:    보스 깜빡임 (alpha 1↔0, 200ms × 5회 = 1000ms)
t=1000ms: 대형 폭발 파티클 50개 + 화면 백색 플래시 (alpha 0→1→0, 400ms)
t=1400ms: "VICTORY!" 텍스트 (scale 0→1.5→1.0, 600ms, easeOutBack, 골드색 #ffd700)
t=2000ms: 보석 비 30개 낙하 (stagger 50ms, 각 보석 300ms 낙하)
t=2500ms: 도감 카드 슬라이드인 (아래→중앙, 400ms)
t=3000ms: beginTransition(BOSS_DEFEAT, LEVEL_COMPLETE)
```

### §6.3 DDA (Dynamic Difficulty Adjustment) — V6 확장

```javascript
const DDA = {
  // 기존 (C44 검증)
  failStreak: 0,           // 연속 실패 횟수
  gemWeightBoost: 0.03,    // 목표색 가중치 보정 (3연패 시 적용)

  // 신규 (보스전 확장)
  bossHpReduction: 0,      // 보스 HP 감소율 (2연패마다 5% 누적, 최대 20%)
  bossCounterDelay: 0,     // 보스 반격 1턴 지연 횟수 (3연패 시 첫 턴 반격 스킵)
  extraTurns: 0,           // 추가 턴 (4연패 시 +2턴)

  onLevelFail() {
    this.failStreak++;
    if (this.failStreak >= 2) this.bossHpReduction = Math.min(0.20, this.failStreak * 0.05);
    if (this.failStreak >= 3) this.bossCounterDelay = 1;
    if (this.failStreak >= 4) this.extraTurns = 2;
  },
  onLevelComplete() {
    this.failStreak = 0;
    this.bossHpReduction = 0;
    this.bossCounterDelay = 0;
    this.extraTurns = 0;
  }
};
```

| 보스 데미지 1점당 | 1점 |

### §7.2 콤보 보너스
- 캐스케이드 연쇄 시 콤보 카운터 증가
- `comboMultiplier = 1.0 + (comboCount - 1) × 0.5`
- 최대 ×5.0 (콤보 9 이상)
- 콤보 텍스트 팝업: "×2!", "×3!", ... (scale 애니메이션 + 골드색)

### §7.3 보스전 점수
- 보스 데미지 = 점수와 별도 트래킹
- 보스 격파 보너스: 보스 최대 HP × 2 (예: HP 800 보스 → 1600점 보너스)
- 남은 턴 보너스: 남은 턴 × 100

### §7.4 하이스코어
- localStorage에 레벨별 최고 점수 + 최고 별 저장
- `try-catch` 래핑 필수 (V1 표준)
- 세이브 키: `gemSiege_level_{n}_score`, `gemSiege_level_{n}_stars`

## §14. 기술 제약 및 규칙 (코더 참조)

1. **단일 HTML+Canvas+Vanilla JS** — 외부 라이브러리 0건
2. **IX Engine 통합**: TweenManager, ObjectPool, TransitionGuard, SoundManager, InputManager, AssetLoader 사용
3. **setTimeout 0건** — 모든 시간 기반 로직은 tween onComplete 또는 Web Audio ctx.currentTime
4. **render()에서 상태 변경 0건** — update/render 완전 분리
5. **`state = STATE.X` 직접 할당 0건** — beginTransition() / directTransition() 전면 사용
6. **`_ready` TDZ 방어 패턴** — 엔진 생성자 콜백에서 미완료 인스턴스 참조 금지
7. **파라미터 섀도잉 금지** — 전역 변수명과 함수 파라미터 충돌 방지
8. **에셋 manifest.json 동적 로드** — `fetch('assets/manifest.json')` + catch 폴백
9. **모든 에셋에 Canvas 폴백** — onerror=resolve 패턴, 폴백 드로잉 함수 개별 구현
10. **게임 루프 try-catch 래핑** — catch 밖에 rAF 배치
11. **SeededRNG** — Math.random() 0건
12. **에셋-코드 교차 검증** — manifest의 모든 에셋 ID가 코드에서 참조, 역참조도 확인
