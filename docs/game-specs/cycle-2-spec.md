## 1. 게임 개요 및 핵심 재미 요소

### 컨셉
우주를 배경으로 **플레이어 전투기를 조종해 웨이브별로 밀려오는 적 함대를 격파**하는 종스크롤 슈팅 게임입니다. 단순한 이동+사격에 **파워업 수집**, **웨이브 기반 난이도 상승**, **보스전**을 더해 "한 판 더"의 중독성을 제공합니다.

### 핵심 재미 요소
1. **즉각적 타격감** — 총알이 적에 명중할 때 폭발 이펙트 + 화면 미세 흔들림(screen shake)으로 시각적 쾌감
2. **파워업 수집의 쾌감** — 적 격파 시 드롭되는 파워업으로 화력이 순간적으로 강해지는 파워 판타지
3. **웨이브 서바이벌** — 5웨이브마다 등장하는 보스를 클리어하고 다음 스테이지로 진입하는 성취감
4. **짧고 강렬한 세션** — 한 판 3~7분의 집중적 플레이. 죽어도 즉시 재시작 가능
5. **점진적 긴장감** — 웨이브가 오를수록 적 밀도와 속도 증가, "얼마나 버티느냐"의 도전

## 2. 게임 규칙 및 목표

### 2.1 기본 규칙
- **종스크롤(위→아래) 슈팅** — 플레이어는 화면 하단, 적은 상단에서 등장
- 플레이어 전투기는 화면 내에서 **상하좌우 자유 이동** (화면 밖 이탈 불가)
- **자동 발사 모드**: 게임 중 총알이 자동으로 발사됨 (터치 환경 편의성)
- 키보드 환경에서는 **Space/Z키로 수동 발사** 가능 (자동+수동 병행)
- 적과 충돌하거나 적 총알에 맞으면 **HP 1 감소**
- HP가 0이 되면 **게임 오버**

### 2.2 웨이브 시스템
- 게임은 **무한 웨이브**로 진행
- 각 웨이브는 **적 편대 패턴**이 정해져 있고, 모든 적을 격파하면 다음 웨이브로 진입
- **5웨이브마다 보스전** 발생 (웨이브 5, 10, 15, ...)
- 웨이브 간 **2초 휴식 시간** — "WAVE N" 텍스트 연출 (tween scaleUp + fadeOut)
- 적 스폰은 동시 X → `spawnInterval = max(400, 1800 - wave * 60)` ms 간격으로 순차 스폰

### 2.3 적 유형 (4종 + 보스)

| 적 유형 | 색상 | HP | 이동 패턴 | 공격 | 점수 | 등장 웨이브 |
|---------|------|-----|----------|------|------|------------|
| **스카우트** (Scout) | `#5DADE2` (하늘) | 1 | 직선 하강 (`80px/s`) | 없음 | 100 | 1+ |
| **파이터** (Fighter) | `#F39C12` (주황) | 2 | 사인파 좌우 (`amplitude 40px, period 2s`) + 하강 (`60px/s`) | 단발 직선탄 (2초 간격) | 200 | 3+ |
| **탱크** (Tank) | `#E74C3C` (빨강) | 4 | 느린 직선 하강 (`40px/s`) | 3방향 확산탄 (-20°, 0°, +20°, 3초 간격) | 350 | 6+ |
| **다트** (Dart) | `#9B59B6` (보라) | 1 | 고속 대각선 돌진 (`200px/s`, 스폰 시 각도 결정) | 없음 (자폭형) | 150 | 8+ |
| **보스** (Boss) | `#2C3E50` + `#E74C3C` 코어 | wave×8 | 좌우 왕복 (`50px/s`) + 상단 고정 (`y=70`) | 복합 패턴 (3페이즈) | 1000×스테이지 | 5, 10, 15... |

### 2.4 보스 공격 패턴 (3페이즈 순환, 각 4초)
1. **페이즈 A — 직선 연사**: 0.4초 간격으로 플레이어 방향 직선탄 5발
2. **페이즈 B — 원형 확산**: 12방향(30° 간격) 방사형 탄막 1회 → 1.5초 후 다시 1회 (2회)
3. **페이즈 C — 추적탄**: 플레이어를 느리게 따라오는 유도탄 2발 (회전 속도 `2rad/s`, 3초 후 자폭)
- **HP 50% 이하**: 페이즈 전환 속도 ×1.5 + 탄속 ×1.2
- 보스 체력바: 보스 상단에 너비 100px 바 (`#2ECC71` → `#E74C3C` HP비례)

### 2.5 게임 목표
- **최고 점수 달성** + **최고 웨이브 도달** (둘 다 기록)
- 하이스코어와 최고 웨이브를 localStorage에 저장

### 2.6 게임 오버 조건
- HP가 0이 되면 게임 오버
- 게임 오버 화면: 최종 점수, 도달 웨이브, 최고 기록, 재시작/타이틀 버튼

## 3. 조작 방법

### 3.1 키보드 (데스크톱)

| 키 | 동작 |
|-----|------|
| `←→↑↓` 또는 `WASD` | 8방향 이동 |
| `Space` / `Z` | 발사 (자동발사와 병행, 홀드 시 연사) |
| `P` / `Esc` | 일시정지 토글 |
| `R` | 재시작 (**Canvas 확인 모달** 표시 후 진행) |
| `Enter` | 타이틀/게임오버에서 시작/재시작 |

> **동시 입력 지원**: `keyDown` Set으로 관리. 대각선 이동 시 속도 정규화: `벡터 = normalize(dx, dy) × speed`. 이동 중 발사도 동시 가능.

### 3.2 터치 (모바일/태블릿)

| 제스처 | 동작 |
|--------|------|
| **왼쪽 하단 영역 터치 & 드래그** | 가상 조이스틱 — 이동 |
| **화면 아무 곳 터치** | 자동 발사 활성 (기본 ON) |
| **오른쪽 상단 일시정지 아이콘 터치** | 일시정지 |

- 가상 조이스틱: 외부 원 반지름 50px (`rgba(255,255,255,0.15)`), 내부 핸들 반지름 20px (`rgba(0,229,255,0.5)`)
- 조이스틱 영역: 좌하단 `(0, canvasH×0.5)` ~ `(canvasW×0.4, canvasH)`
- 터치 시작점 기준 상대 이동, 데드존 8px
- 오토파이어 기본 활성: 터치 환경에서는 별도 발사 버튼 불필요

### 3.3 마우스 (데스크톱 대안)

| 동작 | 설명 |
|------|------|
| 마우스 이동 | 전투기가 마우스 위치를 **부드럽게 추적** (lerp factor `0.12`) |
| 좌클릭 홀드 | 발사 |
| 우클릭 | 없음 (`contextmenu` 이벤트 `preventDefault()`) |

### 4.4 배경 시스템 — 패럴랙스 별 스크롤

```
레이어 1 (먼 별):   별 40개, 반지름 0.5~1px, 스크롤 속도 0.3px/frame
레이어 2 (중간 별): 별 25개, 반지름 1~2px,   스크롤 속도 0.8px/frame
레이어 3 (가까운 별): 별 10개, 반지름 2~3px, 스크롤 속도 1.5px/frame
```

- 별이 화면 하단을 벗어나면 상단에 랜덤 x로 재배치 (무한 루프)
- 레이어 3의 밝은 별은 `alpha` 를 `0.6 + sin(time×2) × 0.2` 로 깜빡임
- 보스전 진입 시 배경 색상: `#0B0E17` → `#1A0A0A` (붉은 톤), 2초간 tween 전환
- 보스 처치 후: `#1A0A0A` → `#0B0E17` 복귀 (1초 tween)

### 4.5 이펙트 상세

| 이펙트 | 구현 |
|--------|------|
| **적 폭발** | 파티클 8~12개 방사 (`#FF6B35`→`#FFD93D`→`#FFFFFF`), 수명 400ms, 속도 `60~120px/s` 랜덤 방향 |
| **보스 폭발** | 파티클 30개 방사 + 백색 원형 확산(scale 0→3, alpha 1→0, 600ms) + 스크린 셰이크 (±6px, 400ms) |

```
function gameLoop(now) {
    requestAnimationFrame(gameLoop);
    const dt = min((now - lastTime) / 16.67, 2.0);  // 60fps 정규화, 최대 2프레임 스킵

        case 'TITLE':         updateTitle(dt);   drawTitle();  break;

        case 'PAUSE':         drawGame(); drawPauseOverlay();  break;
```

### 6.1 Tween 시스템 (Cycle 1 최대 아쉬움 해결)

```javascript
// TweenManager API
class TweenManager {
    tweens = [];  // 활성 tween 목록 (ObjectPool 적용)

    // tween 생성
    add(target, props, duration, easing = 'easeOutQuad', onComplete = null) {
        // target: 애니메이션 대상 객체
        // props: { x: 100, y: 200, alpha: 0, scale: 1.5 } 등 목표값
        // duration: ms
        // easing: 이징 함수명
        // 시작값은 target의 현재 속성값에서 자동 캡처
    }

    update(dt) { /* 모든 활성 tween 진행, 완료 시 콜백 실행 후 pool.release */ }
    cancelAll(target) { /* 특정 target의 모든 tween 제거 */ }
```

| 이름 | 공식 | 용도 |
|------|------|------|
| `linear` | `t` | 배경 스크롤, 깜빡임 |

### 6.2 객체 풀링 시스템 (Cycle 1 개선 제안 반영)

> **Cycle 1 문제**: 파티클을 매 프레임 push({...})로 생성. 이번엔 모든 주요 엔티티에 풀링 적용.

```javascript
class ObjectPool {
    pool = [];      // 비활성 객체 목록
    active = [];    // 활성 객체 목록

    constructor(createFn, initialSize) {
        this.createFn = createFn;
        for (let i = 0; i < initialSize; i++) this.pool.push(createFn());
    }

    acquire() {
        const obj = this.pool.length > 0 ? this.pool.pop() : this.createFn();
        obj.active = true;
        this.active.push(obj);
        return obj;
    }

    release(obj) {
        obj.active = false;
        const idx = this.active.indexOf(obj);
        if (idx !== -1) this.active.splice(idx, 1);
        this.pool.push(obj);
    }

    update(dt, fn) { // active 순회하며 fn(obj, dt) 실행, release가 안전하도록 역순 순회
        for (let i = this.active.length - 1; i >= 0; i--) {
            fn(this.active[i], dt, this);
        }
    }

    draw(ctx, fn) { // active 순회하며 fn(ctx, obj) 실행
        for (const obj of this.active) fn(ctx, obj);
    }
}
```

#### 풀 사이즈 설정

| 엔티티 | 초기 풀 크기 | 최대 동시 활성 (예상) |
|--------|-------------|---------------------|
| 플레이어 총알 | 30 | ~20 |
| 적 총알 | 50 | ~35 |
| 적 (일반) | 20 | ~12 |
| 파워업 | 5 | ~3 |
| 폭발 파티클 | 80 | ~50 |
| 플로팅 텍스트 | 10 | ~5 |
| Tween 객체 | 40 | ~25 |

### 6.3 충돌 감지 (AABB)

```javascript
function checkAABB(a, b) {
    return a.x - a.hw < b.x + b.hw &&
           a.x + a.hw > b.x - b.hw &&
           a.y - a.hh < b.y + b.hh &&
           a.y + a.hh > b.y - b.hh;
}
// hw = halfWidth, hh = halfHeight (중심 기준 AABB)
```

**충돌 매트릭스**:

| A | B | 결과 |
|---|---|------|

| 파워업 | 플레이어 | 파워업 효과 적용, 파워업 pool.release, 획득 이펙트 |

- 히트박스 크기(플레이어): 시각 크기 40×48 → 판정 크기 **24×24** (중심 기준, 관대한 판정)

### 8.2 웨이브 16+ 스케일링 공식

```

보스 HP          = wave × 8
적탄 속도        = 120 + min(wave × 3, 80) px/s
```

### 12.1 기본 스탯

| 항목 | 값 |
|------|-----|
| 초기 HP | 3 |
| 최대 HP | 5 |
| 이동 속도 | `300px/s` (대각선 이동 시 정규화) |
| 피격 무적 시간 | 1.5초 |
| 히트박스 | 중심 기준 `24×24px` (시각 크기 40×48보다 작음 = 관대한 판정) |
| 시각 크기 | `40×48px` |

### 12.2 피격 연출 시퀀스
1. 쉴드 있으면 → 쉴드 파괴 이펙트 (파란 파티클 8개) + HP 감소 없음
2. 쉴드 없으면:
   - HP -= 1
   - 붉은 오버레이 플래시 (`rgba(255,0,0,0.25)`, tween 150ms)
   - 스크린 셰이크 (±4px, tween 200ms)
   - 무적 1.5초 (플레이어 alpha `sin(time×20) × 0.5 + 0.5` 깜빡임)
   - 무기 단계 1단계 하락 (최소 0)
3. HP <= 0 → GAMEOVER 전환

### 12.3 쉴드 연출
- 활성 시: 플레이어 주위 반투명 원형 (`#3498DB`, alpha 0.3, 반지름 28px)
- 잔여 시간 3초 이하: alpha 깜빡임 (경고)
- 파괴 시: tween(shieldAlpha 0.3→0, 300ms) + 파란 파티클 8개 방사

### 13.2 기술 스택
- **HTML5 Canvas** — 단일 캔버스, 게임 렌더링

### 13.3 반응형 대응
- 게임 좌표계: **480×720** 고정 (종스크롤 최적 2:3 비율)
- Canvas CSS 크기: `min(viewportW - 20, 480)` × `min(viewportH - 20, 720)`
- DPR 대응: `canvas.width = cssWidth × dpr`, `canvas.height = cssHeight × dpr`, `ctx.scale(dpr, dpr)`
- `window.resize` + `orientationchange` 이벤트에서 재조정
- 모든 게임 로직은 480×720 좌표계에서 동작, 렌더링 시 스케일 자동 적용
