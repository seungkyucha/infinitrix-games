## 1. 게임 개요 및 핵심 재미 요소

**네온 서바이버즈**는 32-bit 픽셀아트 스타일의 실시간 서바이벌 슈터입니다. 플레이어는 끝없이 몰려오는 적 무리 속에서 자동 발사되는 무기로 생존하며, 레벨업 시 3지선다 중 하나를 선택해 빌드를 구성합니다.

### 핵심 재미 요소
1. **"한 판 더" 중독성**: 30초 안에 첫 레벨업 → 빌드 선택의 쾌감. 죽어도 즉시 재도전 유도
2. **빌드 다양성**: 매 런마다 다른 업그레이드 조합으로 다른 플레이 경험
3. **성장의 쾌감**: 1레벨의 약한 캐릭터가 웨이브를 거듭하며 화면 가득 탄막을 뿌리는 성장 곡선
4. **시각적 피드백**: 32-bit 픽셀아트의 풍부한 이펙트 — 적 처치 시 경험치 보석 비산, 레벨업 시 화면 플래시
5. **리스크-리워드**: 경험치 보석을 모으려면 적에게 가까이 가야 함 → 긴장감

## 2. 게임 규칙 및 목표

### 목표
- **최대한 오래 생존**하며 높은 점수를 달성
- 8 웨이브 + 보스 웨이브를 클리어하면 승리

### 기본 규칙
- 플레이어 HP: 100 (기본). 0이 되면 게임 오버
- 무기는 **자동 발사** — 플레이어는 이동에만 집중
- 적 처치 시 **경험치 보석(XP Gem)** 드롭
- 경험치가 가득 차면 **레벨업** → 3개 업그레이드 중 1개 선택
- 웨이브는 60초 간격으로 전환 (총 8+1 웨이브)
- 맵은 2000×2000px 오픈 필드 (카메라가 플레이어 중심 추적)

### 게임 오버 조건
- HP가 0 이하로 떨어짐
- 게임 오버 화면에서 최종 점수, 생존 시간, 처치 수 표시
- **즉시 재시작** 버튼 (R키 / 터치 / 클릭)

### 승리 조건
- 9번째 웨이브(보스 웨이브)의 보스를 처치하면 클리어
- 클리어 화면에서 총 점수 + 등급(S/A/B/C) 표시

## 4. 시각적 스타일 가이드

### 색상 팔레트
| 용도 | 색상 | HEX |
|------|------|-----|
| 배경 (어두운 바닥) | 딥 네이비 | `#0d1117` |
| 플레이어 하이라이트 | 사이안 | `#00d4ff` |
| 적 기본색 | 크림슨 | `#ff4444` |
| 경험치 보석 | 에메랄드 | `#00ff88` |
| UI 강조 | 골드 | `#ffd700` |
| 보스 | 진보라 | `#8b00ff` |
| 체력바 | 라임 그린 | `#44ff44` |

### 배경
- 황폐한 네온 시티의 광장 (위에서 내려다보는 탑다운 뷰)
- 바닥은 금이 간 콘크리트 + 네온 배선이 깔린 그리드 패턴
- 화면 가장자리에 부서진 건물 실루엣 (중경)
- 먼 배경은 보라빛 밤하늘 + 희미한 빌딩 스카이라인

### 오브젝트 형태
- **플레이어**: 네온 파츠가 부착된 소형 로봇/사이버 전사 (탑다운)
- **적**: 유기체+기계 혼합 크리처 (글리치 효과가 간헐적으로 깜빡임)
- **보석**: 작은 다이아몬드 형태, 색상별로 XP 양 구분
- **탄환**: 에너지 구체/레이저 — 무기 타입별 형태 차이

## 4.5. 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**pixel-art-32bit** — Detailed 32-bit pixel art, rich palette, soft shading, late-90s arcade / PS1 sprite era.

### 세부 디렉션
- **해상도**: 모든 캐릭터 스프라이트는 32×32px 기본 단위, 게임 내에서 2~3배 스케일 렌더링
- **셰이딩**: 2~3단계 소프트 셰이딩 (하이라이트, 미드톤, 그림자)
- **팔레트**: 제한된 팔레트(64색 이내)로 통일감 유지, 네온 강조색은 1~2색만
- **아웃라인**: 1px 다크 아웃라인 (순수 검정 아닌 어두운 톤 사용)
- **애니메이션**: 2~4프레임의 심플 루프 (아이들, 이동, 피격)
- **이펙트**: 적 처치 시 픽셀 파티클 비산, 레벨업 시 원형 펄스

### 레퍼런스 게임
1. **Vampire Survivors** — 탑다운 서바이벌 구도, 적 웨이브 밀도, 경험치 보석 비산 연출
2. **Brotato** — 32-bit 스타일 캐릭터, 무기 이펙트, UI 레이아웃

## 5. 핵심 게임 루프 (프레임 기준 로직 흐름)

```
매 프레임 (60fps, requestAnimationFrame):
│
├─ 1. Input 처리
│   └─ 조이스틱/키보드 → 이동 벡터 계산 (정규화)
│
├─ 2. 플레이어 업데이트
│   ├─ 위치 이동 (속도 × deltaTime)
│   ├─ 맵 경계 클램프
│   └─ 무적 시간 카운트다운 (피격 후 0.5초)
│
├─ 3. 무기 업데이트 (자동 발사)
│   ├─ 각 무기의 쿨다운 체크
│   ├─ 가장 가까운 적 탐색 (또는 무기별 타겟팅 로직)
│   └─ 탄환 생성 → 오브젝트 풀에서 가져오기
│
├─ 4. 탄환 업데이트
│   ├─ 위치 이동
│   ├─ 적과 충돌 판정 (원형 히트박스)
│   ├─ 충돌 시: 데미지 적용, 이펙트 재생, 풀로 반환
│   └─ 화면 밖 → 풀로 반환
│
├─ 5. 적 업데이트
│   ├─ 플레이어 방향으로 이동
│   ├─ 플레이어와 충돌 판정
│   │   └─ 충돌 시: 플레이어 데미지 + 넉백 + 무적 부여
│   ├─ HP ≤ 0 → 사망 처리
│   │   ├─ 경험치 보석 스폰
│   │   ├─ 점수 추가
│   │   └─ 사망 이펙트 재생, 풀로 반환
│   └─ 스폰 타이머 체크 → 화면 밖 랜덤 위치에 생성
│
├─ 6. 아이템 업데이트
│   ├─ 경험치 보석: 플레이어 근접 시 자석 효과로 흡수
│   ├─ HP 회복 아이템: 충돌 시 HP 회복
│   └─ 자석 아이템: 화면 내 모든 보석 즉시 흡수
│
├─ 7. 웨이브 관리
│   ├─ 웨이브 타이머 카운트
│   ├─ 시간 도달 시 → 다음 웨이브 전환
│   │   ├─ 적 종류/수/속도 증가
│   │   └─ 웨이브 알림 UI 표시
│   └─ 웨이브 9 → 보스 스폰
│
├─ 8. 레벨업 체크
│   ├─ XP ≥ 필요량 → 게임 일시정지
│   ├─ 3개 업그레이드 카드 무작위 표시
│   ├─ 선택 후 → 효과 적용, 게임 재개
│   └─ 다음 레벨 필요 XP 증가 (1.2배)
│
├─ 9. 렌더링
│   ├─ 카메라 위치 계산 (플레이어 중심, 부드러운 추적)
│   ├─ 배경 타일 렌더 (카메라 오프셋)
│   ├─ 적 렌더 (활성 오브젝트만)
│   ├─ 탄환 렌더
│   ├─ 아이템 렌더
│   ├─ 플레이어 렌더
│   ├─ 이펙트/파티클 렌더
│   └─ UI 오버레이 (HP바, XP바, 웨이브, 점수, 처치수)
│
└─ 10. 게임 상태 체크
    ├─ HP ≤ 0 → GAME_OVER 씬 전환
    └─ 보스 처치 → VICTORY 씬 전환
```

### 성능 최적화 핵심
- **오브젝트 풀링**: 탄환(200개), 적(100개), 보석(300개), 파티클(200개) 사전 할당
- **공간 해싱(Spatial Hash)**: 충돌 판정을 O(n²) → O(n) 수준으로 최적화. 그리드 셀 64×64px
- **컬링**: 카메라 뷰포트 밖 오브젝트는 렌더링 스킵
- **deltaTime 기반**: 프레임 드랍 시에도 일관된 게임 속도

- **1페이즈** (HP 100%~50%): 느린 추적 + 3초마다 전방위 탄환 8발 발사 + 10초마다 슬라임 4마리 소환

## 10. 기술 구현 노트

### 필수 구현 패턴 (Cycle 1 검증)
- **IX 엔진 API 전면 사용**: 자체 setTimeout/addEventListener 금지
- **에셋 타임아웃 + fallback**: `assets.load(map, { timeoutMs: 10000 })` + 도형 fallback
- **3회 연속 재시작 테스트**: resetGameState()에서 모든 배열/맵/카운터 명시적 초기화
- **Web Audio 톤 합성**: 외부 오디오 파일 없이 효과음 실시간 생성

### 이번 사이클 신규 기술 과제
- **오브젝트 풀링**: 탄환/적/보석/파티클 사전 할당 + acquire/release 패턴
- **공간 해싱**: 충돌 판정 최적화, 64×64 그리드 셀
- **deltaTime 보정**: 프레임 드랍 시에도 일관된 게임 속도
- **카메라 시스템**: 플레이어 중심 부드러운 추적 (lerp 0.1)
- **가상 조이스틱**: 터치 시작 위치 기준 동적 생성, 데드존 10px

### QA 체크리스트
- [ ] 3회 연속 재시작 후 메모리/상태 누수 없음
- [ ] 모든 버튼 48px 이상
- [ ] 씬 전환 후 200ms 입력 무시 동작
- [ ] 웨이브 1~9(보스) 전 구간 플레이 가능
- [ ] 게임 오버 → 재시작 즉시 가능
- [ ] 승리 → 재시작 즉시 가능
- [ ] 오브젝트 풀 소진 시 graceful 처리 (새 생성 대신 스킵)
- [ ] 모바일 가상 조이스틱 반응 정상
- [ ] 에셋 로딩 실패 시 fallback 도형 표시
- [ ] 보스 2페이즈 전환 정상 동작
