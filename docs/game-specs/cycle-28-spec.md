# 네온 펄스 (Neon Pulse) — 사이클 #28 게임 기획서

> **1페이지 요약**: 비트에 맞춰 적을 처치하고 네온 던전을 돌파하는 **리듬 아케이드 로그라이트**. 프로시저럴 BPM 동기화 전투에서 Perfect/Great/Good 타이밍 판정으로 적을 공격하고, 연속 Perfect 콤보로 스코어 배율을 폭발시킨다. 5개 음악 존(Synthwave/Dubstep/Lo-fi/Drum&Bass/Glitch) × 3 스테이지 = 15 기본 스테이지 + 히든 2스테이지(Remix Zone, True Silence) = **총 17 스테이지**. 존 보스 5체 + 히든 보스 "Void DJ" = **6체 보스**. 웨이브 간 3택 사운드 칩(유물) 선택(common 6/rare 4/epic 3 = 13종), 영구 업그레이드 3트리(Rhythm/Power/Flow), SeededRNG 비트 패턴·적 배치, 3단 난이도(비기너/DJ/마에스트로), DDA 동적 밸런스, 다국어(한/영). **arcade+casual 조합을 1→2개로 강화**하며, Poki 2026.3 인기 공식 "원터치 조작 + 높은 리플레이성"을 계승한다.

> **MVP 경계**: Phase 1(핵심 루프: 비트 판정 → 공격 → 콤보 → 적 처치 → 보상, 존 1~2 + 보스 2체 + 업그레이드 기본 트리 + 사운드 칩 6종) + Phase 2(존 3~5 + 보스 3체 + 히든 보스 + 전체 내러티브 + 도전 모드 + 사운드 칩 7종) 순차 구현. Phase 1만으로도 완전한 게임 경험 제공 필수.

## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
네온 펄스는 비트에 맞춰 적을 처치하고 네온 던전을 돌파하는 **리듬 아케이드 로그라이트**다. Web Audio API로 프로시저럴 생성된 BPM 동기화 비트에 맞춰 공격/회피하면 보너스 대미지와 콤보 배율을 획득한다. **리듬 게임의 타이밍 쾌감**, **아케이드의 즉각적 피드백**, **로그라이트의 빌드 다양성**이 삼중 축을 이룬다.

### 1.2 핵심 재미 3축
1. **비트 전투 (Feel the Beat)**: 화면 하단의 비트 레인에 맞춰 타이밍 입력 → Perfect(±50ms)/Great(±100ms)/Good(±150ms)/Miss 4단 판정. Perfect 연속 시 콤보 배율 급증 + 네온 글로우 화면 전체 펄스 이펙트. "비트를 타는 쾌감"이 핵심.
2. **원터치 액션**: 키보드 한 키(Space/↑) 또는 화면 탭 한 번으로 공격. 좌우 이동(←→/스와이프)으로 적 회피. 직관적 조작에 깊은 타이밍 전략.
3. **로그라이트 빌드**: 존 클리어 시 3택 사운드 칩(유물) 선택으로 매 런마다 다른 빌드. "콤보 특화", "회피 특화", "파워 특화" 등 다중 전략 경로.

### 1.3 스토리/내러티브
- **배경**: 디지털 오디오 차원 "펄스 넥서스"가 바이러스 "사일런스"에 감염되어 모든 비트가 왜곡되었다. 마지막 DJ "네온"이 비트의 힘으로 5개 음악 존을 정화하고 차원을 구원해야 한다.
- **존별 이야기**: 각 존에는 감염 전 수호 DJ의 기록이 남아있다. Synthwave의 "레트로", Dubstep의 "워블", Lo-fi의 "칠", Drum&Bass의 "브레이크", Glitch의 "에러". 스테이지 간 짧은 텍스트 대화(Canvas 연출 3줄)로 이야기 전달.
- **엔딩 분기**:
  - 5존 정화 → **"비트 부활"** 일반 엔딩: 사일런스를 퇴치하고 펄스 넥서스에 음악이 돌아온다.
  - 히든 보스 "Void DJ" 격파 → **"영원한 비트"** 진엔딩: 사일런스는 사실 과부하된 비트를 정리하려는 시스템 프로세스였고, 네온은 새로운 하모니를 창조하여 진정한 균형을 이룬다.
- **스토리 전달**: 스테이지 간 수호 DJ 기록(3줄 텍스트 + 배경 연출), 보스 등장/격파 컷신(Canvas 카메라 줌 + 대사 5초), 히든 보스 후 에필로그(스크롤 텍스트 10초).

### 1.4 장르 강화 효과
- **arcade + casual = 1→2개** (최소 빈도 조합 강화)
- arcade 전체: 10→11개, casual 전체: 7→8개
- 기존 arcade+casual(neon-dash-runner: 단순 엔들리스 러너)와 완전 차별화: 리듬 전투 + 로그라이트 진행 — 완전히 다른 핵심 메카닉

## §2. 게임 규칙 및 목표

### 2.1 기본 규칙
- **비트 레인 기반 전투**: 화면 하단에 비트 마커가 BPM에 맞춰 좌→우로 이동. 판정선에 도달할 때 공격 입력
- **타이밍 판정**: Perfect(±50ms) = 150% 대미지 + 콤보 유지, Great(±100ms) = 120% 대미지 + 콤보 유지, Good(±150ms) = 100% 대미지 + 콤보 리셋, Miss = 0% 대미지 + 콤보 리셋 + 적 공격
- **콤보 배율**: 연속 Perfect/Great → 콤보 카운터 증가 → 1 + (combo × 0.1) 배율 (최대 ×3.0 = 20콤보)
- **플레이어 HP**: 초기 100. Miss 시 적 공격으로 HP 감소. HP 0 → 게임 오버
- **적 HP**: 각 적에게 HP가 있으며, 판정별 대미지 × 콤보 배율로 감소
- **영구 진행**: 게임 오버 시 획득한 "비트 크리스탈"은 보존 → 영구 업그레이드 투자

### 2.2 비트 시스템 상세

| 비트 유형 | 표시 | 입력 | 효과 |
|----------|------|------|------|
| 기본 비트 | ● (단색) | Space/탭 | 기본 공격 |
| 더블 비트 | ●● (연속) | 2연타 | 2연속 공격, 두 번 다 Perfect 시 보너스 |
| 홀드 비트 | ●━━ (길게) | 길게 누름 | 지속 대미지, 유지 시간 비례 배율 |
| 회피 비트 | ◆ (마름모) | ←/→ 또는 스와이프 | 적 공격 회피, 회피 성공 시 반격 찬스 |
| 보스 비트 | ★ (별) | 비트 유형에 따라 다름 | 보스전 전용, 패턴 기억 필요 |

### 2.3 존 구조

| 존 | 테마 | BPM 범위 | 스테이지 수 | 보스 |
|----|------|----------|-----------|------|
| 1. Synthwave | 80s 레트로 | 100~120 | 3 | DJ 레트로 (홀드 비트 특화) |
| 2. Dubstep | 베이스 드롭 | 120~140 | 3 | DJ 워블 (더블 비트 + 베이스 드롭 페이즈) |
| 3. Lo-fi | 잔잔한 힙합 | 80~100 | 3 | DJ 칠 (느린 BPM + 변칙 타이밍) |
| 4. Drum & Bass | 고속 드럼 | 150~174 | 3 | DJ 브레이크 (초고속 연타 + 브레이크 구간) |
| 5. Glitch | 글리치 아트 | 100~160 (변속) | 3 | DJ 에러 (BPM 실시간 변속 + 반전 패턴) |
| H1. Remix Zone | 혼합 | 가변 | 1 | — (서바이벌) |
| H2. True Silence | 무음 | 0→200 점진적 | 1 | Void DJ (모든 패턴 종합 + 무음 구간) |

### 2.4 승리/패배 조건
- **승리**: 현재 스테이지의 모든 적 처치 (HP 0으로 만들기)
- **패배**: 플레이어 HP 0 → 게임 오버, 획득 비트 크리스탈 보존
- **존 클리어**: 3스테이지 + 보스전 클리어 → 다음 존 해금
- **진엔딩**: 5존 클리어 후 히든 존 2개 해금 → Void DJ 격파

## §3. 조작 방법

### 3.1 키보드 조작
| 키 | 동작 |
|----|------|
| Space / ↑ / W | 비트 공격 (기본/더블/보스 비트) |
| Space (길게) | 홀드 비트 유지 |
| ← / A | 왼쪽 회피 |
| → / D | 오른쪽 회피 |
| P / Escape | 일시정지 |
| Enter | 확인 (메뉴/선택) |
| 1/2/3 | 사운드 칩 선택 (3택 화면) |

### 3.2 터치/마우스 조작
| 입력 | 동작 |
|------|------|
| 화면 탭 | 비트 공격 |
| 화면 길게 누름 | 홀드 비트 유지 |
| 좌측 스와이프 | 왼쪽 회피 |
| 우측 스와이프 | 오른쪽 회피 |
| 일시정지 버튼 탭 | 일시정지 |
| 칩 카드 탭 | 사운드 칩 선택 |

### 3.3 UI 크기 및 히트 영역 규칙
- 모든 터치 버튼: `Math.max(CONFIG.MIN_TOUCH, computedH)` 적용 (CONFIG.MIN_TOUCH = 48px)
- 히트 영역 판정: **hitTest(x, y, rect)** 단일 함수 사용 (F16, F60, F69)
- 드로잉 좌표와 히트 영역을 동일 rect 객체 `{ x, y, w, h }` 로 관리
- 일시정지 버튼: 최소 48×48px, 우상단 고정
- 비트 판정 영역: 화면 하단 20% 전체 (터치 시 넓은 판정)
- 스와이프 인식: 30px 이상 수평 이동 + 200ms 이내

### 4.5 배경 비주얼
- **패럴랙스 3레이어**: 별/네온 라인(느림) → 도시 실루엣/파형(중간) → 비트 파티클(빠름)
- **존별 고유 환경 이펙트**:
  - Synthwave: 레트로 그리드 바닥 + 석양 그라디언트
  - Dubstep: 파형 물결 + 스피커 콘 진동
  - Lo-fi: 비 내리는 창가 + VHS 노이즈 오버레이
  - D&B: 회전하는 레코드 판 + 에너지 파동
  - Glitch: 화면 분할/반전/왜곡 글리치 아트
- **비트 동기화 펄스**: 매 비트마다 배경 밝기 ±15% 펄스
- **카메라 연출**: 보스 등장 줌인(1.0→1.5 scale, 0.5초), 보스 격파 줌아웃(1.5→1.0, 1초)

### 4.6 Canvas 해상도
- `devicePixelRatio` 적용: 실제 렌더링 해상도 = CSS 크기 × dpr
- 동적 리사이즈: `window.resize` 이벤트 + `resizeCanvas()` 호출
- 최소 해상도: 320×480, 최대: 제한 없음 (동적 스케일링)
- 렌더링 좌표: 논리 좌표(800×600 기준) → 실제 픽셀 변환

### 5.1 초기화 순서 (F12)

```
4. DOM 요소 할당 (canvas, ctx)

6. 게임 루프 시작 (requestAnimationFrame)
```

### 5.2 핵심 엔진 시스템

#### SeededRNG (F18)
- `G.rng = new SeededRNG(seed)` — 모든 랜덤은 `G.rng.next()` 경유
- **Math.random 문자열 0건** — 스모크 테스트 FAIL 게이트

#### TweenManager (F4, F5, F13, F14)
- setTimeout 0건 — 17사이클 연속 목표
- `clearImmediate()` API: cancelAll과 add의 경쟁 조건 방지
- 가드 플래그 패턴: `if (G._transitioning) return;` 로 1회 실행 보장
- **BPM 값은 `G.bpm` 단일 변수로만 관리** (F70): tween으로만 갱신, 직접 대입 경로 0건

#### ObjectPool
- 파티클, 비트 마커, 판정 이펙트용 풀 3종
- `acquire()` / `release()` 패턴, try-catch 래핑

#### SoundManager (Web Audio API)
- **프로시저럴 BGM**: 존별 BPM + 음계 패턴으로 생성
- **효과음 8종+**: 비트 히트(Perfect/Great/Good), 콤보(5/10/20), 회피 성공, 보스 등장, 레벨업, 칩 획득, 피격, 게임오버
- **BPM 동기화**: `audioCtx.currentTime` 기반 비트 스케줄링
- setTimeout 대신 `oscillator.start(ctx.currentTime + delay)` 네이티브 스케줄링

#### InputManager
- hitTest(x, y, rect) 단일 함수 (F16, F69)
- 키보드/마우스/터치 통합
- 스와이프 인식: `{ startX, startY, startTime }` → 종료 시 판정

## §7. 핵심 게임 루프

### 7.1 비트 시스템 (F70)

#### 7.1.1 BPM 동기화 엔진
```
비트 간격 = 60000 / G.bpm (밀리초)
비트 타임스탬프[n] = stageStartTime + n × beatInterval
현재 비트 인덱스 = floor((currentTime - stageStartTime) / beatInterval)
```

- BPM 값은 **G.bpm 단일 변수**로만 관리 (F14, F70)
- 존 전환 시 BPM 변경: `tw.add({ target: G, prop: 'bpm', from: oldBpm, to: newBpm, dur: 2000 })`
- 직접 `G.bpm = xxx` 대입 0건 (스모크 테스트 체크)

#### 7.1.2 비트 패턴 생성 (SeededRNG)
- 각 스테이지의 비트 패턴은 `G.rng` 기반 프로시저럴 생성
- **비트 밀도** (초당 비트 수): 존/스테이지별 CONFIG 테이블 참조
- **비트 유형 비율**: 기본 60%, 더블 15%, 홀드 15%, 회피 10% (존별 변형)
- 보스전: 패턴 기반 고정 시퀀스 + 랜덤 변형 (시드 기반)

#### 7.1.3 타이밍 판정 알고리즘
```
inputTime = 사용자 입력 시각
nearestBeat = 가장 가까운 비트 타임스탬프
delta = abs(inputTime - nearestBeat)

if delta <= 50:  PERFECT (150% dmg, combo++)
elif delta <= 100: GREAT (120% dmg, combo++)
elif delta <= 150: GOOD (100% dmg, combo=0)
else: MISS (0% dmg, combo=0, player.hp -= enemy.atk)
```

### 7.2 프레임 루프 (60fps 기준)

```
gameLoop(timestamp):
  1. rawDt = timestamp - lastTimestamp
  2. dt = Math.min(rawDt, 33.33)  // 2프레임 스킵 캡
  3. lastTimestamp = timestamp

  4. InputManager.poll()
  5. TweenManager.update(dt)

  6. switch(G.state):
     PLAYING / BOSS_FIGHT:
       a. SoundManager.updateBGM(dt)
       b. RhythmEngine.update(dt)  — 비트 진행, 판정 윈도우 업데이트
       c. CombatSystem.update(dt)  — 대미지 적용, 적 HP 체크, 보상
       d. DDA.update(dt)           — 동적 난이도 조정
       e. ParticlePool.update(dt)
     기타 상태: 매트릭스(§6.2) 참조

  7. Renderer.draw(ctx, G, dt)   — 현재 상태에 맞는 드로잉
  8. requestAnimationFrame(gameLoop)
```

### 7.3 전투 루프 (1비트 단위)

```
각 비트 도착 시:
  1. 판정 윈도우 열림 (비트 타임 ± 150ms)
  2. 플레이어 입력 대기
  3. 입력 발생 → 판정(§7.1.3)
     - Perfect/Great: 적에게 대미지, 콤보++, 이펙트 표시
     - Good: 적에게 대미지, 콤보 리셋
     - Miss (입력 없음/범위 초과): 적이 플레이어 공격
  4. 적 HP ≤ 0 → 적 사망 처리 + 보상(비트 크리스탈 + 점수)
  5. 모든 적 사망 → 스테이지 클리어
  6. 플레이어 HP ≤ 0 → 게임 오버 (가드 플래그 체크: F5)
```

### 7.4 보스전 패턴

### 8.2 밸런스 산식

#### DPS (Damage Per Second)
```
baseDmg = CONFIG.PLAYER_BASE_DMG × difficultyMul
judgeMul = (perfectRate × 1.5 + greatRate × 1.2 + goodRate × 1.0)
comboMul = 1 + avgCombo × 0.1  // 캡: 3.0
chipMul = 1 + chipBonusDmg      // 캡: 2.0 (DPS 캡)
DPS = baseDmg × judgeMul × comboMul × chipMul × (bpm / 60)
```

**밸런스 가정 (난이도별)**:
| 가정 | 비기너 | DJ | 마에스트로 |
|------|--------|-----|-----------|
| Perfect율 | 30% | 50% | 70% |
| Great율 | 40% | 30% | 20% |
| Good율 | 20% | 15% | 8% |
| Miss율 | 10% | 5% | 2% |
| 평균 콤보 | 3 | 8 | 15 |

#### EHP (Effective HP)
```
EHP = playerHP / (1 - dodgeRate) / missRate
dodgeRate = 회피 비트 성공률
```

#### 사운드 칩 누적 캡 (F67)

## §11. 점수 시스템

### 11.1 점수 계산 (F8: 판정 먼저, 저장 나중에)
```
beatScore = baseScore × judgeMul × comboMul
  baseScore = 100
  judgeMul: Perfect=1.5, Great=1.2, Good=1.0, Miss=0
  comboMul: 1 + combo × 0.1 (캡 3.0)

stageScore = sum(beatScore) + clearBonus
  clearBonus = 500 × stageNumber

bossScore = sum(beatScore) + bossDefeatBonus
  bossDefeatBonus = 2000 × zoneNumber

totalScore = sum(stageScore + bossScore) + crystalBonus
```

**점수 흐름**:
1. 비트 판정 → 즉시 점수 계산 (판정 먼저)
2. 화면에 점수 표시
3. 스테이지/존 클리어 시 보너스 합산
4. 게임 오버/클리어 시 **최종 점수 판정 → localStorage 저장** (저장 나중에)

### 11.2 하이스코어
- 난이도별 독립 하이스코어
- 최고 도달 존/스테이지 기록
- 최고 콤보 기록
