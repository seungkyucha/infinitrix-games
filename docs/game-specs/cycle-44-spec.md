> **사이클 #44 — 매치3 집중 모드 1/10 라운드**

## §0. MVP 범위 정의

### Phase 1 — 이번 사이클 구현 범위 (MVP)
- 8×8 그리드 기반 6색 보석 스왑 매칭 (3/4/5/L/T자 패턴)
- 스페셜 보석 3종 (줄 파괴, 폭탄 3×3, 무지개) + 스페셜 조합 폭발
- 캐스케이드 루프 (매칭 → 제거 → 낙하 → 리필 → 재매칭)
- 15+ 레벨 시스템 (턴 제한 + 다양한 목표)
- 장애물 3종 (얼음 1~2겹, 나무 상자, 체인 보석)
- 별 평가 (1~3성), 부스터 3종
- 캐슬 복원 메타 레이어 (별로 캐슬 구역 해금)
- Web Audio BGM + 효과음 8종+
- Canvas 풀스크린 + devicePixelRatio + 동적 리사이즈

### Phase 2+ — 후속 라운드 확장 (이번에 미구현)
- 추가 장애물 (독 타일, 텔레포터, 컨베이어 벨트)
- 보스 스테이지 (턴 기반 보스 HP 깎기)
- 히든 스테이지, 일일 챌린지
- 캐슬 꾸미기 선택지 (가구/장식 택1)

## §1. 게임 개요 및 핵심 재미 요소

### 컨셉
판타지 왕국의 보석을 매칭하여 잊혀진 왕의 성을 복원하는 **프리미엄 캐주얼 매치3 퍼즐**. Royal Match의 검증된 공식을 HTML5 Canvas로 재현하되, 화려한 캐스케이드 연쇄와 스페셜 보석 조합의 시각적 쾌감에 집중한다.

### 핵심 재미 요소
1. **연쇄 폭발의 쾌감**: 매칭 → 낙하 → 자연 캐스케이드 → 콤보 카운터 폭발. 화면 가득 파티클이 터지는 시각적 보상
2. **전략적 스페셜 조합**: 줄 파괴 + 폭탄 = 십자 3줄 파괴, 무지개 + 폭탄 = 전체 폭탄화 등 조합의 깊이
3. **레벨별 고유 목표**: "빨간 보석 20개 수집", "얼음 12개 깨기", "점수 5000점 달성" 등 매판 새로운 도전
4. **캐슬 복원 성취감**: 레벨 클리어 → 별 획득 → 캐슬 구역 해금 → 시각적 진행감

## §2. 게임 규칙 및 목표

### §2.1 보석 그리드 시스템

- **그리드 크기**: 8×8 (64칸)
- **보석 종류**: 6색 기본 보석 + 3종 스페셜 보석
- **초기 보드 생성 규칙**:
  1. 랜덤으로 6색 보석 배치 (균등 가중치 각 ~16.7%)
  2. **3매치 없음 검증**: 가로/세로 연속 3개 동색 검출 시 해당 보석 재배치
  3. **유효 이동 검증**: 최소 1개 이상의 유효 스왑이 존재하는지 BFS 탐색
  4. 두 조건 모두 통과할 때까지 반복 (최대 100회, 초과 시 전체 재생성)

### §2.2 매칭 규칙 (검출 우선순위 — F9 해결)

매치 검출은 반드시 아래 우선순위를 따른다. 같은 보석이 여러 패턴에 포함될 경우 **상위 우선순위 패턴이 소비**한다:

| 우선순위 | 패턴 | 생성되는 스페셜 보석 | 점수 배율 |
|----------|-------|---------------------|-----------|
| 1 | **5매치** (가로 또는 세로 5연속) | 무지개 보석 (Rainbow) | ×5 |
| 2 | **T자 매치** (가로3+세로3 교차, 5개) | 폭탄 보석 (Bomb) | ×4 |
| 3 | **L자 매치** (가로3+세로3 꺾임, 5개) | 폭탄 보석 (Bomb) | ×4 |
| 4 | **4매치** (가로 또는 세로 4연속) | 줄 파괴 보석 (Line, 방향=매치 직교) | ×3 |
| 5 | **3매치** (가로 또는 세로 3연속) | 없음 | ×1 |

**검출 알고리즘**:
1. 전체 그리드를 스캔하여 모든 가능한 매치를 리스트에 수집
2. 우선순위 높은 것부터 처리하며, 이미 소비된 보석은 하위 매치에서 제외
3. 스페셜 보석은 스왑된 보석 위치에 생성 (스왑 없는 캐스케이드 매치 시 매치 중앙에 생성)

### §2.3 스페셜 보석 효과

| 보석 | 활성화 | 효과 |
|------|--------|------|
| **줄 파괴 (Line)** | 매칭 또는 스왑 참여 | 가로 또는 세로 한 줄 전체 제거 (화살표 방향) |
| **폭탄 (Bomb)** | 매칭 또는 스왑 참여 | 3×3 범위 폭발 제거 |
| **무지개 (Rainbow)** | 아무 보석과 스왑 | 스왑 대상과 같은 색의 보석을 보드 전체에서 모두 제거 |

### §2.4 스페셜 + 스페셜 조합

| 조합 | 효과 |
|------|------|
| 줄 파괴 + 줄 파괴 | 가로+세로 십자 2줄 동시 제거 |
| 줄 파괴 + 폭탄 | 가로+세로 각 3줄 (5×5 십자) 제거 |
| 폭탄 + 폭탄 | 5×5 범위 대폭발 |
| 무지개 + 줄 파괴 | 랜덤 1색 전체를 줄 파괴로 변환 후 전부 발동 |
| 무지개 + 폭탄 | 랜덤 1색 전체를 폭탄으로 변환 후 전부 발동 |
| 무지개 + 무지개 | 보드 전체 제거 (Ultimate Clear) |

### §2.5 장애물

| 장애물 | HP | 제거 방법 | 레벨 도입 |
|--------|-----|----------|-----------|
| **얼음 (Ice)** | 1~2겹 | 인접 매치 1회당 1겹 제거 | Lv 3+ |
| **나무 상자 (Crate)** | 2 | 인접 매치 2회로 파괴 | Lv 6+ |
| **체인 보석 (Chain)** | 1 | 해당 보석 매치 시 체인 해제 + 보석 제거 | Lv 9+ |

### §2.6 레벨 목표 유형

| 목표 유형 | 설명 | 예시 |
|-----------|------|------|
| **점수 달성** | 목표 점수 도달 | "3500점 이상" |
| **보석 수집** | 특정 색 보석 N개 매칭 | "루비 15개 + 사파이어 10개" |
| **장애물 제거** | 장애물 전부 파괴 | "얼음 8개 깨기" |
| **복합 목표** | 위 조합 | "나무 상자 4개 + 점수 2000점" |

## §3. 조작 방법

### §3.1 마우스 조작
- **클릭 스왑**: 보석 A 클릭 → 인접 보석 B 클릭 → 스왑 시도
- **드래그 스왑**: 보석 위에서 마우스 다운 → 인접 방향으로 드래그 → 마우스 업 시 스왑
- **부스터 사용**: 하단 부스터 버튼 클릭 → 대상 보석/영역 클릭

### §3.2 키보드 조작
- **방향키 (↑↓←→)**: 그리드 내 커서 이동
- **Space**: 보석 선택/선택 해제, 두 번째 Space로 스왑 실행
- **Enter**: 부스터 메뉴 토글
- **Escape**: 일시정지 메뉴
- **1/2/3**: 부스터 1/2/3번 직접 활성화

### §3.3 터치 조작
- **탭 스왑**: 보석 탭 → 인접 보석 탭
- **스와이프 스왑**: 보석 위에서 터치 → 인접 방향으로 스와이프
- **부스터**: 부스터 버튼 탭 → 대상 탭
- **터치 타겟**: 모든 버튼/인터랙티브 요소 최소 48×48px (F8 해결)
  - 렌더링 함수에서 `Math.max(MIN_TOUCH, actualSize)` 강제 적용

## §5. 핵심 게임 루프 (상태 머신)

### §5.1 게임 상태 (7 상태)

```
TITLE → CASTLE_MAP → LEVEL_INTRO → PLAY → LEVEL_COMPLETE → LEVEL_FAIL → PAUSE
```

| 상태 | 설명 |
|------|------|
| **TITLE** | 타이틀 화면, 시작 버튼 |
| **CASTLE_MAP** | 캐슬 맵 + 레벨 선택 |
| **LEVEL_INTRO** | 레벨 목표 표시 (1.5초 tween) |
| **PLAY** | 매치3 플레이 (서브 상태: IDLE, SWAP, MATCH, CASCADE, SETTLE, CHECK_GOAL) |
| **LEVEL_COMPLETE** | 별 평가 + 결과 화면 |
| **LEVEL_FAIL** | 실패 + 재시도 |
| **PAUSE** | 일시정지 오버레이 |

### §5.1.1 PLAY 서브 상태

```
IDLE → SWAP → MATCH_CHECK → MATCH_RESOLVE → CASCADE → SETTLE → CHECK_GOAL → IDLE (또는 COMPLETE/FAIL)
```

| 서브 상태 | 설명 | 가드 플래그 |
|-----------|------|-------------|
| **IDLE** | 플레이어 입력 대기 | — |
| **SWAP** | 보석 스왑 tween (300ms) | `swapLocked = true` |
| **MATCH_CHECK** | 매치 검출 (우선순위 순) | — |
| **MATCH_RESOLVE** | 매칭 보석 제거 tween + 점수 팝업 (400ms) | `resolving = true` |
| **CASCADE** | 낙하 + 리필 tween (350ms/칸) | `cascadeInProgress = true` (F5) |
| **SETTLE** | 재매칭 검사 → 매치 있으면 MATCH_CHECK로, 없으면 CHECK_GOAL | — |
| **CHECK_GOAL** | 목표 달성 확인 / 턴 소진 확인 | `goalChecking = true` |

**유효하지 않은 스왑 처리**: SWAP 후 MATCH_CHECK에서 매치 없음 → 보석 원위치 tween (300ms) → IDLE

### §5.3 TRANSITION_TABLE (허용 전환 정의 — F12 해결)

```javascript
const TRANSITION_TABLE = {
  TITLE:          ['CASTLE_MAP'],
  CASTLE_MAP:     ['LEVEL_INTRO', 'TITLE'],
  LEVEL_INTRO:    ['PLAY'],
  PLAY:           ['LEVEL_COMPLETE', 'LEVEL_FAIL', 'PAUSE'],
  LEVEL_COMPLETE: ['CASTLE_MAP'],
  LEVEL_FAIL:     ['CASTLE_MAP', 'PLAY'], // 재시도 = PLAY
  PAUSE:          ['PLAY', 'CASTLE_MAP']   // 재개 또는 나가기
};

function beginTransition(from, to) {
  if (!TRANSITION_TABLE[from]?.includes(to)) {
    console.warn(`Invalid transition: ${from} → ${to}`);
    return false;
  }
  // tween fadeOut → enterState(to) → tween fadeIn
}
```

### §10.2 초기 보드 검증 알고리즘 (F10 해결)

```
function generateValidBoard(level):
  for attempt in 1..100:
    board = fillRandom(level.colors)
    placeObstacles(board, level.obstacles)

    // 1. 초기 3매치 제거
    while hasMatches(board):
      for each match in findMatches(board):
        board[match.center] = randomColor(level.colors, excludeMatch=true)

    // 2. 유효 이동 존재 확인
    if countValidMoves(board) >= 1:
      return board

  // 100회 실패 시 전체 재생성
  return generateValidBoard(level)
```

### §11.2 낙하 보간 (Tween)

```javascript
// 보석 낙하: easeOutBounce로 자연스러운 바운스
function easeOutBounce(t) {
  if (t < 1/2.75) return 7.5625 * t * t;
  else if (t < 2/2.75) return 7.5625 * (t -= 1.5/2.75) * t + 0.75;
  else if (t < 2.5/2.75) return 7.5625 * (t -= 2.25/2.75) * t + 0.9375;
  else return 7.5625 * (t -= 2.625/2.75) * t + 0.984375;
}

// 보석 스왑: easeInOutQuad로 부드러운 교환
function easeInOutQuad(t) {
  return t < 0.5 ? 2 * t * t : 1 - Math.pow(-2 * t + 2, 2) / 2;
}
```
