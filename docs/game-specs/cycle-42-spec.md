## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
**듄 크로니클(Dune Chronicle)**은 고대 사막 왕국 '아크나르'의 멸망한 문명을 탐험하며, 매치-3 퍼즐 전투로 사막의 수호자와 저주받은 존재들을 격퇴하고, 5조각으로 흩어진 '예언의 서(Codex Prophecy)'를 복원하는 퍼즐 로그라이트 어드벤처다.

### 1.2 핵심 재미 요소
1. **직관적 전투**: 매치-3는 모든 연령대가 즉시 이해하는 메카닉. 스왑 한 번으로 공격/방어/힐이 발동되는 즉각적 피드백.
2. **전략적 깊이**: 4원소(불🔥/물💧/바람🌪️/모래⏳) 조합, 보드 관리(폭탄 타일 위치 선정), 유물 시너지 빌드.
3. **로그라이트 리플레이**: SeededRNG 기반 유적 배치, 유물 드롭, 적 조합이 매 런마다 변화. 3축 업그레이드로 다중 전략 경로.
4. **내러티브 몰입**: 사막 유목민 후예 '자라(Zara)'가 고대 왕국의 비밀을 밝혀가는 스토리. 구역마다 예언의 서 1조각 획득.
5. **캐주얼 접근성**: 터치/클릭 스왑, 자동 힌트, 턴제이므로 시간 압박 없음. casual 장르 본연의 편안함 유지.

### 1.3 레퍼런스
- **Puzzle Quest** — 매치-3 + RPG 전투의 원조. 젬 매칭이 마나 생성 → 스킬 발동.
- **Slay the Spire** — 로그라이트 선택지 구조, 유물 시너지 빌드.
- **Titanium Court** — 2026 매치-3×로그라이트 트렌드의 대표작.
- **Monument Valley** — 고대 문명 건축 비주얼의 미학적 레퍼런스.

## §2. 게임 규칙 및 목표

### 2.1 최종 목표
5개 사막 구역을 탐험하여 예언의 서 5조각을 수집하고, 미라 파라오를 물리쳐 고대 왕국 아크나르를 복원한다.

### 2.2 구역 구성 (MVP Phase 1: 구역 1~3)

| 구역 | 이름 | 테마 | 스테이지 수 | 보스 |
|------|------|------|------------|------|
| 1 | 오아시스 마을 | 튜토리얼+기본 | 3 | — |
| 2 | 모래폭풍 지대 | 시야 제한+바람 원소 | 3 | 전갈 여왕 |
| 3 | 고대 피라미드 | 함정+퍼즐 타일 | 3 | 모래 골렘 |
| 4 | 용암 협곡 (Phase 2) | 불 원소+파괴 타일 | 3 | — |
| 5 | 미라 왕의 무덤 (Phase 2) | 저주+부활 | 3 | 미라 파라오 |
| H1 | 사막의 신기루 미궁 (Phase 2) | 히든 | 1 | — |
| H2 | 하늘의 정원 (Phase 2) | 히든 | 1 | — |

### 2.3 전투 규칙
- **6×6 보드**, 4원소 타일 + 특수 타일(폭탄, 저주, 석화)
- **턴제**: 플레이어 스왑 1회 → 매치 처리 → 캐스케이드 → 적 턴 (적 공격/특수 능력)
- **매치 효과**:
  - 🔥 불(Fire): 공격 (적 HP 감소)
  - 💧 물(Water): 방어 (쉴드 생성)
  - 🌪️ 바람(Wind): 특수 (보드 셔플, 타일 이동, 콤보 보너스)
  - ⏳ 모래(Sand): 힐 (플레이어 HP 회복)
- **매치 등급**: 3매치=기본(1×), 4매치=범위(1.5×+폭탄타일 생성), 5매치=전체(2×+전체 공격/방어)
- **캐스케이드**: 매치 후 빈 칸에 타일 낙하 → 추가 매치 시 콤보 배율 +0.5× per cascade

### 2.4 승리/패배 조건
- **스테이지 승리**: 적 HP ≤ 0
- **스테이지 패배**: 플레이어 HP ≤ 0 → 획득 자원 50% 보존, 스테이지 재도전 가능
- **보스 승리**: 보스 HP ≤ 0 → 예언의 서 조각 획득 + 에픽 유물 선택지
- **런 종료**: 모든 구역 클리어 또는 3회 연속 패배

- **Canvas 해상도**: 풀스크린 + `devicePixelRatio` + 동적 리사이즈
- **에셋**: Gemini API PNG 에셋 + manifest.json 기반 동적 로드 (F1, F22)

### 4.3 아키텍처 패턴 — EventBus (F28 해결)

monkey-patch 확장 대신 **이벤트 버스 패턴**으로 시스템 간 통신:

```javascript
// EventBus 패턴 (monkey-patch 대체)
const bus = {
  _handlers: {},
  on(event, fn) { (this._handlers[event] ||= []).push(fn); },
  off(event, fn) { /* 핸들러 제거 */ },
  emit(event, data) { (this._handlers[event] || []).forEach(fn => fn(data)); }
};

// 사용 예시 — 시스템 간 결합도 0
bus.on('match', ({ element, count }) => { /* 전투 시스템 */ });
bus.on('match', ({ element, count }) => { /* 사운드 시스템 */ });
bus.on('match', ({ element, count }) => { /* 파티클 시스템 */ });
```

**검증**: `grep "origFn\|_orig\|monkey" index.html` → 0건

## §6. 상태 머신

### 6.1 TRANSITION_TABLE (F6, F21)

```javascript
const TRANSITION_TABLE = {
  TITLE:       { targets: ['MAP'],                    priority: 0 },
  MAP:         { targets: ['PRE_BATTLE', 'UPGRADE', 'BOSS', 'TITLE'], priority: 1 },
  PRE_BATTLE:  { targets: ['BATTLE'],                 priority: 2 },
  BATTLE:      { targets: ['POST_BATTLE', 'GAMEOVER'],priority: 3 },
  POST_BATTLE: { targets: ['MAP', 'UPGRADE'],         priority: 2 },
  UPGRADE:     { targets: ['MAP'],                    priority: 1 },
  BOSS:        { targets: ['POST_BATTLE', 'GAMEOVER'],priority: 4 },
  GAMEOVER:    { targets: ['TITLE'],                  priority: 10 }, // 최고 우선순위
  VICTORY:     { targets: ['TITLE'],                  priority: 10 },
  PAUSED:      { targets: ['*_PREV'],                 priority: 5 },
};

// beginTransition은 priority 비교 — 높은 priority가 낮은 것을 차단하지 않음 (F29)
function beginTransition(from, to) {
  if (from === to) return false;
  const entry = TRANSITION_TABLE[from];
  if (!entry || !entry.targets.includes(to)) {
    // GAMEOVER는 어디서든 전환 가능 (priority 10)
    if (to === 'GAMEOVER' || to === 'VICTORY') { /* 허용 */ }
    else return false;
  }
  if (_transitioning) return false;
  _transitioning = true;
  // fade-out → enterState(to) → fade-in → _transitioning = false
}
```

```
검출 순서 (높은 우선순위부터):
1. 5매치 (직선 5연속) → "사막 폭풍" 전체 공격
```

## §10. 탐험 시스템 (맵)

### 10.1 맵 구조
각 구역은 노드 기반 맵 (Slay the Spire 방식):
```
[시작] → [전투] → [이벤트/전투] → [상점/전투] → [보스]
                ↘ [이벤트] → [전투] ↗
```

노드 유형:
- ⚔️ **전투**: 일반 적과 매치-3 전투
- 🎲 **이벤트**: 랜덤 이벤트 (보물 발견, NPC 만남, 함정)
- 🏪 **상점**: 유물 구매 / HP 회복 (자원 소모)
- 🏛️ **보스**: 구역 보스전

### 10.2 프로시저럴 맵 생성
```
1. SeededRNG로 노드 3~4개 행 × 2~3열 생성
2. 시작→끝 경로 최소 2개 보장 (BFS 검증)
3. 노드 유형 분배: 전투 50%, 이벤트 25%, 상점 15%, 보스 10%(마지막 행 고정)
4. 인접 노드 연결 (최소 1개 다음 행 노드와 연결)
```

### 10.3 날씨 시스템 (Phase 1: 2종, Phase 2: 4종)

| 날씨 | 효과 | 적용 구역 |
|------|------|----------|
