# 유령 탐정 (Spectral Sleuth) — 사이클 #32 게임 기획서

> **1페이지 요약**: 살해당한 탐정이 유령이 되어 초자연적 능력(투시·감응·시간역행)으로 보이지 않는 단서를 발견하고, 논리 퍼즐로 범인을 추리하는 **미스터리 퍼즐 어드벤처**. 5구역×3사건 = 15 기본 스테이지 + 히든 미제 사건 = **총 16 스테이지**. 구역 보스(용의자 대결) 5체 + 히든 보스 "그림자 조종자" = **6체 보스전**. 유령 능력 트리 3계열(투시/감응/시간역행) × 5단계, 증거 분석 도구 8종, 도감 시스템 40+항목. 단서 배치 랜덤화 + 다중 해결 경로 + 용의자 행동 변형으로 높은 리플레이 가치. **puzzle+casual 조합을 2→3개로 강화**, 플랫폼 최초 탐정 테마 도입. 8사이클 연속 서로 다른 장르 조합 달성(#25~#32).

> **MVP 경계**: Phase 1(핵심 루프: 탐색→단서수집→추리→대질→보스, 구역 1~2 + 보스 2체 + 기본 유령 능력 3종 + 도구 4종 + 도감 기본) → Phase 2(구역 3~5 + 보스 3체 + 히든 보스 + 전체 내러티브 + 능력 진화 + 도구 4종 추가). **Phase 1만으로도 완전한 게임 경험 제공 필수.**

## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
유령 탐정(Spectral Sleuth)은 억울하게 살해당한 명탐정 "루시안 그레이(Lucian Grey)"가 유령이 되어 자신의 죽음의 진실과 도시를 뒤흔드는 초자연 범죄 연쇄를 파헤치는 **미스터리 퍼즐 어드벤처**다. 플레이어는 투시(Clairvoyance)·감응(Empathy)·시간역행(Rewind) 3가지 유령 능력으로 산 자들이 볼 수 없는 단서를 발견하고, 증거를 조합하여 진범을 추리한다.

핵심 차별점은 "유령 능력 탐색 × 증거 조합 퍼즐 × 대질 논쟁 보스전"의 3축 시스템이다. 전투가 아닌 **추리와 논증**으로 보스를 이기는 비전투 보스전은 플랫폼 내 혁신적 경험이다.

### 1.2 핵심 재미 3축
1. **유령 능력 탐색 (Spectral Investigation)**: 투시로 숨겨진 지문/혈흔 발견, 감응으로 잔류 감정 읽기, 시간역행으로 과거 장면 재현. 능력 조합으로만 접근 가능한 비밀 단서 존재.
2. **증거 조합 퍼즐 (Evidence Puzzle)**: 수집한 단서 3~5개를 논리적으로 연결하여 "추리 체인"을 완성. 동일 사건에 대해 다중 해결 경로(최소 2개)가 존재하여 리플레이 가치 보장.
3. **대질 논쟁 보스전 (Confrontation Boss)**: 용의자의 알리바이/증언에서 모순을 찾아 증거를 제시하는 턴제 논쟁. Phoenix Wright식 대질의 캐주얼 적응. 오답 시 "신뢰도(Credibility)" 감소 — 0이 되면 사건 실패.

### 1.3 스토리 개요
- **배경**: 1920년대 아르데코 양식의 항구 도시 "미스트헤이븐(Misthaven)"
- **도입**: 명탐정 루시안 그레이가 자택에서 피살. 유령으로 깨어나 자신의 살인범을 추적.
- **전개**: 5개 구역 각각에서 관련 사건을 해결하며, 자신의 죽음과 연결된 거대한 음모 발견.
- **결말**: 히든 보스 "그림자 조종자"가 도시의 모든 범죄 배후. 최종 대질로 진실 폭로.
- **환경 스토리텔링**: 대화 3줄 이내 + 배경 연출(유령 시야에서만 보이는 과거 잔상)로 텍스트 최소화하면서 깊이 유지. [F82 반영]

## §2. 게임 규칙 및 목표

### 2.1 메인 게임 루프
```
[탐색 모드] → 유령 능력으로 현장 조사
     ↓
[단서 수집] → 증거 인벤토리에 추가
     ↓
[증거 조합 퍼즐] → 3~5개 단서로 추리 체인 완성
     ↓
[대질 보스전] → 용의자 증언의 모순에 증거 제시
     ↓
[사건 해결] → 보상(능력 포인트, 도구, 도감 해금)
     ↓
[능력/도구 업그레이드] → 다음 사건으로
```

### 2.2 게임 목표
- **즉각 목표**: 현재 사건의 범인을 추리하여 대질에서 증명
- **구역 목표**: 구역 3사건 모두 해결 → 구역 보스(핵심 용의자) 대질 해금
- **최종 목표**: 5구역 모두 해결 + 히든 사건 발견 → 자신의 죽음의 진실 규명
- **완전 클리어**: 도감 100% + 모든 대체 해결 경로 발견

### 2.3 핵심 자원
| 자원 | 설명 | 획득 | 소모 |
|------|------|------|------|
| 에테르(Ether) | 유령 능력 사용 비용 | 단서 발견 +5, 사건 해결 +20 | 투시 -3, 감응 -5, 시간역행 -8 |
| 신뢰도(Credibility) | 대질 보스전 HP | 사건 시작 시 100 | 오답 -15~25, 힌트 사용 -10 |
| 영혼 파편(Soul Fragment) | 영구 업그레이드 화폐 | 보스 클리어 +3, 히든 단서 +1 | 능력 업그레이드 1~5개 |
| 도감 포인트(Archive Point) | 도감 해금 보조 | 신규 단서 발견 +1 | 자동 소비 |

### 2.4 유령 능력 시스템
| 능력 | Lv1 효과 | Lv3 효과 | Lv5 (최종) 효과 | 에테르 비용 |
|------|----------|----------|-----------------|------------|
| 투시 (Clairvoyance) | 숨겨진 지문 발견 | 벽 너머 물체 감지 | 봉인된 문서 해독 | 3 |
| 감응 (Empathy) | 잔류 감정(기쁨/분노/공포) 읽기 | 과거 대화 편린 청취 | 용의자 거짓말 자동 탐지 | 5 |
| 시간역행 (Rewind) | 3초 전 장면 재현 | 30초 전 재현 + 오브젝트 이동 추적 | 1시간 전 완전 재현 | 8 |

### 5.3 10 REGION 코드 구조 [F87]

| # | REGION | 책임 | export 함수 목록 |
|---|--------|------|-----------------|
| 1 | CONFIG | 상수, 밸런스 수치, 다국어 텍스트 | `C`, `LANG`, `BALANCE` |
| 2 | ENGINE | TweenManager, ObjectPool, SeededRNG, SoundManager | `tw`, `pool`, `rng`, `snd` |
| 3 | ENTITY | 루시안, 용의자, NPC, 단서 오브젝트 | `createLucian()`, `createSuspect()`, `createClue()` |
| 4 | DRAW | 모든 순수 드로잉 함수 | `drawLucian()`, `drawClue()`, `drawBuilding()`, `drawSuspect()`, `drawBoss()`, `drawWeather()`, `drawHUD()` |
| 5 | ABILITY | 유령 능력 시스템 (투시/감응/시간역행) | `useClairvoyance()`, `useEmpathy()`, `useRewind()`, `canUseAbility()` |
| 6 | PUZZLE | 증거 조합 퍼즐 로직 | `checkChain()`, `getValidChains()`, `evaluateDeduction()` |
| 7 | CONFRONT | 대질 보스전 시스템 | `startConfrontation()`, `presentEvidence()`, `checkContradiction()`, `getBossDialogue()` |
| 8 | STATE | 상태 머신, 전환 가드 | `beginTransition()`, `ESCAPE_ALLOWED`, `RESTART_ALLOWED`, `ACTIVE_SYSTEMS` |
| 9 | SAVE | 로컬 스토리지 저장/불러오기 | `saveProgress()`, `loadProgress()`, `resetProgress()` |
| 10 | MAIN | 초기화, 메인 루프, 이벤트 바인딩 | `init()`, `loop()`, `resize()` |

**의존 방향**: CONFIG → ENGINE → ENTITY → DRAW → ABILITY → PUZZLE → CONFRONT → STATE → SAVE → MAIN (단방향, 순환 참조 0건)

## §7. 사건 및 구역 설계

### 7.1 구역 구조: 5구역 × 3사건 + 히든

| 구역 | 테마 | 사건 1 | 사건 2 | 사건 3 | 보스 (용의자) |
|------|------|--------|--------|--------|-------------|
| 1. 항구 | 밀수 | 선원 실종 | 화물 변조 | 밀수 조직 거래 | "독사" 빅터 |
| 2. 뒷골목 | 사기 | 위조 지폐 | 보험 사기 | 갱단 자금 세탁 | "거울" 마리안 |
| 3. 부유층 | 배신 | 유산 분쟁 | 이중 정체 | 비밀 결사 의식 | "백작" 레오폴드 |
| 4. 창고 지구 | 은폐 | 증거 인멸 | 증인 실종 | 경찰 매수 | "철면" 감독관 |
| 5. 시청 | 음모 | 시장 협박 | 선거 조작 | 도시 전복 계획 | "인형사" 시의원 |
| 히든 | 진실 | 루시안의 죽음 | — | — | "그림자 조종자" |

### 7.2 사건 구조 상세
각 사건은 아래 구조를 따른다:
1. **브리핑** (CASE_BRIEFING): 사건 개요, 피해자/관련자 소개 (대화 3줄 이내)
2. **탐색** (EXPLORATION): 3~5개 단서를 유령 능력으로 발견
   - 필수 단서: 3개 (정답 체인에 반드시 포함)
   - 선택 단서: 1~2개 (대체 체인용 또는 도감 포인트)
3. **추리** (PUZZLE_BOARD): 단서를 연결하여 추리 체인 완성
4. **대질** (CONFRONTATION): 용의자와 증언 대결

### 7.3 구역별 환경 위험 [F84]
| 구역 | 환경 위험 | 효과 | 수치 | 대응 |
|------|----------|------|------|------|
| 1. 항구 | 짙은 안개 | 시야 범위 감소 | 시야 반경 60% → 투시 사용 시 100% | 투시 Lv2+ |
| 2. 뒷골목 | 혼령 잔재 | 에테르 자연 감소 | -1/3초 | 감응 Lv2+(감소 무효화) |
| 3. 부유층 | 마법 결계 | 능력 쿨다운 증가 | 쿨다운 ×1.5 | 시간역행 Lv3+ (쿨다운 감소) |
| 4. 창고 | 봉인 구역 | 특정 영역 진입 불가 | 투시 Lv3+ 필요 | 투시 Lv3+ |
| 5. 시청 | 에테르 교란 | 감응 정확도 저하 | 거짓 감정 30% 혼입 | 감응 Lv4+ (필터링) |
| 히든 | 시간 왜곡 | 시간역행 범위 제한 | 최대 10초 재현 | 시간역행 Lv5 (해제) |

### 7.4 단서 시스템
- **단서 유형**: 물리적(지문/무기/편지), 감정적(잔류 감정/공포 흔적), 시간적(과거 장면/이동 경로)
- **발견 조건**: 유령 능력 유형 + 레벨 요구사항
- **랜덤화**: SeededRNG로 단서 배치 위치 변형 (3~5곳 중 랜덤 선택) [F18]
- **다중 해결 경로**: 각 사건 최소 2가지 추리 체인 (필수 단서 3 + 선택 단서 조합)

### 7.5 대질 보스전 상세
- **구조**: 3~5라운드 턴제. 라운드마다 용의자 증언 → 모순 탐지 → 증거 제시.
- **모순 유형**: 시간 불일치, 알리바이 충돌, 증거와 증언 불일치, 동기 모순
- **정답 시**: 용의자 동요 → 신규 증언 → 다음 라운드
- **오답 시**: 신뢰도 -15(일반) / -25(핵심 증거 오제시). 신뢰도 0 = 사건 실패 → RESTART_ALLOWED
- **보스 보상**: 영혼 파편 3개 + 도구 1개 + 도감 해금 (bossRewardGiven 플래그로 1회 보장) [F17]
- **히든 보스 "그림자 조종자"**: 7라운드 대질. 이전 5보스의 증거를 모두 활용해야 하는 통합 추리.

| 탐정 (Medium) | 100 | +1/초 | -15~25 | 사건당 3회(-10) | 능력 사용 시만 |
| 전설 탐정 (Hard) | 75 | +0.5/초 | -20~30 | 사건당 1회(-15) | 정확한 능력+위치 |

### 8.2 DDA 동적 난이도 보정 [F88]
3변수 산식:
```
난이도 점수 = (필수단서수 × 3) + (시간제한 보정) + (오답허용 역수 × 5)
```

| DDA 조건 | 보정 |
|----------|------|
| 연속 2사건 S랭크 | 환경 위험 수치 ×1.2, 선택 단서 -1 |
| 연속 2사건 실패 | 힌트 +1, 오답 페널티 ×0.8 |
| 대질 1라운드 내 정답 | 다음 보스 라운드 +1 |
| 대질 신뢰도 30% 이하 | 보너스 힌트 제공 |

산식의 가정: 플레이어가 힌트를 50% 확률로 사용 [Cycle 24 교훈]. 가정이 틀릴 경우 DDA가 ±1단계 추가 보정.

### 8.3 극단 케이스 검증 [F88]
| 시나리오 | 예상 결과 | 폴백 |
|----------|----------|------|
| 모든 힌트 소진 + 신뢰도 30% | DDA 보너스 힌트 발동 | 사건 재시작 허용 |

## §10. 증거 조합 퍼즐 상세 설계

### 10.1 추리 보드 UI
- 수집된 증거 카드가 좌측에 나열
- 우측에 "추리 체인" 슬롯 3~5개
- 드래그 앤 드롭으로 카드를 슬롯에 배치
- 카드 간 연결선이 자동 생성 (논리적 관계 표시)
- "검증" 버튼으로 체인 평가

### 10.2 추리 체인 검증 알고리즘
```
1. 슬롯에 배치된 증거 카드 수 확인 (최소 3개)
2. 각 카드 쌍의 논리적 연결 유효성 검사:
   - 시간 순서 정합 (사건 발생 시간 순)
   - 인물 연관성 (동일 인물 또는 관계자)
   - 장소 일관성 (같은 구역 또는 이동 가능 거리)
3. 완성된 체인이 사전 정의된 유효 체인 목록과 매칭
4. 매칭 시: 추리 성공 → DEDUCTION_RESULT → CONFRONTATION_INTRO
5. 미매칭 시: "추리가 완성되지 않았습니다" 피드백 + 잘못된 연결 하이라이트
```

**다중 해결 경로 보장**: 각 사건에 최소 2개의 유효 체인을 정의. 선택 단서 포함 체인은 보너스 보상.

### 10.3 BFS/DFS 도달 가능성 검증 [Cycle 23 교훈]
- 탐색 맵 생성 시, 모든 필수 단서 위치까지 **이동 경로가 존재하는지 BFS로 검증**
- 능력 레벨 요구사항과 현재 능력 레벨 비교 → 접근 불가 단서는 "잠김" 표시
- 유효 체인 구성에 필요한 필수 단서가 모두 접근 가능한지 사전 확인

## §11. 점수 시스템

### 11.1 사건별 점수 [F8: 판정 먼저, 저장 나중에]
```
사건 점수 = 기본점수 + 시간 보너스 + 정확도 보너스 + 탐색 보너스

기본점수: 1000 × 구역번호
시간 보너스: max(0, (제한시간 - 소요시간) × 10)
정확도 보너스: (잔여 신뢰도 / 최대 신뢰도) × 500
탐색 보너스: (발견 단서 / 전체 단서) × 300
```

**판정 순서**: 점수 계산 → 랭크 판정 → 최고 기록 비교 → 저장 [F8]

### 11.2 랭크 시스템
| 랭크 | 조건 | 보상 배율 |
|------|------|----------|
| S | 오답 0회 + 전체 단서 발견 + 시간 50% 이내 | ×2.0 |
| A | 오답 1회 이하 + 필수 단서 전발견 | ×1.5 |
| B | 사건 해결 + 신뢰도 50% 이상 | ×1.0 |
| C | 사건 해결 | ×0.7 |

### 11.3 누적 점수
- 총점 = Σ(사건별 점수 × 랭크 배율)
- 로컬 스토리지 저장 (최고 총점 + 사건별 최고 랭크)
