# 글리프 래버린스 (Glyph Labyrinth) — 사이클 #25 게임 기획서

> **1페이지 요약**: 고대 유적의 상호 연결된 미궁을 탐험하며 글리프(마법 문양)를 수집·조합하여 새로운 능력을 해금하고, 환경 퍼즐과 보스전을 공략하는 퍼즐 메트로이드바니아. 5개 바이옴(화염 신전, 빙결 동굴, 고대 숲, 심연 폐허, 천공 탑) × 4개 방 + 5개 보스방 + 히든 바이옴 "시간의 균열" = 총 26개 방. 글리프 업그레이드 트리, 비밀 조합 경로, SeededRNG 프로시저럴 방 배치로 매 플레이가 다른 경험. **arcade+puzzle 조합을 1→2개로 강화**한다.

> **MVP 경계**: Phase 1(핵심 루프: 탐험→퍼즐→능력 해금→보스전, 바이옴 1~2 + 보스 2체) + Phase 2(바이옴 3~5 + 보스 3체 + 히든 바이옴 + 전체 내러티브) 순차 구현. Phase 1만으로도 완전한 게임 경험 제공 필수.

## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
글리프 래버린스는 잊혀진 고대 유적의 미궁을 탐험하는 퍼즐 메트로이드바니아다. 고대 문명 "아르카나"가 남긴 다섯 신전에서 글리프를 수집하고, 이를 조합하여 새로운 능력을 해금하며, 이전에 접근할 수 없던 경로를 열어나간다. 보스전은 단순 액션이 아닌 환경 퍼즐과 전투의 결합이며, 비밀 글리프 조합으로 해금되는 히든 바이옴 "시간의 균열"에서는 모든 능력이 동시에 필요한 최종 도전이 기다린다.

### 1.2 핵심 재미 3축
1. **탐험과 발견의 쾌감**: 상호 연결된 26개 방을 오가며, 새 능력으로 이전에 지나친 곳의 비밀 경로를 발견하는 "아하!" 모먼트. 메트로이드바니아 장르 특유의 역방향 탐험 재미.
2. **글리프 조합 퍼즐**: 5종 기본 글리프 + 15종 조합 글리프로 총 20개 능력. 같은 퍼즐도 다른 글리프 조합으로 해결할 수 있는 다중 풀이 경로. "이 조합이면 저기 갈 수 있지 않을까?"의 실험 욕구.
3. **환경 퍼즐 보스전**: 각 보스는 단순히 때려잡는 것이 아니라, 바이옴 환경을 활용하고 글리프 능력을 창의적으로 사용해야 약점이 노출된다. 패턴 관찰 → 환경 활용 → 타격의 사이클.

### 1.3 스토리/내러티브
- **배경**: 고대 문명 "아르카나"는 글리프에 지식과 힘을 새겨 문명을 일으켰으나, 금지된 글리프 "크로노스"의 남용으로 시간이 뒤틀리며 멸망했다. 유적에 남은 글리프들은 아르카나의 기억과 경고를 품고 있다.
- **목표**: 5개 신전의 수호자(보스)를 극복하고 핵심 글리프를 모아 크로노스의 폭주를 봉인한다.
- **스토리 전달**: 방마다 벽면에 새겨진 "아르카나 기록"(Canvas 텍스트 연출)으로 단편적인 이야기를 전달. 글리프 조합 시 짧은 기억 플래시백(화면 필터 전환 + 텍스트 3줄). 보스 격파 후 컷신(Canvas 연출 3~5초).
- **엔딩**: 봉인 완료 후 미궁 밖으로 탈출하는 결말. 히든 바이옴 클리어 시 "아르카나의 진실" — 크로노스 글리프가 실은 시간을 되돌려 문명의 실수를 바로잡으려는 마지막 희망이었다는 반전 엔딩.

### 1.4 장르 강화 효과
- **arcade + puzzle = 1→2개** (6개 최소 빈도 조합 중 하나를 강화)
- puzzle 전체: 7→8개
- 기존 arcade+puzzle(neon-hex-drop)과 차별화: 낙하 퍼즐 vs 메트로이드바니아 탐험 퍼즐 — 완전히 다른 플레이 경험

## §2. 게임 규칙 및 목표

### 2.1 기본 규칙
- 탑다운 뷰의 2D 미궁 탐험. 한 화면 = 한 방(Room)
- 방 이동 시 화면 전환(tween 기반 슬라이드, 300ms)
- 적과 접촉 또는 환경 함정에 피격 시 HP 감소
- HP 0 → 게임 오버 (수집한 글리프·해금된 능력은 영구 보존)
- 각 바이옴의 보스를 격파하면 핵심 글리프 획득 + 다음 바이옴 해금

### 2.2 승리/패배 조건
- **승리**: 5개 핵심 글리프 수집 → 최종 봉인 퍼즐 완료
- **패배**: HP 0 → 마지막 세이브 포인트에서 재개 (글리프·능력 유지, 현재 방 초기화)
- **히든 엔딩**: 20개 글리프 전부 수집 + 모든 아르카나 기록 발견 → "시간의 균열" 해금

### 2.3 바이옴 구성
| 바이옴 | 방 수 | 테마 | 핵심 퍼즐 요소 | 보스 |
|--------|------|------|--------------|------|
| 1. 화염 신전 | 4+보스 | 용암, 화염 기둥, 열기파 | 화염 경로 타이밍, 용암 흐름 방향 전환 | 이그니스 (용암 골렘) |
| 2. 빙결 동굴 | 4+보스 | 빙판, 고드름, 냉기 안개 | 미끄러짐 물리 퍼즐, 얼음 반사 | 글라시아 (빙결 서펀트) |
| 3. 고대 숲 | 4+보스 | 덩굴, 포자, 생물발광 | 성장/축소 퍼즐, 덩굴 경로 활성화 | 실바누스 (고대 나무정령) |
| 4. 심연 폐허 | 4+보스 | 무중력, 어둠, 에코 | 중력 반전 퍼즐, 소리 기반 탐색 | 닉스 (심연의 그림자) |
| 5. 천공 탑 | 4+보스 | 부유석, 번개, 시간 왜곡 | 시간 정지 퍼즐, 플랫폼 타이밍 | 크로노스 (시간의 수호자) |
| 히든. 시간의 균열 | 1 | 전 바이옴 혼합 | 모든 능력 활용 종합 퍼즐 | — (퍼즐 전용) |

### 2.4 난이도 3종
| 난이도 | 플레이어HP | 적 강화 | 퍼즐 힌트 | 세이브 빈도 | 잠금 조건 |
|--------|----------|---------|----------|-----------|----------|
| 탐험가 (쉬움) | ×1.5 | ×0.6 | 힌트 글리프 표시 | 방마다 자동 | 없음 |
| 모험가 (보통) | ×1.0 | ×1.0 | 없음 | 바이옴 입구만 | 없음 |
| 전설 (어려움) | ×0.7 | ×1.4 | 없음 | 바이옴 입구만 | 바이옴 3 이상 클리어 |

- **순수 Canvas 코드 드로잉만 사용** — 외부 SVG/PNG/이미지 파일 로드 0건 (F1)

### 4.5 시각 연출
- **방 전환**: 슬라이드 + 페이드(tween 300ms). 이동 방향에 따른 슬라이드 방향
- **글리프 획득**: 화면 중앙 줌인 + 능력 설명 텍스트 + 발광 파티클 폭발(tween 2초)
- **보스 등장**: 화면 줌인(1.0→1.3, 1초) + 보스명 타이틀 텍스트 + 바이옴 색상 플래시
- **보스 페이즈 전환**: 화면 셰이크(0.5초) + 색상 변화 + 변신 파티클
- **피격**: 캐릭터 빨간 플래시(100ms) + 화면 살짝 셰이크(50ms)
- **대시**: 잔상 3개(alpha 0.6→0.3→0.1) + 모션 블러 라인
- **환경광**: 바이옴별 파티클(화염 불꽃, 눈 결정, 포자, 에코 파동, 번개 섬광) 상시 배경 레이어

## §7. 핵심 게임 루프 (초당 프레임 기준 로직 흐름)

### 7.1 메인 루프 (60fps, rAF 기반)
```
function gameLoop(timestamp) {
    const dt = Math.min(timestamp - lastTime, 50);  // dt 캡 50ms
    lastTime = timestamp;

    // 1. 입력 처리 (ACTIVE_SYSTEMS[state].input)
    processInput(dt);

    // 2. 트윈 업데이트 (항상)
    tweenManager.update(dt);

    // 3. 물리/이동 (ACTIVE_SYSTEMS[state].physics)
    updatePhysics(dt);

    // 4. 적/보스 AI (ACTIVE_SYSTEMS[state].enemy/boss)
    updateEnemies(dt);
    updateBoss(dt);

    // 5. 퍼즐 시스템 (ACTIVE_SYSTEMS[state].puzzle)
    updatePuzzles(dt);

    // 6. 충돌 판정
    checkCollisions();

    // 7. 카메라 (ACTIVE_SYSTEMS[state].camera)
    updateCamera(dt);

    // 8. 파티클 (ACTIVE_SYSTEMS[state].particle)
    updateParticles(dt);

    // 9. 세이브 체크 (ACTIVE_SYSTEMS[state].save)
    checkSavePoints();

    // 10. 렌더링 (항상)
    render();

    requestAnimationFrame(gameLoop);
}
```

### 8.2 클리어 가능성 산식 (F51)

**플레이어 DPS 산식**:
```
baseDPS = playerATK / attackInterval
effectiveDPS = baseDPS × (1 + glyphBonus) × comboMultiplier
```

**보스 클리어 시간 산식**:
```
clearTime = bossHP / (effectiveDPS × hitRate)
hitRate 가정: 탐험가 0.7, 모험가 0.5, 전설 0.4
```

**생존 시간 산식**:
```
survivalTime = playerHP / (bossATK × (1 - dodgeRate) / bossAttackInterval)
dodgeRate 가정: 탐험가 0.3, 모험가 0.4, 전설 0.5
```

**밸런스 조건**: `survivalTime ≥ clearTime × 1.3` (30% 여유)

> 가정이 틀릴 경우의 폴백: **동적 난이도 보정(DDA)** — 같은 방에서 3회 연속 사망 시 적 HP 10% 감소, 5회 연속 사망 시 20% 감소. "모험가" 기준에서만 적용 (탐험가는 이미 쉬움, 전설은 의도적으로 어려움).

### 8.3 동적 난이도 조정 (DDA)
- **3회 연속 사망** (같은 방): 적 HP ×0.9, 힌트 글리프 1개 표시
- **5회 연속 사망**: 적 HP ×0.8, 적 ATK ×0.9, 힌트 글리프 추가
- **무손상 보스 격파**: 다음 바이옴 적 HP ×1.1 (보상적 강화)
- DDA 보정값은 세이브에 저장하지 않음 (런 단위 리셋)

## §9. 보스 전투 상세

### 9.1 보스별 컨셉

#### 보스 1: 이그니스 (화염 신전)
- **형태**: 용암 골렘, 몸 곳곳에 균열에서 용암 분출
- **페이즈 1** (HP 100%~50%): 용암 덩어리 투척(3방향), 돌진 공격. 바닥 용암 웅덩이 생성
- **페이즈 2** (HP 50%~0%): 용암 분출 패턴 복잡화 + 화염 기둥 연쇄 폭발. 벽면 균열에서 냉각수 유도 가능
- **약점 공략**: 방의 냉각수 배관을 글리프로 활성화 → 이그니스에 명중 시 3초간 경직 + 2배 데미지

#### 보스 2: 글라시아 (빙결 동굴)
- **형태**: 거대한 빙결 서펀트, 몸이 방 전체를 감싸는 크기
- **페이즈 1**: 빙판 위 미끄럼 돌진, 고드름 비(랜덤 위치), 냉기 브레스(부채꼴)
- **페이즈 2**: 분체화(3개 소형 서펀트로 분리) → 하나씩 격파 후 본체 약점 노출
- **약점 공략**: 방의 열원(화염 글리프로 활성화)에 서펀트를 유인 → 빙갑 해제 시 약점 노출

#### 보스 3: 실바누스 (고대 숲)
- **형태**: 거대한 고대 나무정령, 뿌리가 방 전체에 퍼짐
- **페이즈 1**: 뿌리 공격(바닥에서 솟아오름), 포자 구름(시야 방해), 덩굴 속박
- **페이즈 2**: 이동 시작 + 뿌리 회전 공격 + 치유(포자 흡수)
- **페이즈 3** (전설 난이도만): 미니 분신 소환 + 방 전체 뿌리 웨이브
- **약점 공략**: 포자 구름을 실바누스에게 역류(바람 글리프) → 중독 + 치유 차단

#### 보스 4: 닉스 (심연 폐허)
- **형태**: 그림자로 이루어진 형체, 실체와 분신 구별 어려움
- **페이즈 1**: 그림자 분신 3체 소환 + 텔레포트, 에코 공격(원형 파동)
- **페이즈 2**: 방 조명 소등 → 에코 파동으로만 위치 파악, 실체 공격 시 대미지
- **페이즈 3**: 빛/어둠 교대 → 빛일 때만 실체 보임, 어둠일 때 분신 강화
- **약점 공략**: 에코 글리프로 실체 위치 감지 + 빛 글리프로 강제 조명 → 약점 노출

#### 보스 5: 크로노스 (천공 탑) — 최종 보스
- **형태**: 시간의 수호자, 거대한 시계 기어로 이루어진 인간형
- **페이즈 1**: 시간 감속 필드(플레이어 속도 ×0.5), 과거 공격 재현(2초 전 패턴 반복)
- **페이즈 2**: 시간 되감기(플레이어 위치를 3초 전으로 강제 이동), 기어 투사체
- **페이즈 3**: 시간 정지 폭탄(특정 영역 정지, 접촉 시 3초 무력화) + 전 페이즈 패턴 혼합
- **약점 공략**: 시간 글리프로 감속 필드 무효화 + 되감기 시 역으로 유리한 위치 이동 → 기어 노출 시 타격

### 9.2 보스 페이즈 전환 상태 다이어그램 (F22 검증된 패턴)

### 10.1 방 구조
각 방은 16×12 타일 그리드 (타일 크기: Canvas너비/16)

- **문 타일**: 다른 방으로의 연결점 (상하좌우 최대 4개)
- **특수 타일**: 함정, 글리프, 세이브 포인트, 아르카나 기록

### 10.2 프로시저럴 방 배치 + BFS 도달 검증 (F54)
SeededRNG 기반 방 배치 알고리즘:
```
1. 바이옴 템플릿에서 4개 방 구조 랜덤 선택
2. 방 간 연결 (최소 스패닝 트리 + 1~2 추가 연결)
3. 글리프/아이템 배치
4. 적 스폰 포인트 배치
5. ★ BFS 도달 가능성 검증:
   - 시작방에서 모든 방으로의 경로 존재 확인
   - 글리프 능력 순서를 고려한 경로 가능성 검증
   - 검증 실패 시 3단계로 시드 재생성 (최대 10회)
   - 10회 모두 실패 시 고정 레이아웃 폴백
6. 비밀 통로 배치 (벽 타일 중 1~2개를 숨겨진 문으로)
```

### 10.3 미니맵
- 탭(Tab) 토글 또는 HUD 우상단 고정 (60×45px)
- 방문한 방: 밝게 표시, 미방문: 어둡게
- 현재 위치: 깜박이는 점
- 보스방: 특수 아이콘(바이옴 색상 별)

```javascript
    }
};
```

### 15.2 저장/로드 규칙 (F8)
- **판정 먼저, 저장 나중**: 글리프 획득 판정 → UI 업데이트 → 점수 갱신 → localStorage 저장

## §19. 썸네일 SVG 가이드 (20KB+)

시네마틱 구도: 미궁 입구에 선 탐험가(뒷모습)가 거대한 글리프 문양이 새겨진 석문을 바라보는 장면.
- 전경: 탐험가 실루엣(금빛 윤곽선)
- 중경: 글리프 문양이 빛나는 거대 석문(시안 발광)
- 후경: 어둠 속 미궁 복도 + 바이옴별 색상 힌트(5가지 색 빛줄기)
- 분위기: 신비롭고 웅장한 탐험의 시작
- SVG viewBox: "0 0 480 360" (4:3 비율)
- 복합 filter 체인: glow + shadow 조합으로 깊이감
