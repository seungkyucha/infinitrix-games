## 1. 게임 개요 및 핵심 재미 요소

**빛의 직조자**는 빛과 그림자를 전환하여 플랫폼을 생성·소멸시키며 퍼즐을 풀어나가는 2D 퍼즐 플랫포머입니다.

### 배경 스토리
빛과 어둠이 공존하던 세계에서 그림자가 폭주하여 빛의 결정들이 산산조각났습니다. 빛의 직조자 '루미'는 흩어진 빛의 조각을 모아 세계의 균형을 되돌려야 합니다. 각 레벨은 빛과 그림자로 이루어진 이중 구조의 방이며, 광원을 조작하여 길을 열고 출구에 도달하는 것이 목표입니다.

### 핵심 재미 요소
1. **"보이는 것만 존재한다"** — Closure에서 영감받은 핵심 메카닉. 빛 영역에서만 '빛 플랫폼'이 실체화되고, 그림자 영역에서만 '그림자 플랫폼'이 실체화됩니다. 같은 공간이 빛/그림자 상태에 따라 완전히 다른 지형으로 변합니다.
2. **"아하!" 순간** — 레벨마다 빛과 그림자의 전환 타이밍, 광원 위치 조작으로 "이렇게 하면 되는구나!"라는 발견의 쾌감을 제공합니다.
3. **정밀 조작의 긴장감** — Celeste에서 영감받은 탄탄한 플랫포밍. 점프·벽타기·대시를 조합하여 좁은 통로를 통과하는 쾌감.
4. **별 수집 도전** — 각 레벨 3개 별(빛의 조각). 기본 클리어 + 숨겨진 별 수집으로 반복 플레이 유도.

### 레퍼런스
- **Closure** (빛/그림자 존재 메카닉) + **Celeste** (정밀 플랫포밍, 난이도 곡선) + **INSIDE/Limbo** (분위기, 환경 퍼즐)
- 아트: **Thomas Was Alone** (기하 캐릭터) + **Pikuniku** (플랫 벡터 스타일)

## 2. 게임 규칙 및 목표

### 2.1 기본 규칙
- 플레이어 캐릭터 '루미'는 빛의 존재로, 빛과 그림자 양쪽 플랫폼에서 이동 가능
- **빛 모드(Light Mode)**: 밝은 색상 플랫폼만 실체화 (충돌 있음). 어두운 플랫폼은 반투명 유령 상태 (통과 가능)
- **그림자 모드(Shadow Mode)**: 어두운 색상 플랫폼만 실체화. 밝은 플랫폼은 유령 상태
- **중립 플랫폼**: 모드와 무관하게 항상 실체화 (벽, 바닥 등)
- 가시·함정에 닿으면 해당 레벨 시작점 또는 최근 체크포인트로 즉시 리스폰 (사망 카운터 표시)
- 레벨 출구(빛의 문)에 도달하면 클리어

### 2.2 목표
- **기본 목표**: 모든 레벨의 출구에 도달하여 6개 월드 클리어
- **도전 목표**: 각 레벨에 숨겨진 빛의 조각(별) 3개 수집 (총 24개)
- **마스터 목표**: 최소 사망 횟수 + 최소 시간 클리어 (퍼스널 베스트 기록)

### 2.3 광원 상호작용
- **고정 광원**: 레벨에 배치된 이동 불가 광원. 반경 내 빛 모드 활성
- **이동 광원**: 플레이어가 밀거나 옮길 수 있는 광원 오브젝트
- **스위치**: 밟으면 광원 ON/OFF 토글. 활성 상태 시각적으로 명확히 표시 (빛나는 테두리 + 파티클)
- **빛 구슬(레벨 4부터)**: 플레이어가 던져서 착지 지점에 임시 광원 생성 (5초 지속). 1회 사용 후 재충전 필요
- **프리즘(레벨 5부터)**: 빛을 반사/굴절시켜 특정 방향으로 빛 영역 확장

## 3. 조작 방법

### 키보드
| 키 | 동작 |
|---|---|
| ← → (또는 A/D) | 좌우 이동 |
| ↑ (또는 W / Space) | 점프 (벽 근처에서 벽 점프) |
| Shift | 대시 (공중 1회, 착지 시 리셋) |
| E (또는 X) | 빛/그림자 모드 전환 |
| F (또는 Z) | 빛 구슬 던지기 (레벨 4부터) |
| R | 레벨 즉시 재시작 |
| ESC | 일시정지 메뉴 |

### 마우스
| 입력 | 동작 |
|---|---|
| 화면 좌/우 반쪽 클릭 | 좌우 이동 |
| 상단 클릭 | 점프 |
| 더블클릭 | 대시 |
| 우클릭 | 빛/그림자 모드 전환 |
| 빛 구슬 UI 버튼 | 빛 구슬 던지기 |

### 터치 (모바일)
| 입력 | 동작 |
|---|---|
| 좌측 가상 조이스틱 | 이동 (데드존 10px) |
| 우측 하단 A 버튼 | 점프 |
| 우측 하단 B 버튼 | 대시 |
| 우측 상단 토글 버튼 | 빛/그림자 모드 전환 (활성 상태: 빛나는 테두리 + 색상 변경) |
| 우측 상단 구슬 버튼 | 빛 구슬 던지기 |

⚠️ 모든 토글 버튼은 현재 상태를 시각적으로 명확히 표시:
- 빛 모드: 버튼 배경 황금색 + ☀️ 아이콘 + 부드러운 글로우
- 그림자 모드: 버튼 배경 남색 + 🌙 아이콘 + 미세한 그림자 효과
- 모바일 버튼 최소 크기: `Math.max(48, ...)` 가드 필수

## 4. 시각적 스타일 가이드

### 4.1 색상 팔레트

**빛 모드 팔레트:**
- 배경: `#F5F0E8` (따뜻한 크림)
- 빛 플랫폼: `#FFD93D` (골든 옐로우) / `#FFA726` (오렌지 악센트)
- 플레이어: `#FFFFFF` (순백) + `#FFD93D` (황금 글로우)
- UI 악센트: `#FF6B35` (코랄 오렌지)

**그림자 모드 팔레트:**
- 배경: `#1A1A2E` (딥 네이비)
- 그림자 플랫폼: `#533483` (딥 퍼플) / `#0F3460` (다크 블루)
- 플레이어: `#E0E0E0` (연회색) + `#B388FF` (라벤더 글로우)
- UI 악센트: `#00BCD4` (시안)

**공통:**
- 중립 플랫폼: `#546E7A` (블루그레이)
- 별(빛의 조각): `#FFD700` (빛나는 골드) + 펄스 파티클
- 위험 요소: `#FF1744` (위험 레드)

### 4.2 배경
- **원경**: 기하학적 산맥/도시 실루엣. 빛 모드에서는 밝은 그라데이션, 그림자 모드에서는 어두운 톤으로 전환
- **중경**: 기둥, 아치, 창문 등 건축 실루엣. 패럴랙스 0.5배속
- **근경**: 타일 기반 플랫폼 + 장식 파티클 (먼지/빛 입자)

### 4.3 오브젝트 형태
- **플레이어 '루미'**: 둥근 삼각형 몸체 + 큰 원형 눈 2개 + 작은 팔다리. 기하학적이지만 귀여움. 빛 모드에서 황금 글로우, 그림자 모드에서 라벤더 글로우
- **빛 플랫폼**: 깨끗한 직사각형, 둥근 모서리, 골든 옐로우 계열. 활성 시 밝은 테두리
- **그림자 플랫폼**: 같은 형태, 딥 퍼플/다크 블루 계열. 활성 시 어두운 글로우
- **유령 플랫폼 (비활성)**: 점선 윤곽선, 20% 불투명도. 통과 가능함을 시각적으로 명확히 전달
- **광원**: 원형, 방사형 광선 표현. 반경 내 빛 영역 시각화 (부드러운 원형 그라데이션)
- **스위치**: 정사각형 기반 + 레버 표현. ON=빛남, OFF=어두움
- **빛의 문 (출구)**: 큰 아치형, 빛나는 파티클 방출
- **가시/함정**: 삼각형 가시, 빨간 악센트

## 4.5. 아트 디렉션 (Art Direction)

### 아트 스타일 키워드
**flat-vector** — Flat vector art: clean geometric shapes, bold outlines, limited gradients, modern mobile-game look.

### 디자인 원칙
1. **기하학적 단순함**: 모든 오브젝트는 원, 삼각형, 사각형 기본 도형의 조합. 불필요한 디테일 배제.
2. **두꺼운 아웃라인**: 3~4px 굵기의 어두운 아웃라인으로 오브젝트 구분 명확화
3. **제한된 그라데이션**: 그라데이션은 배경과 광원 효과에만 사용. 오브젝트는 플랫 색상
4. **빛/그림자 이원 구조**: 화면이 빛 모드↔그림자 모드 전환 시 팔레트가 부드럽게(300ms) 전환. 두 팔레트 모두 동일한 기하학적 형태를 유지하되 색감만 변경
5. **가독성 최우선**: 활성 플랫폼 vs 유령 플랫폼 구분이 0.1초 내에 인지 가능해야 함

### 레퍼런스 게임
- **Thomas Was Alone** (Mike Bithell) — 기하학적 캐릭터, 미니멀 환경, 감성적 스토리텔링
- **Pikuniku** (Sectordub) — 플랫 벡터 아트, 깨끗한 도형, 밝고 대비 강한 색상

## 5. 핵심 게임 루프 (60fps 기준 로직 흐름)

### 5.1 프레임 루프
```
매 프레임 (16.67ms):
  1. Input 수집
     - 키보드/터치/마우스 입력 → 이동 벡터, 점프, 대시, 모드 전환 명령
  2. Physics Update
     - 중력 적용 (gravityY = 980 px/s²)
     - 수평 이동 (속도: 240 px/s, 가속: 1600 px/s², 감속: 2400 px/s²)
     - 점프 (초기 속도: -420 px/s, 가변 높이: 키 유지 시 중력 50% 적용)
     - 대시 (8프레임간 480 px/s 수평 이동, 중력 무시)
     - 벽 슬라이드 (벽 접촉 시 낙하 속도 30%로 감속)
     - 벽 점프 (대각선 방향 발사: vx=±300, vy=-380)
  3. 모드 상태 판정
     - 현재 빛/그림자 모드에 따라 각 플랫폼의 collider 활성/비활성 전환
     - 유령 플랫폼: 충돌 비활성 + 불투명도 20%로 전환 (150ms 트윈)
     - 활성 플랫폼: 충돌 활성 + 불투명도 100%로 전환 (150ms 트윈)
  4. Collision Detection
     - 활성 플랫폼 + 중립 플랫폼에 대해서만 충돌 판정
     - AABB 기반 타일맵 충돌 (16×16px 또는 32×32px 타일)
     - 광원 반경 판정: 원형 범위 체크
     - 별(빛의 조각) 수집 판정
     - 가시/함정 접촉 판정
     - 출구(빛의 문) 도달 판정
  5. 광원 시스템 업데이트
     - 고정/이동 광원 위치 기반 빛 영역 계산
     - 빛 구슬 잔여 시간 카운트다운
     - 스위치 상태 변경 시 연결된 광원 토글
  6. Render
     - 배경 3레이어 패럴랙스 (원경×0.3, 중경×0.6, 근경×1.0)
     - 빛/그림자 모드에 따른 배경색 전환 (lerp 300ms)
     - 중립 플랫폼 렌더링
     - 활성 플랫폼 (100% 불투명) 렌더링
     - 유령 플랫폼 (20% 불투명, 점선 윤곽) 렌더링
     - 광원 범위 시각화 (부드러운 원형 글로우)
     - 플레이어 캐릭터 (현재 모드에 맞는 글로우 색상)
     - 별, 스위치, 출구 등 인터랙티브 오브젝트
     - 파티클 (빛 입자, 먼지, 전환 이펙트)
     - UI 오버레이 (별 카운트, 사망 횟수, 타이머, 모드 표시기)
  7. Audio
     - 점프: 밝은 '핑' (800Hz→1200Hz, 100ms)
     - 착지: 부드러운 '퍽' (200Hz, 50ms)
     - 모드 전환: 상승 아르페지오(빛) / 하강 아르페지오(그림자) (300ms)
     - 별 수집: 반짝이는 차임 (1000Hz→1500Hz→2000Hz, 200ms)
     - 사망: 낮은 '우웅' (150Hz, 300ms)
     - 스위치: '딸깍' 클릭 (500Hz→300Hz, 80ms)
     - 출구 도달: 승리 팡파르 (도-미-솔-도↑, 500ms)
```

## 6. 난이도 시스템

### 6.1 난이도 선택 (3단계)
게임 시작 시 선택. localStorage에 저장.

| 항목 | 쉬움 | 보통 | 어려움 |

| 빛 구슬 지속 | 8초 | 5초 | 3초 |
| 가시 피격 판정 | 75% 크기 | 100% | 110% |
| 체크포인트 간격 | 2개/레벨 | 1개/레벨 | 0개 (시작점만) |
| 유령 플랫폼 가시성 | 40% 불투명 | 20% 불투명 | 10% 불투명 |
| 대시 쿨다운 | 없음 | 없음 | 0.3초 |
| 힌트 표시 | 자동 | 요청 시 | 없음 |

### 6.2 레벨별 난이도 곡선 (6레벨 + 보스 2개)

| 레벨 | 제목 | 신규 메카닉 | 복잡도 |
|---|---|---|---|
| Tutorial | 빛의 시작 | 이동, 점프, 빛/그림자 플랫폼 개념 | ★ |
| 1 | 첫 빛 | 빛/그림자 모드 수동 전환 | ★★ |
| 2 | 반전된 길 | 스위치로 광원 토글 | ★★☆ |
| 3 | 떠다니는 빛 | 이동 광원 밀기 | ★★★ |
| Boss 1 | 빛의 수호자 | 자동 전환 + 추격 퍼즐 (2페이즈) | ★★★☆ |
| 4 | 빛의 구슬 | 빛 구슬 던지기 능력 획득 | ★★★ |
| 5 | 프리즘 미궁 | 프리즘으로 빛 굴절 | ★★★★ |
| 6 | 그림자 심연 | 복합 퍼즐 (전 메카닉 활용) | ★★★★☆ |

### 6.3 보스 퍼즐 설계

**Boss 1 — 빛의 수호자** (레벨 3 이후):
- 페이즈 1: 거대 빛 구체가 방 안을 떠돌며 빛/그림자 영역을 주기적으로 변경. 플레이어는 3개 스위치를 올바른 타이밍에 활성화하여 빛의 문 도달
- 페이즈 2 (스위치 2개 활성 후): 구체 이동 속도 1.5배, 가시 추가 활성화, 스위치 토글 간격 단축

**Boss 2 — 먹구름의 심장** (레벨 6 이후):
- 페이즈 1: 어둠의 핵이 중앙에 위치. 주변에 그림자 장벽이 회전. 빛 구슬을 핵에 3회 명중시켜야 함
- 페이즈 2 (HP 50%): 핵이 이동 시작 + 프리즘 퍼즐 추가. 프리즘을 조정하여 빛을 핵에 반사시켜야 함. 그림자 장벽 패턴 복잡화

## 11. 구현 가이드 (개발자 참고)

### 11.1 핵심 시스템
1. **타일맵 엔진**: C4 메트로배니아 타일맵 패턴 재활용. 32×32px 타일 기반.
2. **광원 시스템**: 각 광원은 `{x, y, radius, on}`. 타일이 광원 반경 내인지 원형 범위 체크.
3. **모드 전환**: 전역 상태 `currentMode: 'light' | 'shadow'`. 전환 시 모든 플랫폼의 collider 상태 즉시 재계산.
4. **물리 엔진**: 간단한 AABB + 중력. 오브젝트 풀링은 파티클에만 적용 (퍼즐 특성상 대량 엔티티 불필요).

### 11.2 파티클 풀
- lightParticles: 50개 (빛 입자, 글로우 효과)
- shadowParticles: 50개 (그림자 입자)
- deathParticles: 20개 (사망 파편)

### 11.3 코드 구조 목표
- **코드 본문**: 980~1100줄 (로직 + 렌더링 + UI)
- **JSON 레벨 데이터**: 별도 변수로 코드 상단에 선언 (200~300줄)
