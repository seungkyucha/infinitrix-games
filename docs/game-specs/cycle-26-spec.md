# 보이드 아키텍트 (Void Architect) — 사이클 #26 게임 기획서

> **1페이지 요약**: 차원 균열에서 쏟아지는 보이드 피조물을 테트로미노 형태의 방어 구조물로 막아내는 **로그라이크 타워 디펜스**. 7종 테트로미노 블록 × 5원소(화염/빙결/독/번개/보이드) 배치로 타워를 건설하고, 15+1 웨이브를 방어하며, 5차원 보스 + 히든 보스 "차원 파괴자"를 공략한다. 웨이브 간 로그라이크 선택지(타워 업그레이드/새 블록/차원 능력) 3택으로 매 플레이가 다른 전략 경로를 제공한다. 영구 진행(업그레이드 트리 3갈래: 공격/방어/유틸), SeededRNG 프로시저럴 맵 + BFS 경로 검증, 3단 난이도(견습생/건축가/차원장인), 내러티브(차원 수호자 스토리). **arcade+strategy 조합을 1→2개로 강화**한다.

> **MVP 경계**: Phase 1(핵심 루프: 테트로미노 배치→웨이브 방어→로그라이크 선택→보스전, 차원 1~2 + 보스 2체 + 업그레이드 기본 트리) + Phase 2(차원 3~5 + 보스 3체 + 히든 보스 + 전체 내러티브 + 도전 모드) 순차 구현. Phase 1만으로도 완전한 게임 경험 제공 필수.

## §1. 게임 개요 및 핵심 재미 요소

### 1.1 컨셉
보이드 아키텍트는 차원의 균열을 통해 밀려오는 보이드 피조물에 맞서, 테트로미노 형태의 방어 블록을 전략적으로 배치하여 차원문을 봉인하는 로그라이크 타워 디펜스다. 테트로미노의 **공간 퍼즐**과 타워 디펜스의 **경로 전략**, 로그라이크의 **리플레이 가치**가 삼중 축을 이룬다.

### 1.2 핵심 재미 3축
1. **테트로미노 공간 퍼즐**: 7종 블록의 회전·반전으로 제한된 그리드에 최적 타워 배치를 찾는 공간 퍼즐. 인접 동일 원소 블록 연결 시 시너지 범위 공격 해금 — "테트리스 + 타워 디펜스" 하이브리드의 독특한 쾌감.
2. **로그라이크 전략 분기**: 웨이브마다 3택(새 블록/타워 업그레이드/차원 능력) 로그라이크 선택지. 같은 맵이라도 전략 경로가 매번 달라 재도전 욕구 극대화. "이번엔 빙결 집중 빌드로 가볼까?"
3. **차원 보스 공략**: 5차원 보스는 특정 타워 배치 패턴에서만 약점이 노출되는 공간 퍼즐 전투. 단순 DPS가 아닌 "올바른 위치에 올바른 타워"를 세우는 지적 만족감.

### 1.3 스토리/내러티브
- **배경**: "아키텍트"는 다차원을 건설·유지하는 고대 존재들이다. 보이드 — 차원 사이의 무(無) — 가 깨어나 차원을 삼키기 시작했다. 마지막 아키텍트인 플레이어는 차원의 잔해(테트로미노 블록)를 이용해 방벽을 세우고, 5개 차원의 균열을 봉인해야 한다.
- **차원별 이야기**: 각 차원에는 멸망한 문명의 기록이 남아있다. 화염 차원의 대장장이, 빙결 차원의 학자, 독 차원의 연금술사, 번개 차원의 공학자, 보이드 차원의 이전 아키텍트. 웨이브 사이에 짧은 텍스트 대화(Canvas 연출 3줄)로 이야기 전달.
- **엔딩**: 5차원 봉인 완료 → "차원의 안정" 일반 엔딩. 히든 보스 "차원 파괴자" 격파 → "아키텍트의 계승" 진엔딩 — 보이드가 사실은 낡은 차원을 정리하여 새 차원의 탄생을 돕는 자연현상이었고, 아키텍트의 역할은 파괴가 아닌 재건이었다는 반전.
- **스토리 전달**: 웨이브 간 차원 기록(3줄 텍스트 + 배경 연출), 보스 등장/격파 컷신(Canvas 카메라 줌 + 대사 5초), 히든 보스 후 에필로그(스크롤 텍스트 10초).

### 1.4 장르 강화 효과
- **arcade + strategy = 1→2개** (최소 빈도 조합 강화)
- arcade 전체: 9→10개, strategy 전체: 9→10개
- 기존 arcade+strategy(chrono-siege)와 차별화: 전통 시즈 방어 vs 테트로미노 공간 퍼즐 TD — 완전히 다른 핵심 메카닉

## §2. 게임 규칙 및 목표

### 2.1 기본 규칙
- **그리드 기반 전략**: 12×8 헥스 그리드에 테트로미노 블록을 배치하여 타워를 건설
- **웨이브 디펜스**: 적은 맵 좌측 스폰 포인트에서 출현하여 우측 차원문(코어)으로 진행
- **코어 HP**: 차원문 코어 HP = 10(견습생)/7(건축가)/5(차원장인). 적이 코어에 도달하면 HP -1
- **코어 HP 0 → 게임 오버**. 획득한 영구 자원(차원석)은 보존
- **적 경로**: 적은 타워 블록을 우회하여 BFS 최단 경로로 이동 — 블록 배치가 곧 경로 설계

### 2.2 승리/패배 조건
- **스테이지 승리**: 해당 차원의 모든 웨이브(3웨이브) + 보스 웨이브 클리어
- **런 승리**: 5개 차원 순차 클리어 (총 15 일반 웨이브 + 5 보스 웨이브 = 20웨이브)
- **히든 보스**: 3가지 조건 달성 시 16번째 스테이지 "차원의 심연" 해금
  - 코어 HP 잔여 50% 이상
  - 5가지 원소 타워 모두 건설한 적 있음
  - 영구 업그레이드 "차원 감지" 해금 완료
- **패배**: 코어 HP 0 → 게임 오버 화면 → 차원석 정산 → 타이틀

### 2.3 차원 구성

| 차원 | 웨이브 수 | 테마 | 환경 효과 | 보스 |
|------|----------|------|----------|------|
| 1. 화염 차원 | 3+보스 | 용암, 화염 기둥, 열기파 | 화염 지형: 위를 지나는 적에게 DoT | 이그니스 (용암 거인) |
| 2. 빙결 차원 | 3+보스 | 빙판, 고드름, 냉기 안개 | 빙결 지형: 적 이동속도 -30% | 글라시아 (빙결 히드라) |
| 3. 독 차원 | 3+보스 | 독안개, 포자, 부식 늪 | 독 지형: 매 2초 적 HP -3% | 톡시아 (맹독 와이번) |
| 4. 번개 차원 | 3+보스 | 전기장, 도체, 방전 | 번개 지형: 3초마다 랜덤 적에게 벼락 | 볼타 (뇌전 피닉스) |
| 5. 보이드 차원 | 3+보스 | 무(無), 왜곡, 소멸 | 보이드: 타워 공격력 -20%, 적 투명화 | 닐 (보이드 군주) |
| 히든. 차원의 심연 | 보스만 | 전 차원 혼합 | 무작위 환경 전환 (10초 주기) | 차원 파괴자 |

### 2.4 난이도 3종

| 난이도 | 코어HP | 적 HP배율 | 웨이브당 적 수 | 로그라이크 선택지 | 잠금 조건 |
|--------|--------|----------|--------------|----------------|----------|
| 견습생 (쉬움) | 10 | ×0.7 | 기본 | 4택 (1 레어 보장) | 없음 |
| 건축가 (보통) | 7 | ×1.0 | ×1.2 | 3택 | 없음 |
| 차원장인 (어려움) | 5 | ×1.4 | ×1.5 | 3택 (커먼만) | 차원 3 이상 클리어 |

## §5. 핵심 게임 루프

### 5.1 초기화 순서 (F12 — TDZ 방지)
```
1. CONFIG 상수 선언
2. 전역 상태 객체(G) 선언
3. Canvas DOM 할당 (canvas, ctx)
4. 유틸리티 클래스 초기화 (TweenManager, ObjectPool, SoundManager)
5. 이벤트 리스너 등록 (키보드, 마우스, 터치, 리사이즈)
6. 게임 루프 시작 (requestAnimationFrame)
```

### 5.2 메인 게임 루프 (60fps)
```
function gameLoop(timestamp) {
  const dt = Math.min(timestamp - lastTime, 33.33); // 최대 2프레임 스킵
  lastTime = timestamp;

  // 1. 입력 처리 (inputManager.process)
  // 2. TweenManager 업데이트 (F4: setTimeout 0건, F13: clearImmediate)
  // 3. 상태별 업데이트 — §6.2 ACTIVE_SYSTEMS 매트릭스 참조
  //    - PLACEMENT: 블록 미리보기, 그리드 유효성 검사
  //    - WAVE: 적 이동(BFS), 타워 공격, 투사체, 파티클
  //    - BOSS: 보스 AI + 적 이동 + 타워 공격
  //    - ROGUELIKE_SELECT: 카드 호버/선택 애니메이션
  // 4. 렌더링 (배경 → 그리드 → 타워 → 적 → 보스 → 투사체 → 파티클 → UI)
  // 5. ObjectPool 정리 (비활성 객체 반환)

  requestAnimationFrame(gameLoop);
}
```

> **가드 플래그 패턴 (F5)**: 웨이브 클리어, 보스 격파 등 상태 전환 tween에는 반드시 `isTransitioning = true` 가드를 설정하여 콜백이 1회만 실행되도록 보호한다.

> **단일 갱신 경로 (F14)**: 모든 애니메이션 값(alpha, scale, position 등)은 TweenManager를 통해서만 갱신한다. 직접 대입과 tween을 동시에 사용하지 않는다.

## §7. 핵심 메카닉 상세

### 7.1 테트로미노 시스템

7종 기본 블록 (각 블록 = 4셀 타워):

```
I: ████      O: ██     T:  █      S:  ██     Z: ██
                ██        ██         ██          ██

L: █         J:   █
   █            █
   ██           ██
```

- **회전**: 0°/90°/180°/270° (SRS 회전 시스템 간소화)
- **반전**: 좌우 대칭 (S↔Z, L↔J 변환 효과)
- **원소**: 각 블록에 원소(화염/빙결/독/번개/보이드) 부여 — 원소에 따라 타워 공격 속성 결정
- **시너지**: 인접한 동일 원소 블록이 2개 이상 연결되면 시너지 발동
  - 2연결: 공격 범위 +20%
  - 3연결: 범위 공격 해금 (스플래시)
  - 4연결: 원소 폭발 (10초 쿨다운, 화면 전체 원소 효과)

### 7.2 타워 공격 시스템

| 은신 (Stealth) | ×0.7 | 1.0 | 반투명, 인접 3셀 내에서만 타겟팅 가능 |
| 치유 (Healer) | ×1.0 | 0.7 | 주변 아군 HP 회복 (초당 5%) |
| 보이드 (Void) | ×1.8 | 0.9 | 타워 공격력 -30% 오라 |

### 7.4 로그라이크 선택지 (웨이브 간)

3장의 카드 중 1장 선택:

| 등급 | 확률 | 예시 |
|------|------|------|
| 커먼 (흰색) | 60% | 새 테트로미노 블록 1개, 타워 DPS +10%, 사거리 +0.5셀 |
| 레어 (파랑) | 30% | 원소 변환 (기존 블록 원소 교체), 시너지 범위 +50%, 코어 HP +1 |
| 에픽 (보라) | 10% | 차원 능력 (전체 적 감속 5초), 메가 블록 (5셀 L+1), 골드 타워 (전 원소 복합) |

### 7.5 BFS 경로 시스템 (F54 — SeededRNG + 도달 가능성 검증)
```
1. 블록 배치 시: 임시 그리드에 블록 추가
2. BFS(spawnPoint → corePoint) 실행
3. 경로 존재 → 배치 확정, 적 경로 재계산
4. 경로 미존재 → 배치 거부 (빨간색 미리보기 + 경고음)
5. 비행형 적은 BFS 대신 직선 경로(A*)로 별도 계산
```

> 모든 블록 배치는 BFS 경로 검증을 통과해야 확정된다. "경로 차단" 상황은 원천 불가.

## §8. 난이도 시스템 / 밸런스

### 8.1 웨이브별 밸런스 테이블 (3구간)

**기본 난이도(건축가) 기준**

| 구간 | 웨이브 | 적 수 | 적 HP | 적 속도 | 보스 HP | 보상 (차원석) |
|------|--------|------|-------|---------|---------|-------------|
| 초반 (화염+빙결) | W1~W6 | 8→14 | 30→55 | 0.8→1.0 | 200/250 | 5~10 |
| 중반 (독+번개) | W7~W12 | 14→22 | 55→90 | 1.0→1.2 | 350/450 | 10~18 |
| 후반 (보이드+히든) | W13~W16 | 22→30 | 90→150 | 1.1→1.3 | 600/1000 | 18~30 |

### 8.2 DPS/EHP 밸런스 산식 (F57)

```
타워 총 DPS = Σ(각 타워 DPS × 원소 상성 배율 × 시너지 보너스) × 난이도 보정
적 총 EHP = 적 HP × 적 수 × (1 + Healer 보정) × 난이도 HP배율
클리어 예상 시간 = 적 총 EHP / 타워 총 DPS

목표: 클리어 예상 시간 < 웨이브 소요 시간(경로 길이 / 적 평균 속도)
안전 마진: 1.3배 (타워 DPS가 필요 DPS의 1.3배 이상이면 클리어 가능)

가정:
- 플레이어가 그리드의 60% 이상을 타워로 채움
- 시너지 2연결 평균 2쌍 (DPS +20% × 2)
- 경로 길이: BFS 최단 경로의 1.5배 (블록 배치로 우회 유도)

가정 오류 폴백 (DDA):
- 코어 HP가 3연속 웨이브에서 감소 → 다음 웨이브 적 HP -15%
- 코어 HP가 3연속 웨이브에서 무손상 → 다음 웨이브 적 수 +20%
```

### 8.3 난이도별 보정 매트릭스

| 파라미터 | 견습생 | 건축가 | 차원장인 |
|---------|--------|--------|---------|
| 코어 HP | 10 | 7 | 5 |
| 적 HP 배율 | ×0.7 | ×1.0 | ×1.4 |
| 적 수 배율 | ×1.0 | ×1.2 | ×1.5 |
| 보스 HP 배율 | ×0.8 | ×1.0 | ×1.3 |
| 로그라이크 선택 수 | 4택 | 3택 | 3택 |
| 레어 보장 | 매 선택 1개 | 없음 | 없음 |
| DDA 활성 | ✅ | ✅ | ❌ |
| 환경 효과 강도 | ×0.5 | ×1.0 | ×1.5 |

## §10. 프로시저럴 시스템

### 10.1 SeededRNG 맵 생성
- 각 런 시작 시 시드 생성 (난이도 + 타임스탬프 기반)
- 그리드 12×8에서 고정 요소: 스폰 포인트(좌측 중앙), 코어(우측 중앙)
- 랜덤 요소: 환경 오브젝트 위치(3~5개), 초기 장애물 블록(2~3개), 인터랙티브 요소 위치
- 보스 약점 위치도 시드 기반 — 단, 도달 가능한 위치만 유효

### 10.2 BFS 도달 가능성 검증 (F54)
```
1. 맵 생성 직후: BFS(spawn → core) 실행
2. 경로 미존재 → 장애물 1개 제거 후 재검증 (최대 3회 반복)
3. 3회 반복 후에도 미존재 → 새 시드로 재생성
4. 보스 약점 위치: 플레이어가 블록 배치로 해당 위치에 도달 가능한지 검증
   - 약점 위치에 인접한 셀이 최소 2개 빈 셀이어야 함 (블록 배치 공간 확보)
5. 비행형 적 경로: 직선 A*로 별도 검증 (장애물 무시, 맵 경계만 준수)
```

### 10.3 로그라이크 선택지 생성
- 웨이브마다 카드 풀에서 3장 랜덤 추출 (등급 확률 §7.4 준수)
- 중복 제거: 직전 웨이브에서 선택한 카드와 동일 카드 제외
- 밸런스 보정: 타워가 단일 원소에 편중(70%+)되면 다른 원소 카드 가중치 ×2

## §11. 점수 시스템

### 11.1 점수 계산 (F8 — 판정 먼저, 저장 나중에)

```
런 점수 = (적 처치 점수 + 보스 점수 + 잔여 코어 HP 보너스 + 속도 보너스) × 난이도 배율

속도 보너스: 총 플레이 시간 < 15분이면 ×1.2, < 10분이면 ×1.5
난이도 배율: 견습생 ×0.7, 건축가 ×1.0, 차원장인 ×1.5
```

### 14.2 자동 검증 스크립트 (F59 — FAIL/WARN 2단계)

**FAIL (필수 — 위반 시 리뷰 제출 불가):**
1. `assets/` 디렉토리에 manifest.json, thumbnail.svg 외 파일 존재
2. `setTimeout` / `setInterval` 사용 (0건 필수)
3. `alert(` / `confirm(` / `prompt(` 사용
4. `new Image(` / `ASSET_MAP` / `SPRITES` / `preloadAssets` 패턴
5. `<link` / `<script src=` / `@import url` / `googleapis` 패턴
6. BFS 경로 검증 없는 블록 배치 코드

**WARN (권고 — 위반 시 경고 표시, 리뷰 제출 가능):**
7. `"Google Fonts"` / `"CDN"` / `"external"` 주석 잔존
8. 선언만 되고 참조되지 않는 상수/변수 (데드코드 F56)
9. 전역 변수 직접 참조 함수 (순수 함수 위반 F9)
10. 터치 타겟 48px 미만 하드코딩
