# 골드러시 택틱스 (Gold Rush Tactics) — 사이클 #37 게임 기획서

> **1페이지 요약**: 1849년 서부 골드러시 배경. 8×8 광산 그리드에서 채굴 블록을 전략적으로 배치하여 금맥을 캐내고, 획득한 자원으로 마을을 건설하는 **퍼즐+전략 하이브리드**. 5개 지역(골짜기→강변→사막→산악→전설의 광산) × 3개 광산 = 15 메인 스테이지 + 무법자 보스 5종 + 히든 골드러시 이벤트 3개 = **총 23 스테이지**. 도구 업그레이드 트리(3갈래×5단계) + 마을 건물 8종(×3레벨) + SeededRNG 기반 광맥 배치 + 다중 전략 경로(속도 클리어/완전 채굴/건물 우선/도구 우선). **puzzle+strategy 조합 8게임 최장 미사용 해소 + Western 미사용 테마 차별화 + Poki Top 10 내 puzzle+strategy 0개 블루오션.** Block Blast(823K 검색) 트렌드 + Blue Prince식 퍼즐 로그라이크 직접 대응.

> **MVP 경계**: Phase 1(핵심 루프: 광산 퍼즐→보상→마을 건설→다음 스테이지, 지역 1~3 + 보스 3체 + 도구 Lv1~3 + 건물 5종 + 블록 8종 + DDA 3단계) → Phase 2(지역 4~5 + 보스 2체 + 히든 3스테이지 + 도구 Lv4~5 + 건물 8종 완성 + 블록 12종 + 날씨/시간대 + 전체 내러티브 + 다국어). **Phase 1만으로도 완전한 퍼즐 전략 경험 제공 필수.**

## §1. 게임 개요 및 핵심 재미 요소

### 핵심 컨셉

**"10초 안에 이해, 10시간 파고들기"** — Block Blast의 직관적 그리드 배치 + Blue Prince의 전략적 선택 + 골드러시 내러티브.

1849년, 캘리포니아 골드러시. 동부에서 온 주인공 "잭 모건"이 황무지에 도착해 금광을 찾아 마을을 세우는 이야기. 플레이어는 8×8 광산 그리드에 채굴 블록을 배치하여 행/열을 완성하고, 금맥에서 자원을 채굴한다. 획득한 자원(금/은/동)으로 마을에 건물을 짓고, 도구를 업그레이드하며, 무법자 보스의 습격을 방어한다.

## §2. 게임 규칙 및 목표

### 2.1 광산 퍼즐 (핵심 플레이)

**그리드**: 8×8 정사각형 그리드. 각 셀은 빈 칸, 광석(금/은/동), 장애물(바위/지하수/독가스), 또는 금맥(특수 보상 셀).

**블록 배치**:
- 매 턴 3개의 채굴 블록이 제공됨 (1~4셀 크기의 다양한 형태)
- 블록을 드래그하여 그리드 빈 칸에 배치
- **블록은 회전 불가** (Block Blast 핵심 규칙 — 배치 전략의 긴장감)
- 3개 블록을 모두 배치해야 다음 3개가 나옴
- 배치 불가능한 상태가 되면 **광산 붕괴** (스테이지 실패)

**행/열 클리어**:
- 가로 행 또는 세로 열이 완전히 채워지면 해당 행/열이 클리어됨
- 클리어 시 해당 행/열의 모든 광석이 자원으로 변환:
  - 금 광석 → 금 ×3
  - 은 광석 → 은 ×2
  - 동 광석 → 동 ×1
- **콤보 보너스**: 동시에 2줄 이상 클리어 시 자원 ×1.5배 (3줄: ×2.0, 4줄+: ×3.0)
- 장애물은 클리어로 제거되지 않음 (도구로만 제거 가능)

**금맥 셀**:
- 특수 황금빛 셀. 금맥이 포함된 행/열을 클리어하면 **보너스 금 ×10** + 무작위 부스트 아이템 1개
- 스테이지마다 1~3개 금맥 존재, SeededRNG로 위치 결정

### 2.2 마을 건설 (메타 전략)

광산에서 획득한 자원으로 마을 건물을 건설/업그레이드:

| 건물 | 비용 (Lv1) | 효과 | 최대 Lv |
|------|-----------|------|---------|
| 대장간 | 금5 은10 | 도구 업그레이드 해금 | 3 |
| 잡화점 | 금3 은5 동10 | 스테이지 시작 시 부스트 아이템 1개 제공 | 3 |
| 역마차 | 금8 | 새로운 지역 개방 | 3 |
| 은행 | 금10 은5 | 스테이지 클리어 시 보유 금 ×5% 이자 | 3 |
| 보안관 사무실 | 금5 동15 | 보스전 방어 시간 +5초 | 3 |
| 주점 | 은8 동8 | 광부 고용 → 블록 선택지 +1개 (3→4개) | 3 |
| 교회 | 금3 은3 동3 | 광산 붕괴 시 1회 부활 (런당 1회) | 3 |
| 전보국 | 금15 | 히든 스테이지 위치 힌트 표시 | 3 |

### 2.3 무법자 보스전

5스테이지(지역 보스)마다 무법자가 마을을 습격. 보스전은 **타이머 퍼즐 모드**로 전환:

- 30초 제한 시간 내에 그리드에 **방어 블록**을 배치하여 방어벽 완성
- 보스마다 다른 약점 패턴: 특정 형태의 방어벽을 완성해야 격퇴
- 보스 체력 바 = 남은 약점 셀 수
- 시간 내 방어벽 미완성 시 마을 피해 (자원 50% 손실, 건물 1단계 하락)

| 보스 | 지역 | 약점 패턴 | 특수 능력 |
|------|------|----------|----------|
| 빌리 더 키드 | 골짜기 | 2×4 직선 방어벽 | 5초마다 랜덤 셀 파괴 |
| 검은 바트 | 강변 | L자 방어벽 2개 | 시야 50% 차단 (안개) |
| 제시 제임스 | 사막 | 십자(+) 방어벽 | 블록 1개를 랜덤 변형 |
| 와일드 번치 | 산악 | 테두리 1줄 완성 | 3인 동시 습격 (3개 약점) |
| 황금 유령 (최종) | 전설 광산 | T자 + L자 + 직선 복합 | 8초마다 그리드 회전 90° |

### 2.4 골드러시 이벤트 (히든 스테이지)

특정 조건 달성 시 황금 광맥 발견:
- **히든 1**: 금맥 셀 5개를 한 런에서 모두 클리어 → "엘도라도 광맥" (모든 셀이 금 광석)
- **히든 2**: 콤보 ×3.0 이상을 3회 달성 → "마더 로드" (블록이 모두 1×1, 50개 배치 가능)
- **히든 3**: 전 보스 무피해 격퇴 → "유령의 금광" (장애물 0, 시간 제한 60초 내 최대 채굴)

### 2.5 승리/패배 조건

- **스테이지 클리어**: 스테이지별 목표 자원량 달성 (예: 금 20 이상)
- **스테이지 실패**: 블록 배치 불가능 (광산 붕괴) 또는 보스전 시간 초과
- **게임 클리어**: 15개 메인 스테이지 + 최종 보스(황금 유령) 격퇴
- **영구 진행**: 실패해도 건물·도구 업그레이드는 유지됨 (로그라이트)

## §6. 게임 상태 머신

### 6.1 상태 정의 및 전환 테이블

> **F6/F21/F23 핵심 대응**: STATE_PRIORITY 8번째 재발 근절을 위해, 게임 상태를 **5개로 단순화**하고 모든 전환을 **TRANSITION_TABLE 단일 객체**에 정의한다.

```javascript
// 게임 상태 (5개만)
const STATE = {
  TITLE: 'TITLE',
  PUZZLE: 'PUZZLE',    // 광산 퍼즐 + 보스전 포함
  MAP: 'MAP',          // 마을 지도 + 건물 배치
  SHOP: 'SHOP',        // 도구 업그레이드 상점
  GAMEOVER: 'GAMEOVER'
};

// ⚠️ 모든 전환은 이 테이블에서만 정의. beginTransition()은 이 테이블만 참조.
const TRANSITION_TABLE = {
  'TITLE→PUZZLE':    { duration: 800, easing: 'easeInOut' },
  'TITLE→MAP':       { duration: 600, easing: 'easeIn' },
  'PUZZLE→MAP':      { duration: 500, easing: 'easeOut' },
  'PUZZLE→SHOP':     { duration: 400, easing: 'easeOut' },
  'PUZZLE→GAMEOVER': { duration: 1000, easing: 'easeIn' },
  'MAP→PUZZLE':      { duration: 500, easing: 'easeIn' },
  'MAP→SHOP':        { duration: 400, easing: 'easeOut' },
  'SHOP→MAP':        { duration: 400, easing: 'easeOut' },
  'SHOP→PUZZLE':     { duration: 500, easing: 'easeIn' },
  'GAMEOVER→TITLE':  { duration: 800, easing: 'easeInOut' },
  'GAMEOVER→MAP':    { duration: 600, easing: 'easeIn' }   // 로그라이트 계속
};

// beginTransition은 TRANSITION_TABLE에 없는 전환을 시도하면 console.error + 무시
function beginTransition(from, to) {
  const key = `${from}→${to}`;
  if (!TRANSITION_TABLE[key]) {
    console.error(`[FSM] Invalid transition: ${key}`);
    return false;
  }
  if (G._transitioning) return false; // 가드 플래그
  G._transitioning = true;
  const cfg = TRANSITION_TABLE[key];
  tweenManager.add(G._transProxy, 'alpha', 0, 1, cfg.duration, cfg.easing, () => {
    G.state = to;
    tweenManager.add(G._transProxy, 'alpha', 1, 0, cfg.duration, cfg.easing, () => {
      G._transitioning = false; // 가드 해제
    });
  });
  return true;
}
```

**자동 검증**: 스모크 테스트에서 `Object.keys(TRANSITION_TABLE)` 내 모든 `from→to` 쌍이 실제 코드에서 `beginTransition(STATE.X, STATE.Y)` 형태로 호출되는지 grep 검증.

## §7. 보스전 상세 설계

### 7.1 보스전 진입

지역 마지막 광산(3번째) 클리어 시 자동 보스전 진입:
1. 화면 흔들림 + 경고음
2. 보스 등장 컷신 (SVG 확대 + 이름 표시, 3초)
3. 퍼즐 모드 → 보스 퍼즐 모드 전환 (그리드 리셋, 방어 블록 제공)
4. 30초 타이머 시작

### 7.2 보스 약점 패턴

각 보스는 그리드 위에 **약점 셀** (빨간 테두리)을 표시. 해당 셀에 블록을 배치하면 보스에게 데미지:

```

. . . . . . . .            . . ■ . . . . .

. . ■ ■ ■ ■ . .            . . ■ ■ . . . .
. . . . . . . .            . . . . . . . .
. . . . . . . .            . . . . ■ . . .
. . . . . . . .            . . . . ■ . . .
. . . . . . . .            . . . . ■ ■ . .
```

### 7.3 보스 DPS 캡 (F26)

보스전 데미지 계산:
- 약점 셀 1개 배치 = 1 데미지
- 도구 보너스(곡괭이 Lv3): +0.5 데미지/셀
- **DPS 캡: 약점 셀 수 × 2.0** (도구 보너스 상한)
- 보스 체력 = 약점 셀 수 × 1.5 (도구 없이 불가능한 보스는 없음 보장)

## §8. 난이도 시스템

### 8.1 스테이지별 난이도 곡선 (수학적 정의)

```
stage = 1~15 (메인 스테이지)
region = ceil(stage / 3)  // 1~5

장애물 비율 = 0.05 + (stage - 1) × 0.02          // 5%~33%
금맥 수     = max(1, 4 - floor(stage / 4))         // 3→2→1
블록 복잡도 = min(4, 1 + floor(stage / 4))         // 1~4셀 블록
목표 자원   = 10 + stage × 5                        // 15~85
보스 체력   = region × 8                             // 8~40 (약점 셀 수)
보스 시간   = max(20, 35 - region × 3)              // 32→29→26→23→20초
```

### 8.2 DDA (Dynamic Difficulty Adjustment) 3단계

| DDA 레벨 | 조건 | 조정 |
|----------|------|------|
| 쉬움 | 연속 2회 실패 | 장애물 -20%, 블록 선택지 +1개, 보스 시간 +10초 |
| 보통 | 기본 | 기획 수치 그대로 |
| 어려움 | 연속 3회 무피해 클리어 | 장애물 +10%, 금맥 -1개, 보스 특수 능력 강화 |

DDA는 **스테이지 시작 시에만** 적용 (플레이 중 변경 없음).

### 8.3 극단 빌드 검증 (부록A 요약)

| 빌드 | 도구 | 건물 | 예상 클리어 시간 |
|------|------|------|----------------|
| 속도 클리어 (도구 최우선) | 곡괭이 Lv5 | 대장간 Lv3만 | 스테이지 15: 45초 |
| 완전 채굴 (자원 최대화) | 곡괭이 Lv3 | 은행 Lv3 + 교회 Lv3 | 스테이지 15: 90초 |
| 건물 우선 (안전 플레이) | 곡괭이 Lv2 | 전체 Lv2 | 스테이지 15: 110초 |

> 모든 극단 빌드의 클리어 시간: **30~120초 범위 내** (범위 외이면 밸런스 조정 필요).

## §10. 프로시저럴 생성

### 10.1 광맥 배치 알고리즘

```
generateMineGrid(seed, stage):
  1. SeededRNG(seed + stage) 초기화
  2. 8×8 그리드 빈 칸으로 초기화
  3. 장애물 배치: 장애물비율 × 64개 셀, 랜덤 위치
  4. 금맥 배치: 금맥 수만큼 비장애물 셀에 랜덤 배치
  5. 광석 배치: 나머지 빈 셀에 금(15%)/은(35%)/동(50%) 확률로 배치
  6. ⚠️ 도달 가능성 검증 (§10.2)
  7. 검증 실패 시 seed+1로 재생성 (최대 10회)
```

### 10.2 도달 가능성 검증 (BFS) — F27

> **Cycle 36 BFS 기술 계승 + Cycle 23 교훈**: 생성된 그리드에서 최소 1개 행 또는 열이 클리어 가능한지 BFS로 검증.

```
validateGrid(grid, blocks):
  // 현재 블록 세트로 최소 1개 행/열 완성 가능한지 시뮬레이션
  for each row/col:
    빈 셀 수 계산
    if (빈 셀 수 <= 가용 블록 총 셀 수):
      return VALID  // 최소한 클리어 가능성 존재
  return INVALID → 재생성
```

## §16. 내러티브 / 스토리

### 16.1 배경 설정

1849년, 캘리포니아. 잭 모건은 동부의 가난한 농부였으나, 금이 발견되었다는 소문을 듣고 서부로 향한다. 도착한 곳은 "데드맨스 크릭" — 아직 아무것도 없는 황무지. 여기서 금을 캐고, 마을을 세우고, 무법자로부터 지켜내야 한다.

### 16.2 지역별 내러티브

| 지역 | 스토리 요소 | 분위기 |
|------|-----------|--------|
| 1. 골짜기 | 첫 금 발견, 작은 캠프 시작 | 희망적, 밝은 |
| 2. 강변 | 경쟁 광부 등장, 영역 다툼 | 긴장감, 갈등 |
| 3. 사막 | 물 부족, 생존 위기 | 절박한, 거친 |
| 4. 산악 | 전설의 광맥 소문, 위험한 등반 | 경이로운, 위험 |
| 5. 전설의 광산 | 원주민 전설의 황금 동굴 발견 | 신비로운, 장대한 |

### 16.3 컷신 (텍스트 + 캐릭터 SVG)

각 지역 시작/종료 + 보스 등장 시 짧은 텍스트 컷신:
- 캐릭터 SVG 포즈 전환 + 대사 박스
- 터치/클릭으로 진행
- Skip 버튼 (첫 플레이 후 해금)

